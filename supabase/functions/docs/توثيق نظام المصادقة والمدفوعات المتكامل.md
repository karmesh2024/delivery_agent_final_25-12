# توثيق نظام المصادقة والمدفوعات المتكامل

## مقدمة

يهدف هذا المستند إلى توثيق التكامل بين نظام المصادقة المحسّن ونظام المدفوعات والمحافظ المالية في تطبيق لوحة تحكم مندوبي التوصيل. يستعرض المستند الهيكل الحالي لكلا النظامين، مدى توافقهما، والتغييرات المطلوبة لضمان تكاملهما بشكل سلس مع التركيز على استخدام Supabase Edge Functions.

## 1. تحليل الوضع الحالي

### 1.1 نظام المصادقة الحالي

نظام المصادقة الحالي يعتمد على:

- استخدام JWT المخزن في localStorage للحفاظ على حالة المصادقة
- إنشاء عميل Supabase مع JWT للمصادقة
- آلية لتجديد JWT عند انتهاء صلاحيته
- معترض للطلبات لمعالجة أخطاء المصادقة
- AuthProvider للتحقق من وجود JWT في localStorage

### 1.2 نظام المدفوعات والمحافظ المالية الحالي

نظام المدفوعات الحالي يعتمد على:

- جداول قاعدة بيانات: `wallets`, `wallet_transactions`, `transactions`, `payout_requests`, `payment_methods`, `user_payment_methods`
- خدمات في الواجهة الخلفية (`paymentsService.ts`) تتفاعل مباشرة مع قاعدة البيانات
- استخدام سياسات RLS للتحكم في الوصول
- عمليات معقدة تتطلب صلاحيات مرتفعة (`service_role_key`)
- واجهات مستخدم لإدارة المحافظ والمعاملات

### 1.3 مدى التوافق بين الأسكيما الحالية والتطبيق

بعد مراجعة الوثائق والأكواد الحالية، تبين أن:

1. **هيكل قاعدة البيانات متوافق مع التطبيق الحالي**:
   - جدول `admins` يحتوي على الحقول اللازمة لإدارة المصادقة والصلاحيات
   - جداول المدفوعات (`wallets`, `transactions`, الخ) تتوافق مع متطلبات التطبيق

2. **نظام الصلاحيات مناسب**:
   - تم إضافة صلاحية `wallets:manage` للسماح بإدارة المحافظ
   - آلية التحقق من الصلاحيات تعمل بشكل صحيح

3. **بعض التحسينات مطلوبة**:
   - تحسين آلية التكامل بين المصادقة والمدفوعات
   - تحويل العمليات الحساسة إلى Edge Functions

## 2. خطة ترحيل العمليات الحساسة إلى Edge Functions

### 2.1 العمليات المرشحة للترحيل

استناداً إلى تحليل الوثائق، يجب ترحيل العمليات التالية إلى Edge Functions:

1. **عمليات إدارة المحافظ**:
   - إنشاء محفظة جديدة
   - تعديل رصيد المحفظة (إضافة/خصم)
   - تغيير حالة المحفظة (تعليق/تنشيط)

2. **عمليات المعاملات المالية**:
   - إنشاء معاملة جديدة
   - معالجة طلبات السحب (الموافقة/الرفض)
   - تنفيذ التحويلات بين المحافظ

3. **عمليات المصادقة الحساسة**:
   - تغيير صلاحيات المستخدمين
   - إدارة أدوار المستخدمين

### 2.3 تكامل المصادقة مع Edge Functions

لضمان أمان Edge Functions، سيتم تنفيذ الآلية التالية:

1. **التحقق من JWT**:
   - استقبال JWT في ترويسة الطلب (`Authorization`)
   - التحقق من صحة JWT باستخدام `SupabaseClient`
   - استخراج `user_id` وبيانات المستخدم

2. **التحقق من الصلاحيات**:
   - الاستعلام عن صلاحيات المستخدم بناءً على `user_id`
   - التحقق من وجود الصلاحيات المطلوبة للعملية
   - رفض الطلبات غير المصرح بها

3. **تسجيل العمليات**:
   - تسجيل جميع العمليات الحساسة مع هوية المستخدم المنفذ
   - تخزين سجلات التدقيق في جدول مخصص

### 5.2 آلية التحقق من الصلاحيات في Edge Functions

```typescript
// مثال للتحقق من الصلاحيات في Edge Function
async function checkPermission(userId: string, permission: string): Promise<boolean> {
  const { data, error } = await supabaseAdmin
    .from('admins')
    .select('*')
    .eq('user_id', userId)
    .single();
  
  if (error || !data) return false;
  
  // التحقق من الصلاحية المباشرة
  if (data.permissions && data.permissions[permission] === true) return true;
  
  // التحقق من صلاحيات الدور
  const { data: rolePermissions } = await supabaseAdmin
    .rpc('check_admin_permission_enhanced', {
      admin_id: data.id,
      permission_code: permission
    });
  
  return !!rolePermissions;
}
```

## 6. مثال تفصيلي: Edge Function لإنشاء طلب سحب

### 6.1 هيكل ملف `create-payout-request.ts`

```typescript
// supabase/functions/payouts/create-payout-request.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2.38.4";
import { corsHeaders } from "../_shared/cors.ts";

const supabaseUrl = Deno.env.get("SUPABASE_URL") || "";
const supabaseServiceKey = Deno.env.get("SUPABASE_SERVICE_ROLE_KEY") || "";

interface PayoutRequestBody {
  amount: number;
  payment_method_type: string;
  payment_method_details: Record<string, any>;
  user_notes?: string;
}

serve(async (req) => {
  // التعامل مع طلبات CORS
  if (req.method === "OPTIONS") {
    return new Response("ok", { headers: corsHeaders });
  }

  try {
    // التحقق من الطريقة
    if (req.method !== "POST") {
      return new Response(
        JSON.stringify({ error: "Method not allowed" }),
        { status: 405, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    // استخراج JWT من الترويسة
    const authHeader = req.headers.get("Authorization");
    if (!authHeader || !authHeader.startsWith("Bearer ")) {
      return new Response(
        JSON.stringify({ error: "Unauthorized" }),
        { status: 401, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    const jwt = authHeader.replace("Bearer ", "");
    
    // إنشاء عملاء Supabase
    const supabaseClient = createClient(supabaseUrl, supabaseServiceKey, {
      auth: { persistSession: false }
    });
    
    const supabaseAuthClient = createClient(supabaseUrl, supabaseServiceKey, {
      auth: { 
        persistSession: false,
        autoRefreshToken: false,
      }
    });

    // التحقق من JWT واستخراج user_id
    const { data: { user }, error: userError } = await supabaseAuthClient.auth.getUser(jwt);
    
    if (userError || !user) {
      return new Response(
        JSON.stringify({ error: "Invalid token" }),
        { status: 401, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    const userId = user.id;

    // جلب بيانات الطلب
    const requestData: PayoutRequestBody = await req.json();
    
    // التحقق من البيانات المطلوبة
    if (!requestData.amount || requestData.amount <= 0) {
      return new Response(
        JSON.stringify({ error: "Invalid amount" }),
        { status: 400, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    if (!requestData.payment_method_type || !requestData.payment_method_details) {
      return new Response(
        JSON.stringify({ error: "Payment method information is required" }),
        { status: 400, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    // البحث عن محفظة المستخدم
    const { data: walletData, error: walletError } = await supabaseClient
      .from("wallets")
      .select("id, balance, status")
      .eq("user_id", userId)
      .eq("status", "ACTIVE")
      .single();

    if (walletError || !walletData) {
      return new Response(
        JSON.stringify({ error: "Wallet not found or inactive" }),
        { status: 404, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    // التحقق من كفاية الرصيد
    if (walletData.balance < requestData.amount) {
      return new Response(
        JSON.stringify({ error: "Insufficient balance" }),
        { status: 400, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    // البحث عن طريقة الدفع
    const { data: paymentMethodData, error: paymentMethodError } = await supabaseClient
      .from("payment_methods")
      .select("id")
      .eq("code", requestData.payment_method_type)
      .eq("is_active", true)
      .single();

    if (paymentMethodError || !paymentMethodData) {
      return new Response(
        JSON.stringify({ error: "Invalid payment method" }),
        { status: 400, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    // البحث عن طريقة دفع المستخدم أو إنشاء واحدة جديدة
    let userPaymentMethodId: string;
    
    const { data: existingPaymentMethod } = await supabaseClient
      .from("user_payment_methods")
      .select("id")
      .eq("user_id", userId)
      .eq("payment_method_id", paymentMethodData.id)
      .eq("status", "ACTIVE")
      .maybeSingle();
      
    if (existingPaymentMethod) {
      userPaymentMethodId = existingPaymentMethod.id;
    } else {
      // إنشاء طريقة دفع جديدة للمستخدم
      const { data: newPaymentMethod, error: newPaymentMethodError } = await supabaseClient
        .from("user_payment_methods")
        .insert({
          user_id: userId,
          payment_method_id: paymentMethodData.id,
          details: requestData.payment_method_details,
          status: "ACTIVE"
        })
        .select("id")
        .single();
        
      if (newPaymentMethodError || !newPaymentMethod) {
        return new Response(
          JSON.stringify({ error: "Failed to create payment method" }),
          { status: 500, headers: { ...corsHeaders, "Content-Type": "application/json" } }
        );
      }
      
      userPaymentMethodId = newPaymentMethod.id;
    }

    // بدء معاملة قاعدة البيانات
    const { data: payoutRequest, error: payoutError } = await supabaseClient.rpc(
      "create_payout_request",
      {
        p_user_id: userId,
        p_wallet_id: walletData.id,
        p_amount: requestData.amount,
        p_payment_method_id: paymentMethodData.id,
        p_user_payment_method_id: userPaymentMethodId,
        p_user_notes: requestData.user_notes || ""
      }
    );

    if (payoutError) {
      return new Response(
        JSON.stringify({ error: "Failed to create payout request", details: payoutError.message }),
        { status: 500, headers: { ...corsHeaders, "Content-Type": "application/json" } }
      );
    }

    // تسجيل العملية في سجل التدقيق
    await supabaseClient
      .from("audit_logs")
      .insert({
        user_id: userId,
        action: "CREATE_PAYOUT_REQUEST",
        entity_type: "PAYOUT_REQUEST",
        entity_id: payoutRequest.id,
        details: {
          amount: requestData.amount,
          payment_method_type: requestData.payment_method_type
        }
      });

    // إرجاع الاستجابة
    return new Response(
      JSON.stringify({ 
        success: true, 
        data: { 
          payout_request_id: payoutRequest.id,
          status: "PENDING_APPROVAL",
          message: "Payout request created successfully"
        }
      }),
      { status: 201, headers: { ...corsHeaders, "Content-Type": "application/json" } }
    );

  } catch (error) {
    // معالجة الأخطاء غير المتوقعة
    console.error("Error in create-payout-request:", error);
    
    return new Response(
      JSON.stringify({ error: "Internal server error", details: error.message }),
      { status: 500, headers: { ...corsHeaders, "Content-Type": "application/json" } }
    );
  }
});
```

### 6.2 دالة قاعدة البيانات المساعدة `create_payout_request`

```sql
CREATE OR REPLACE FUNCTION create_payout_request(
  p_user_id UUID,
  p_wallet_id UUID,
  p_amount NUMERIC,
  p_payment_method_id UUID,
  p_user_payment_method_id UUID,
  p_user_notes TEXT DEFAULT ''
) RETURNS UUID
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_payout_request_id UUID;
  v_transaction_id UUID;
BEGIN
  -- بدء المعاملة
  BEGIN
    -- إنشاء طلب سحب جديد
    INSERT INTO payout_requests (
      user_id,
      wallet_id,
      amount_requested,
      currency,
      payment_method_id,
      user_payment_method_id,
      status,
      user_notes
    ) VALUES (
      p_user_id,
      p_wallet_id,
      p_amount,
      'EGP',  -- يمكن جعل هذا معلمة إذا كان هناك دعم لعملات متعددة
      p_payment_method_id,
      p_user_payment_method_id,
      'PENDING_APPROVAL',
      p_user_notes
    ) RETURNING id INTO v_payout_request_id;
    
    -- إنشاء معاملة مرتبطة
    INSERT INTO transactions (
      user_id,
      amount,
      currency,
      type,
      status,
      description,
      payout_request_id
    ) VALUES (
      p_user_id,
      p_amount,
      'EGP',
      'PAYOUT_REQUEST',
      'PENDING_APPROVAL',
      'طلب سحب بقيمة ' || p_amount || ' جنيه',
      v_payout_request_id
    ) RETURNING id INTO v_transaction_id;
    
    -- تحديث طلب السحب بمعرف المعاملة
    UPDATE payout_requests
    SET transaction_id = v_transaction_id
    WHERE id = v_payout_request_id;
    
    -- إرجاع معرف طلب السحب
    RETURN v_payout_request_id;
  EXCEPTION
    WHEN OTHERS THEN
      -- إلغاء المعاملة في حالة حدوث خطأ
      RAISE;
  END;
END;
$$;
```

### 6.3 كيفية استخدام Edge Function من واجهة المستخدم

```typescript
// مثال لاستخدام Edge Function من واجهة المستخدم
async function createPayoutRequest(amount: number, paymentMethodType: string, paymentMethodDetails: any) {
  try {
    const token = localStorage.getItem('supabase.auth.token');
    
    if (!token) {
      throw new Error('User not authenticated');
    }
    
    const response = await fetch(
      'https://your-project-ref.supabase.co/functions/v1/payouts/create-payout-request',
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${token}`
        },
        body: JSON.stringify({
          amount,
          payment_method_type: paymentMethodType,
          payment_method_details: paymentMethodDetails
        })
      }
    );
    
    const data = await response.json();
    
    if (!response.ok) {
      throw new Error(data.error || 'Failed to create payout request');
    }
    
    return data;
  } catch (error) {
    console.error('Error creating payout request:', error);
    throw error;
  }
}
```

## 8. الهيكل المالي المتكامل للإدارات والنظام

### 8.1 نظرة عامة على الهيكل المالي

يهدف النظام المالي المتكامل إلى إنشاء بنية هرمية للمحافظ المالية تعكس الهيكل التنظيمي للشركة، مع آليات محكمة للصلاحيات والتحكم في تدفق الأموال. يتكون هذا النظام من:

1. **محافظ النظام الرئيسية**: تمثل سيولة النظام الكلية وتخضع لإدارة أصحاب أدوار مجلس الإدارة
2. **محافظ الإدارات**: محافظ مخصصة لكل إدارة بميزانيات محددة
3. **محافظ المسؤولين الشخصية**: محافظ شخصية للمسؤولين لإدارة مصاريفهم ومكافآتهم

### 8.2 هيكل قاعدة البيانات للنظام المالي المتكامل

```sql
-- جدول محافظ النظام الرئيسية
CREATE TABLE IF NOT EXISTS public.system_wallets (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  wallet_type text NOT NULL, -- 'main_treasury', 'revenue', 'expenses', etc.
  name text NOT NULL, -- اسم المحفظة (خزينة النظام الرئيسية، إيرادات، مصروفات، إلخ)
  balance numeric NOT NULL DEFAULT 0,
  currency text NOT NULL DEFAULT 'SAR',
  description text,
  is_active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT system_wallets_pkey PRIMARY KEY (id),
  CONSTRAINT system_wallets_type_unique UNIQUE (wallet_type)
);

-- جدول محافظ الإدارات
CREATE TABLE IF NOT EXISTS public.department_wallets (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  department_id uuid NOT NULL REFERENCES public.departments(id) ON DELETE CASCADE,
  wallet_type text NOT NULL DEFAULT 'operational', -- 'operational', 'bonus', 'petty_cash', etc.
  name text NOT NULL, -- اسم المحفظة
  balance numeric NOT NULL DEFAULT 0,
  currency text NOT NULL DEFAULT 'SAR',
  budget_limit numeric, -- الميزانية المخصصة (اختياري)
  budget_period text, -- فترة الميزانية (شهري، ربع سنوي، سنوي)
  description text,
  is_active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT department_wallets_pkey PRIMARY KEY (id),
  CONSTRAINT department_wallets_dept_type_unique UNIQUE (department_id, wallet_type)
);

-- جدول المعاملات المالية (موحد لكل أنواع المحافظ)
CREATE TABLE IF NOT EXISTS public.financial_transactions (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  transaction_type text NOT NULL, -- 'deposit', 'withdrawal', 'transfer', 'budget_allocation', etc.
  amount numeric NOT NULL,
  currency text NOT NULL DEFAULT 'SAR',
  
  -- حقول المصدر والوجهة (تستخدم حسب نوع المعاملة)
  source_wallet_id uuid, -- معرف المحفظة المصدر (إذا كان تحويل)
  source_wallet_type text, -- نوع المحفظة المصدر ('system', 'department', 'admin')
  destination_wallet_id uuid, -- معرف المحفظة الوجهة (إذا كان تحويل)
  destination_wallet_type text, -- نوع المحفظة الوجهة ('system', 'department', 'admin')
  
  -- معلومات إضافية
  reference_id uuid, -- معرف مرجعي (مثل معرف طلب أو معاملة)
  reference_type text, -- نوع المرجع ('order', 'payout', 'budget_allocation', etc.)
  description text, -- وصف المعاملة
  notes text, -- ملاحظات إضافية
  
  -- معلومات الحالة
  status text NOT NULL DEFAULT 'completed', -- 'pending', 'completed', 'failed', 'cancelled'
  requires_approval boolean NOT NULL DEFAULT false, -- هل تتطلب موافقة
  approved_by uuid REFERENCES public.admins(id), -- من وافق على المعاملة
  approved_at timestamptz, -- وقت الموافقة
  
  -- معلومات المنشئ والتوقيت
  created_by uuid NOT NULL REFERENCES public.admins(id), -- من أنشأ المعاملة
  created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT financial_transactions_pkey PRIMARY KEY (id)
);

-- جدول صلاحيات المحافظ (من يستطيع الوصول لأي محفظة وبأي صلاحية)
CREATE TABLE IF NOT EXISTS public.wallet_permissions (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  
  -- يمكن أن تكون الصلاحية لدور أو لمسؤول محدد (أحدهما فقط يجب أن يكون غير فارغ)
  role_id uuid REFERENCES public.roles(id),
  admin_id uuid REFERENCES public.admins(id),
  
  -- المحفظة المستهدفة (يمكن أن تكون محفظة نظام أو إدارة)
  wallet_id uuid NOT NULL, -- معرف المحفظة
  wallet_type text NOT NULL, -- نوع المحفظة ('system', 'department')
  
  -- الصلاحيات
  can_view boolean NOT NULL DEFAULT false, -- يمكنه عرض الرصيد والمعاملات
  can_deposit boolean NOT NULL DEFAULT false, -- يمكنه إيداع أموال
  can_withdraw boolean NOT NULL DEFAULT false, -- يمكنه سحب أموال
  can_transfer boolean NOT NULL DEFAULT false, -- يمكنه تحويل أموال
  can_approve boolean NOT NULL DEFAULT false, -- يمكنه الموافقة على المعاملات
  transaction_limit numeric, -- الحد الأقصى لقيمة المعاملة (اختياري)
  
  -- معلومات إضافية
  granted_by uuid NOT NULL REFERENCES public.admins(id), -- من منح هذه الصلاحية
  valid_from timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP, -- تاريخ بدء الصلاحية
  valid_until timestamptz, -- تاريخ انتهاء الصلاحية (اختياري)
  is_active boolean NOT NULL DEFAULT true,
  created_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamptz NOT NULL DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT wallet_permissions_pkey PRIMARY KEY (id)
);
```

5. **تطوير أول Edge Function للتحقق من الصلاحيات**:
   ```typescript
   // supabase/functions/_shared/permissions.ts
   import { createClient } from "https://esm.sh/@supabase/supabase-js@2.38.4";

   const supabaseUrl = Deno.env.get("SUPABASE_URL") || "";
   const supabaseServiceKey = Deno.env.get("SUPABASE_SERVICE_ROLE_KEY") || "";

   export async function checkPermission(userId: string, permission: string): Promise<boolean> {
     const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey, {
       auth: { persistSession: false }
     });
     
     const { data, error } = await supabaseAdmin
       .from('admins')
       .select('*')
       .eq('user_id', userId)
       .single();
     
     if (error || !data) return false;
     
     // التحقق من الصلاحية المباشرة
     if (data.permissions && data.permissions[permission] === true) return true;
     
     // التحقق من صلاحيات الدور
     const { data: rolePermissions } = await supabaseAdmin
       .rpc('check_admin_permission_enhanced', {
         admin_id: data.id,
         permission_code: permission
       });
     
     return !!rolePermissions;
   }

   export async function checkWalletPermission(
     userId: string,
     walletId: string,
     walletType: string,
     permissionType: 'view' | 'deposit' | 'withdraw' | 'transfer' | 'approve'
   ): Promise<boolean> {
     const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey, {
       auth: { persistSession: false }
     });
     
     // الحصول على معرف المسؤول
     const { data: adminData, error: adminError } = await supabaseAdmin
       .from('admins')
       .select('id, role_id')
       .eq('user_id', userId)
       .single();
     
     if (adminError || !adminData) return false;
     
     // التحقق من صلاحيات المسؤول المباشرة
     const { data: directPermissions } = await supabaseAdmin
       .from('wallet_permissions')
       .select(`can_${permissionType}`)
       .eq('admin_id', adminData.id)
       .eq('wallet_id', walletId)
       .eq('wallet_type', walletType)
       .eq('is_active', true)
       .lt('valid_from', 'now()')
       .or(`valid_until.is.null,valid_until.gt.now()`)
       .maybeSingle();
     
     if (directPermissions && directPermissions[`can_${permissionType}`]) return true;
     
     // التحقق من صلاحيات الدور
     if (adminData.role_id) {
       const { data: rolePermissions } = await supabaseAdmin
         .from('wallet_permissions')
         .select(`can_${permissionType}`)
         .eq('role_id', adminData.role_id)
         .eq('wallet_id', walletId)
         .eq('wallet_type', walletType)
         .eq('is_active', true)
         .lt('valid_from', 'now()')
         .or(`valid_until.is.null,valid_until.gt.now()`)
         .maybeSingle();
       
       if (rolePermissions && rolePermissions[`can_${permissionType}`]) return true;
     }
     
     return false;
   }
   ```
