# دليل توثيق نظام إدارة المدفوعات والمحافظ المالية

هذا الدليل يجمع جميع وثائق نظام إدارة المدفوعات والمحافظ المالية في تطبيق لوحة تحكم مسؤول التوصيل.

## قائمة المحتويات

1. **[التوثيق التفصيلي المقترح لنظام إدارة المدفوعات والمحافظ المالية](./\#%20توثيق%20تفصيلي%20مقترح%20لنظام%20إدارة%20المدفوعات%20والمحافظ%20المالية.md)**
   - توثيق شامل يتضمن تصميم قاعدة البيانات، الجداول، الأعمدة، الأنواع المخصصة، الدوال، وسير العمل المقترح
   - يحتوي على تفاصيل الجداول: المحافظ، معاملات المحفظة، طرق الدفع، وطلبات السحب
   - يشمل سياسات الأمان وإدارة الصلاحيات (RLS)

2. **[توثيق إضافة صلاحية إدارة المحافظ المالية](./إضافة%20صلاحية%20إدارة%20المحافظ.md)**
   - يشرح عملية إضافة صلاحية `wallets:manage` إلى نظام الصلاحيات
   - يوثق خطوات إضافة السجل إلى جدول `permissions` وربطه بالأدوار المناسبة
   - يوضح كيفية تكامل الصلاحية مع دوال التحقق المستخدمة في سياسات RLS

3. **[توثيق تنفيذ واجهة وخدمات المدفوعات](./توثيق%20تنفيذ%20واجهة%20وخدمات%20المدفوعات.md)**
   - يصف تنفيذ واجهة المستخدم ومكونات الـ React لصفحات المدفوعات
   - يشرح تنفيذ حالة Redux وخدمات API للمدفوعات
   - يوثق المشكلات التي واجهتنا وكيفية حلها (مثل تطابق القيم وأذونات الوصول)

## نظرة عامة على النظام

### المكونات الرئيسية

1. **الجداول في قاعدة البيانات**:
   - `wallets`: محافظ المستخدمين لإدارة الأرصدة المالية
   - `wallet_transactions`: سجل المعاملات المالية للمحافظ
   - `payment_methods`: طرق الدفع المتاحة في النظام
   - `user_payment_methods`: طرق الدفع المضافة لكل مستخدم
   - `payouts`: طلبات سحب الأموال من المحافظ

2. **نظام الصلاحيات**:
   - صلاحية `wallets:manage` للوصول إلى المحافظ وإدارتها
   - تكامل مع نظام الأدوار والصلاحيات الموجود
   - دالة `check_wallet_permission()` للتحقق من الصلاحيات في سياسات RLS

3. **واجهة المستخدم**:
   - صفحة رئيسية للمدفوعات مع علامات تبويب متعددة
   - مكونات لعرض الإحصائيات، الرسوم البيانية، وجداول المعاملات
   - تعريب كامل للواجهة وتنسيق RTL

4. **خدمات API**:
   - `paymentsService.ts` للاتصال المباشر بـ Supabase
   - استعلامات للإحصائيات، اتجاهات المعاملات، والمعاملات الأخيرة
   - دوال ترجمة لتحويل القيم المخزنة بالإنجليزية إلى العربية للعرض

### سير العمل

1. **عرض البيانات**:
   - المستخدم يدخل إلى صفحة المدفوعات من الشريط الجانبي
   - يتم التحقق من صلاحية `wallets:manage`
   - تقوم المكونات المختلفة باستدعاء خدمات API وعرض البيانات

2. **إدارة المدفوعات**:
   - يمكن للمسؤول عرض الإحصائيات وتتبع المعاملات واتجاهاتها
   - يمكنه إدارة المحافظ والموافقة على طلبات السحب
   - يتم تنفيذ جميع العمليات مع مراعاة سياسات الأمان
