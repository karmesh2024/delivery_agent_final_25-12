# توثيق إضافة صلاحية إدارة المحافظ المالية

## مقدمة

يوثق هذا الملف عملية إضافة صلاحية "إدارة المحافظ" (`wallets:manage`) إلى نظام الصلاحيات في تطبيق لوحة تحكم مسؤول التوصيل. تم إضافة هذه الصلاحية للسماح للمشرفين بإدارة المحافظ المالية وعرض المعاملات والتحكم في طلبات السحب.

## نظرة عامة على التغييرات

1. إضافة صلاحية `wallets:manage` إلى جدول `permissions` الرسمي في النظام
2. ربط الصلاحية بالأدوار الإدارية المناسبة
3. تكامل الصلاحية الجديدة مع دالة التحقق `check_wallet_permission()`

## خطوات التنفيذ

### 1. إضافة صلاحية جديدة إلى جدول الصلاحيات

تمت إضافة سجل جديد في جدول `permissions` يمثل صلاحية إدارة المحافظ:

```sql
INSERT INTO permissions (id, name, code, description, resource_id, action_id, created_at, updated_at)
VALUES 
(uuid_generate_v4(), 'إدارة المحافظ', 'wallets:manage', 'إدارة المحافظ المالية وتتبع المعاملات', 
 '00000000-0000-0000-0000-000000000009', '00000000-0000-0000-0000-000000000007', now(), now());
```

حيث:
- **اسم الصلاحية**: "إدارة المحافظ"
- **رمز الصلاحية**: "wallets:manage"
- **وصف الصلاحية**: "إدارة المحافظ المالية وتتبع المعاملات"
- **المورد (resource_id)**: المالية (finance) برمز '00000000-0000-0000-0000-000000000009'
- **الإجراء (action_id)**: إدارة (manage) برمز '00000000-0000-0000-0000-000000000007'

### 2. ربط الصلاحية بالأدوار الإدارية

تم ربط صلاحية `wallets:manage` بالأدوار الإدارية التالية:

```sql
INSERT INTO role_permissions (id, role_id, permission_id, created_at)
VALUES 
(uuid_generate_v4(), 'b98dc2ff-d97d-4f5a-a7bb-c9b864268c4c', '3016f1f2-617c-40bb-878d-813808c6833a', now()),
(uuid_generate_v4(), '6786b241-8c2f-49b5-9947-6377fb5348fe', '3016f1f2-617c-40bb-878d-813808c6833a', now());
```

الأدوار التي تم ربط الصلاحية بها:
- **مدير النظام** (super_admin) - معرف الدور: 'b98dc2ff-d97d-4f5a-a7bb-c9b864268c4c'
- **رئيس مجلس الإدارة** (admin) - معرف الدور: '6786b241-8c2f-49b5-9947-6377fb5348fe'

### 3. التكامل مع دالة التحقق من صلاحيات المحافظ

تم التحقق من دالة `check_wallet_permission()` المستخدمة في سياسات الأمان (RLS):

```sql
CREATE OR REPLACE FUNCTION public.check_wallet_permission()
RETURNS BOOLEAN
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
BEGIN
  RETURN EXISTS (
    SELECT 1 
    FROM admins a
    WHERE a.user_id = auth.uid()
    AND a.is_active = true
    AND (
      (a.permissions->>'wallets:manage')::boolean = true
      OR check_admin_permission_enhanced(a.id, 'wallets:manage')
    )
  );
END;
$$;
```

هذه الدالة تتحقق من صلاحيات المستخدم بطريقتين:
1. **مباشرة من حقل `permissions`**: حيث يتم تخزين الصلاحيات كـ JSONB في جدول `admins`
2. **من خلال نظام الأدوار**: عبر استدعاء دالة `check_admin_permission_enhanced()` التي تتحقق من:
   - الصلاحيات المخصصة للمستخدم
   - الصلاحيات المتعلقة بالدور المعين للمستخدم
   - الصلاحيات المتعلقة بمجموعات المستخدم
   - الصلاحيات المتوارثة في التسلسل الهرمي

## نتائج التغييرات

بعد تطبيق هذه التغييرات:
1. أصبحت صلاحية `wallets:manage` جزءاً من نظام الصلاحيات الرسمي في التطبيق
2. يمكن للأدوار الإدارية الوصول إلى صفحة الدفعات ومعاملات المحافظ المالية
3. يمكن إدارة هذه الصلاحية بنفس الطريقة التي تدار بها بقية صلاحيات النظام

## الاختبار والتحقق

تم التحقق من صحة التغييرات بالطرق التالية:
1. **تسجيل الدخول كمدير نظام**: التأكد من إمكانية الوصول إلى صفحة المدفوعات
2. **مراجعة القيم المسترجعة**: التأكد من أن البيانات تظهر بشكل صحيح في واجهة المستخدم
3. **التحقق من الأذونات**: مراجعة قيم `permissions` المخزنة في جدول `admins` لمستخدم المسؤول

## الخطوات المستقبلية

1. **إضافة واجهة مستخدم لإدارة الصلاحيات**: تطوير واجهة تتيح للمسؤول الرئيسي منح/إلغاء صلاحية `wallets:manage` للمسؤولين الآخرين
2. **توسيع نطاق الصلاحيات**: إضافة صلاحيات أكثر تحديداً مثل `wallets:view`، `payouts:approve` لتحكم أدق
3. **تعزيز التسجيل والتدقيق**: تسجيل جميع العمليات المتعلقة بإدارة المحافظ والصلاحيات
