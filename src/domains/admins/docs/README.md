## المكونات الرئيسية

### 1. نظام المصادقة

- **تسجيل المستخدمين**: آلية تسجيل مستخدمين جدد مع ربط حساباتهم بجدول المسؤولين
- **تسجيل الدخول**: آلية تسجيل دخول المستخدمين والتحقق من صلاحياتهم
- **إدارة الجلسات**: آلية إدارة جلسات المستخدمين وتجديدها

### 2. نظام الصلاحيات

- **الأدوار**: نظام الأدوار المختلفة (مدير، مشرف، مشاهد، إلخ)
- **الصلاحيات**: صلاحيات محددة لكل وظيفة في النظام
- **التحقق من الصلاحيات**: آلية التحقق من صلاحيات المستخدم للقيام بعمليات معينة

### 3. قاعدة البيانات

- **جداول المصادقة**: auth.users, admins, roles, permissions
- **محفزات**: لربط نظام Supabase Auth بجدول المسؤولين
- **RLS**: سياسات أمان على مستوى الصف لحماية البيانات

## مخطط سريع لنظام المصادقة والصلاحيات

```
1. المستخدم يقوم بالتسجيل أو تسجيل الدخول
2. نظام Supabase Auth يتحقق من بيانات الاعتماد
3. يتم البحث عن المستخدم في جدول admins
4. يتم جلب دور المستخدم وصلاحياته
5. تُخزن بيانات المستخدم في Redux Store
6. مكونات AuthProvider و PermissionGuard تتحقق من المصادقة والصلاحيات
7. يتم عرض المحتوى المناسب للمستخدم حسب صلاحياته
```

## للمساهمة في التوثيق

عند إجراء تغييرات على نظام المصادقة أو الصلاحيات، يرجى تحديث الوثائق المناسبة للحفاظ على توثيق دقيق ومحدث.
