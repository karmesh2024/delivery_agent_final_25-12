# دليل إعداد قاعدة البيانات وحل مشاكل التسجيل

هذا الدليل يشرح كيفية إعداد قاعدة البيانات بشكل صحيح لنظام المسؤولين والمصادقة، ويتناول المشاكل الشائعة التي قد تواجهها، وكيفية حلها.

## التحضير المسبق لقاعدة البيانات

قبل بدء استخدام نظام تسجيل المسؤولين، يجب إعداد قاعدة البيانات بالشكل الصحيح:

### 1. إنشاء الجداول الأساسية

تأكد من إنشاء الجداول التالية:

```sql
-- جدول المسؤولين
CREATE TABLE IF NOT EXISTS public.admins (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  email text NOT NULL,
  username text,
  full_name text,
  is_active boolean DEFAULT true,
  role_id uuid,
  created_at timestamptz DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamptz DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT admins_pkey PRIMARY KEY (id),
  CONSTRAINT admins_email_key UNIQUE (email)
);

-- جدول الأدوار
CREATE TABLE IF NOT EXISTS public.roles (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  name text NOT NULL,
  code text NOT NULL,
  description text,
  created_at timestamptz DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamptz DEFAULT CURRENT_TIMESTAMP,
  CONSTRAINT roles_pkey PRIMARY KEY (id),
  CONSTRAINT roles_code_key UNIQUE (code)
);

-- إضافة العلاقة بين المسؤولين والأدوار
ALTER TABLE public.admins
  ADD CONSTRAINT admins_role_id_fkey FOREIGN KEY (role_id) REFERENCES public.roles(id);
```

### 2. إنشاء الأدوار الأساسية

أدخل بعض الأدوار الأساسية في النظام:

```sql
-- إدراج الأدوار الأساسية
INSERT INTO public.roles (name, code, description)
VALUES
  ('مدير النظام', 'super_admin', 'صلاحيات كاملة على جميع أجزاء النظام'),
  ('مسؤول', 'admin', 'صلاحيات واسعة مع بعض القيود'),
  ('مدير', 'manager', 'إدارة العمليات اليومية'),
  ('مشرف', 'supervisor', 'الإشراف على مجموعة أو قسم'),
  ('دعم فني', 'support', 'معالجة استفسارات وطلبات المستخدمين'),
  ('مشاهد', 'viewer', 'وصول للقراءة فقط')
ON CONFLICT (code) DO NOTHING;
```

### 3. تنفيذ محفزات المصادقة

قم بتنفيذ ملف `admin-auth-triggers.sql` لإعداد المحفزات التي تربط نظام Supabase للمصادقة مع جدول المسؤولين.

## حل مشاكل التسجيل

### مشكلة: "Database error saving new user"

إذا واجهتك هذه المشكلة عند التسجيل، فيمكن أن يكون سببها أحد الأمور التالية:

#### 1. الجداول غير موجودة أو غير مكتملة

**الحل:**
- تأكد من إنشاء جدول `admins` وجدول `roles` قبل تسجيل أي مستخدم.
- تحقق من هيكل الجداول ومطابقته للتصميم المطلوب.

#### 2. محفزات المصادقة لم يتم تنفيذها بشكل صحيح

**الحل:**
- أعد تنفيذ ملف `admin-auth-triggers.sql`.
- تأكد من عدم وجود أخطاء عند تنفيذ المحفزات.

#### 3. مشكلة في المعرف الخارجي للدور (role_id)

**الحل:**
- تأكد من وجود دور بكود 'viewer' في جدول `roles`.
- يمكنك تنفيذ الاستعلام التالي لإضافة الدور إذا لم يكن موجودًا:

```sql
INSERT INTO public.roles (name, code, description)
VALUES ('مشاهد', 'viewer', 'وصول للقراءة فقط')
ON CONFLICT (code) DO NOTHING;
```

#### 4. اختلاف في بنية الجدول

**الحل:**
قد تكون بنية جدول `admins` لديك مختلفة عن البنية المتوقعة في المحفز. يمكنك تعديل ملف `admin-auth-triggers.sql` ليتوافق مع بنية الجدول الخاصة بك.

### مشكلة: المستخدم مسجل في Supabase ولكن لا يظهر في admins

**الحل:**
- تحقق من وجود تسجيل خطأ في لوحة تحكم Supabase.
- يمكنك إضافة المستخدم يدويًا إلى جدول المسؤولين:

```sql
INSERT INTO public.admins (user_id, email, is_active)
SELECT id, email, true
FROM auth.users
WHERE email = 'البريد_الإلكتروني_للمستخدم';
```

## التحسينات المطبقة في المحفزات

لقد قمنا بتحسين المحفزات لتكون أكثر مرونة ولتجاوز الأخطاء الشائعة:

1. **التحقق من وجود الجداول المطلوبة**:
   - يتحقق المحفز من وجود جدول `admins` و `roles` قبل محاولة الإدراج.
   - إذا لم تكن الجداول موجودة، فلن يحاول الإدراج وسيسمح للتسجيل بالاستمرار.

2. **معالجة الأخطاء**:
   - تم إضافة معالجة استثناءات متعددة للتعامل مع أخطاء قاعدة البيانات.
   - استخدام نماذج إدراج مبسطة كخطة بديلة في حال فشل النموذج الكامل.

3. **حماية لعملية التسجيل**:
   - عملية التسجيل في Supabase (auth.users) ستنجح حتى لو فشلت إضافة المستخدم إلى `admins`.
   - يمكن إضافة المستخدم إلى `admins` يدويًا لاحقًا.

## الخطوات المقترحة عند مواجهة مشاكل

1. تأكد من إعداد قاعدة البيانات بشكل صحيح باستخدام النصوص SQL المذكورة أعلاه.
2. نفذ ملف `admin-auth-triggers.sql` المحسّن الذي يتعامل مع الأخطاء المحتملة.
3. تحقق من وجود الدور الافتراضي 'viewer' في جدول `roles`.
4. إذا استمرت المشكلة، يمكنك استخدام الطريقة اليدوية لإضافة المستخدم إلى جدول `admins`.

## الخطوات المستقبلية لتحسين الثبات

- إضافة نظام تسجيل الأخطاء للمحفزات لتسهيل تشخيص المشاكل.
- تطوير واجهة إدارة لتدبير المستخدمين الذين لم يتم إضافتهم بنجاح إلى جدول `admins`.
- إنشاء عملية تسجيل منتظمة للتأكد من تزامن auth.users مع جدول `admins`.
