# نظام المصادقة والصلاحيات المتكامل

## نظرة عامة

يوفر نظام المصادقة والصلاحيات المتكامل إطارًا شاملًا لإدارة المستخدمين المسؤولين وصلاحياتهم في لوحة التحكم. تم تصميم النظام ليكون:

- **متكامل مع Supabase**: يستفيد من نظام المصادقة الأساسي الذي توفره Supabase
- **قابل للتوسعة**: يدعم إضافة أدوار وصلاحيات جديدة بسهولة
- **مرن**: يسمح بتخصيص الصلاحيات لكل مسؤول بشكل فردي
- **آمن من حيث التنميط**: استخدام TypeScript لتوفير أمان النوع والاكتشاف المبكر للأخطاء
- **متوافق مع DDD والريدوكس**: متكامل مع هيكلة المشروع الحالية

## هيكل قاعدة البيانات

### الجداول الأساسية

#### جدول المسؤولين (admins)

```sql
CREATE TABLE public.admins (
  id uuid NOT NULL DEFAULT extensions.uuid_generate_v4(),
  user_id uuid NULL,
  department_id uuid NULL,
  role_id uuid NULL,
  manager_id uuid NULL,
  email text NOT NULL,
  username text NULL,
  full_name text NULL,
  phone text NULL,
  job_title text NULL,
  profile_image_url text NULL,
  permissions jsonb NULL,
  is_active boolean NOT NULL DEFAULT true,
  last_login timestamp with time zone NULL,
  created_at timestamp with time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at timestamp with time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT admins_pkey PRIMARY KEY (id),
  CONSTRAINT admins_email_unique UNIQUE (email),
  CONSTRAINT admins_username_unique UNIQUE (username),
  CONSTRAINT admins_manager_id_fkey FOREIGN KEY (manager_id) REFERENCES admins (id),
  CONSTRAINT admins_department_id_fkey FOREIGN KEY (department_id) REFERENCES departments (id),
  CONSTRAINT admins_role_id_fkey FOREIGN KEY (role_id) REFERENCES roles (id),
  CONSTRAINT admins_user_id_fkey FOREIGN KEY (user_id) REFERENCES auth.users (id) ON DELETE CASCADE
);
```

حقل `permissions` من نوع JSONB يخزن الصلاحيات المخصصة للمسؤول.

### المحفزات لربط المستخدمين بالمسؤولين

تم تنفيذ محفزات في قاعدة البيانات لضمان إنشاء سجل مسؤول تلقائيًا عند إنشاء مستخدم جديد في Supabase:

```sql
CREATE OR REPLACE FUNCTION public.process_auth_user_created()
RETURNS TRIGGER AS $$
BEGIN
  -- إدراج المستخدم في جدول المسؤولين
  INSERT INTO public.admins (user_id, email, username, full_name, is_active, permissions)
  VALUES (
    NEW.id, 
    NEW.email, 
    SPLIT_PART(NEW.email, '@', 1), 
    COALESCE(NEW.raw_user_meta_data->>'full_name', 'مستخدم جديد'), 
    true,
    json_build_object(
      'dashboard:view', true,
      'orders:view', true
    )
  );
  
  RETURN NEW;
EXCEPTION
  WHEN OTHERS THEN
    RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- إنشاء المحفز لتشغيل الإجراء عند إنشاء مستخدم جديد
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.process_auth_user_created();
```

هناك أيضًا محفز آخر لتحديث بيانات المسؤول عند تحديث بيانات المستخدم.

## نظام إدارة الصلاحيات

### هيكل البيانات

تم تصميم هيكل الصلاحيات باستخدام النهج التالي:

1. **الصلاحيات العامة**: تخزن كزوج مفتاح-قيمة (مثلًا `{ "orders:view": true }`)
2. **صلاحيات النطاق**: تسمح بتقييد الصلاحيات لكيانات محددة (مثلًا السماح بإدارة مندوبين في منطقة معينة فقط)

مثال لهيكل البيانات المخزن في حقل `permissions`:

```json
{
  "dashboard:view": true,
  "orders:view": true,
  "orders:create": false,
  "scoped_permissions": {
    "region": {
      "agents:manage": ["region1", "region2"]
    },
    "branch": {
      "reports:view": ["branch1"]
    }
  }
}
```

### أنواع TypeScript

تم تعريف الأنواع التالية لدعم نظام الصلاحيات:

```typescript
// نوع الصلاحيات البسيطة
export type AdminPermissions = Record<string, boolean>;

// نوع صلاحيات النطاق
export interface ScopedPermissions {
  [scopeType: string]: {
    [permissionCode: string]: string[];
  };
}

// نوع صلاحيات المسؤول الموسعة
export interface ExtendedPermissions {
  permissions: AdminPermissions;
  scoped_permissions?: ScopedPermissions;
}
```

### الدوال المساعدة

تم توفير مجموعة من الدوال المساعدة للتعامل مع نظام الصلاحيات:

```typescript
// إضافة/تعديل صلاحية
function addPermission(
  extendedPermissions: ExtendedPermissions,
  permissionCode: string,
  value: boolean = true
): ExtendedPermissions;

// إضافة/تعديل صلاحية نطاق
function addScopedPermission(
  extendedPermissions: ExtendedPermissions,
  permissionCode: string,
  scopeType: string,
  scopeValue: string
): ExtendedPermissions;

// التحقق من وجود صلاحية
function hasPermission(
  extendedPermissions: ExtendedPermissions,
  permissionCode: string
): boolean;

// التحقق من وجود صلاحية نطاق
function hasPermissionWithScope(
  extendedPermissions: ExtendedPermissions,
  permissionCode: string,
  scopeType: string,
  scopeValue: string
): boolean;

// تحويل من JSONB الخام إلى ExtendedPermissions
function fromRawJsonbPermissions(
  rawPermissions: RawPermissionsData | null | undefined
): ExtendedPermissions;
```

## دمج النظام مع Redux

تم تنفيذ شريحة Redux (`integratedAuthSlice.ts`) للتعامل مع حالة المصادقة والصلاحيات:

### الأحداث غير المتزامنة (Thunks)

- `login`: تسجيل الدخول للمسؤولين
- `register`: تسجيل مسؤول جديد
- `logout`: تسجيل الخروج
- `checkAuth`: التحقق من حالة المصادقة
- `resetPassword`: طلب إعادة تعيين كلمة المرور
- `updatePassword`: تحديث كلمة المرور
- `checkPermission`: التحقق من وجود صلاحية محددة

### الإجراءات

- `clearError`: مسح الأخطاء
- `clearResetEmailSent`: إعادة تعيين حالة إرسال البريد الإلكتروني
- `setCurrentAdmin`: تعيين المسؤول الحالي
- `updateAdminPermissions`: تحديث صلاحيات المسؤول

## كيفية استخدام النظام

### 1. التحقق من المصادقة

```tsx
import { useEffect } from 'react';
import { useAppDispatch, useAppSelector } from '@/store/hooks';
import { checkAuth } from '@/domains/admins/store/integratedAuthSlice';

function MyComponent() {
  const dispatch = useAppDispatch();
  const { isAuthenticated, currentAdmin, loading } = useAppSelector(state => state.auth);
  
  useEffect(() => {
    dispatch(checkAuth());
  }, [dispatch]);
  
  if (loading) {
    return <div>جاري التحميل...</div>;
  }
  
  if (!isAuthenticated || !currentAdmin) {
    return <div>يرجى تسجيل الدخول</div>;
  }
  
  return <div>مرحبًا {currentAdmin.full_name || currentAdmin.username}!</div>;
}
```

### 2. التحقق من الصلاحيات

استخدام هوك `usePermission` (الذي يجب تحديثه ليستخدم النظام الجديد):

```tsx
import { usePermission } from '@/domains/admins/hooks/usePermission';

function OrderActions({ orderId }) {
  const { hasAccess: canViewOrders } = usePermission('orders:view');
  const { hasAccess: canEditOrders } = usePermission('orders:edit');
  
  // التحقق من صلاحية محددة بنطاق
  const { hasAccess: canManageAgentInRegion } = usePermission('agents:manage', {
    scope: { type: 'region', value: 'region1' }
  });
  
  return (
    <div>
      {canViewOrders && <ViewOrderButton orderId={orderId} />}
      {canEditOrders && <EditOrderButton orderId={orderId} />}
      {canManageAgentInRegion && <AssignAgentButton orderId={orderId} />}
    </div>
  );
}
```

### 3. تحديث صلاحيات المسؤول

```tsx
import { useAppDispatch } from '@/store/hooks';
import { updateAdminPermissions } from '@/domains/admins/store/integratedAuthSlice';

function PermissionsManager({ adminId }) {
  const dispatch = useAppDispatch();
  
  const handleUpdatePermissions = () => {
    dispatch(updateAdminPermissions({
      permissions: {
        'orders:view': true,
        'orders:edit': true
      },
      scopedPermissions: {
        'region': {
          'agents:manage': ['region1', 'region2']
        }
      }
    }));
  };
  
  return (
    <button onClick={handleUpdatePermissions}>
      تحديث الصلاحيات
    </button>
  );
}
```

## توسيع النظام في المستقبل

### إضافة أدوار جديدة

يدعم النظام الحالي استخدام جدول `roles` لتعريف الأدوار، ويمكن توسيعه لتحميل الصلاحيات الافتراضية من الدور:

```sql
CREATE TABLE public.roles (
  id uuid PRIMARY KEY DEFAULT extensions.uuid_generate_v4(),
  name text NOT NULL,
  code text UNIQUE NOT NULL,
  description text,
  permissions jsonb,
  level integer
);
```

### ربط الصلاحيات بالأدوار

يمكن تحديث ملف `integratedAuthSlice.ts` لجلب الصلاحيات من الدور والدمج مع الصلاحيات المخصصة للمسؤول.

### إضافة نطاقات جديدة للصلاحيات

النظام مصمم ليدعم أي نوع جديد من النطاقات، مثل:
- `department`: لتقييد الصلاحيات لأقسام معينة
- `customer`: لتقييد الصلاحيات لعملاء معينين
- `project`: لتقييد الصلاحيات لمشاريع معينة
- أي نطاق آخر قد تحتاجه في المستقبل

## ملخص

يوفر نظام المصادقة والصلاحيات المتكامل الجديد:

1. **تسجيل وإدارة المستخدمين**: باستخدام Supabase مع التكامل مع جدول المسؤولين
2. **نظام صلاحيات مرن**: يدعم الصلاحيات العامة وصلاحيات النطاق
3. **توافق مع Redis والـ DDD**: متكامل مع هيكلية المشروع
4. **قابلية التوسعة**: سهولة إضافة أدوار وصلاحيات وأنواع نطاقات جديدة
5. **أمان النوع**: استخدام TypeScript لتوفير أمان النوع والاكتشاف المبكر للأخطاء

هذا النظام يمثل أساسًا متينًا يمكن البناء عليه وتطويره مع نمو احتياجات التطبيق.
