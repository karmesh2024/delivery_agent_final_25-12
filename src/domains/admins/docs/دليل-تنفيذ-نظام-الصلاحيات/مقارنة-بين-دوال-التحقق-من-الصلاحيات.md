# مقارنة بين دوال التحقق من الصلاحيات في نظام إدارة المسؤولين

## الدالتان الرئيسيتان للتحقق من الصلاحيات

يتضمن نظام إدارة المسؤولين والصلاحيات دالتين رئيسيتين للتحقق من الصلاحيات:

1. `check_admin_permission` - الدالة الأساسية
2. `check_admin_permission_enhanced` - النسخة المحسنة من الدالة

## الفروق الرئيسية بين الدالتين

| المميزات                      | check_admin_permission (الدالة الأساسية) | check_admin_permission_enhanced (الدالة المحسنة) |
|-------------------------------|------------------------------------------|---------------------------------------------------|
| التحقق من صلاحيات الدور       | ✅                                        | ✅                                                |
| التحقق من صلاحيات المجموعة    | ✅                                        | ✅                                                |
| التحقق من الصلاحيات المخصصة   | ✅                                        | ✅                                                |
| دعم التسلسل الهرمي للصلاحيات  | ❌                                        | ✅                                                |
| عمق هرمي قابل للتكوين         | ❌                                        | ✅                                                |
| تسجيل محاولات الوصول للتدقيق  | ❌                                        | ✅                                                |
| سهولة الاستخدام                | عالية                                     | عالية                                            |
| الأداء مع الصلاحيات البسيطة   | أفضل (استعلامات أقل)                      | جيد                                              |
| الأداء مع هياكل تنظيمية معقدة | منخفض (لا يستفيد من التسلسل الهرمي)       | أفضل (يستفيد من التسلسل الهرمي)                   |

## هل يمكن تنفيذ الدالتين معًا؟

**نعم**، يمكن تنفيذ الدالتين معًا بدون تعارض، لأنهما دالتان منفصلتان بأسماء مختلفة. كل دالة منهما مستقلة عن الأخرى وتستخدم وفق الحاجة.

## مثال لتنفيذ الدالتين معًا

عند تنفيذ الملفات SQL لنظام الصلاحيات بالترتيب التالي:

1. `admin-system-schema.sql` - يحتوي على `check_admin_permission`
2. `admin-system-enhancements.sql` - يحتوي على `check_admin_permission_enhanced`
3. `admin-system-refinements.sql` - تحسينات إضافية
4. `admin-system-fixes.sql` - إصلاحات خاصة

سيتم إنشاء كلتا الدالتين في قاعدة البيانات ويمكن استخدامهما بدون تعارض.

## متى تستخدم كل دالة؟

### استخدم `check_admin_permission` عندما:

- تحتاج إلى نظام صلاحيات بسيط
- تتعامل مع هيكل تنظيمي مسطح (بدون تسلسل هرمي معقد)
- تريد الحفاظ على توافق قديم مع تطبيقات موجودة تستخدم هذه الدالة
- تريد استعلامات أبسط وأداء أفضل للحالات البسيطة

### استخدم `check_admin_permission_enhanced` عندما:

- تحتاج إلى الاستفادة من التسلسل الهرمي للصلاحيات (وراثة الصلاحيات من المدير المباشر)
- تريد التحكم في عمق التسلسل الهرمي عبر الإعدادات
- تحتاج إلى تسجيل محاولات الوصول للتدقيق
- تتعامل مع هيكل تنظيمي معقد متعدد المستويات

## مثال عملي لاستخدام الدالتين

### استخدام الدالة الأساسية:

```sql
-- التحقق من صلاحية معينة للمسؤول
SELECT check_admin_permission(
  'fcb32a2f-7f7c-4a68-8d33-9e89b3c72884', -- معرف المسؤول
  'dashboard:view' -- رمز الصلاحية
);
```

### استخدام الدالة المحسنة:

```sql
-- التحقق من صلاحية معينة للمسؤول مع دعم التسلسل الهرمي
SELECT check_admin_permission_enhanced(
  'fcb32a2f-7f7c-4a68-8d33-9e89b3c72884', -- معرف المسؤول
  'dashboard:view' -- رمز الصلاحية
);
```

## توصيات لتكامل النظام

### 1. استراتيجية التنفيذ

في معظم الحالات، ينصح باستخدام الدالة المحسنة `check_admin_permission_enhanced` لإعداد جديد، لأنها توفر جميع المميزات بما في ذلك التوافق الخلفي. وهي تقوم بجميع وظائف الدالة الأساسية وأكثر.

### 2. استراتيجية التحول التدريجي

إذا كان لديك نظام حالي يعتمد على الدالة الأساسية، يمكنك اتباع استراتيجية تحول تدريجي:

1. تنفيذ كلتا الدالتين
2. استخدام الدالة المحسنة في الميزات الجديدة فقط
3. نقل الميزات القديمة تدريجيًا إلى الدالة المحسنة
4. الاحتفاظ بالدالة الأساسية للتوافق الخلفي

### 3. تنفيذ طبقة تجريد

يمكنك إنشاء طبقة تجريد في التطبيق الخاص بك لتوحيد الاستخدام:

```typescript
// في ملف src/domains/admins/utils/permissions.ts
import { supabase } from '@/lib/supabase';
import { Admin } from '../types';

/**
 * التحقق من امتلاك المسؤول للصلاحية
 * استخدام الدالة المحسنة، مع الرجوع إلى الدالة الأساسية كخيار احتياطي
 */
export async function hasPermission(admin: Admin, permissionCode: string): Promise<boolean> {
  try {
    // محاولة استخدام الدالة المحسنة أولاً
    const { data: enhancedResult, error: enhancedError } = await supabase.rpc(
      'check_admin_permission_enhanced',
      {
        admin_id: admin.id,
        permission_code: permissionCode
      }
    );
    
    if (enhancedError) {
      console.warn('فشل استخدام الدالة المحسنة، الرجوع إلى الدالة الأساسية:', enhancedError);
      
      // الرجوع إلى الدالة الأساسية
      const { data: basicResult } = await supabase.rpc(
        'check_admin_permission',
        {
          admin_id: admin.id,
          permission_code: permissionCode
        }
      );
      
      return !!basicResult;
    }
    
    return !!enhancedResult;
  } catch (error) {
    console.error('خطأ في التحقق من الصلاحية:', error);
    return false;
  }
}
```

## الخلاصة

- يمكن تنفيذ الدالتين معًا بدون تعارض
- الدالة المحسنة `check_admin_permission_enhanced` توفر جميع مميزات الدالة الأساسية وأكثر
- للأنظمة الجديدة، يفضل استخدام الدالة المحسنة فقط
- للأنظمة الموجودة، يمكن استخدام الدالتين معًا مع التحول التدريجي
- طبقة التجريد في التطبيق تسهل إدارة استخدام الدالتين
