## نظرة عامة

نظام الصلاحيات يتكون من عدة طبقات متكاملة تغطي:

1. **طبقة قاعدة البيانات**: تخزين الصلاحيات، الأدوار، ودوال التحقق
2. **طبقة API**: واجهة برمجية للتفاعل مع نظام الصلاحيات
3. **طبقة Redux**: إدارة حالة الصلاحيات في تطبيق الواجهة
4. **طبقة الواجهة**: مكونات للتحقق من الصلاحيات وعرض/إخفاء العناصر بناءً عليها

يدعم النظام أنواعًا متعددة من التحقق من الصلاحيات:
- **الصلاحيات العامة**: مثل `orders:view`، `users:manage`
- **الصلاحيات المحددة بنطاق**: تسمح بتقييد الصلاحيات لكيانات محددة، مثلًا: صلاحية إدارة مندوبين محددين فقط

## بنية النظام

### 1. طبقة قاعدة البيانات

تتكون من ثلاثة جداول رئيسية:

#### جدول المسؤولين (admins)
```sql
create table public.admins (
  id uuid primary key,
  user_id uuid references auth.users(id),
  role_id uuid references public.roles(id),
  email text,
  permissions jsonb,
  is_active boolean default true
  -- حقول أخرى
);
```

#### جدول الأدوار (roles)
```sql
create table public.roles (
  id uuid primary key,
  name text not null,
  code text unique not null,
  description text,
  permissions jsonb,
  level integer
  -- حقول أخرى
);
```

#### جدول الصلاحيات (permissions)
```sql
create table public.permissions (
  id uuid primary key,
  code text unique not null,
  name text not null,
  description text,
  group text
  -- حقول أخرى
);
```

#### تخزين الصلاحيات
الصلاحيات تخزن بتنسيق JSON كالتالي:

```json
{
  "orders:view": true,
  "orders:create": true,
  "orders:delete": false,
  "scoped_permissions": {
    "region": {
      "agents:manage": ["123", "456"]
    },
    "branch": {
      "reports:view": ["B001"]
    }
  }
}
```

### 2. طبقة API

واجهة برمجية موحدة للتفاعل مع نظام الصلاحيات موجودة في ملف `src/domains/admins/api/permissionsApi.ts`، وتوفر الدوال التالية:

- `getAvailablePermissions()`: جلب قائمة الصلاحيات المتاحة
- `getRoles()`: جلب قائمة الأدوار
- `getAdminPermissions(adminId)`: جلب صلاحيات مسؤول معين
- `updateAdminPermissions(adminId, permissions)`: تحديث صلاحيات مسؤول
- `assignRoleToAdmin(adminId, roleId)`: تعيين دور لمسؤول
- `updateScopedPermissions(adminId, scopeType, permissionCode, scopeValues)`: تحديث صلاحيات النطاق
- `hasPermission(adminId, permissionCode)`: التحقق من امتلاك صلاحية عامة
- `hasPermissionWithScope(adminId, permissionCode, scopeType, scopeValue)`: التحقق من صلاحية محددة بنطاق

### 3. طبقة Redux

تم تنفيذ شريحة Redux لإدارة حالة الصلاحيات في ملف `src/domains/admins/store/permissionsSlice.ts`، وتوفر:

- إجراءات async لجلب وتحديث الصلاحيات والأدوار
- أفعال لتعيين المسؤول المحدد ومسح البيانات
- حالة منظمة للصلاحيات والأدوار في التطبيق

### 4. هوكس React المخصصة

تم تنفيذ هوكس React مخصصة في ملف `src/domains/admins/hooks/usePermission.ts` لتسهيل التحقق من الصلاحيات في المكونات:

- `usePermission(permissionCode, options)`: للتحقق من صلاحية واحدة
- `useAnyPermission(permissionCodes, options)`: للتحقق من وجود أي صلاحية من مجموعة
- `useAllPermissions(permissionCodes, options)`: للتحقق من وجود جميع الصلاحيات المطلوبة

### 5. مكونات حماية العناصر والمسارات

- `PermissionGuard`: حماية أي عنصر في واجهة المستخدم (`src/domains/admins/components/PermissionGuard.tsx`)
- `RouteGuard`: حماية المسارات والصفحات (`src/shared/components/RouteGuard.tsx`)

## استخدام النظام في واجهة المستخدم

### 1. حماية عنصر في واجهة المستخدم

استخدم مكون `PermissionGuard` لحماية أي عنصر بناءً على الصلاحيات:

```tsx
import { PermissionGuard } from '@/domains/admins/components/PermissionGuard';

// حماية بسيطة لعنصر
<PermissionGuard permissionCode="orders:view">
  <ViewOrdersButton />
</PermissionGuard>

// حماية مع محتوى بديل
<PermissionGuard 
  permissionCode="orders:edit"
  fallback={<p>ليس لديك صلاحية تعديل الطلبات</p>}
>
  <EditOrderButton />
</PermissionGuard>

// حماية بصلاحية محددة بنطاق
<PermissionGuard 
  permissionCode="orders:edit"
  scope={{ type: 'order', value: orderId }}
>
  <EditSpecificOrderButton />
</PermissionGuard>
```

### 2. حماية صفحة أو مسار

استخدم مكون `RouteGuard` لحماية المسارات:

```tsx
// في ملف layout.tsx
import { RouteGuard } from '@/shared/components/RouteGuard';

export default function AdminsLayout({ children }) {
  return (
    <RouteGuard 
      requiredPermission="admins:view"
      redirectTo="/unauthorized"
    >
      {children}
    </RouteGuard>
  );
}

// حماية بأي من عدة صلاحيات
<RouteGuard 
  requiredPermission={['orders:view', 'orders:manage']}
  requireAll={false}
>
  {children}
</RouteGuard>

// حماية تتطلب جميع الصلاحيات المحددة
<RouteGuard 
  requiredPermission={['admins:view', 'admins:manage']}
  requireAll={true}
>
  {children}
</RouteGuard>
```

## التحقق من الصلاحيات في الكود

### 1. استخدام هوكس React المخصصة

```tsx
import { usePermission, useAnyPermission, useAllPermissions } from '@/domains/admins/hooks/usePermission';

function UserActions() {
  // التحقق من صلاحية واحدة
  const { hasAccess: canViewUsers, loading: isCheckingView } = usePermission('users:view');
  
  // التحقق من أي صلاحية من مجموعة
  const { hasAccess: canManageUsers } = useAnyPermission(['users:create', 'users:update']);
  
  // التحقق من وجود جميع الصلاحيات
  const { hasAccess: isAdmin } = useAllPermissions(['admins:view', 'admins:manage']);
  
  if (isCheckingView) return <Loading />;
  
  return (
    <div>
      {canViewUsers && <UsersList />}
      {canManageUsers && <ManageUsersPanel />}
      {isAdmin && <AdminControls />}
    </div>
  );
}
```

### 2. استخدام دوال مباشرة

```tsx
import { hasPermission, hasPermissionWithScope } from '@/domains/admins/utils/permissions';

// التحقق المباشر من صلاحية عامة
const canDeleteOrders = hasPermission(currentAdmin, 'orders:delete');

// التحقق من صلاحية محددة بنطاق
const canEditOrder = hasPermissionWithScope(
  currentAdmin, 
  'orders:edit', 
  'order', 
  orderId
);
```

## إدارة الصلاحيات

### 1. تعيين صلاحيات مباشرة لمسؤول

استخدم مكون `PermissionManager` لإدارة صلاحيات المسؤولين:

```tsx
import { PermissionManager } from '@/domains/admins/components/PermissionManager';

function AdminPermissionsPage({ params }) {
  const adminId = params.id;
  
  const handleSave = (adminId, permissions) => {
    console.log('تم حفظ الصلاحيات:', permissions);
  };
  
  return (
    <div>
      <h1>إدارة صلاحيات المسؤول</h1>
      <PermissionManager 
        adminId={adminId} 
        onSave={handleSave} 
      />
    </div>
  );
}
```

### 2. تعيين دور لمسؤول

استخدم واجهة API مباشرة أو من خلال Redux:

```tsx
import { assignRoleToAdmin } from '@/domains/admins/store/permissionsSlice';

// باستخدام Redux
dispatch(assignRoleToAdmin({ adminId, roleId }));

// أو باستخدام API مباشرة
import { permissionsApi } from '@/domains/admins/api/permissionsApi';
await permissionsApi.assignRoleToAdmin(adminId, roleId);
```

### 3. إدارة صلاحيات النطاق

```tsx
// باستخدام Redux
dispatch(updateScopedPermissions({
  adminId,
  scopeType: 'region',
  permissionCode: 'agents:manage',
  scopeValues: ['123', '456', '789']
}));

// أو باستخدام API مباشرة
await permissionsApi.updateScopedPermissions(
  adminId,
  'region',
  'agents:manage',
  ['123', '456', '789']
);
```

## التوسعة والتطوير المستقبلي

### 1. إضافة صلاحيات جديدة

1. أضف الصلاحية في جدول `permissions` في قاعدة البيانات
2. استخدم الصلاحية في كود التطبيق

### 2. إضافة نوع جديد من نطاقات الصلاحيات

النظام مصمم ليدعم أنواعًا متعددة من النطاقات، مثل:
- نطاق المنطقة (region)
- نطاق الفرع (branch)
- نطاق المستخدم (user)
- نطاق الطلب (order)
- أي نطاق آخر

لإضافة نوع جديد، ببساطة استخدمه في دالة `hasPermissionWithScope` مع قيم الكيانات.

### 3. توسيع وظائف النظام

يمكن توسيع النظام بسهولة من خلال:
- إضافة دوال تحقق جديدة
- إضافة أنواع من نطاقات الصلاحيات
- تحسين واجهة المستخدم لإدارة الصلاحيات

## أمثلة عملية

### مثال 1: حماية صفحة الطلبات

```tsx
// في ملف app/orders/layout.tsx
import { RouteGuard } from '@/shared/components/RouteGuard';

export default function OrdersLayout({ children }) {
  return (
    <RouteGuard requiredPermission="orders:view">
      <div className="orders-layout">
        {children}
      </div>
    </RouteGuard>
  );
}
```

### مثال 2: عرض/إخفاء أزرار بناءً على الصلاحيات

```tsx
function OrderActions({ order }) {
  const { hasAccess: canEdit } = usePermission('orders:edit');
  const { hasAccess: canDelete } = usePermission('orders:delete');
  
  return (
    <div className="actions">
      <ViewButton order={order} />
      
      {canEdit && (
        <EditButton order={order} />
      )}
      
      {canDelete && (
        <DeleteButton order={order} />
      )}
    </div>
  );
}
```

### مثال 3: صلاحيات محددة بنطاق

```tsx
function AgentActions({ agent }) {
  const { hasAccess: canManage } = usePermission('agents:manage', {
    scope: {
      type: 'region',
      value: agent.regionId
    }
  });
  
  return (
    <div>
      {canManage && (
        <ManageAgentButton agent={agent} />
      )}
    </div>
  );
}
```

## استكشاف الأخطاء وإصلاحها

### 1. الصلاحيات لا تتحديث في واجهة المستخدم

1. تأكد من أن `currentAdmin` متوفر في متجر Redux
2. تأكد من جلب الصلاحيات بشكل صحيح من الخادم
3. تحقق من Redux DevTools للتأكد من حالة `permissions`

### 2. أخطاء في التحقق من الصلاحيات

1. تأكد من تمرير رمز الصلاحية الصحيح (`permissionCode`)
2. تحقق من بيانات الصلاحيات في قاعدة البيانات
3. استخدم خيار `serverCheck: true` للتأكد من التحقق من الخادم

### 3. مشاكل في صلاحيات النطاق

1. تأكد من تنسيق بيانات `scoped_permissions` في الصلاحيات
2. تحقق من تمرير `scopeType` و `scopeValue` بشكل صحيح
3. تأكد من وجود قيمة النطاق في مصفوفة القيم المسموح بها
