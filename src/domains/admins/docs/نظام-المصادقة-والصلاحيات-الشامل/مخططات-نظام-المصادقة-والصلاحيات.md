# مخططات نظام المصادقة والصلاحيات

هذا المستند يقدم مجموعة من المخططات التوضيحية والتصورات البصرية لفهم نظام المصادقة والصلاحيات في تطبيق لوحة تحكم مندوبي التوصيل. هذه المخططات تكمّل التوثيق النصي المفصل الموجود في ملف [نظام-المصادقة-والصلاحيات-الشامل.md](./نظام-المصادقة-والصلاحيات-الشامل.md).

## مخطط العلاقات لقاعدة البيانات (ER Diagram)

فيما يلي مخطط العلاقات بين الجداول الرئيسية في نظام المصادقة والصلاحيات:

```
                         +------------------+
                         |    auth.users    |
                         +------------------+
                         | id (PK)          |
                         | email            |
                         | password (hash)  |
                         | email_confirmed  |
                         | ...              |
                         +--------+---------+
                                  |
                                  | 1:1
                                  |
                                  v
+------------+            +------------------+            +---------------+
|  permissions|<--------- |    admins        | ---------> |    roles      |
+------------+            +------------------+            +---------------+
| id (PK)     |   n:m     | id (PK)          |   n:1      | id (PK)       |
| code        | <-------- | user_id (FK)     | ---------> | name          |
| name        |   via     | email            |            | description   |
| description | permissions| is_active       |            | permissions   |
| group_name  |   field   | role_id (FK)     |            | is_system     |
+------------+            | permissions      |            +---------------+
                          | ...              |
                          +------------------+
```

## مخطط تدفق عملية تسجيل الدخول

```
+-------------+    +----------------+    +------------------+    +------------------------+
| المستخدم    |    | نموذج تسجيل   |    | authApi.login()  |    | Supabase Auth         |
| يدخل بيانات |-->| الدخول        |-->| إرسال بيانات     |-->| المصادقة             |
| الاعتماد    |    |                |    | الاعتماد        |    | signInWithPassword()  |
+-------------+    +----------------+    +------------------+    +------------------------+
                                                |
                                                v
+------------------+    +---------------------+    +-------------------------+
| Redux Store      |    | قاعدة البيانات     |    | التحقق من المسؤول      |
| تحديث حالة      |<---| البحث في جدول      |<---| في جدول admins         |
| المصادقة        |    | admins              |    | وأنه نشط              |
+------------------+    +---------------------+    +-------------------------+
        |
        v
+------------------------+    +----------------------+
| توجيه المستخدم        |    | تحميل بيانات        |
| إلى الصفحة الرئيسية   |--->| المسؤول والصلاحيات |
| أو المقصودة           |    |                      |
+------------------------+    +----------------------+
```

## مخطط تدفق عملية التسجيل

```
+-------------+    +----------------+    +------------------+    +------------------------+
| المستخدم    |    | نموذج         |    | authApi.register |    | Supabase Auth         |
| يدخل بيانات |-->| التسجيل       |-->| إرسال بيانات     |-->| إنشاء مستخدم جديد     |
| التسجيل     |    |                |    | التسجيل         |    | signUp()              |
+-------------+    +----------------+    +------------------+    +------------------------+
                                                |
                                                v
+------------------------+    +---------------------+    +-------------------------+
| إرسال بريد            |    | تشغيل محفز         |    | إنشاء سجل في جدول      |
| تأكيد (اختياري)      |<---| on_auth_user_create |<---| admins مع دور افتراضي  |
|                        |    | في قاعدة البيانات  |    | (عادة "viewer")        |
+------------------------+    +---------------------+    +-------------------------+
        |
        v
+------------------------+    +----------------------+
| توجيه المستخدم        |    | عودة ردود إيجابية    |
| إلى صفحة              |--->| أو سلبية            |
| التأكيد أو الدخول     |    | مع أي أخطاء         |
+------------------------+    +----------------------+
```

## مخطط آلية التحقق من الصلاحيات

```
+------------------+    +----------------------+    +------------------------+
| مكون التطبيق     |    | PermissionGuard     |    | hasPermission()       |
| أو الصفحة        |-->| مع رمز الصلاحية     |-->| التحقق من الصلاحية    |
|                  |    | المطلوبة           |    | في القاعدة أو الذاكرة |
+------------------+    +----------------------+    +------------------------+
                                                            |
                                                            v
+------------------+    +----------------------+    +------------------------+
| عرض المحتوى      |    | الرد بـ true        |    | فحص الصلاحيات:        |
| أو الطبقة البديلة|<---| أو false            |<---| 1. الصلاحيات المخصصة  |
| (fallback)       |    |                      |    | 2. صلاحيات الدور      |
+------------------+    +----------------------+    +------------------------+
```

## ملاحظات استخدام المخططات

1. **إضافة المخططات للتوثيق العام**: يمكن استخدام هذه المخططات في صفحات التوثيق العامة للمطورين
2. **تحويل المخططات إلى صور**: يمكن استخدام أدوات مثل Mermaid أو PlantUML لتحويل هذه المخططات النصية إلى صور
3. **تحديث المخططات**: يجب تحديث هذه المخططات كلما تم إجراء تغييرات جوهرية على نظام المصادقة أو الصلاحيات

## أدوات مقترحة لإنشاء المخططات

- [Mermaid](https://mermaid-js.github.io/mermaid/#/) - أداة لإنشاء المخططات من النص
- [PlantUML](https://plantuml.com/) - أداة متقدمة لإنشاء مخططات UML من النص
- [draw.io](https://app.diagrams.net/) - أداة رسم مخططات تفاعلية

يمكن تحويل المخططات النصية الموجودة في هذا المستند إلى مخططات رسومية باستخدام هذه الأدوات لتحسين قابلية قراءة وفهم نظام المصادقة والصلاحيات.
