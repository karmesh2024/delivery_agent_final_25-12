# دليل نظام تسجيل الدخول - سير العمل وآلية الظهور

يشرح هذا المستند كيفية ظهور صفحة تسجيل الدخول وتكاملها مع نظام التسجيل والصلاحيات، بما في ذلك متى وكيف يتم توجيه المستخدم إليها، وسير عمل المصادقة بشكل كامل.

## آلية ظهور صفحة تسجيل الدخول

### 1. عند الزيارة المباشرة

صفحة تسجيل الدخول متاحة عند الرابط:
```
/login
```

يمكن للمستخدم الوصول إليها مباشرة عن طريق كتابة العنوان في متصفح الويب أو النقر على رابط تسجيل الدخول.

### 2. التوجيه التلقائي

يتم توجيه المستخدم تلقائيًا إلى صفحة تسجيل الدخول في الحالات التالية:

1. **عند محاولة الوصول إلى صفحة محمية بدون تسجيل الدخول**:
   - عندما يحاول المستخدم غير المصادق الوصول إلى أي صفحة داخل لوحة التحكم
   - عندما تنتهي صلاحية جلسة المستخدم ويحاول الوصول إلى صفحة محمية

2. **عند تسجيل الخروج**:
   - بعد النقر على زر تسجيل الخروج، يتم توجيه المستخدم تلقائيًا إلى صفحة تسجيل الدخول

### 3. عند الصفحة الرئيسية

يمكن أن تظهر صفحة تسجيل الدخول أيضًا في السيناريو التالي:

- عندما يزور المستخدم الصفحة الرئيسية للتطبيق (`/`) وهو غير مسجل دخوله
- يتم فحص حالة المصادقة وتوجيهه إما إلى لوحة التحكم (إذا كان مصادقًا) أو إلى صفحة تسجيل الدخول

## كيفية عمل نظام توجيه المصادقة

### 1. مكون AuthProvider

مكون `AuthProvider` هو المسؤول الرئيسي عن تنفيذ منطق التوجيه وإدارة حالة المصادقة في التطبيق. يتم وضعه في التخطيط الرئيسي للتطبيق (`app/layout.tsx`) حتى يتم تطبيقه على جميع الصفحات.

```tsx
export default function RootLayout({ children }: RootLayoutProps) {
  return (
    <html lang="ar" dir="rtl" className="h-full">
      <body className="h-full">
        <StoreProvider>
          <ThemeProvider>
            <AuthProvider> 
              {children}
            </AuthProvider>
          </ThemeProvider>
        </StoreProvider>
      </body>
    </html>
  );
}
```

### 2. التحقق من المصادقة عند بدء التطبيق

عندما يتم تحميل التطبيق، يقوم `AuthProvider` بالتالي:

1. استدعاء `checkAuth()` من شريحة المصادقة (redux) للتحقق من وجود جلسة نشطة
2. قراءة حالة المصادقة (`isAuthenticated`) من المتجر
3. فحص المسار الحالي ومقارنته بقائمة المسارات العامة (غير المحمية)
4. اتخاذ قرار بشأن توجيه المستخدم استنادًا إلى الحالة والمسار

### 3. قواعد التوجيه

يتم تنفيذ قواعد التوجيه التالية:

- **إذا كان المستخدم مصادقًا ويزور مسارًا عامًا**:
  توجيهه إلى لوحة التحكم (`/dashboard`)

- **إذا كان المستخدم غير مصادق ويزور مسارًا محميًا**:
  توجيهه إلى صفحة تسجيل الدخول (`/login`) مع حفظ المسار الأصلي للعودة إليه بعد تسجيل الدخول

## تكامل RouteGuard مع AuthProvider

بالإضافة إلى `AuthProvider` العام، نستخدم مكون `RouteGuard` لتأمين المسارات المحددة وإضافة فحص الصلاحيات:

```tsx
// src/app/(dashboard)/layout.tsx
export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <RouteGuard>
      <DashboardLayout>
        {children}
      </DashboardLayout>
    </RouteGuard>
  );
}
```

يعمل مكون `RouteGuard` على النحو التالي:

1. يفحص حالة المصادقة و(اختياريًا) يفحص الصلاحية المحددة
2. إذا كان المستخدم غير مصادق، يوجهه إلى صفحة تسجيل الدخول
3. إذا كان المستخدم مصادقًا لكن ليس لديه الصلاحية المطلوبة، يوجهه إلى صفحة غير مصرح (`/unauthorized`)

## حالات ظهور صفحة تسجيل الدخول مع أمثلة

### مثال 1: زيارة صفحة محمية مباشرة

```
المستخدم -> يزور /dashboard -> AuthProvider يكتشف أنه غير مصادق -> توجيه إلى /login
```

### مثال 2: انتهاء صلاحية الجلسة

```
المستخدم المصادق -> توقف نشاط لفترة طويلة -> انتهاء صلاحية الجلسة -> 
محاولة الوصول إلى صفحة محمية -> توجيه إلى /login
```

### مثال 3: تسجيل الدخول ثم العودة إلى الصفحة المطلوبة

```
المستخدم -> يزور /orders -> توجيه إلى /login?returnUrl=%2Forders -> 
تسجيل دخول ناجح -> إعادة توجيه تلقائي إلى /orders
```

## تخصيص سلوك المصادقة

يمكن تخصيص سلوك المصادقة وإعدادات التوجيه عن طريق:

1. **تعديل قائمة المسارات العامة**:
   تعديل متغير `PUBLIC_PATHS` في ملف `AuthProvider.tsx`

2. **تغيير سلوك التوجيه بعد تسجيل الدخول**:
   تعديل المنطق في نتيجة نجاح تسجيل الدخول في `login.tsx`

3. **إضافة فحص الصلاحيات المخصصة**:
   استخدام مكون `RouteGuard` مع تحديد `requiredPermission` للصفحات التي تتطلب صلاحيات خاصة

## كيفية استخدام مكون PermissionGuard داخل الصفحات

بالإضافة إلى حماية المسارات، يمكن استخدام مكون `PermissionGuard` لإخفاء أو إظهار أجزاء معينة من واجهة المستخدم بناءً على صلاحيات المستخدم:

```tsx
// مثال لاستخدام PermissionGuard في واجهة المستخدم
<PermissionGuard permissionCode="admins:create">
  <Button onClick={handleOpenCreateModal}>إضافة مسؤول جديد</Button>
</PermissionGuard>
```

يمكن أيضًا تحديد محتوى بديل ليتم عرضه إذا لم يكن للمستخدم الصلاحية المطلوبة:

```tsx
<PermissionGuard 
  permissionCode="reports:export" 
  fallback={<Button disabled>تصدير التقرير (غير مصرح)</Button>}
>
  <Button onClick={handleExport}>تصدير التقرير</Button>
</PermissionGuard>
```

## الخلاصة

نظام تسجيل الدخول المنفذ يتميز بالمرونة والأمان، حيث:

1. **يظهر بشكل تلقائي** عند الحاجة إلى المصادقة
2. **يحمي جميع المسارات** من الوصول غير المصرح
3. **يدعم التوجيه الذكي** للعودة إلى الصفحة المطلوبة بعد تسجيل الدخول
4. **يتكامل مع نظام الصلاحيات** للتحكم الدقيق في وصول المستخدمين
5. **قابل للتخصيص** ليناسب احتياجات التطبيق المختلفة
