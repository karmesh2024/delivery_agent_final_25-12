# لوحة الطلبيات - Orders Fulfillment Board

## نظرة عامة

لوحة الطلبيات هي نظام مشابه لـ Kitchen Display System (KDS) مصمم خصيصاً لمتابعة وتنفيذ طلبات التجميع والتغليف في المخازن.

## المميزات

### 1. عرض الطلبات
- عرض جميع الطلبات الواردة من المتاجر
- ترتيب تلقائي حسب حالة التجميع وتاريخ الإنشاء
- تحديث تلقائي كل 30 ثانية

### 2. مراحل التجميع
الطلبات تمر بـ 6 مراحل:

1. **في انتظار التجميع (pending)**: الطلب جديد وانتظار البدء
2. **قيد التجميع (collecting)**: جارٍ تجميع المنتجات من المخزن
3. **قيد التحقق (verifying)**: التحقق من الكميات والجودة
4. **قيد التغليف (packaging)**: تغليف المنتجات وإرفاق الفاتورة
5. **جاهز للتسليم (ready)**: الطلب جاهز وانتظار المندوب/الوكيل
6. **تم التسليم (completed)**: تم تسليم الطلب بنجاح

### 3. الفلترة والبحث
- فلترة حسب حالة التجميع
- عرض عدد الطلبات لكل حالة
- تحديث يدوي أو تلقائي

### 4. تحديث الحالة
- تحديث الحالة من البطاقة مباشرة (زر "التالي")
- تحديث الحالة من نافذة التفاصيل
- اختيار أي مرحلة مباشرة

## كيفية الاستخدام

### الوصول إلى اللوحة
1. اذهب إلى **إدارة المخازن**
2. اضغط على بطاقة **لوحة الطلبيات**
3. أو اذهب مباشرة إلى `/warehouse-management/orders-board`

### تحديث حالة الطلب

#### من البطاقة:
1. اضغط على زر **التالي** في بطاقة الطلب
2. سيتم تحديث الحالة تلقائياً إلى المرحلة التالية

#### من نافذة التفاصيل:
1. اضغط على بطاقة الطلب لفتح نافذة التفاصيل
2. اختر المرحلة المطلوبة من قسم "تحديث حالة التجميع"
3. اضغط على **حفظ التغييرات**

### عرض التفاصيل
- اضغط على أي بطاقة طلب لعرض التفاصيل الكاملة:
  - معلومات الوكيل/العميل
  - قائمة المنتجات مع الصور
  - المبلغ الإجمالي
  - طريقة الدفع
  - ملاحظات الطلب

## التكامل التلقائي

عند إنشاء طلب جديد في المتجر:
- يتم تعيين `fulfillment_status = 'pending'` تلقائياً
- يظهر الطلب فوراً في لوحة الطلبيات
- لا حاجة لأي إجراء يدوي

## API Endpoints

### GET `/api/warehouse/orders-board`
جلب الطلبات للوحة التجميع

**Query Parameters:**
- `fulfillment_status`: فلترة حسب الحالة (pending, collecting, verifying, packaging, ready, all)
- `shop_id`: فلترة حسب المتجر

### PATCH `/api/warehouse/orders-board/[orderId]`
تحديث حالة التجميع

**Body:**
```json
{
  "fulfillment_status": "collecting"
}
```

## قاعدة البيانات

### Migration
تم إضافة حقل `fulfillment_status` إلى جدول `store_orders`:
- Type: `store_order_fulfillment_status` enum
- Default: `'pending'`
- Index: موجود لتحسين الأداء

### تطبيق Migration
```bash
npm run migrate
```

أو قم بتطبيق الملف:
`src/migrations/add_fulfillment_status_to_store_orders.sql`

## ملاحظات مهمة

1. **الطلبات المكتملة**: الطلبات التي وصلت إلى حالة `completed` لا تظهر في اللوحة تلقائياً
2. **التحديث التلقائي**: اللوحة تقوم بتحديث البيانات كل 30 ثانية
3. **الصوت**: يمكن تفعيل/تعطيل التنبيهات الصوتية من زر الصوت
4. **الأداء**: تم إضافة فهارس على `fulfillment_status` لتحسين سرعة الاستعلامات
