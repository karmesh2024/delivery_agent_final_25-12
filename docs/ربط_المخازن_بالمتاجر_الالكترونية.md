# ربط المخازن بالمتاجر الإلكترونية

## المشكلة الأساسية

كان هناك انفصال كامل بين نظام إدارة المخازن (`warehouse_inventory`) ونظام إدارة المتاجر الإلكترونية (`store_products`). عند إضافة منتج جديد في المتجر، لم يكن هناك طريقة لمعرفة الكمية المتاحة من هذا المنتج في المخازن.

## الحل المطبق

### 1. تحديث قاعدة البيانات

#### إضافة حقل `store_product_id` في `warehouse_inventory`

تم إضافة حقل جديد `store_product_id` في جدول `warehouse_inventory` للربط المباشر مع جدول `store_products`:

```sql
ALTER TABLE public.warehouse_inventory
ADD COLUMN IF NOT EXISTS store_product_id UUID REFERENCES public.store_products(id);
```

**الفوائد:**
- ربط مباشر بين المنتجات في المتاجر والمخزون في المخازن
- إمكانية تتبع المخزون لكل منتج في كل مخزن
- دعم المنتجات من نوع `waste_data_admin` والمنتجات من `store_products` في نفس الجدول

#### تحديث Schema Prisma

تم تحديث `prisma/schema.prisma` لإضافة:
- حقل `store_product_id` في `warehouse_inventory`
- علاقة `store_products` في `warehouse_inventory`
- علاقة `warehouse_inventory` في `store_products`
- فهرس على `store_product_id` لتحسين الأداء
- Constraint فريد على `(warehouse_id, store_product_id)`

### 2. إنشاء API Endpoints

#### `/api/products/[productId]/warehouse-inventory`
- **الوظيفة:** جلب كمية المنتج من جميع المخازن
- **المعاملات:** `productId` (UUID) أو `sku` (query parameter)
- **الاستجابة:** 
  ```json
  {
    "product_id": "uuid",
    "total_quantity": 150,
    "warehouses": [
      {
        "id": 1,
        "warehouse_id": 1,
        "quantity": 100,
        "unit": "piece",
        "warehouses": {
          "id": 1,
          "name": "مخزن المعادي",
          "location": "المعادي"
        }
      }
    ],
    "count": 1
  }
  ```

#### `/api/products/search-warehouse-inventory?sku=XXX`
- **الوظيفة:** البحث عن المخزون باستخدام SKU أو اسم المنتج
- **المعاملات:** `sku` أو `name` (query parameters)
- **الاستخدام:** مفيد عند إضافة منتج جديد للتحقق من وجوده في المخازن

### 3. تحديث `inventoryService`

تم إضافة ثلاث دوال جديدة في `inventoryService`:

```typescript
// جلب المخزون باستخدام store_product_id
getInventoryByStoreProductId(storeProductId: string)

// جلب المخزون باستخدام SKU
getInventoryBySKU(sku: string)

// جلب إجمالي الكمية من جميع المخازن
getTotalInventoryByStoreProductId(storeProductId: string): Promise<number>
```

### 4. تحديث `InventoryForm`

تم إضافة ميزات جديدة في نموذج المخزون:

#### البحث التلقائي عن المخزون
- عند إدخال SKU، يتم البحث تلقائياً بعد ثانية واحدة (debounced)
- عرض إجمالي الكمية المتاحة في جميع المخازن
- عرض تفاصيل الكمية في كل مخزن على حدة

#### واجهة المستخدم
- قسم مخصص لعرض المخزون من المخازن
- زر "استخدام الكمية من المخازن" لملء حقل `stock_quantity` تلقائياً
- رسائل خطأ واضحة عند عدم وجود مخزون
- مؤشر تحميل أثناء البحث

## كيفية الاستخدام

### عند إضافة منتج جديد:

1. املأ حقل **SKU** في تبويب "المعلومات الأساسية"
2. انتقل إلى تبويب "التسعير والمخزون" → "المخزون"
3. سيتم البحث تلقائياً عن المخزون بعد إدخال SKU
4. إذا وُجد مخزون، سيتم عرض:
   - إجمالي الكمية في جميع المخازن
   - تفاصيل الكمية في كل مخزن
   - زر لاستخدام الكمية من المخازن كقيمة أولية

### عند البحث يدوياً:

1. املأ حقل SKU
2. اضغط على زر "البحث عن المخزون"
3. سيتم عرض النتائج فوراً

## الخطوات التالية (اختياري)

1. **تحديث `recordInMovement`** لدعم `store_product_id` عند تسجيل حركات المخزون
2. **إضافة ميزة المزامنة التلقائية** - عند تحديث المخزون في المخازن، تحديث `stock_quantity` في `store_products` تلقائياً
3. **إضافة تنبيهات** عند انخفاض المخزون عن الحد الأدنى
4. **إضافة تقارير** تظهر العلاقة بين مخزون المتاجر ومخزون المخازن

## ملاحظات مهمة

- الحقل `product_id` في `warehouse_inventory` لا يزال موجوداً لدعم المنتجات من نوع `waste_data_admin`
- يمكن استخدام `store_product_id` و `product_id` معاً في نفس السجل (لكن ليس مطلوباً)
- عند البحث باستخدام SKU، يتم البحث في جميع المنتجات التي تطابق SKU (قد يكون هناك أكثر من منتج بنفس SKU في متاجر مختلفة)
