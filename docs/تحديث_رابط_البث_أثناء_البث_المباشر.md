# تحديث رابط البث أثناء البث المباشر

**التاريخ:** 2025-01-22  
**الإصدار:** V1.5.1

---

## 🔴 المشكلة

عند تغيير رابط البث في الداش بورد، لم يتم تشغيل الرابط الجديد في تطبيق الموبايل، بل تم تشغيل الرابط القديم.

**السبب:**
- لا توجد دالة لتحديث `stream_url` و `listen_url` في قاعدة البيانات أثناء البث المباشر
- الموبايل قد يكون يحتفظ بالرابط في cache
- لا يوجد Real-time subscription لتحديث رابط البث تلقائياً

## ✅ الحل المطبق

### 1. إضافة دالة `updateStreamUrl` في Service

**الملف:** `src/domains/club-zone/services/clubRadioService.ts`

```typescript
/**
 * ✅ V1.5.1: تحديث رابط البث أثناء البث المباشر
 */
async updateStreamUrl(activityId: string, streamUrl: string, listenUrl?: string): Promise<void> {
  const updateData: any = {
    stream_url: streamUrl,
    updated_at: new Date().toISOString(),
  };

  // إذا تم توفير listen_url، استخدمه، وإلا استخدم stream_url
  if (listenUrl) {
    updateData.listen_url = listenUrl;
  } else {
    updateData.listen_url = streamUrl;
  }

  const { error } = await supabase
    .from('club_activities')
    .update(updateData)
    .eq('id', activityId)
    .eq('is_live', true)
    .eq('is_active', true);

  if (error) {
    console.error('[Radio] Error updating stream URL:', error);
    throw error;
  }

  console.log('[Radio] Stream URL updated successfully:', {
    activityId,
    streamUrl,
    listenUrl: updateData.listen_url,
  });
}
```

### 2. إضافة Redux Action

**الملف:** `src/domains/club-zone/store/clubZoneSlice.ts`

```typescript
export const updateStreamUrl = createAsyncThunk(
  'clubZone/updateStreamUrl',
  async (params: {
    activityId: string;
    streamUrl: string;
    listenUrl?: string;
  }) => {
    await clubRadioService.updateStreamUrl(params.activityId, params.streamUrl, params.listenUrl);
    // إعادة جلب البث المحدث
    return await clubRadioService.getCurrentStream();
  }
);

// في extraReducers:
.addCase(updateStreamUrl.pending, (state) => {
  state.loading = true;
  state.error = null;
})
.addCase(updateStreamUrl.fulfilled, (state, action: PayloadAction<RadioStream | null>) => {
  state.loading = false;
  state.currentStream = action.payload;
})
.addCase(updateStreamUrl.rejected, (state, action) => {
  state.loading = false;
  state.error = action.error.message || 'Failed to update stream URL';
});
```

### 3. إضافة Real-time Subscription

**الملف:** `src/app/club-zone/radio/page.tsx`

```typescript
// ✅ V1.5.1: Real-time subscription لتحديث رابط البث
useEffect(() => {
  if (!currentStream?.id) return;

  console.log('[Radio Stream] Setting up subscription for stream updates:', currentStream.id);

  const streamChannel = supabase
    .channel(`radio_stream_${currentStream.id}`)
    .on(
      'postgres_changes',
      {
        event: 'UPDATE',
        schema: 'public',
        table: 'club_activities',
        filter: `id=eq.${currentStream.id}`,
      },
      (payload) => {
        const updatedStream = payload.new as any;
        console.log('[Radio Stream] Stream URL updated:', {
          id: updatedStream.id,
          stream_url: updatedStream.stream_url,
          listen_url: updatedStream.listen_url,
        });

        // تحديث رابط البث في Redux store
        dispatch(fetchCurrentStream());

        // تحديث رابط البث في audio player إذا كان يعمل
        if (audioRef.current && !audioRef.current.paused && updatedStream.listen_url) {
          const wasPlaying = !audioRef.current.paused;
          audioRef.current.src = updatedStream.listen_url;
          if (wasPlaying) {
            audioRef.current.play().catch((err) => {
              console.error('[Radio Stream] Error playing updated URL:', err);
            });
          }
        }
      }
    )
    .subscribe((status) => {
      console.log('[Radio Stream] Subscription status:', status);
    });

  return () => {
    console.log('[Radio Stream] Cleaning up subscription');
    supabase.removeChannel(streamChannel);
  };
}, [currentStream?.id, dispatch]);
```

## 🔄 كيف يعمل الحل

### في الداش بورد:

1. **تحديث الرابط يدوياً:**
   - اضغط على زر "تحديث الرابط" أثناء البث المباشر
   - أدخل رابط البث الجديد
   - يتم تحديث `stream_url` و `listen_url` في قاعدة البيانات

2. **Real-time Updates:**
   - عند تحديث الرابط، يتم إرسال تحديث Real-time إلى جميع المشتركين
   - الداش بورد يتلقى التحديث ويحدث رابط البث في audio player تلقائياً

### في تطبيق الموبايل:

1. **قراءة الرابط من قاعدة البيانات:**
   - الموبايل يقرأ `stream_url` و `listen_url` من `club_activities` عند بدء الاستماع
   - يجب أن يقرأ الرابط من قاعدة البيانات وليس من cache

2. **Real-time Subscription (مقترح):**
   - الموبايل يجب أن يتابع تحديثات `club_activities` عبر Supabase Realtime
   - عند تحديث الرابط، يجب أن يتوقف الموبايل عن الاستماع للرابط القديم ويبدأ الرابط الجديد

### 2. Real-time Subscription (مقترح)

```dart
// في Flutter
supabase
  .channel('radio_stream_updates')
  .onPostgresChanges(
    event: PostgresChangeEvent.update,
    schema: 'public',
    table: 'club_activities',
    filter: PostgresChangeFilter(
      type: PostgresChangeFilterType.eq,
      column: 'id',
      value: currentStreamId,
    ),
    callback: (payload) {
      final updatedStream = payload.newRecord;
      final newStreamUrl = updatedStream['listen_url'] ?? updatedStream['stream_url'];
      
      // تحديث رابط البث في audio player
      audioPlayer.setUrl(newStreamUrl);
      
      // إذا كان البث يعمل، إعادة التشغيل بالرابط الجديد
      if (isPlaying) {
        audioPlayer.play();
      }
    },
  )
  .subscribe();
```

## 🔍 التحقق من التنفيذ

### 1. اختبار تحديث الرابط في الداش بورد

1. ابدأ بث مباشر
2. اضغط على زر "تحديث الرابط"
3. أدخل رابط جديد
4. تحقق من Console - يجب أن ترى:
   ```
   [Radio] Stream URL updated successfully: { ... }
   [Radio Stream] Stream URL updated: { ... }
   ```

### 2. التحقق من قاعدة البيانات

```sql
-- في Supabase SQL Editor
SELECT id, stream_url, listen_url, updated_at
FROM club_activities
WHERE activity_type = 'radio_stream'
  AND is_active = true
  AND is_live = true;
```

### 3. اختبار Real-time

- افتح Console في المتصفح
- راقب الرسائل: `[Radio Stream] Stream URL updated`
- تحقق من أن رابط البث في audio player يتحدث تلقائياً

## 🎯 الخطوات التالية للموبايل

1. **تأكد من قراءة الرابط من قاعدة البيانات:**
   - لا تستخدم cache للرابط
   - اقرأ `listen_url` من `club_activities` عند بدء الاستماع

2. **إضافة Real-time Subscription (مقترح):**
   - تابع تحديثات `club_activities` عبر Supabase Realtime
   - عند تحديث الرابط، قم بتحديث audio player تلقائياً

3. **معالجة تغيير الرابط:**
   - عند تحديث الرابط، أوقف البث القديم
   - ابدأ البث الجديد بالرابط المحدث
   - أبلغ المستخدم بتغيير الرابط (اختياري)
