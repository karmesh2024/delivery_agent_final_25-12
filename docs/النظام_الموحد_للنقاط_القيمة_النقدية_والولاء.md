## 🎯 المفهوم الأساسي

### النظام الهجين: **محفظة مالية + نقاط متجر + مضاعف ذكي**

النظام في كارمش يجمع بين:
1. **محفظة مالية (Wallet)** - فلوس حقيقية قابلة للسحب النقدي
2. **نقاط المتجر (Store Points)** - للتحفيز والتسويق (منتجات/خصومات/هدايا)
3. **مضاعف ذكي (Smart Multiplier)** - يتغير حسب سلوك العميل (`trust_score`)

### الفصل الواضح:

```
💰 wallet_balance → سحب نقدي مباشر (فلوس حقيقية)
🎁 store_points → منتجات / خصومات / هدايا (تحفيز)
🧠 trust_score → مضاعف ذكي (0.5 إلى 1.5) حسب سلوك العميل
```

**المبدأ الأساسي:**
- **المحفظة المالية** = القيمة الفعلية للمخلفات (قابلة للسحب)
- **نقاط المتجر** = البونص والتحفيز (غير قابلة للسحب النقدي)
- **المضاعف الذكي** = يؤثر فقط على نقاط المتجر (لا يؤثر على المحفظة)

## ⚠️ تحذير: المخاطر الحقيقية

قبل البدء في التنفيذ، يجب فهم المخاطر التالية:

### 🔴 الخطر رقم 1: ربط النقاط بسعر البورصة مباشرة
- **المشكلة**: لو السعر ارتفع فجأة → النقاط ترتفع → العملاء يستبدلون فوراً وأنت لم تبع المخلفات بعد
- **الحل**: ✅ تجميد السعر عند الجلسة في `unit_price`

### 🔴 الخطر رقم 2: multiplier = تضخيم فلوس حقيقية
- **المشكلة**: `multiplier = 2.0` يعني العميل حصل على 27 جنيه وأنت اشتريت بـ 13.5 جنيه (دعم نقدي مباشر)
- **الحل**: ✅ فصل `financial_points` (قابل للسحب) عن `bonus_points` (للمتجر فقط)

### 🔴 الخطر رقم 3: الاستبدال الفوري بدون ضوابط
- **المشكلة**: استبدال فوري بدون Lock, Queue, Daily limit, KYC
- **الحل**: ✅ قيود يومية، فترة انتظار، موافقة إدارية، قفل على الجلسات غير المقفلة

## 📐 المعادلات الرسمية (Accounting Rules)

### 🏗️ المعمارية النهائية: النظام الهجين + المضاعف الذكي

### 1️⃣ القيمة المالية (Total Value)

```sql
total_value = Σ(actual_weight × unit_price)
```

**حيث:**
- `actual_weight`: الوزن الفعلي المقاس بالكيلوجرام
- `unit_price`: سعر البورصة المجمد عند الجلسة (لا يتغير)

---

### 2️⃣ النقاط الأساسية (Base Points)

```sql
base_points = FLOOR(total_value × points_per_egp)
```

**حيث:**
- `total_value`: القيمة المالية الإجمالية بالجنيه
- `points_per_egp`: معامل التحويل (افتراضي: 100 نقطة = 1 جنيه)

**النتيجة:**
- `base_points` → تذهب إلى **المحفظة المالية** (`wallet_balance`)
- كل نقطة = 0.01 جنيه (قابلة للسحب النقدي)

---

### 3️⃣ المضاعف الذكي (Smart Multiplier)

```sql
final_multiplier = base_multiplier × trust_score × volume_factor
```

**حيث:**
- `base_multiplier`: المضاعف الأساسي من المنتج (افتراضي: 1.0)
- `trust_score`: درجة الثقة في العميل (0.5 إلى 1.5)
- `volume_factor`: عامل الحجم (اختياري - حسب الكمية)

**مثال:**
```sql
base_multiplier = 1.2 (عرض خاص)
trust_score = 1.1 (عميل موثوق)
volume_factor = 1.0 (لا تأثير)

final_multiplier = 1.2 × 1.1 × 1.0 = 1.32
```

---

### 4️⃣ نقاط المتجر (Store Points - البونص)

```sql
bonus_points = FLOOR(base_points × (final_multiplier - 1.0))
```

**حيث:**
- `base_points`: النقاط الأساسية
- `final_multiplier`: المضاعف النهائي

**المنطق:**
- إذا `final_multiplier = 1.0` → `bonus_points = 0` (لا بونص)
- إذا `final_multiplier = 1.5` → `bonus_points = base_points × 0.5` (50% بونص)
- `bonus_points` → تذهب إلى **نقاط المتجر** (`store_points`)

---

### 📊 الصيغة الكاملة:

```sql
-- 1. حساب القيمة المالية
total_value = Σ(actual_weight × unit_price)

-- 2. حساب النقاط الأساسية (للمحفظة)
base_points = FLOOR(total_value × points_per_egp)
wallet_balance += base_points

-- 3. حساب المضاعف الذكي
final_multiplier = base_multiplier × trust_score × volume_factor

-- 4. حساب نقاط المتجر (البونص)
IF final_multiplier > 1.0 THEN
    bonus_points = FLOOR(base_points × (final_multiplier - 1.0))
    store_points += bonus_points
END IF
```

**المبدأ:**
- ✅ **المحفظة المالية** = القيمة الفعلية (قابلة للسحب)
- ✅ **نقاط المتجر** = البونص فقط (غير قابلة للسحب النقدي)
- ✅ **المضاعف الذكي** = يؤثر فقط على نقاط المتجر

## 🔄 تدفق المستخدم (User Flow)

### 📱 في التطبيق (قبل الجمع)

```
العميل → يدخل المخلفات في التطبيق
  ↓
النظام يعرض:
  💰 قيمة تقديرية (total_value)
  🎁 نقاط متجر متوقعة (bonus_points)
  📊 المضاعف المتوقع (final_multiplier)
  ↓
العميل يؤكد الطلب
```

**مثال في التطبيق:**
```
زجاجات بلاستيك PET: 20 قطعة
─────────────────────────────
💰 القيمة المتوقعة: 13.5 جنيه
🎁 نقاط متجر متوقعة: 270 نقطة
📊 مضاعف: 1.2x (عميل موثوق)
─────────────────────────────
✅ إجمالي النقاط: 1620 نقطة
   (1350 محفظة + 270 متجر)
```

---

### 🚚 أثناء الجمع

```
عامل التوصيل → وزن المخلفات
  ↓
إدخال الوزن الفعلي (actual_weight)
  ↓
النظام يحسب:
  - total_value = actual_weight × unit_price (مجمد)
  - base_points = FLOOR(total_value × points_per_egp)
  - final_multiplier = base_multiplier × trust_score
  - bonus_points = FLOOR(base_points × (final_multiplier - 1.0))
  ↓
إغلاق الجلسة (status = 'completed')
```

---

### ✅ بعد الإغلاق (قبل التسوية)

```
عند status = 'completed':
  ↓
تحديث profiles:
  wallet_balance += base_points (معلق - لا يمكن السحب)
  store_points += bonus_points (متاح فوراً)
  ↓
تسجيل في points_transactions
  ↓
⚠️ لا يمكن السحب النقدي حتى is_settled = true
```

## 🔄 السيناريو الكامل (التفصيلي)

### المرحلة 1: إعداد النظام

#### 1.1 إنشاء/تحديث إعدادات النقاط
```
المسؤول → إدارة النقاط → إعدادات النقاط
  ↓
تحديد points_per_egp (مثال: 100 نقطة = 1 جنيه)
  ↓
حفظ في points_configuration
```

**مثال:**
- `points_per_egp = 100` يعني: **100 نقطة = 1 جنيه**
- `points_per_egp = 200` يعني: **200 نقطة = 1 جنيه** (نقاط أكثر لكل جنيه)

#### 1.2 تحديث أسعار البورصة (يومياً)
```
المسؤول → إدارة البورصة → تحديث الأسعار
  ↓
تحديث stock_exchange.buy_price لكل فئة فرعية
  ↓
النظام يستخدم السعر الحالي لحساب النقاط
```

**مثال:**
- بلاستيك PET: `buy_price = 15` جنيه/كجم
- ورق: `buy_price = 8` جنيه/كجم
- معادن: `buy_price = 25` جنيه/كجم

#### 1.3 إضافة/تحديث منتج
```
المسؤول → إدارة المنتجات → منتج جديد
  ↓
اختيار الفئة الفرعية
  ↓
تحديد counting_method (pieces/weight)
  ↓
إدخال الوزن (للمنتجات المعدودة)
  ↓
تحديد points_multiplier (للعروض)
```

#### 2.3 إضافة عناصر المخلفات

**السيناريو A: منتج معدود بالقطعة**
```
عامل التوصيل → اختيار المنتج (زجاجة بلاستيك)
  ↓
إدخال عدد القطع: 20 قطعة
  ↓
النظام يحسب تلقائياً:
  - total_weight = (weight / 1000) × quantity
  - monetary_value = total_weight × buy_price
  - earned_points = FLOOR(monetary_value × points_per_egp × multiplier)
```

**السيناريو B: منتج موزون بالكيلو**
```
عامل التوصيل → اختيار المنتج (كرتون)
  ↓
وزن المخلفات: 5.5 كجم
  ↓
النظام يحسب تلقائياً:
  - total_weight = actual_weight
  - monetary_value = total_weight × buy_price
  - earned_points = FLOOR(monetary_value × points_per_egp × multiplier)
```

#### 2.4 إتمام الجلسة
```
عامل التوصيل → إنهاء التجميع
  ↓
تحديث waste_collection_session:
  status = "completed"
  completed_at = NOW()
  total_weight = SUM(total_weight) من جميع العناصر
  total_amount = SUM(monetary_value) من جميع العناصر
  total_points = SUM(earned_points) من جميع العناصر
```

#### 2.5 منح النقاط للعميل (قبل التسوية)
```
عند إتمام الجلسة (status = 'completed')
  ↓
تحديث رصيد نقاط العميل:

  profiles.wallet_balance += base_points (معلق - لا يمكن السحب)
  ↓
تسجيل في points_transactions:
  - type = 'store' (للبونص)
  - type = 'wallet' (للمحفظة - معلق)
  ↓
⚠️ لا يمكن السحب النقدي حتى is_settled = true
```

#### 2.6 تسوية الجلسة (Settlement)
```
المسؤول → مراجعة الجلسة
  ↓
التحقق من الوزن والسعر
  ↓
اعتماد الجلسة
  ↓
تحديث is_settled = true
  ↓
تحديث profiles:
  wallet_balance += base_points (متاح للسحب الآن)
  ↓
تسجيل في points_transactions:
  - type = 'wallet' (متاح الآن)
  ↓
✅ السماح بالسحب النقدي
```

### المرحلة 3: استخدام النقاط

#### 3.1 سحب نقدي من المحفظة
```
العميل → تطبيق الموبايل → سحب من المحفظة
  ↓
التحقق من:
  - wallet_balance >= المبلغ المطلوب
  - لا توجد جلسات غير مقفلة (is_settled = false)
  ↓
إدخال المبلغ: 100 جنيه
  ↓
النظام يحسب النقاط:
  points = amount_egp × points_per_egp
  = 100 × 100 = 10000 نقطة
  ↓
إنشاء points_redemption
  ↓
خصم من wallet_balance
  ↓
تحويل المبلغ للعميل (نقدي/تحويل بنكي/محفظة)
```

#### 3.2 شراء منتج من المتجر (باستخدام نقاط المتجر)
```
العميل → المتجر الإلكتروني → اختيار منتج
  ↓
المنتج: قميص كارمش - السعر: 20000 نقطة متجر
  ↓
التحقق من:
  - store_points >= 20000
  ↓
العميل يدفع بنقاط المتجر
  ↓
خصم من store_points (وليس wallet_balance)
  ↓
إتمام الطلب
```

**ملاحظة:**
- يمكن دفع جزء بالنقاط المالية وجزء بنقاط المتجر
- الأولوية: نقاط المتجر أولاً

#### 3.3 استبدال بهدية
```
العميل → نادي كارمش → اختيار هدية
  ↓
الهدية: كوب كارمش - قيمة النقاط: 5000 نقطة
  ↓
خصم 5000 نقطة من رصيد العميل
  ↓
إرسال الهدية للعميل
```

#### 3.4 التبرع
```
العميل → نادي كارمش → التبرع
  ↓
إدخال عدد النقاط: 10000 نقطة
  ↓
خصم 10000 نقطة من رصيد العميل
  ↓
تسجيل التبرع
```

## 🏗️ هيكل قاعدة البيانات الموحد (النظام الهجين)

### 1. تحديث `profiles` (المحفظة الهجينة)

```sql
ALTER TABLE profiles
ADD COLUMN wallet_balance NUMERIC(12,2) DEFAULT 0,
ADD COLUMN store_points INTEGER DEFAULT 0,
ADD COLUMN trust_score NUMERIC(3,2) DEFAULT 1.0;

COMMENT ON COLUMN profiles.wallet_balance IS 
'المحفظة المالية - فلوس حقيقية قابلة للسحب النقدي (بالجنيه)';
COMMENT ON COLUMN profiles.store_points IS 
'نقاط المتجر - للتحفيز والتسويق (منتجات/خصومات/هدايا) - غير قابلة للسحب النقدي';
COMMENT ON COLUMN profiles.trust_score IS 
'درجة الثقة في العميل - من 0.5 إلى 1.5 - يؤثر على المضاعف الذكي';
```

**المعنى:**
- `wallet_balance`: المحفظة المالية (بالجنيه) - قابلة للسحب النقدي
- `store_points`: نقاط المتجر - للتحفيز فقط
- `trust_score`: درجة الثقة (0.5 = منخفضة، 1.0 = عادية، 1.5 = عالية)

### 4. جدول `points_transactions` (Audit Log إلزامي)

```sql
CREATE TABLE points_transactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    profile_id UUID NOT NULL REFERENCES profiles(id),
    
    type TEXT NOT NULL,
    -- wallet: معاملة محفظة مالية
    -- store: معاملة نقاط متجر
    
    amount INTEGER NOT NULL,
    -- موجب للإيداع، سالب للسحب
    
    before_balance INTEGER NOT NULL,
    after_balance INTEGER NOT NULL,
    -- حسب النوع: wallet_balance أو store_points
    
    source TEXT,
    -- waste_collection_session, redemption, admin_adjustment, etc.
    
    reference_id UUID,
    -- معرف المرجع (session_id, redemption_id, etc.)
    
    description TEXT,
    created_by UUID REFERENCES profiles(id),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_points_transactions_profile ON points_transactions(profile_id);
CREATE INDEX idx_points_transactions_created ON points_transactions(created_at);
CREATE INDEX idx_points_transactions_reference ON points_transactions(reference_id);
```

### المرحلة 1: حساب الوزن الإجمالي

#### أ. المنتجات المعدودة بالقطعة (`counting_method = 'pieces'`)

```sql
-- مثال: زجاجة بلاستيك PET
-- وزن القطعة: 45 جرام = 0.045 كجم
-- عدد القطع: 20

total_weight = (weight / 1000) × quantity
total_weight = (45 / 1000) × 20
total_weight = 0.045 × 20 = 0.9 كجم
```

**الكود:**
```typescript
if (countingMethod === 'pieces') {
  const weightPerPieceKg = product.weight / 1000; // تحويل من جرام لكيلو
  totalWeight = weightPerPieceKg * quantity;
}
```

---

#### ب. المنتجات الموزونة بالكيلو (`counting_method = 'weight'`)

```sql
-- مثال: كرتون مستعمل
-- الوزن الفعلي المقاس: 5.5 كجم

total_weight = actual_weight
total_weight = 5.5 كجم
```

**الكود:**
```typescript
if (countingMethod === 'weight') {
  totalWeight = actualWeight;
}
```

### المرحلة 2: حساب القيمة المالية (Total Value)

```sql
-- جلب سعر البورصة وتجميده عند الجلسة
SELECT buy_price 
FROM stock_exchange 
WHERE sub_category_id = :subcategory_id
ORDER BY last_update DESC 
LIMIT 1;

-- حساب القيمة المالية للعنصر
item_value = actual_weight × unit_price

-- القيمة الإجمالية للجلسة
total_value = Σ(item_value) من جميع العناصر
```

**مثال:**
```sql
-- بلاستيك PET
actual_weight = 0.9 كجم
unit_price = 15 جنيه/كجم (مجمد عند الجلسة)

item_value = 0.9 × 15 = 13.5 جنيه
```

**⚠️ مهم:**
- `unit_price` يُحفظ في `waste_collection_items.unit_price` عند INSERT
- أي تحديث لاحق لسعر البورصة **لا يؤثر** على الجلسات السابقة

### المرحلة 3: حساب النقاط الأساسية (Base Points)

```sql
-- جلب معامل التحويل
SELECT points_per_egp 
FROM points_configuration 
WHERE is_active = true 
LIMIT 1;

-- حساب النقاط الأساسية
base_points = FLOOR(total_value × points_per_egp)
```

**مثال:**
```sql
-- البيانات:
total_value = 13.5 جنيه
points_per_egp = 100 (100 نقطة = 1 جنيه)

-- الحساب:
base_points = FLOOR(13.5 × 100)
base_points = FLOOR(1350)
base_points = 1350 نقطة
```

**النتيجة:**
- `base_points` → تذهب إلى **المحفظة المالية** (`wallet_balance`)
- كل نقطة = 0.01 جنيه (قابلة للسحب النقدي)

### المرحلة 4: حساب المضاعف الذكي (Smart Multiplier)

```sql
-- جلب درجة الثقة في العميل
SELECT trust_score FROM profiles WHERE id = :customer_id;

-- جلب المضاعف الأساسي من المنتج
SELECT points_multiplier FROM waste_data_admin WHERE id = :product_id;

-- حساب المضاعف النهائي
final_multiplier = base_multiplier × trust_score × volume_factor
```

**حساب `trust_score`:**

| السلوك | التأثير |
|--------|---------|
| دقة الوزن | +0.05 |
| التزام بالمواعيد | +0.05 |
| عدم الإلغاء | +0.05 |
| شكاوى / فروق | -0.1 |
| عميل قديم (أكثر من 6 أشهر) | +0.1 |

**النطاق:** `trust_score ∈ [0.5, 1.5]`

**مثال:**
```sql
-- عميل موثوق:
-- دقة الوزن: +0.05
-- التزام بالمواعيد: +0.05
-- عميل قديم: +0.1
-- trust_score = 1.0 + 0.05 + 0.05 + 0.1 = 1.2

base_multiplier = 1.0 (لا يوجد عرض خاص)
trust_score = 1.2
volume_factor = 1.0

final_multiplier = 1.0 × 1.2 × 1.0 = 1.2
```

---

### المرحلة 5: حساب نقاط المتجر (Store Points - البونص)

```sql
-- حساب نقاط البونص
IF final_multiplier > 1.0 THEN
    bonus_points = FLOOR(base_points × (final_multiplier - 1.0))
ELSE
    bonus_points = 0
END IF
```

**مثال:**
```sql
-- البيانات:
base_points = 1350 نقطة
final_multiplier = 1.2

-- الحساب:
bonus_points = FLOOR(1350 × (1.2 - 1.0))
bonus_points = FLOOR(1350 × 0.2)
bonus_points = FLOOR(270)
bonus_points = 270 نقطة
```

**النتيجة:**
- `base_points = 1350` → تذهب للمحفظة المالية
- `bonus_points = 270` → تذهب لنقاط المتجر
- **المجموع = 1620 نقطة** (1350 مالية + 270 متجر)

**⚠️ مهم:**
- `bonus_points` **غير قابلة للسحب النقدي**
- يمكن استخدامها فقط في: المتجر، الهدايا، الخصومات

```sql
CREATE OR REPLACE FUNCTION calculate_collection_item()
RETURNS TRIGGER AS $$
DECLARE
    v_product RECORD;
    v_buy_price DECIMAL(10,2);
    v_total_weight DECIMAL(10,3);
    v_item_value DECIMAL(10,2);
BEGIN
    -- 1. جلب بيانات المنتج
    SELECT 
        wda.weight,
        wda.subcategory_id,
        wda.counting_method,
        wda.has_fixed_weight
    INTO v_product
    FROM waste_data_admin wda
    WHERE wda.id = NEW.waste_data_id;
    
    IF NOT FOUND THEN
        RAISE EXCEPTION 'المنتج غير موجود: %', NEW.waste_data_id;
    END IF;
    
    -- 2. نسخ طريقة العد
    NEW.counting_method := v_product.counting_method;
    
    -- 3. حساب الوزن الإجمالي
    IF NEW.counting_method = 'pieces' THEN
        IF v_product.weight IS NULL OR v_product.weight = 0 THEN
            RAISE EXCEPTION 'المنتج يتطلب وزن محدد للعد بالقطعة';
        END IF;
        
        IF NEW.quantity IS NULL OR NEW.quantity <= 0 THEN
            RAISE EXCEPTION 'يجب إدخال عدد القطع';
        END IF;
        
        v_total_weight := (v_product.weight / 1000.0) * NEW.quantity;
        
    ELSIF NEW.counting_method = 'weight' THEN
        IF NEW.actual_weight IS NULL OR NEW.actual_weight <= 0 THEN
            RAISE EXCEPTION 'يجب إدخال الوزن الفعلي';
        END IF;
        
        v_total_weight := NEW.actual_weight;
        NEW.quantity := 1;
        
    ELSE
        RAISE EXCEPTION 'طريقة عد غير صالحة: %', NEW.counting_method;
    END IF;
    
    NEW.total_weight := v_total_weight;
    NEW.actual_weight := v_total_weight; -- للتوحيد
    
    -- 4. جلب وتجميد سعر البورصة (فقط عند INSERT)
    IF TG_OP = 'INSERT' THEN
        SELECT buy_price INTO v_buy_price
        FROM stock_exchange
        WHERE sub_category_id = v_product.subcategory_id
        ORDER BY last_update DESC
        LIMIT 1;
        
        v_buy_price := COALESCE(v_buy_price, 0);
        NEW.unit_price := v_buy_price; -- ⭐ تجميد السعر
    ELSE
        -- في UPDATE: استخدام السعر المجمد
        v_buy_price := COALESCE(NEW.unit_price, OLD.unit_price, 0);
    END IF;
    
    -- 5. حساب القيمة المالية للعنصر
    v_item_value := v_total_weight * v_buy_price;
    NEW.total_price := v_item_value;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- حذف الـ trigger القديم
DROP TRIGGER IF EXISTS trigger_calculate_item_points ON waste_collection_items;
DROP TRIGGER IF EXISTS trigger_calculate_collection_item ON waste_collection_items;

-- إنشاء trigger جديد
CREATE TRIGGER trigger_calculate_collection_item
BEFORE INSERT OR UPDATE ON waste_collection_items
FOR EACH ROW
EXECUTE FUNCTION calculate_collection_item();

COMMENT ON FUNCTION calculate_collection_item IS 
'حساب القيمة المالية للعنصر وتجميد السعر عند الجلسة';
```

### 3. Function: حساب trust_score (المضاعف الذكي) - تفصيلي

**كيف يُحسب `trust_score`:**

| السلوك | التأثير | الشرط |
|--------|---------|-------|
| دقة الوزن | +0.05 | الفرق بين المتوقع والفعلي < 5% |
| التزام بالمواعيد | +0.05 | متوسط التأخير < 30 دقيقة |
| عدم الإلغاء | +0.05 | نسبة الإلغاء < 5% |
| شكاوى / فروق | -0.1 | كل شكوى محققة = -0.1 |
| عميل قديم | +0.1 | أكثر من 6 أشهر |

**النطاق:** `trust_score ∈ [0.5, 1.5]`

**مثال عملي:**
```sql
-- عميل جديد (0 أشهر):
trust_score = 1.0 (افتراضي)

-- عميل موثوق:
-- دقة الوزن: +0.05
-- التزام بالمواعيد: +0.05
-- عدم الإلغاء: +0.05
-- عميل قديم: +0.1
-- trust_score = 1.0 + 0.05 + 0.05 + 0.05 + 0.1 = 1.25

-- عميل مشكوك فيه:
-- شكوى واحدة: -0.1
-- trust_score = 1.0 - 0.1 = 0.9

-- عميل سيء جداً:
-- 3 شكاوى: -0.3
-- trust_score = GREATEST(0.5, 1.0 - 0.3) = 0.7 (الحد الأدنى)
```

**Function:**

```sql
CREATE OR REPLACE FUNCTION calculate_trust_score(p_customer_id UUID)
RETURNS NUMERIC(3,2) AS $$
DECLARE
    v_base_score NUMERIC(3,2) := 1.0;
    v_accuracy_bonus NUMERIC(3,2) := 0.0;
    v_punctuality_bonus NUMERIC(3,2) := 0.0;
    v_no_cancellation_bonus NUMERIC(3,2) := 0.0;
    v_complaints_penalty NUMERIC(3,2) := 0.0;
    v_loyalty_bonus NUMERIC(3,2) := 0.0;
    v_final_score NUMERIC(3,2);
BEGIN
    -- 1. دقة الوزن (الفرق بين المتوقع والفعلي < 5%)
    SELECT 
        CASE 
            WHEN AVG(ABS(estimated_weight - actual_weight) / NULLIF(estimated_weight, 0)) < 0.05 
            THEN 0.05 
            ELSE 0.0 
        END
    INTO v_accuracy_bonus
    FROM waste_collection_sessions
    WHERE customer_id = p_customer_id
      AND status = 'completed'
      AND estimated_weight IS NOT NULL
      AND actual_weight IS NOT NULL;
    
    -- 2. التزام بالمواعيد (لا تأخير أكثر من 30 دقيقة)
    SELECT 
        CASE 
            WHEN AVG(EXTRACT(EPOCH FROM (actual_start_time - scheduled_time)) / 60) < 30 
            THEN 0.05 
            ELSE 0.0 
        END
    INTO v_punctuality_bonus
    FROM delivery_orders
    WHERE customer_id = p_customer_id
      AND status = 'completed'
      AND scheduled_time IS NOT NULL
      AND actual_start_time IS NOT NULL;
    
    -- 3. عدم الإلغاء (نسبة الإلغاء < 5%)
    SELECT 
        CASE 
            WHEN (COUNT(*) FILTER (WHERE status = 'cancelled')::DECIMAL / NULLIF(COUNT(*), 0)) < 0.05 
            THEN 0.05 
            ELSE 0.0 
        END
    INTO v_no_cancellation_bonus
    FROM delivery_orders
    WHERE customer_id = p_customer_id;
    
    -- 4. شكاوى / فروق (كل شكوى = -0.1)
    SELECT 
        COALESCE(-COUNT(*) * 0.1, 0)
    INTO v_complaints_penalty
    FROM customer_complaints
    WHERE customer_id = p_customer_id
      AND status = 'resolved'
      AND resolution_type = 'customer_right';
    
    -- 5. عميل قديم (أكثر من 6 أشهر)
    SELECT 
        CASE 
            WHEN MIN(created_at) < NOW() - INTERVAL '6 months' 
            THEN 0.1 
            ELSE 0.0 
        END
    INTO v_loyalty_bonus
    FROM profiles
    WHERE id = p_customer_id;
    
    -- 6. حساب النتيجة النهائية
    v_final_score := v_base_score 
                   + v_accuracy_bonus 
                   + v_punctuality_bonus 
                   + v_no_cancellation_bonus 
                   + v_complaints_penalty 
                   + v_loyalty_bonus;
    
    -- 7. تطبيق النطاق [0.5, 1.5]
    v_final_score := GREATEST(0.5, LEAST(1.5, v_final_score));
    
    RETURN v_final_score;
END;
$$ LANGUAGE plpgsql;

COMMENT ON FUNCTION calculate_trust_score IS 
'حساب درجة الثقة في العميل بناءً على سلوكه (0.5 إلى 1.5)';
```

### 4. Function: حساب النقاط عند إتمام الجلسة (النظام الهجين)

```sql
CREATE OR REPLACE FUNCTION calculate_session_points()
RETURNS TRIGGER AS $$
DECLARE
    v_total_value DECIMAL(12,2);
    v_points_per_egp INTEGER;
    v_base_points INTEGER;
    v_trust_score NUMERIC(3,2);
    v_base_multiplier NUMERIC(3,2) := 1.0;
    v_volume_factor NUMERIC(3,2) := 1.0;
    v_final_multiplier NUMERIC(4,2);
    v_bonus_points INTEGER;
BEGIN
    -- فقط عند تغيير الحالة إلى completed
    IF NEW.status = 'completed' AND 
       (OLD.status IS NULL OR OLD.status != 'completed') THEN
        
        -- 1. حساب القيمة المالية الإجمالية
        SELECT COALESCE(SUM(total_price), 0) INTO v_total_value
        FROM waste_collection_items
        WHERE session_id = NEW.id;
        
        NEW.total_value := v_total_value;
        
        -- 2. جلب معامل التحويل
        SELECT points_per_egp INTO v_points_per_egp
        FROM points_configuration
        WHERE is_active = true
        ORDER BY created_at DESC
        LIMIT 1;
        
        v_points_per_egp := COALESCE(v_points_per_egp, 100);
        
        -- 3. حساب النقاط الأساسية
        v_base_points := FLOOR(v_total_value * v_points_per_egp);
        NEW.base_points := v_base_points;
        
        -- 4. حساب trust_score
        v_trust_score := calculate_trust_score(NEW.customer_id);
        
        -- 5. جلب base_multiplier (من المنتج أو العرض)
        -- يمكن جلبها من جدول عروض أو من المنتج
        v_base_multiplier := 1.0; -- افتراضي
        
        -- 6. حساب volume_factor (اختياري - حسب الكمية)
        v_volume_factor := 1.0; -- يمكن حسابه بناءً على total_weight
        
        -- 7. حساب المضاعف النهائي
        v_final_multiplier := v_base_multiplier * v_trust_score * v_volume_factor;
        NEW.final_multiplier := v_final_multiplier;
        
        -- 8. حساب نقاط المتجر (البونص)
        IF v_final_multiplier > 1.0 THEN
            v_bonus_points := FLOOR(v_base_points * (v_final_multiplier - 1.0));
        ELSE
            v_bonus_points := 0;
        END IF;
        
        NEW.bonus_points := v_bonus_points;
        NEW.total_points := v_base_points + v_bonus_points;
        
        -- 9. منح نقاط المتجر فوراً (متاح للاستخدام)
        UPDATE profiles
        SET 
            store_points = COALESCE(store_points, 0) + v_bonus_points,
            updated_at = NOW()
        WHERE id = NEW.customer_id;
        
        -- 10. تسجيل نقاط المتجر في Audit Log
        IF v_bonus_points > 0 THEN
            INSERT INTO points_transactions (
                profile_id,
                type,
                amount,
                before_balance,
                after_balance,
                source,
                reference_id,
                description
            )
            SELECT 
                NEW.customer_id,
                'store',
                v_bonus_points,
                COALESCE(p.store_points, 0),
                COALESCE(p.store_points, 0) + v_bonus_points,
                'waste_collection_session',
                NEW.id,
                'نقاط متجر من جلسة تجميع (بونص)'
            FROM profiles p
            WHERE p.id = NEW.customer_id;
        END IF;
        
        -- 11. تسجيل نقاط المحفظة (معلق - لا يمكن السحب حتى is_settled = true)
        INSERT INTO points_transactions (
            profile_id,
            type,
            amount,
            before_balance,
            after_balance,
            source,
            reference_id,
            description
        )
        SELECT 
            NEW.customer_id,
            'wallet',
            v_base_points,
            COALESCE(p.wallet_balance, 0),
            COALESCE(p.wallet_balance, 0), -- لا يزال معلق
            'waste_collection_session',
            NEW.id,
            'نقاط محفظة من جلسة تجميع (معلق حتى التسوية)'
        FROM profiles p
        WHERE p.id = NEW.customer_id;
        
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### 5. Function: تسوية الجلسة (Settlement)

```sql
CREATE OR REPLACE FUNCTION settle_collection_session(p_session_id UUID)
RETURNS BOOLEAN AS $$
DECLARE
    v_session RECORD;
    v_base_points INTEGER;
BEGIN
    -- 1. جلب بيانات الجلسة
    SELECT 
        id,
        customer_id,
        base_points,
        is_settled
    INTO v_session
    FROM waste_collection_sessions
    WHERE id = p_session_id;
    
    IF NOT FOUND THEN
        RAISE EXCEPTION 'الجلسة غير موجودة: %', p_session_id;
    END IF;
    
    IF v_session.is_settled = true THEN
        RAISE EXCEPTION 'الجلسة مقفلة بالفعل';
    END IF;
    
    IF v_session.status != 'completed' THEN
        RAISE EXCEPTION 'الجلسة لم تكتمل بعد';
    END IF;
    
    v_base_points := COALESCE(v_session.base_points, 0);
    
    -- 2. تحديث المحفظة المالية (متاح للسحب الآن)
    UPDATE profiles
    SET 
        wallet_balance = COALESCE(wallet_balance, 0) + v_base_points,
        updated_at = NOW()
    WHERE id = v_session.customer_id;
    
    -- 3. تحديث حالة الجلسة
    UPDATE waste_collection_sessions
    SET 
        is_settled = true,
        updated_at = NOW()
    WHERE id = p_session_id;
    
    -- 4. تحديث Audit Log (تغيير الحالة من معلق إلى متاح)
    UPDATE points_transactions
    SET 
        after_balance = (
            SELECT wallet_balance 
            FROM profiles 
            WHERE id = v_session.customer_id
        ),
        description = description || ' - تم التسوية'
    WHERE reference_id = p_session_id
      AND type = 'wallet'
      AND source = 'waste_collection_session';
    
    RETURN true;
END;
$$ LANGUAGE plpgsql;

COMMENT ON FUNCTION settle_collection_session IS 
'تسوية الجلسة - جعل نقاط المحفظة متاحة للسحب النقدي';
```

### 6. Function: التسوية الآلية (Auto Settlement) ⭐

```sql
CREATE OR REPLACE FUNCTION auto_settle_sessions()
RETURNS INTEGER AS $$
DECLARE
    v_auto_settle_hours INTEGER := 24; -- بعد 24 ساعة
    v_settled_count INTEGER := 0;
    v_session RECORD;
BEGIN
    -- جلب الجلسات المؤهلة للتسوية الآلية
    FOR v_session IN
        SELECT 
            id, 
            customer_id, 
            base_points,
            completed_at
        FROM waste_collection_sessions
        WHERE status = 'completed'
          AND is_settled = false
          AND completed_at < NOW() - (v_auto_settle_hours || ' hours')::INTERVAL
        ORDER BY completed_at ASC
        LIMIT 100 -- معالجة دفعات
    LOOP
        -- تحقق بسيط: هل الجلسة صحيحة؟
        -- (يمكن إضافة شروط إضافية هنا مثل: التحقق من الوزن، السعر، etc.)
        
        -- تسوية الجلسة
        PERFORM settle_collection_session(v_session.id);
        
        v_settled_count := v_settled_count + 1;
        
        -- إشعار العميل (اختياري - يمكن إرساله عبر Backend)
        -- يمكن إضافة جدول notifications هنا
        
    END LOOP;
    
    -- تسجيل في Log
    IF v_settled_count > 0 THEN
        INSERT INTO system_logs (
            log_type,
            message,
            metadata
        ) VALUES (
            'auto_settlement',
            format('تم تسوية %s جلسة تلقائياً', v_settled_count),
            json_build_object('count', v_settled_count, 'timestamp', NOW())
        );
    END IF;
    
    RETURN v_settled_count;
END;
$$ LANGUAGE plpgsql;

COMMENT ON FUNCTION auto_settle_sessions IS 
'تسوية تلقائية للجلسات بعد 24 ساعة - يحسن تجربة المستخدم';
```

### 6. Function: سحب من المحفظة المالية (النظام الهجين)

```sql
CREATE OR REPLACE FUNCTION withdraw_from_wallet(
    p_customer_id UUID,
    p_amount_egp DECIMAL(10,2)
)
RETURNS UUID AS $$
DECLARE
    v_points_per_egp INTEGER;
    v_points_required INTEGER;
    v_current_wallet DECIMAL(12,2);
    v_unsettled_sessions INTEGER;
    v_redemption_id UUID;
    v_config RECORD;
    v_daily_withdrawn DECIMAL(10,2);
    v_last_withdrawal TIMESTAMPTZ;
BEGIN
    -- 1. التحقق من رصيد المحفظة
    SELECT wallet_balance INTO v_current_wallet
    FROM profiles
    WHERE id = p_customer_id;
    
    IF v_current_wallet < p_amount_egp THEN
        RAISE EXCEPTION 'رصيد المحفظة غير كافٍ. المتاح: % ج.م، المطلوب: % ج.م', 
            v_current_wallet, p_amount_egp;
    END IF;
    
    -- 2. التحقق من الجلسات غير المقفلة
    SELECT COUNT(*) INTO v_unsettled_sessions
    FROM waste_collection_sessions
    WHERE customer_id = p_customer_id
      AND status = 'completed'
      AND is_settled = false;
    
    IF v_unsettled_sessions > 0 THEN
        RAISE EXCEPTION 'يوجد % جلسة غير مقفلة. لا يمكن السحب إلا بعد تسوية الجلسات', 
            v_unsettled_sessions;
    END IF;
    
    -- 3. جلب إعدادات القيود
    SELECT 
        points_per_egp,
        max_daily_redeem_amount,
        require_approval_above,
        redemption_cooldown_hours
    INTO v_config
    FROM points_configuration
    WHERE is_active = true
    ORDER BY created_at DESC
    LIMIT 1;
    
    v_points_per_egp := COALESCE(v_config.points_per_egp, 100);
    v_points_required := FLOOR(p_amount_egp * v_points_per_egp);
    
    -- 4. التحقق من الحد اليومي
    SELECT COALESCE(SUM(amount_egp), 0) INTO v_daily_withdrawn
    FROM points_redemptions
    WHERE customer_id = p_customer_id
      AND redemption_type = 'cash'
      AND status IN ('pending', 'approved', 'completed')
      AND DATE(created_at) = CURRENT_DATE;
    
    IF (v_daily_withdrawn + p_amount_egp) > COALESCE(v_config.max_daily_redeem_amount, 5000) THEN
        RAISE EXCEPTION 'تم تجاوز الحد اليومي للسحب. الحد: % ج.م، المسحوب اليوم: % ج.م', 
            v_config.max_daily_redeem_amount, v_daily_withdrawn;
    END IF;
    
    -- 5. التحقق من فترة الانتظار (Cooldown)
    SELECT MAX(created_at) INTO v_last_withdrawal
    FROM points_redemptions
    WHERE customer_id = p_customer_id
      AND redemption_type = 'cash'
      AND status IN ('pending', 'approved', 'completed');
    
    IF v_last_withdrawal IS NOT NULL AND 
       (NOW() - v_last_withdrawal) < (COALESCE(v_config.redemption_cooldown_hours, 24) || ' hours')::INTERVAL THEN
        RAISE EXCEPTION 'يجب الانتظار % ساعة بين عمليات السحب', 
            v_config.redemption_cooldown_hours;
    END IF;
    
    -- 6. تحديد الحالة (يتطلب موافقة أم لا)
    DECLARE
        v_status VARCHAR(20);
    BEGIN
        IF p_amount_egp > COALESCE(v_config.require_approval_above, 1000) THEN
            v_status := 'pending_approval';
        ELSE
            v_status := 'pending';
        END IF;
        
        -- 7. إنشاء سجل السحب
        v_redemption_id := uuid_generate_v4();
        
        INSERT INTO points_redemptions (
            id,
            customer_id,
            points_redeemed,
            amount_egp,
            redemption_type,
            status
        ) VALUES (
            v_redemption_id,
            p_customer_id,
            v_points_required,
            p_amount_egp,
            'cash',
            v_status
        );
        
        -- 8. خصم من المحفظة (فقط إذا كانت pending مباشرة)
        IF v_status = 'pending' THEN
            UPDATE profiles
            SET 
                wallet_balance = wallet_balance - p_amount_egp,
                updated_at = NOW()
            WHERE id = p_customer_id;
            
            -- تسجيل في Audit Log
            INSERT INTO points_transactions (
                profile_id,
                type,
                amount,
                before_balance,
                after_balance,
                source,
                reference_id,
                description
            )
            SELECT 
                p_customer_id,
                'wallet',
                -v_points_required,
                p.wallet_balance,
                p.wallet_balance - p_amount_egp,
                'points_redemption',
                v_redemption_id,
                'سحب نقدي من المحفظة: ' || p_amount_egp || ' ج.م'
            FROM profiles p
            WHERE p.id = p_customer_id;
        END IF;
        
        RETURN v_redemption_id;
    END;
END;
$$ LANGUAGE plpgsql;

COMMENT ON FUNCTION withdraw_from_wallet IS 
'سحب نقدي من المحفظة المالية - مع قيود كاملة';
```

### 7. Function: استخدام نقاط المتجر

```sql
CREATE OR REPLACE FUNCTION use_store_points(
    p_customer_id UUID,
    p_points INTEGER,
    p_redemption_type VARCHAR DEFAULT 'product'
)
RETURNS UUID AS $$
DECLARE
    v_current_store_points INTEGER;
    v_redemption_id UUID;
BEGIN
    -- 1. التحقق من رصيد نقاط المتجر
    SELECT store_points INTO v_current_store_points
    FROM profiles
    WHERE id = p_customer_id;
    
    IF v_current_store_points < p_points THEN
        RAISE EXCEPTION 'رصيد نقاط المتجر غير كافٍ. المتاح: %, المطلوب: %', 
            v_current_store_points, p_points;
    END IF;
    
    -- 2. إنشاء سجل الاستبدال
    v_redemption_id := uuid_generate_v4();
    
    INSERT INTO points_redemptions (
        id,
        customer_id,
        points_redeemed,
        amount_egp,
        redemption_type,
        status
    ) VALUES (
        v_redemption_id,
        p_customer_id,
        p_points,
        0, -- نقاط المتجر لا قيمة نقدية
        p_redemption_type,
        'pending'
    );
    
    -- 3. خصم من نقاط المتجر
    UPDATE profiles
    SET 
        store_points = store_points - p_points,
        updated_at = NOW()
    WHERE id = p_customer_id;
    
    -- 4. تسجيل في Audit Log
    INSERT INTO points_transactions (
        profile_id,
        type,
        amount,
        before_balance,
        after_balance,
        source,
        reference_id,
        description
    )
    SELECT 
        p_customer_id,
        'store',
        -p_points,
        p.store_points,
        p.store_points - p_points,
        'points_redemption',
        v_redemption_id,
        'استخدام نقاط المتجر: ' || p_redemption_type
    FROM profiles p
    WHERE p.id = p_customer_id;
    
    RETURN v_redemption_id;
END;
$$ LANGUAGE plpgsql;

COMMENT ON FUNCTION use_store_points IS 
'استخدام نقاط المتجر في شراء منتجات أو هدايا';
```

```sql
    updated_at = NOW()
WHERE id = NEW.customer_id;
```

#### ✅ الحل 1: تسريع التسوية الآلية (الأفضل للبداية) ⭐⭐⭐⭐⭐

**الفكرة:**
- تسوية تلقائية للجلسات بعد فترة زمنية محددة (مثلاً 24 ساعة)
- تحقق بسيط من البيانات قبل التسوية
- أمان كامل بدون مخاطر مالية

**الميزات:**
- ✅ **أمان كامل** - لا مخاطر مالية
- ✅ **تجربة مستخدم ممتازة** - نقاط متاحة بسرعة
- ✅ **سهل التنفيذ** - لا يحتاج تغييرات معقدة
- ✅ **لا يحتاج إدارة ديون** - كل شيء آمن

**التنفيذ:**
```sql
-- Function للتسوية الآلية
CREATE OR REPLACE FUNCTION auto_settle_sessions()
RETURNS INTEGER AS $$
DECLARE
    v_auto_settle_hours INTEGER := 24; -- بعد 24 ساعة
    v_settled_count INTEGER := 0;
    v_session RECORD;
BEGIN
    -- جلب الجلسات المؤهلة للتسوية الآلية
    FOR v_session IN
        SELECT id, customer_id, base_points
        FROM waste_collection_sessions
        WHERE status = 'completed'
          AND is_settled = false
          AND completed_at < NOW() - (v_auto_settle_hours || ' hours')::INTERVAL
        ORDER BY completed_at ASC
        LIMIT 100 -- معالجة دفعات
    LOOP
        -- تحقق بسيط: هل الجلسة صحيحة؟
        -- (يمكن إضافة شروط إضافية هنا)
        
        -- تسوية الجلسة
        PERFORM settle_collection_session(v_session.id);
        
        v_settled_count := v_settled_count + 1;
        
        -- إشعار العميل
        PERFORM notify_customer(
            v_session.customer_id,
            'wallet_available',
            json_build_object(
                'amount', v_session.base_points / 100,
                'message', 'تم اعتماد جلسة التجميع الخاصة بك. المحفظة متاحة الآن.'
            )
        );
    END LOOP;
    
    RETURN v_settled_count;
END;
$$ LANGUAGE plpgsql;

-- Cron Job للتشغيل كل ساعة
-- يمكن استخدام pg_cron أو cron خارجي
```

```sql
ADD COLUMN credit_used NUMERIC(10,2) DEFAULT 0;

-- حساب الحد الائتماني بناءً على trust_score
CREATE OR REPLACE FUNCTION calculate_credit_limit(p_customer_id UUID)
RETURNS NUMERIC(10,2) AS $$
DECLARE
    v_trust_score NUMERIC(3,2);
    v_credit_limit NUMERIC(10,2);
BEGIN
    SELECT trust_score INTO v_trust_score
    FROM profiles
    WHERE id = p_customer_id;
    
    v_credit_limit := CASE
        WHEN v_trust_score >= 1.2 THEN 100.00  -- عملاء موثوقون
        WHEN v_trust_score >= 1.0 THEN 50.00    -- عملاء عاديون
        ELSE 0.00                               -- عملاء جدد/مشبوهون
    END;
    
    RETURN v_credit_limit;
END;
$$ LANGUAGE plpgsql;

-- تحديث الحد الائتماني تلقائياً
CREATE OR REPLACE FUNCTION update_credit_limits()
RETURNS VOID AS $$
BEGIN
    UPDATE profiles
    SET credit_limit = calculate_credit_limit(id)
    WHERE trust_score IS NOT NULL;
END;
$$ LANGUAGE plpgsql;
```

### 🔧 Function: الشراء من المتجر (مع دعم الائتمان)

```sql
CREATE OR REPLACE FUNCTION purchase_product(
    p_customer_id UUID,
    p_product_price_egp DECIMAL(10,2),
    p_allow_credit BOOLEAN DEFAULT false
)
RETURNS JSON AS $$
DECLARE
    v_customer RECORD;
    v_points_per_egp INTEGER;
    v_product_price_points INTEGER;
    v_store_points_to_use INTEGER;
    v_wallet_available_to_use DECIMAL(10,2);
    v_wallet_pending_to_use DECIMAL(10,2);
    v_credit_to_use DECIMAL(10,2);
    v_total_available DECIMAL(10,2);
    v_result JSON;
BEGIN
    -- 1. جلب بيانات العميل
    SELECT 
        store_points,
        wallet_balance,
        credit_limit,
        credit_used,
        trust_score
    INTO v_customer
    FROM profiles
    WHERE id = p_customer_id;
    
    IF NOT FOUND THEN
        RETURN json_build_object(
            'success', false,
            'message', 'العميل غير موجود'
        );
    END IF;
    
    -- 2. جلب معامل التحويل
    SELECT points_per_egp INTO v_points_per_egp
    FROM points_configuration
    WHERE is_active = true
    ORDER BY created_at DESC
    LIMIT 1;
    
    v_points_per_egp := COALESCE(v_points_per_egp, 100);
    v_product_price_points := (p_product_price_egp * v_points_per_egp)::INTEGER;
    
    -- 3. حساب ما يمكن استخدامه من كل مصدر
    
    -- A. نقاط المتجر (أولاً)
    v_store_points_to_use := LEAST(
        v_customer.store_points, 
        v_product_price_points
    );
    
    -- B. المحفظة المتاحة (ثانياً)
    -- التحقق من الجلسات المعلقة
    DECLARE
        v_unsettled_sessions INTEGER;
        v_wallet_available DECIMAL(10,2);
    BEGIN
        SELECT COUNT(*) INTO v_unsettled_sessions
        FROM waste_collection_sessions
        WHERE customer_id = p_customer_id
          AND status = 'completed'
          AND is_settled = false;
        
        -- المحفظة المتاحة = wallet_balance إذا كانت جميع الجلسات مقفلة
        IF v_unsettled_sessions = 0 THEN
            v_wallet_available := v_customer.wallet_balance;
        ELSE
            v_wallet_available := 0; -- لا يمكن استخدام المحفظة إذا كانت هناك جلسات معلقة
        END IF;
        
        v_wallet_available_to_use := LEAST(
            v_wallet_available,
            (v_product_price_points - v_store_points_to_use)::DECIMAL / v_points_per_egp
        );
    END;
    
    -- C. الائتمان من المحفظة المعلقة (ثالثاً - إن سُمح)
    IF p_allow_credit AND v_customer.trust_score >= 1.2 THEN
        DECLARE
            v_credit_available DECIMAL(10,2);
            v_remaining_needed DECIMAL(10,2);
            v_pending_wallet DECIMAL(10,2);
        BEGIN
            v_credit_available := v_customer.credit_limit - v_customer.credit_used;
            
            -- حساب المحفظة المعلقة
            SELECT COALESCE(SUM(base_points), 0) / 100 INTO v_pending_wallet
            FROM waste_collection_sessions
            WHERE customer_id = p_customer_id
              AND status = 'completed'
              AND is_settled = false;
            
            v_remaining_needed := p_product_price_egp - 
                                  (v_store_points_to_use::DECIMAL / v_points_per_egp) -
                                  v_wallet_available_to_use;
            
            -- يمكن استخدام الائتمان حتى حد المحفظة المعلقة والائتمان المتاح
            v_wallet_pending_to_use := LEAST(
                v_pending_wallet,
                v_credit_available,
                v_remaining_needed
            );
        END;
    ELSE
        v_wallet_pending_to_use := 0;
    END IF;
    
    -- 4. حساب الإجمالي المتاح
    v_total_available := 
        (v_store_points_to_use::DECIMAL / v_points_per_egp) +
        v_wallet_available_to_use +
        v_wallet_pending_to_use;
    
    -- 5. التحقق من الكفاية
    IF v_total_available < p_product_price_egp THEN
        RETURN json_build_object(
            'success', false,
            'message', 'الرصيد غير كافٍ',
            'breakdown', json_build_object(
                'store_points_available', v_customer.store_points,
                'wallet_available', v_wallet_available,
                'credit_available', COALESCE(v_customer.credit_limit - v_customer.credit_used, 0),
                'total_available', v_total_available,
                'needed', p_product_price_egp,
                'shortage', p_product_price_egp - v_total_available
            )
        );
    END IF;
    
    -- 6. إجراء الخصم (نفس الكود من المستند الأصلي)
    -- ... (يتم الخصم من كل مصدر)
    
    RETURN json_build_object(
        'success', true,
        'payment_breakdown', json_build_object(
            'store_points_used', v_store_points_to_use,
            'wallet_available_used', v_wallet_available_to_use,
            'credit_used', v_wallet_pending_to_use,
            'total_paid', p_product_price_egp
        )
    );
    
END;
$$ LANGUAGE plpgsql;
```

- ✅ **عدم التأثر بالتحديثات اللاحقة**: أي تحديث لاحق لسعر البورصة لا يؤثر على جلسات سابقة

```sql
    FROM stock_exchange
    WHERE sub_category_id = v_product.subcategory_id

    LIMIT 1;

ELSE
    -- في UPDATE: استخدام السعر المجمد الموجود

END IF;
```

```sql
CREATE OR REPLACE FUNCTION calculate_collection_item()
RETURNS TRIGGER AS $$
DECLARE
    v_product RECORD;
    v_buy_price DECIMAL(10,2);
    v_points_per_egp INTEGER;
    v_total_weight DECIMAL(10,3);
    v_monetary_value DECIMAL(10,2);
    v_multiplier DECIMAL(3,2);
BEGIN
    -- 1. جلب بيانات المنتج
    SELECT 
        wda.weight,
        wda.subcategory_id,
        wda.counting_method,
        wda.points_multiplier,
        wda.has_fixed_weight
    INTO v_product
    FROM waste_data_admin wda
    WHERE wda.id = NEW.waste_data_id;
    
    IF NOT FOUND THEN
        RAISE EXCEPTION 'المنتج غير موجود: %', NEW.waste_data_id;
    END IF;
    
    -- 2. نسخ طريقة العد
    NEW.counting_method := v_product.counting_method;
    
    -- 3. حساب الوزن الإجمالي
    IF NEW.counting_method = 'pieces' THEN
        IF v_product.weight IS NULL OR v_product.weight = 0 THEN
            RAISE EXCEPTION 'المنتج يتطلب وزن محدد للعد بالقطعة';
        END IF;
        
        IF NEW.quantity IS NULL OR NEW.quantity <= 0 THEN
            RAISE EXCEPTION 'يجب إدخال عدد القطع';
        END IF;
        
        v_total_weight := (v_product.weight / 1000.0) * NEW.quantity;
        
    ELSIF NEW.counting_method = 'weight' THEN
        IF NEW.actual_weight IS NULL OR NEW.actual_weight <= 0 THEN
            RAISE EXCEPTION 'يجب إدخال الوزن الفعلي';
        END IF;
        
        v_total_weight := NEW.actual_weight;
        NEW.quantity := 1;
        
    ELSE
        RAISE EXCEPTION 'طريقة عد غير صالحة: %', NEW.counting_method;
    END IF;
    
    NEW.total_weight := v_total_weight;
    
    -- 4. جلب وتجميد سعر البورصة (فقط عند INSERT)
    IF TG_OP = 'INSERT' THEN
        SELECT buy_price INTO v_buy_price
        FROM stock_exchange
        WHERE sub_category_id = v_product.subcategory_id
        ORDER BY last_update DESC
        LIMIT 1;
        
        v_buy_price := COALESCE(v_buy_price, 0);
        NEW.unit_price := v_buy_price; -- ⭐ تجميد السعر
    ELSE
        -- في UPDATE: استخدام السعر المجمد
        v_buy_price := COALESCE(NEW.unit_price, OLD.unit_price, 0);
    END IF;
    
    -- 5. حساب القيمة النقدية
    v_monetary_value := v_total_weight * v_buy_price;
    NEW.monetary_value := v_monetary_value;
    NEW.total_price := v_monetary_value;
    
    -- 6. جلب معامل التحويل
    SELECT points_per_egp INTO v_points_per_egp
    FROM points_configuration
    WHERE is_active = true
    ORDER BY created_at DESC
    LIMIT 1;
    
    v_points_per_egp := COALESCE(v_points_per_egp, 100);
    
    -- 7. جلب معامل المضاعفة
    v_multiplier := COALESCE(v_product.points_multiplier, 1.0);
    
    -- 8. حساب النقاط (فصل المالي عن البونص)
    NEW.earned_points_financial := FLOOR(v_monetary_value * v_points_per_egp);
    NEW.earned_points_bonus := FLOOR(v_monetary_value * v_points_per_egp * (v_multiplier - 1.0));
    NEW.earned_points := NEW.earned_points_financial + NEW.earned_points_bonus;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

```sql
CREATE OR REPLACE FUNCTION award_points_on_completion()
RETURNS TRIGGER AS $$
DECLARE
    v_financial_points INTEGER;
    v_bonus_points INTEGER;
BEGIN
    IF NEW.status = 'completed' AND 
       (OLD.status IS NULL OR OLD.status != 'completed') THEN
        
        -- حساب النقاط المالية والبونص
        SELECT 
            COALESCE(SUM(earned_points_financial), 0),
            COALESCE(SUM(earned_points_bonus), 0)
        INTO v_financial_points, v_bonus_points
        FROM waste_collection_items
        WHERE session_id = NEW.id;
        
        -- تحديث رصيد النقاط (فصل المالي عن البونص)
        UPDATE profiles
        SET 
            financial_points = COALESCE(financial_points, 0) + v_financial_points,
            bonus_points = COALESCE(bonus_points, 0) + v_bonus_points,
            loyalty_points = COALESCE(loyalty_points, 0) + v_financial_points + v_bonus_points,
            updated_at = NOW()
        WHERE id = NEW.customer_id;
        
        -- تسجيل في Audit Log
        INSERT INTO points_transactions (
            customer_id,
            transaction_type,
            points_amount,
            before_balance_financial,
            after_balance_financial,
            before_balance_bonus,
            after_balance_bonus,
            source,
            reference_id,
            description
        )
        SELECT 
            NEW.customer_id,
            'EARNED',
            v_financial_points + v_bonus_points,
            COALESCE(p.financial_points, 0),
            COALESCE(p.financial_points, 0) + v_financial_points,
            COALESCE(p.bonus_points, 0),
            COALESCE(p.bonus_points, 0) + v_bonus_points,
            'waste_collection_session',
            NEW.id,
            'نقاط من جلسة تجميع المخلفات'
        FROM profiles p
        WHERE p.id = NEW.customer_id;
        
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

### Function: استبدال النقاط (محدث مع القيود)

```sql
CREATE OR REPLACE FUNCTION redeem_points(
    p_customer_id UUID,
    p_points INTEGER,
    p_redemption_type VARCHAR DEFAULT 'cash'
)
RETURNS UUID AS $$
DECLARE
    v_points_per_egp INTEGER;
    v_amount_egp DECIMAL(10,2);
    v_current_financial INTEGER;
    v_current_bonus INTEGER;
    v_redemption_id UUID;
    v_config RECORD;
    v_daily_redeemed DECIMAL(10,2);
    v_last_redemption TIMESTAMPTZ;
    v_unsettled_sessions INTEGER;
BEGIN
    -- 1. التحقق من رصيد النقاط المالية (فقط المالية قابلة للسحب النقدي)
    SELECT financial_points, bonus_points INTO v_current_financial, v_current_bonus
    FROM profiles
    WHERE id = p_customer_id;
    
    IF p_redemption_type = 'cash' THEN
        -- الاستبدال النقدي: فقط من النقاط المالية
        IF v_current_financial < p_points THEN
            RAISE EXCEPTION 'رصيد النقاط المالية غير كافٍ. المتاح: %, المطلوب: %', 
                v_current_financial, p_points;
        END IF;
    ELSE
        -- المتجر/الهدايا: من المالية أو البونص
        IF (v_current_financial + v_current_bonus) < p_points THEN
            RAISE EXCEPTION 'رصيد النقاط غير كافٍ. المتاح: %, المطلوب: %', 
                (v_current_financial + v_current_bonus), p_points;
        END IF;
    END IF;
    
    -- 2. التحقق من الجلسات غير المقفلة (للاستبدال النقدي فقط)
    IF p_redemption_type = 'cash' THEN
        SELECT COUNT(*) INTO v_unsettled_sessions
        FROM waste_collection_sessions
        WHERE customer_id = p_customer_id
          AND status = 'completed'
          AND is_settled = false;
        
        IF v_unsettled_sessions > 0 THEN
            RAISE EXCEPTION 'يوجد % جلسة غير مقفلة. لا يمكن السحب النقدي إلا بعد تسوية الجلسات', 
                v_unsettled_sessions;
        END IF;
    END IF;
    
    -- 3. جلب إعدادات القيود
    SELECT 
        points_per_egp,
        max_daily_redeem_amount,
        require_approval_above,
        redemption_cooldown_hours
    INTO v_config
    FROM points_configuration
    WHERE is_active = true
    ORDER BY created_at DESC
    LIMIT 1;
    
    v_points_per_egp := COALESCE(v_config.points_per_egp, 100);
    v_amount_egp := p_points::DECIMAL / v_points_per_egp;
    
    -- 4. التحقق من الحد اليومي
    IF p_redemption_type = 'cash' THEN
        SELECT COALESCE(SUM(amount_egp), 0) INTO v_daily_redeemed
        FROM points_redemptions
        WHERE customer_id = p_customer_id
          AND redemption_type = 'cash'
          AND status IN ('pending', 'approved', 'completed')
          AND DATE(created_at) = CURRENT_DATE;
        
        IF (v_daily_redeemed + v_amount_egp) > COALESCE(v_config.max_daily_redeem_amount, 5000) THEN
            RAISE EXCEPTION 'تم تجاوز الحد اليومي للاستبدال. الحد: % ج.م، المستبدل اليوم: % ج.م', 
                v_config.max_daily_redeem_amount, v_daily_redeemed;
        END IF;
    END IF;
    
    -- 5. التحقق من فترة الانتظار (Cooldown)
    IF p_redemption_type = 'cash' THEN
        SELECT MAX(created_at) INTO v_last_redemption
        FROM points_redemptions
        WHERE customer_id = p_customer_id
          AND redemption_type = 'cash'
          AND status IN ('pending', 'approved', 'completed');
        
        IF v_last_redemption IS NOT NULL AND 
           (NOW() - v_last_redemption) < (COALESCE(v_config.redemption_cooldown_hours, 24) || ' hours')::INTERVAL THEN
            RAISE EXCEPTION 'يجب الانتظار % ساعة بين عمليات الاستبدال', 
                v_config.redemption_cooldown_hours;
        END IF;
    END IF;
    
    -- 6. تحديد الحالة (يتطلب موافقة أم لا)
    DECLARE
        v_status VARCHAR(20);
    BEGIN
        IF p_redemption_type = 'cash' AND 
           v_amount_egp > COALESCE(v_config.require_approval_above, 1000) THEN
            v_status := 'pending_approval';
        ELSE
            v_status := 'pending';
        END IF;
        
        -- 7. إنشاء سجل الاستبدال
        v_redemption_id := uuid_generate_v4();
        
        INSERT INTO points_redemptions (
            id,
            customer_id,
            points_redeemed,
            amount_egp,
            redemption_type,
            status
        ) VALUES (
            v_redemption_id,
            p_customer_id,
            p_points,
            v_amount_egp,
            p_redemption_type,
            v_status
        );
        
        -- 8. خصم النقاط (فقط إذا كانت pending مباشرة)
        IF v_status = 'pending' THEN
            IF p_redemption_type = 'cash' THEN
                UPDATE profiles
                SET 
                    financial_points = financial_points - p_points,
                    loyalty_points = loyalty_points - p_points,
                    updated_at = NOW()
                WHERE id = p_customer_id;
            ELSE
                -- المتجر/الهدايا: خصم من المالية أولاً، ثم البونص
                UPDATE profiles
                SET 
                    financial_points = GREATEST(0, financial_points - LEAST(p_points, financial_points)),
                    bonus_points = GREATEST(0, bonus_points - GREATEST(0, p_points - financial_points)),
                    loyalty_points = loyalty_points - p_points,
                    updated_at = NOW()
                WHERE id = p_customer_id;
            END IF;
            
            -- تسجيل في Audit Log
            INSERT INTO points_transactions (
                customer_id,
                transaction_type,
                points_amount,
                before_balance_financial,
                after_balance_financial,
                before_balance_bonus,
                after_balance_bonus,
                source,
                reference_id,
                description
            )
            SELECT 
                p_customer_id,
                'REDEEMED',
                -p_points,
                p.financial_points,
                CASE WHEN p_redemption_type = 'cash' 
                     THEN p.financial_points - p_points 
                     ELSE GREATEST(0, p.financial_points - LEAST(p_points, p.financial_points))
                END,
                p.bonus_points,
                CASE WHEN p_redemption_type = 'cash' 
                     THEN p.bonus_points
                     ELSE GREATEST(0, p.bonus_points - GREATEST(0, p_points - p.financial_points))
                END,
                'points_redemption',
                v_redemption_id,
                'استبدال نقاط: ' || p_redemption_type
            FROM profiles p
            WHERE p.id = p_customer_id;
        END IF;
        
        RETURN v_redemption_id;
    END;
END;
$$ LANGUAGE plpgsql;
```

### 🔄 التدفق العام (High-Level Flow)

```
[العميل]
   ↓
[إدخال مخلفات في التطبيق]
   ↓
[حساب تقديري]
(قيمة تقديرية + نقاط متجر متوقعة)
   ↓
[تأكيد الطلب]
   ↓
-----------------------------
| مرحلة الجمع الفعلي |
-----------------------------
   ↓
[وزن حقيقي للمخلفات]
   ↓
[تثبيت سعر الوحدة]
(unit_price) ⭐ مجمد عند الجلسة
   ↓
[حساب القيمة المالية]
(total_value = Σ actual_weight × unit_price)
   ↓
[حساب النقاط الأساسية]
(base_points = FLOOR(total_value × points_per_egp))
   ↓
[تطبيق المضاعف الذكي]
(trust_score, volume_factor)
   ↓
[فصل النقاط]
↙                 ↘
[Wallet Balance]   [Store Points]
(فلوس حقيقية)      (تحفيز)
   ↓                 ↓
[Pending]         [Available]
(معلق)            (متاح فوراً)
   ↓
[اعتماد إداري]
(is_settled = true)
   ↓
[تسجيل محاسبي نهائي]
   ↓
[السحب / الاستخدام]
```

### 🔐 نقاط التحكم الحرجة (Control Points)

| النقطة | الشرط | الحماية |
|--------|-------|---------|
| ❗ تحديث المحفظة | لا يتم إلا بعد `is_settled = true` | Trigger + Function Check |
| ❗ تثبيت السعر | `unit_price` يُثبت وقت الجلسة | INSERT فقط، لا UPDATE |
| ❗ المضاعف | لا يؤثر على Wallet (فقط Store) | Logic في `calculate_session_points()` |
| ❗ السحب النقدي | فقط من `wallet_balance` بعد التسوية | `withdraw_from_wallet()` |
| ❗ نقاط المتجر | غير قابلة للسحب النقدي | `use_store_points()` فقط |

## 📘 السياسة المحاسبية الرسمية (Accounting Policy)

### 1️⃣ التعريفات الرسمية

| المصطلح | التعريف |
|---------|---------|
| **Wallet Balance** | رصيد نقدي مستحق للعميل مقابل مخلفات تم استلامها فعليًا وتم اعتمادها محاسبيًا (`is_settled = true`) |
| **Store Points** | نقاط تحفيزية غير نقدية تُستخدم داخل المنصة فقط (متجر، هدايا، خصومات) |
| **Monetary Value** | القيمة المالية للمخلفات بناءً على الوزن الفعلي والسعر المثبت عند الجلسة (`unit_price`) |
| **Settled Session** | جلسة تم اعتمادها محاسبيًا (`is_settled = true`) ولا يمكن تعديلها أو إلغاء نقاطها |
| **Base Points** | النقاط الأساسية المحسوبة من القيمة المالية الفعلية (تذهب للمحفظة) |
| **Bonus Points** | نقاط البونص المحسوبة من المضاعف الذكي (تذهب لنقاط المتجر) |

---

### 2️⃣ مبادئ محاسبية معتمدة

#### 🔹 مبدأ التحقق (Verification Principle)

> **لا تُسجل أي قيمة مالية إلا بعد وزن فعلي واعتماد إداري.**

**التطبيق:**
- عند `status = 'completed'`: تسجيل `base_points` في `points_transactions` بحالة "معلق"
- عند `is_settled = true`: تحديث `wallet_balance` وتغيير الحالة إلى "متاح"

---

#### 🔹 مبدأ الثبات السعري (Price Locking)

> **السعر المستخدم في الحساب هو السعر المثبت وقت الجلسة وليس سعر السوق لاحقًا.**

**التطبيق:**
```sql
-- في calculate_collection_item()
IF TG_OP = 'INSERT' THEN
    -- جلب السعر الحالي وتثبيته
    SELECT buy_price INTO v_buy_price
    FROM stock_exchange
    WHERE sub_category_id = v_product.subcategory_id
    ORDER BY last_update DESC
    LIMIT 1;
    
    NEW.unit_price := v_buy_price; -- ⭐ تثبيت السعر
END IF;
```

---

#### 🔹 مبدأ الفصل (Separation Principle)

> **يتم الفصل التام بين:**
> - **التزامات نقدية** (Wallet Balance) - قابلة للسحب النقدي
> - **حوافز تسويقية** (Store Points) - للمتجر والهدايا فقط

**التطبيق:**
```sql
-- في calculate_session_points()
base_points → wallet_balance (معلق حتى التسوية)
bonus_points → store_points (متاح فوراً)
```

---

### 3️⃣ آلية الاعتراف بالإيراد / الالتزام

| المرحلة | المعالجة المحاسبية | الحالة |
|---------|-------------------|--------|
| **إدخال الطلب** | لا يوجد قيد محاسبي | تقديري فقط |
| **الوزن الفعلي** | تسجيل `base_points` في `points_transactions` | `type = 'wallet'`, `status = 'pending'` |
| **اعتماد الجلسة** (`is_settled = true`) | تحديث `wallet_balance` | `status = 'available'` |
| **السحب** | خصم من `wallet_balance` | `type = 'wallet'`, `amount = negative` |

---

### 4️⃣ قيود محاسبية مبسطة (Conceptual)

#### عند اعتماد الجلسة (`is_settled = true`):

```
Debit:  Inventory (المخلفات المستلمة)
Credit: Wallet Liability (رصيد العميل المستحق)
```

#### عند السحب:

```
Debit:  Wallet Liability (رصيد العميل)
Credit: Cash / Bank Transfer / Wallet (الخروج النقدي)
```

### 🎯 الهدف

---

### 🧨 التهديدات الرئيسية والحلول

#### 🔴 1. تضخيم النقاط عبر Bug أو Exploit

**الخطر:**
- زيادة رصيد بدون مقابل حقيقي
- تعديل `wallet_balance` مباشرة

**الحماية:**
```sql
-- ✅ Triggers على مستوى DB
CREATE TRIGGER trigger_calculate_session_points
BEFORE UPDATE ON waste_collection_sessions
FOR EACH ROW
EXECUTE FUNCTION calculate_session_points();

-- ✅ Audit Log لكل تغيير
INSERT INTO points_transactions (
    profile_id, type, amount,
    before_balance, after_balance,
    source, reference_id
) VALUES (...);
```

**التنفيذ:**
- منع التحديث المباشر لـ `wallet_balance` و `store_points`
- جميع التحديثات عبر Functions فقط
- تسجيل كل معاملة في `points_transactions`

---

#### 🔴 2. استغلال المضاعف (Multiplier Abuse)

**الخطر:**
- تحويل العروض إلى فلوس حقيقية
- `multiplier = 2.0` → ضعف النقاط المالية

**الحماية:**
```sql
-- ✅ المضاعف يؤثر فقط على Store Points
base_points = FLOOR(total_value × points_per_egp)  -- للمحفظة
bonus_points = FLOOR(base_points × (final_multiplier - 1.0))  -- للمتجر فقط

-- ✅ Cap أعلى للمضاعف
final_multiplier = GREATEST(0.5, LEAST(2.0, base_multiplier × trust_score))
```

**التنفيذ:**
- `base_points` دائماً = القيمة الفعلية (لا يتأثر بالمضاعف)
- `bonus_points` فقط = البونص (غير قابلة للسحب النقدي)

---

#### 🔴 3. السحب قبل الإغلاق المحاسبي

**الخطر:**
- سحب فلوس قبل بيع المخلفات
- سحب من جلسات غير مقفلة

**الحماية:**
```sql
-- ✅ شرط is_settled = true
CREATE OR REPLACE FUNCTION withdraw_from_wallet(...)
RETURNS UUID AS $$
BEGIN
    -- التحقق من الجلسات غير المقفلة
    SELECT COUNT(*) INTO v_unsettled_sessions
    FROM waste_collection_sessions
    WHERE customer_id = p_customer_id
      AND status = 'completed'
      AND is_settled = false;
    
    IF v_unsettled_sessions > 0 THEN
        RAISE EXCEPTION 'يوجد % جلسة غير مقفلة', v_unsettled_sessions;
    END IF;
    
    -- ...
END;
$$ LANGUAGE plpgsql;
```

**التنفيذ:**
- لا يمكن السحب إلا بعد `is_settled = true`
- التحقق في `withdraw_from_wallet()` قبل كل سحب

---

#### 🔴 4. هجوم تكرار الطلبات (Replay / Automation)

**الخطر:**
- توليد جلسات وهمية
- تكرار نفس الجلسة عدة مرات
- بوت يولد: 1000 جلسة/يوم × 10 جنيه = 10,000 جنيه

**الحماية:**
- ✅ **Rate Limiting**: حد أقصى للجلسات اليومية
- ✅ **Device Fingerprinting**: ربط الجلسات بالجهاز
- ✅ **سلوك غير طبيعي → مراجعة**: Alert عند أنماط مشبوهة
- ✅ **Timestamp Validation**: منع إعادة استخدام نفس البيانات
- ✅ **CAPTCHA**: على الطلبات المشبوهة

**التنفيذ:**
```sql
-- A. Rate Limiting على مستوى العميل
ALTER TABLE profiles
ADD COLUMN daily_sessions_count INTEGER DEFAULT 0,
ADD COLUMN last_session_date DATE;

-- B. Device Fingerprinting
ALTER TABLE waste_collection_sessions
ADD COLUMN device_id VARCHAR(100),
ADD COLUMN ip_address INET;

-- C. Anomaly Detection Table
CREATE TABLE suspicious_activities (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES profiles(id),
    session_id UUID REFERENCES waste_collection_sessions(id),
    reason TEXT,
    severity VARCHAR(20), -- low, medium, high
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- D. التحقق في calculate_session_points()
DECLARE
    v_daily_count INTEGER;
    v_duplicate_count INTEGER;
BEGIN
    -- التحقق من الحد اليومي
    SELECT daily_sessions_count INTO v_daily_count
    FROM profiles
    WHERE id = NEW.customer_id;
    
    IF v_daily_count >= 5 THEN -- حد أقصى 5 جلسات/يوم
        INSERT INTO suspicious_activities (
            customer_id,
            session_id,
            reason,
            severity
        ) VALUES (
            NEW.customer_id,
            NEW.id,
            'تجاوز الحد اليومي للجلسات',
            'high'
        );
        
        RAISE EXCEPTION 'تم تجاوز الحد اليومي للجلسات';
    END IF;
    
    -- التحقق من التكرار
    SELECT COUNT(*) INTO v_duplicate_count
    FROM waste_collection_sessions
    WHERE customer_id = NEW.customer_id
      AND DATE(created_at) = CURRENT_DATE
      AND ABS(total_value - v_total_value) < 0.01
      AND ABS(EXTRACT(EPOCH FROM (created_at - NEW.created_at))) < 60;
    
    IF v_duplicate_count > 1 THEN
        INSERT INTO suspicious_activities (
            customer_id,
            session_id,
            reason,
            severity
        ) VALUES (
            NEW.customer_id,
            NEW.id,
            'جلسة مكررة مشبوهة',
            'medium'
        );
        
        RAISE WARNING 'جلسة مشبوهة - مراجعة مطلوبة';
    END IF;
    
    -- تحديث العداد اليومي
    UPDATE profiles
    SET 
        daily_sessions_count = CASE 
            WHEN last_session_date = CURRENT_DATE 
            THEN daily_sessions_count + 1 
            ELSE 1 
        END,
        last_session_date = CURRENT_DATE
    WHERE id = NEW.customer_id;
END;
```

**النتيجة:** 🟡 **يحتاج تعزيز** → ✅ **محمي بعد التنفيذ**

#### 🔴 5. تلاعب داخلي (Insider Threat)

**الخطر:**
- تعديل أوزان أو أسعار
- منح نقاط بدون مقابل

**الحماية:**
- ✅ **سجل تدقيق لا يمكن تعديله**: `points_transactions` Read-Only بعد الإدراج

#### 🔴 3. خطر السيولة (Liquidity Risk)

**السيناريو:**
```
إذا كان:
- إجمالي Wallet Balance = 500,000 ج.م
- السيولة المتاحة = 100,000 ج.م
→ مشكلة سيولة محتملة
```

**الحماية:**
- ✅ **حساب Cash Flow Projection شهري**
- ✅ **وضع حد أقصى للسحب اليومي الإجمالي**
- ✅ **احتياطي نقدي = 20% من إجمالي Wallet Balance**

**التنفيذ:**
```sql
-- إضافة جدول لتتبع السيولة
CREATE TABLE liquidity_tracking (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    date DATE UNIQUE,
    total_wallet_balance DECIMAL(12,2),
    available_cash DECIMAL(12,2),
    liquidity_ratio DECIMAL(5,2), -- available / total
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Function للتحقق من السيولة
CREATE OR REPLACE FUNCTION check_liquidity()
RETURNS BOOLEAN AS $$
DECLARE
    v_total_balance DECIMAL(12,2);
    v_available_cash DECIMAL(12,2);
    v_ratio DECIMAL(5,2);
BEGIN
    SELECT 
        SUM(wallet_balance),
        (SELECT available_cash FROM liquidity_tracking ORDER BY date DESC LIMIT 1)
    INTO v_total_balance, v_available_cash
    FROM profiles;
    
    v_ratio := (v_available_cash / NULLIF(v_total_balance, 0)) * 100;
    
    -- إذا كانت السيولة أقل من 20%
    IF v_ratio < 20 THEN
        RAISE WARNING 'تحذير: نسبة السيولة منخفضة: %', v_ratio;
        RETURN false;
    END IF;
    
    RETURN true;
END;
$$ LANGUAGE plpgsql;
```

```typescript
async function settleSession(sessionId: string) {
  // 1. التحقق من البيانات
  const session = await getSession(sessionId);
  
  if (!session) throw new Error('الجلسة غير موجودة');
  if (session.is_settled) throw new Error('الجلسة معتمدة مسبقاً');
  
  // 2. عرض ملخص للمراجعة
  const summary = {
    totalValue: session.total_value,
    basePoints: session.base_points,
    bonusPoints: session.bonus_points,
    walletImpact: `+${session.base_points / 100} ج.م`,
    storeImpact: `+${session.bonus_points} نقطة متجر`,
    customer: session.customer_name,
    items: session.items
  };
  
  // 3. تأكيد المسؤول
  const confirmed = await showConfirmDialog(summary);
  
  if (confirmed) {
    // 4. استدعاء Function في قاعدة البيانات
    const { data, error } = await supabase.rpc('settle_collection_session', {
      p_session_id: sessionId
    });
    
    if (error) throw error;
    
    // 5. إرسال إشعار للعميل
    await notifyCustomer(session.customer_id, {
      type: 'wallet_updated',
      amount: session.base_points / 100,
      message: 'تم اعتماد جلسة التجميع الخاصة بك'
    });
    
    // 6. تحديث Dashboard
    await refreshDashboard();
    
    return { success: true, message: 'تم اعتماد الجلسة بنجاح' };
  }
}
```

#### Function: معالجة طلب السحب

```typescript
async function processRedemption(
  redemptionId: string, 
  action: 'approve' | 'reject',
  notes?: string
) {
  const redemption = await getRedemption(redemptionId);
  
  if (action === 'approve') {
    // 1. التحقق من السيولة
    const liquidityCheck = await checkLiquidity();
    if (!liquidityCheck) {
      throw new Error('نسبة السيولة منخفضة - لا يمكن الموافقة');
    }
    
    // 2. التحويل البنكي
    const bankTransfer = await bankAPI.transfer({
      to: redemption.customer_bank_account,
      amount: redemption.amount_egp,
      reference: `RED-${redemptionId}`,
      description: `سحب نقاط - ${redemption.customer_name}`
    });
    
    // 3. تحديث الحالة
    await supabase
      .from('points_redemptions')
      .update({
        status: 'completed',
        completed_at: new Date(),
        reference_number: bankTransfer.id,
        notes: notes || 'تمت الموافقة والمعالجة'
      })
      .eq('id', redemptionId);
    
    // 4. إشعار العميل
    await notifyCustomer(redemption.customer_id, {
      type: 'redemption_completed',
      amount: redemption.amount_egp,
      reference: bankTransfer.id,
      message: `تم تحويل ${redemption.amount_egp} ج.م إلى حسابك`
    });
    
    return { success: true, transferId: bankTransfer.id };
    
  } else {
    // رفض + إرجاع النقاط
    await supabase.rpc('refund_redemption', {
      p_redemption_id: redemptionId
    });
    
    await notifyCustomer(redemption.customer_id, {
      type: 'redemption_rejected',
      message: 'تم رفض طلب السحب. سيتم إرجاع النقاط إلى حسابك.'
    });
    
    return { success: true, message: 'تم رفض الطلب وإرجاع النقاط' };
  }
}
```

## ✅ الخلاصة

### المبادئ الأساسية (النظام الهجين):

1. **المحفظة المالية** = القيمة الفعلية للمخلفات (قابلة للسحب النقدي)
2. **نقاط المتجر** = البونص والتحفيز (للمتجر والهدايا فقط)
3. **المضاعف الذكي** = يؤثر فقط على نقاط المتجر (لا يؤثر على المحفظة)
4. **تجميد السعر** = `unit_price` يُحفظ عند الجلسة (لا يتأثر بالتحديثات)
5. **تسوية الجلسات** = `is_settled = true` قبل السماح بالسحب النقدي

### المعادلات الرسمية:

```
1. total_value = Σ(actual_weight × unit_price)
2. base_points = FLOOR(total_value × points_per_egp)
3. final_multiplier = base_multiplier × trust_score × volume_factor
4. bonus_points = FLOOR(base_points × (final_multiplier - 1.0))
```

### الفوائد:

- ✅ **أمان مالي**: فصل المحفظة عن نقاط المتجر
- ✅ **تحفيز قوي**: مضاعف ذكي حسب سلوك العميل
- ✅ **تجربة ممتازة**: العميل يرى القيمة والنقاط المتوقعة
- ✅ **جاهز للاستثمار**: لا خسائر غير محسوبة
- ✅ **تدقيق كامل**: Audit Log لكل معاملة

### الحماية من المخاطر:

- ✅ **تجميد السعر**: `unit_price` مجمد عند الجلسة
- ✅ **فصل البونص**: `bonus_points` غير قابلة للسحب النقدي
- ✅ **قفل الجلسات**: لا سحب إلا بعد `is_settled = true`
- ✅ **قيود الاستبدال**: حدود يومية، فترة انتظار، موافقة إدارية
- ✅ **Audit Log**: تتبع كامل لجميع المعاملات
