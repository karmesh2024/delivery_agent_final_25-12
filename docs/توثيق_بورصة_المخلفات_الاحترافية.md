# توثيق بورصة المخلفات الاحترافية

## 1. مقدمة ورؤية النظام

يهدف هذا النظام إلى بناء بورصة إلكترونية احترافية لإدارة عمليات شراء وبيع المخلفات (مواد قابلة لإعادة التدوير) بشكل ديناميكي وذكي، بحيث يتم تحديد أسعار الشراء من العملاء (الأفراد/الشركات) وأسعار البيع للموردين (مصانع، شركات إعادة تدوير) بناءً على معادلات دقيقة تأخذ في الاعتبار جميع العوامل المؤثرة، مع تخصيص الأسعار حسب المناطق الجغرافية، والتفاعل مع تقلبات السوق في الوقت الفعلي.

## 2. العوامل المؤثرة على التسعير

### 2.1. عوامل العرض (Supply)
- **إجمالي المخزون المتاح** في كل منطقة جغرافية.
- **معدل التجميع اليومي/الأسبوعي** (كمية المخلفات التي يتم جمعها).
- **كفاءة وكلاء التجميع** (عددهم، نشاطهم، جودة التجميع).
- **سعة المخازن** وحدودها.

### 2.2. عوامل الطلب (Demand)
- **عروض الموردين النشطة** (الكمية والسعر المطلوب لكل مادة).
- **متوسط أسعار الشراء من الموردين**.
- **الطلبات التاريخية للموردين** (أنماط الشراء الموسمية).
- **الطلب الصناعي في المنطقة** (يمكن ربطه ببيانات خارجية أو تاريخية).

### 2.3. جودة المواد
- **درجة النقاء/الفرز** (كلما زادت الجودة ارتفع السعر).
- **الرطوبة/التلف** (تؤثر سلبًا على السعر).

### 2.4. عوامل السوق والاقتصاد
- **السعر الأساسي المرجعي** (يتم تحديثه دوريًا).
- **تكلفة النقل والتجميع** (تختلف حسب المنطقة والمسافة).
- **أسعار الطاقة/الوقود** (تؤثر على تكلفة النقل والتشغيل).
- **عوامل اقتصادية كلية** (تضخم، أسعار عالمية للمواد الخام).

### 2.5. عوامل جغرافية
- **المنطقة الجغرافية** (تؤثر على العرض والطلب والتكلفة).
- **المسافة بين نقاط التجميع والمخازن/الموردين**.

### 2.6. عوامل موسمية/زمنية
- **مواسم الذروة** (رمضان، الأعياد، الصيف... إلخ).
- **تغيرات مفاجئة في السوق** (أحداث اقتصادية، قرارات حكومية).

## 3. هيكل قاعدة البيانات (Supabase Schema)

### 3.1. الجداول الأساسية

```sql
-- جدول المناطق الجغرافية
CREATE TABLE regions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  parent_region_id UUID REFERENCES regions(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول المخازن (مرتبط بمنطقة)
CREATE TABLE warehouses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  region_id UUID REFERENCES regions(id),
  location TEXT,
  capacity DECIMAL(10,2),
  current_stock DECIMAL(10,2),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول العملاء (من نشتري منهم)
CREATE TABLE customers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  region_id UUID REFERENCES regions(id),
  phone TEXT,
  type TEXT CHECK (type IN ('individual', 'company')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول الموردين (من نبيع لهم)
CREATE TABLE suppliers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  region_id UUID REFERENCES regions(id),
  rating DECIMAL(2,1),
  contact_phone TEXT,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول المنتجات (مواد المخلفات)
CREATE TABLE products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  category_id UUID,
  subcategory_id UUID,
  quality_grade TEXT,
  unit TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول مخزون المواد في المخازن
CREATE TABLE warehouse_inventory (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  warehouse_id UUID REFERENCES warehouses(id),
  product_id UUID REFERENCES products(id),
  quantity DECIMAL(10,2),
  min_level DECIMAL(10,2),
  max_level DECIMAL(10,2),
  last_updated TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول عروض الموردين
CREATE TABLE supplier_price_offers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  supplier_id UUID REFERENCES suppliers(id),
  product_id UUID REFERENCES products(id),
  proposed_price DECIMAL(10,2),
  quantity DECIMAL(10,2),
  unit TEXT,
  valid_from DATE,
  valid_to DATE,
  status TEXT CHECK (status IN ('pending', 'active', 'rejected', 'expired', 'accepted')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول البورصة (سعر الشراء من العميل وسعر البيع للمورد)
CREATE TABLE stock_exchange (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id UUID REFERENCES products(id),
  region_id UUID REFERENCES regions(id),
  base_price DECIMAL(10,2),
  buy_price DECIMAL(10,2), -- سعر الشراء من العميل
  sell_price DECIMAL(10,2), -- سعر البيع للمورد
  last_update TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  price_change_percentage DECIMAL(5,2),
  demand_level TEXT,
  supply_level TEXT,
  volatility_index DECIMAL(5,2), -- مؤشر تقلب السوق
  notes TEXT
);

-- جدول العمليات (شراء/بيع)
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT CHECK (type IN ('buy', 'sell')),
  product_id UUID REFERENCES products(id),
  region_id UUID REFERENCES regions(id),
  customer_id UUID REFERENCES customers(id),
  supplier_id UUID REFERENCES suppliers(id),
  quantity DECIMAL(10,2),
  unit TEXT,
  price DECIMAL(10,2),
  total DECIMAL(12,2),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- جدول التنبيهات
CREATE TABLE system_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  alert_type TEXT,
  severity TEXT,
  message TEXT,
  region_id UUID REFERENCES regions(id),
  warehouse_id UUID REFERENCES warehouses(id),
  product_id UUID REFERENCES products(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  resolved_at TIMESTAMP WITH TIME ZONE
);
```

## 4. آلية التسعير الديناميكي

### 4.1. معادلة تحديد سعر الشراء من العميل (Buy Price)

```typescript
function calculateBuyPrice({
  basePrice,
  totalStock,
  avgSupplierPrice,
  demandLevel,
  supplyLevel,
  regionVolatility,
  qualityGrade,
  transportCost,
  seasonalityFactor
}) {
  let price = basePrice;

  // تأثير العرض
  if (supplyLevel === 'low') price *= 1.10;
  else if (supplyLevel === 'high') price *= 0.92;

  // تأثير الطلب
  if (demandLevel === 'high') price *= 1.12;
  else if (demandLevel === 'low') price *= 0.95;

  // تأثير جودة المادة
  if (qualityGrade === 'A') price *= 1.08;
  else if (qualityGrade === 'C') price *= 0.90;

  // تأثير تكلفة النقل
  price -= transportCost;

  // تأثير تقلب السوق
  price *= (1 + regionVolatility);

  // تأثير الموسم
  price *= seasonalityFactor;

  // حدود دنيا وعليا
  price = Math.max(price, basePrice * 0.8);
  price = Math.min(price, basePrice * 1.2);

  return Number(price.toFixed(2));
}
```

### 4.2. معادلة تحديد سعر البيع للمورد (Sell Price)

```typescript
function calculateSellPrice({
  buyPrice,
  avgSupplierOffer,
  marketTrend,
  profitMargin,
  regionVolatility
}) {
  let price = buyPrice * (1 + profitMargin);

  // تأثير عروض الموردين
  if (avgSupplierOffer > price) price = (price + avgSupplierOffer) / 2;

  // تأثير اتجاه السوق
  price *= (1 + marketTrend);

  // تأثير تقلب المنطقة
  price *= (1 + regionVolatility);

  // حدود دنيا وعليا
  price = Math.max(price, buyPrice * 1.05);
  price = Math.min(price, buyPrice * 1.5);

  return Number(price.toFixed(2));
}
```

### 4.3. مؤشر تقلب السوق (Volatility Index)
- يُحسب بناءً على تغيرات الأسعار في آخر 30 يومًا، وعدد العروض الجديدة، وحجم التداول.
- كلما زاد المؤشر، زادت حساسية الأسعار للتغيرات.

## 5. دورة حياة عملية الشراء والبيع

1. **تجميع المخلفات من العميل:**
   - تسجيل الكمية والجودة والمنطقة.
   - حساب سعر الشراء من العميل تلقائيًا.
   - إصدار إيصال للعميل.
2. **تخزين المواد في المخزن المناسب حسب المنطقة.**
3. **تحديث المخزون وإعادة حساب الأسعار تلقائيًا.**
4. **استقبال عروض الموردين واحتساب سعر البيع المناسب.**
5. **تنفيذ عملية البيع وتحديث المخزون.**
6. **تسجيل كل عملية في جدول العمليات (transactions).**

## 6. آلية تحديث الأسعار وتقلبات السوق

- **تحديث تلقائي دوري** (كل ساعة/يوم):
  - جلب بيانات المخزون، العروض، الطلب، الجودة، التكاليف.
  - إعادة حساب الأسعار لكل منتج/منطقة.
  - تحديث جدول البورصة.
- **تحديث فوري عند الأحداث الحرجة:**
  - انخفاض حاد في المخزون.
  - وصول عرض جديد كبير من مورد.
  - تغير مفاجئ في تكلفة النقل أو الطاقة.
- **إمكانية التعديل اليدوي من قبل الإدارة مع تسجيل السبب.**

## 7. نظام التنبيهات والتحكم

- **تنبيهات المخزون المنخفض** لكل منتج/منطقة.
- **تنبيهات تقلب الأسعار** عند تجاوز نسبة معينة.
- **تنبيهات عروض الموردين الجديدة.**
- **تنبيهات موسمية/اقتصادية** (يمكن ربطها ببيانات خارجية).
- **واجهة إدارة التنبيهات** مع إمكانية التصفية حسب المنطقة/المنتج/النوع.

## 8. لوحات التحكم والتحليلات

- **لوحة بورصة تفاعلية**: أسعار الشراء والبيع، التغيرات، المؤشرات لكل منطقة.
- **تحليلات المخزون**: الكميات، معدلات الدوران، المواد الحرجة.
- **تحليلات الموردين**: أفضل العروض، أوقات التسليم، تقييم الأداء.
- **تحليلات الربحية**: هوامش الربح لكل منتج/منطقة/فترة زمنية.
- **تقارير موسمية وتوقعات الأسعار** (باستخدام الذكاء الاصطناعي لاحقًا).

## 9. خطة التنفيذ المرحلية

### المرحلة 1: البنية الأساسية (1-2 أسبوع)
- إنشاء الجداول في Supabase.
- تطوير واجهات إدارة المناطق، المخازن، المنتجات.
- ربط عمليات التجميع بالمخازن.

### المرحلة 2: تفعيل البورصة والتسعير (2-3 أسابيع)
- تطوير واجهة بورصة تفاعلية.
- بناء خوارزميات التسعير الديناميكي.
- ربط الأسعار بعمليات الشراء والبيع.
- تفعيل نظام التنبيهات.

### المرحلة 3: التحليلات والتكامل (2-3 أسابيع)
- بناء لوحات تحكم متقدمة.
- ربط النظام بتقارير خارجية/ذكاء اصطناعي.
- تحسين تجربة المستخدم.

## 10. التحديات والحلول

- **تعدد المناطق واختلاف الأسعار:**
  - الحل: ربط كل عملية/مخزن/عميل/مورد بمنطقة محددة، وتخصيص الأسعار والمعادلات.
- **تقلبات السوق المفاجئة:**
  - الحل: مؤشر تقلب ديناميكي، وتنبيهات فورية، وإمكانية تدخل يدوي.
- **جودة البيانات:**
  - الحل: تدريب الوكلاء، مراجعة دورية للبيانات، آليات تحقق تلقائي.
- **أمان البيانات:**
  - الحل: سياسات RLS صارمة في Supabase، مراجعة دورية.
- **تجربة المستخدم:**
  - الحل: واجهات بصرية سهلة، دعم البحث والتصفية، تقارير مبسطة.
