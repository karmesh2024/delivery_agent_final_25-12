# تحليل وتوحيد نظام الفئات والتصنيفات

## 📊 الوضع الحالي - تحليل الجداول

### 1. الجداول المتعلقة بالفئات والتصنيفات

#### أ. نظام التصنيفات الهرمية (Hierarchical Categories)
- **`warehouse_sectors`** (9 سجلات)
  - القطاعات الأساسية (مشترك للمنتجات والمخلفات)
  - الحقول: `id` (UUID), `name`, `code`, `description`, `color`, `is_active`
  
- **`product_classifications`** (23 سجل)
  - التصنيفات (المستوى الثاني)
  - مرتبطة بـ: `warehouse_sectors` (sector_id)
  - الحقول: `id` (UUID), `name`, `description`, `sector_id`, `is_active`
  
- **`main_categories`** (19 سجل)
  - الفئات الأساسية (المستوى الثالث)
  - مرتبطة بـ: `product_classifications` (classification_id)
  - الحقول: `id` (UUID), `name`, `description`, `classification_id`, `parent_id`, `level`, `path`
  
- **`sub_categories`** (25 سجل)
  - الفئات الفرعية (المستوى الرابع)
  - مرتبطة بـ: `main_categories` (main_category_id)
  - الحقول: `id` (UUID), `name`, `description`, `main_category_id`, `parent_id`, `level`, `path`

#### ب. نظام فئات الكتالوج (Catalog Categories)

**للمنتجات:**
- **`product_main_categories`** (6 سجلات)
  - فئات أساسية للمنتجات (منفصلة!)
  - الحقول: `id` (BigInt), `code`, `name`
  - **مشكلة**: لا ترتبط بـ `warehouse_sectors` أو `product_classifications`
  
- **`product_sub_categories`** (12 سجل)
  - فئات فرعية للمنتجات
  - مرتبطة بـ: `product_main_categories` (main_id)
  - الحقول: `id` (BigInt), `code`, `name`, `main_id`

**للمخلفات:**
- **`waste_main_categories`** (20 سجل)
  - فئات أساسية للمخلفات (منفصلة!)
  - الحقول: `id` (BigInt), `code`, `name`
  - **مشكلة**: لا ترتبط بـ `warehouse_sectors`
  
- **`waste_sub_categories`** (34 سجل)
  - فئات فرعية للمخلفات
  - مرتبطة بـ: `waste_main_categories` (main_id)
  - الحقول: `id` (BigInt), `code`, `name`, `main_id`

**فئات عامة (قديمة):**
- **`categories`** (4 سجلات)
  - فئات عامة (منفصلة تماماً)
  - الحقول: `id` (UUID), `name`, `description`
  
- **`subcategories`** (8 سجلات)
  - فئات فرعية عامة
  - مرتبطة بـ: `categories` (category_id)
  - الحقول: `id` (UUID), `name`, `category_id`, `price`, `points_per_kg`

#### ج. البراندز (Brands)

- **`store_brands`** (1 سجل)
  - براندز المتاجر الإلكترونية
  - مرتبطة بـ: `store_shops` (shop_id)
  - الحقول: `id` (UUID), `shop_id`, `name_ar`, `name_en`, `logo_url`
  
- **`warehouse_product_brands`** (9 سجلات)
  - براندز المنتجات في المخازن
  - **مشكلة**: لا ترتبط بأي فئة أو تصنيف
  - الحقول: `id` (BigInt), `name`, `logo_url`, `description`

#### د. الوحدات (Units)

- **`units`** (14 سجل)
  - وحدات موحدة (مشتركة للمنتجات والمخلفات)
  - الحقول: `id` (BigInt), `code`, `name`
  - **مشكلة**: لا ترتبط بالفئات أو التصنيفات

## 🔍 المشاكل الرئيسية

### 1. **تضارب في البنية**
- نظامان منفصلان: "التصنيفات الهرمية" و "فئات الكتالوج"
- نفس المفهوم (فئات أساسية/فرعية) موجود في جداول مختلفة
- لا يوجد ربط بين النظامين

### 2. **عدم الربط بالقطاعات والتصنيفات**
- `product_main_categories` و `waste_main_categories` لا ترتبط بـ `warehouse_sectors` أو `product_classifications`
- لا يمكن معرفة أي قطاع أو تصنيف تنتمي إليه الفئة

### 3. **عدم الربط بالبراندز**
- البراندز لا ترتبط بالفئات أو التصنيفات
- لا يمكن تحديد البراندز المتاحة لكل تصنيف

### 4. **عدم الربط بالوحدات**
- الوحدات لا ترتبط بالفئات أو التصنيفات
- لا يمكن تحديد الوحدات المتاحة لكل تصنيف

### 5. **أنواع بيانات مختلفة**
- بعض الجداول تستخدم `UUID` (warehouse_sectors, product_classifications, main_categories)
- بعضها يستخدم `BigInt` (product_main_categories, waste_main_categories)
- صعوبة في الربط بين الجداول

## 🎯 الحل المقترح: نظام موحد "إدارة التنظيم والتسلسل"

### الهيكل المقترح:

```
القطاع (Sector)
  └── التصنيف (Classification)
      └── الفئة الأساسية (Main Category)
          └── الفئة الفرعية (Sub Category)
              ├── الوحدات المتاحة (Available Units)
              └── البراندز المتاحة (Available Brands)
```

### الجداول المقترحة:

#### 1. **`unified_sectors`** (موحد - بديل لـ warehouse_sectors)
```sql
- id (UUID)
- name
- code
- description
- color
- is_active
- created_at, updated_at
```

#### 2. **`unified_classifications`** (موحد - بديل لـ product_classifications)
```sql
- id (UUID)
- sector_id (FK → unified_sectors)
- name
- description
- item_type (ENUM: 'product', 'waste', 'both') -- نوع العنصر
- is_active
- created_at, updated_at
```

#### 3. **`unified_main_categories`** (موحد - يجمع main_categories, product_main_categories, waste_main_categories)
```sql
- id (UUID)
- classification_id (FK → unified_classifications)
- code
- name
- description
- item_type (ENUM: 'product', 'waste', 'both')
- parent_id (FK → unified_main_categories) -- للهيكل الهرمي
- level (INT)
- path (TEXT) -- للمسار الهرمي
- is_active
- created_at, updated_at
```

#### 4. **`unified_sub_categories`** (موحد - يجمع sub_categories, product_sub_categories, waste_sub_categories)
```sql
- id (UUID)
- main_category_id (FK → unified_main_categories)
- code
- name
- description
- item_type (ENUM: 'product', 'waste', 'both')
- parent_id (FK → unified_sub_categories) -- للهيكل الهرمي
- level (INT)
- path (TEXT)
- is_active
- created_at, updated_at
```

#### 5. **`classification_units`** (ربط الوحدات بالتصنيفات)
```sql
- id (UUID)
- classification_id (FK → unified_classifications)
- unit_id (FK → units)
- is_default (BOOLEAN)
- sort_order (INT)
- created_at, updated_at
```

#### 6. **`classification_brands`** (ربط البراندز بالتصنيفات)
```sql
- id (UUID)
- classification_id (FK → unified_classifications)
- brand_id (FK → unified_brands)
- is_default (BOOLEAN)
- sort_order (INT)
- created_at, updated_at
```

#### 7. **`unified_brands`** (موحد - يجمع store_brands و warehouse_product_brands)
```sql
- id (UUID)
- name_ar
- name_en
- logo_url
- description_ar
- description_en
- website
- is_active
- created_at, updated_at
```
