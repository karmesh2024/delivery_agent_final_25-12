# نظام تتبع أسعار الشراء الفعلي (Professional Price Tracking System)

## نظرة عامة

تم تطبيق نظام احترافي لتتبع آخر سعر تم الشراء به فعلياً من العملاء، مما يعطي مؤشراً دقيقاً للعميل عن التغير في السعر مقارنة بآخر عملية شراء فعلية.

## المشكلة التي تم حلها

### المشكلة السابقة:
- المؤشر كان يحسب الفرق بين `buy_price` الحالي و `old_buy_price` من آخر تحديث في `exchange_price_history`
- هذا يعطي الفرق بين آخر تحديثين يدويين، وليس السعر الذي اشترى به العميل فعلياً

### الحل الاحترافي:
- تتبع آخر سعر تم الشراء به فعلياً من جدول `order_details`
- إضافة حقل `last_actual_purchase_price` في `stock_exchange`
- تحديث هذا الحقل تلقائياً عند كل عملية شراء مكتملة

## البنية التقنية

### 1. حقل جديد في قاعدة البيانات

```sql
ALTER TABLE stock_exchange
ADD COLUMN last_actual_purchase_price DECIMAL(12, 2) DEFAULT NULL;
```

### 2. Functions و Triggers

#### Function: `update_last_actual_purchase_price_on_order_complete()`
- يتم استدعاؤها عند إضافة أو تحديث `order_details`
- تحديث `last_actual_purchase_price` فقط عند إتمام الطلب (`status = 'completed'` أو `'delivered'`)

#### Function: `update_last_actual_purchase_price_on_order_status_change()`
- يتم استدعاؤها عند تغيير حالة الطلب في `delivery_orders`
- تحديث `last_actual_purchase_price` عند إتمام الطلب

#### Triggers:
- `trigger_update_last_actual_purchase_price`: على `order_details`
- `trigger_update_last_actual_purchase_price_on_status_change`: على `delivery_orders`

### 3. آلية الربط

يتم الربط بين `order_details` و `stock_exchange` عبر:
1. **الربط الأساسي**: `waste_data_admin.name = order_details.product_name`
2. **الربط البديل**: `categories.name = order_details.category_name` و `subcategories.name = order_details.subcategory_name`

## كيفية العمل

### السيناريو المطلوب:
1. العميل اشترى يوم السبت بـ 17 جنيه
2. يوم الأحد ارتفعت الأسعار، وتم تعديل السعر إلى 24 جنيه
3. المؤشر يعبر عن: 24 - 17 = +7 جنيه (ارتفاع)
4. نسبة التغير: (24 - 17) / 17 × 100 = +41.18%

### التدفق:
1. عند إتمام طلب شراء (`delivery_orders.status = 'completed'`)
2. يتم تحديث `last_actual_purchase_price` في `stock_exchange` تلقائياً
3. عند حساب المؤشر، يتم استخدام `last_actual_purchase_price` كسعر مرجعي
4. إذا لم يكن هناك `last_actual_purchase_price`، يتم استخدام السعر من `exchange_price_history` كبديل

## الكود

### في `exchangeService.ts`:

```typescript
// جلب last_actual_purchase_price من stock_exchange
let lastActualPurchasePricesMap = new Map<number, number>();
if (stockExchangeIds.length > 0) {
  const { data: actualPurchasePrices } = await supabase!
    .from("stock_exchange")
    .select("id, last_actual_purchase_price")
    .in("id", stockExchangeIds);
  
  actualPurchasePrices?.forEach(price => {
    if (price.last_actual_purchase_price) {
      lastActualPurchasePricesMap.set(Number(price.id), Number(price.last_actual_purchase_price));
    }
  });
}

// استخدام last_actual_purchase_price كسعر مرجعي أولاً
let price24hAgo = lastActualPurchasePricesMap.get(stockExchangeId) || 0;
```

### في `ExchangeDashboardPage.tsx`:

```typescript
// استخدام last_actual_purchase_price (آخر سعر تم الشراء به فعلياً) كسعر مرجعي
if (trend && trend.price_24h_ago) {
  const price24hAgo = Number(trend.price_24h_ago) || 0;
  const diff = buyPrice - price24hAgo; // الفرق بين السعر الحالي وآخر سعر تم الشراء به فعلياً
  const percent = (diff / price24hAgo) * 100; // نسبة التغير
}
```

## المزايا

1. **دقة عالية**: يعكس السعر الفعلي للشراء وليس مجرد تحديثات يدوية
2. **تحديث تلقائي**: يتم تحديث `last_actual_purchase_price` تلقائياً عند كل عملية شراء
3. **مطابق للمعايير**: يعمل كما في البورصات الحقيقية
4. **مؤشر دقيق**: يعطي العميل مؤشراً دقيقاً عن التغير في السعر
