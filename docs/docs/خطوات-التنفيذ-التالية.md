## دليل لتطبيق Redux على مكون موجود

لتحويل مكون حالي يستخدم `useState` إلى استخدام Redux، اتبع هذه الخطوات:

1. **تحديد حالة المكون**
   - ما هي بيانات الحالة التي يديرها المكون؟
   - ما هي العمليات التي تغير هذه الحالة؟

2. **إنشاء (أو تحديث) الـ API**
   - ضع وظائف الاتصال بالخادم في ملف API منفصل

3. **إنشاء (أو تحديث) شريحة Redux**
   - عرّف الحالة الأولية
   - أضف المخفضات للعمليات المحلية
   - أضف thunks للعمليات غير المتزامنة

4. **تحديث المكون**
   - استبدل `useState` بـ `useAppSelector`
   - استبدل دوال تحديث الحالة المحلية بـ `dispatch` لإجراءات Redux

### مثال سريع:

**قبل (باستخدام useState):**
```tsx
function OrdersComponent() {
  const [orders, setOrders] = useState<Order[]>([]);
  const [loading, setLoading] = useState(false);
  
  useEffect(() => {
    const loadOrders = async () => {
      setLoading(true);
      try {
        const data = await getOrders();
        setOrders(data);
      } catch (error) {
        console.error(error);
      } finally {
        setLoading(false);
      }
    };
    
    loadOrders();
  }, []);
  
  // باقي التنفيذ...
}
```

**بعد (باستخدام Redux):**
```tsx
function OrdersComponent() {
  const dispatch = useAppDispatch();
  const { items: orders, loading } = useAppSelector(state => state.orders);
  
  useEffect(() => {
    dispatch(fetchOrders());
  }, [dispatch]);
  
  // باقي التنفيذ...
}
```

1. **تحميل البيانات فقط عند الحاجة**
   - استخدم حالة Redux لتخزين البيانات التي تم جلبها بالفعل
   - تحقق إذا كانت البيانات موجودة قبل إعادة تحميلها

## الهدف النهائي

بعد اكتمال هذه الخطوات، سيتوفر لدينا:

1. **بنية منظمة حسب مجالات الأعمال** - كل مجال له مكوناته وخدماته ومنطقه الخاص
2. **إدارة حالة مركزية** - باستخدام Redux لمشاركة البيانات بين المكونات
3. **فصل واضح للمسؤوليات** - فصل منطق الأعمال وواجهة المستخدم وطبقة البيانات
4. **قابلية للتوسع** - سهولة إضافة مجالات جديدة في المستقبل (الموارد البشرية، المالية، إلخ)

هذه الخطوات ستضمن أن المشروع قادر على النمو والتطور بشكل مستدام وقابل للصيانة.
