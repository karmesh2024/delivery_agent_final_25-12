# 🚀 دليل تطوير ميزة صناعة المحتوى الاحترافية (Content Studio + n8n Workflow)

هذا المستند يحدد خارطة الطريق لتطوير ميزة "Content Studio" في لوحة تحكم زوون،
بالاعتماد على معمارية هجينة تجمع بين واجهة Next.js ومحرك الأتمتة n8n لإنتاج محتوى
احترافي ومتعدد الأساليب.

## 1. الفلسفة التطويرية: من "جلب بيانات" إلى "ورك فلو إبداعي"

الهدف ليس مجرد جلب أخبار وعرضها، بل تحويل "مدخل بسيط" (خبر أو فكرة) إلى "حزمة
محتوى متكاملة" (بوستات متعددة، صور مولدة بالذكاء الاصطناعي، وتحليل نفسي) جاهزة
للنشر.

---

## 2. المعمارية التقنية (The Architecture)

### أ. لوحة التحكم (Next.js Dashboard) - "المايسترو"

- **الوظيفة**: تحديد المصادر، التحكم في الكلمات المفتاحية، مراجعة المسودات،
  وإعطاء أمر النشر النهائي.
- **التقنيات**: React, Shadcn/UI, Supabase Client.

### ب. محرك الأتمتة (n8n Workflow) - "المصنع"

- **الوظيفة**: تنفيذ العمليات الثقيلة (Fetch, Merge, AI Translation, Image
  Generation).
- **لماذا n8n؟**: يدعم الـ Long-running tasks، وسهولة الربط مع خدمات خارجية مثل
  Together AI و Telegram و Notion دون تعقيد الكود المباشر.

### ج. طبقة النماذج (AI Models) - "العقل"

- **Gemini 1.5 Flash**: للتحليل الأولي واستخراج الكلمات المفتاحية الذكية.
- **Qwen 2.5 (عبر Groq)**: لصياغة البوستات بلهجة سكندرية/مصرية عامية دقيقة.
- **Flux.1 Schnell (عبر Together AI)**: لتوليد صور احترافية بدقة 1080x1080.

---

## 3. مراحل العمل (Step-by-Step Workflow)

### المرحلة 1: التحكم الذكي بالمدخلات (Smart Input)

- **استخراج الكلمات**: بدلاً من البحث بـ "أخبار محرم بك"، يقوم النظام باستخراج
  كلمات مثل `Alexandria urban life`, `Egyptian community interaction`.
- **تغذية n8n**: يتم إرسال طلب Webhook من لوحة التحكم إلى n8n يحتوي على:
  - الهدف الأسبوعي للغرفة (Target Goal).
  - التوجه النفسي المطلوب (Psychological Impact).
  - الكلمات المفتاحية المستخرجة.

### المرحلة 2: المعالجة المتعددة (Multi-Format Execution)

يقوم n8n بإنتاج 3 نسخ من المحتوى لكل مدخل واحد:

1. **النسخة الرسمية (👔 Official)**: لغة فصحى بسيطة، أخبار جادة.
2. **النسخة الشبابية (😎 Trending)**: لغة عامية، Emojis، تريندات محلية.
3. **النسخة التفاعلية (💬 Interactive)**: سؤال للجمهور، استطلاع رأي، دعوة
   للمشاركة.

### المرحلة 3: التوليد المرئي المحاكي (Visual Generation)

- استخدام الكلمات المفتاحية لوصف مشهد (Prompt) بالإنجليزية مثل:
  `Professional photo of a vibrant social gathering in Moharram Bek Alexandria, cinematic lighting, 8k`.
- دمج الصورة تلقائياً مع البوست في مسودة واحدة.

### المرحلة 4: المراجعة البشرية (Human-in-the-Loop)

- يتم حفظ النتائج في "طابور المراجعة" (Supabase table: `zoon_posts_queue`).
- تصل رسالة فورية للمدرف عبر **Telegram Bot** برابط المسودة للمراجعة السريعة.

---

## 4. هيكلية البيانات المطلوبة (Database Integration)

يتم تخزين المحتوى المولد في جدول `zoon_posts_queue` مع الحقول التالية:

- `content`: نص المنشور المقترح.
- `image_url`: رابط الصورة المولدة.
- `style`: (official, youthful, interactive).
- `psychological_impact`: JSON يحدد التأثير المتوقع (Openness, Agreeableness,
  etc).
- `source_news_url`: رابط الخبر الأصلي للتوثيق.
- `status`: (pending, approved, rejected).

---

## 5. خطة التنفيذ البرمجية (Implementation Steps)

1. **برمجة الـ Webhook**: إنشاء API في Next.js يرسل البيانات لـ n8n عند ضغط زر
   "تشغيل المصنع الذكي".
2. **بناء الـ n8n Flow**: استيراد ملف الـ JSON المتاح
   (`moharram-bey-content-automation.json`) وربطه بمفاتيح الـ API (Groq,
   Together AI).
3. **تطوير الـ Post Editor**: تعديل واجهة المحرر الحالية لتدعم التبديل بين النسخ
   الـ 3 (رسمي/شبابي/تفاعلي) بضغطة زر.
4. **ربط الصور**: إضافة معرض صور مصغر تحت البوست يسمح للمستخدم بالتبديل بين
   الصورة المولدة بالـ AI وصور DuckDuckGo الأصلية.

---

## 6. المميزات التنافسية لهذا النظام

- **لهجة محلية حقيقية**: بفضل استخدام Qwen 2.5 المدرب جيداً على السياق العربي.
- **تكلفة رمزية**: استخدام Groq (مجاني) و Flux Schnell (تكلفة لا تذكر) و Gemini
  (مجاني).
- **سرعة إنتاج**: تحويل الخبر إلى 3 بوستات وصورة في أقل من 20 ثانية.
