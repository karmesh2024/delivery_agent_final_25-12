## 🎯 نظرة عامة على النظام المالي

### المبدأ الأساسي

النظام المالي في تطبيق العملاء يعتمد على **نظام نقاط هجين** يجمع بين:

1. **💰 المحفظة المالية (Wallet Balance)**
   - قيمة مالية حقيقية قابلة للسحب النقدي
   - كل نقطة = 0.01 جنيه مصري
   - مخزنة في: `new_profiles.wallet_balance`

2. **🎁 نقاط المتجر (Store Points)**
   - نقاط تحفيزية للولاء
   - غير قابلة للسحب النقدي
   - للاستبدال بمنتجات/هدايا/خصومات
   - مخزنة في: `new_profiles.store_points`

### المعادلة الأساسية

```
القيمة المالية للمخلفات (بالجنيه)
    ↓
تحويل إلى نقاط أساسية (× 100)
    ↓
    ├─→ wallet_balance (قابلة للسحب)
    └─→ store_points (للاستبدال في المتجر)
```

---

## 💳 هيكل محفظة العميل

### البيانات المخزنة في `new_profiles`

```sql
new_profiles {
    id: UUID,
    wallet_balance: NUMERIC(12,2),  -- المحفظة المالية (بالجنيه)
    store_points: INTEGER,           -- نقاط المتجر
    -- ... بيانات أخرى
}
```

### مثال عملي

```
العميل: أحمد محمد
├─ wallet_balance: 150.00 ج.م
│  └─ = 15,000 نقطة (قابلة للسحب النقدي)
│
└─ store_points: 5,000 نقطة
   └─ (للاستبدال في المتجر فقط)
```

### الرصيد الإجمالي (للعرض في التطبيق)

```
الرصيد الإجمالي = (wallet_balance × 100) + store_points
                 = (150 × 100) + 5,000
                 = 15,000 + 5,000
                 = 20,000 نقطة
```

**⚠️ ملاحظة مهمة:** في تطبيق العميل، نعرض **رقم واحد فقط** (20,000 نقطة) بدون تفصيل wallet/store.

## 🔄 السيناريوهات المالية الأساسية

### السيناريو 1: جمع المخلفات (Waste Collection)

#### الخطوات:

1. **العميل يدخل المخلفات في التطبيق**
   ```
   - يختار نوع المخلفات
   - يدخل الكمية/الوزن
   - يطلب التجميع
   ```

2. **إنشاء جلسة تجميع**
   ```sql
   waste_collection_sessions {
       id: UUID,
       customer_id: UUID,
       status: 'pending',
       is_settled: false,
       -- ...
   }
   ```

3. **عند وصول المندوب ووزن المخلفات**
   ```sql
   waste_collection_items {
       session_id: UUID,
       waste_data_id: UUID,
       quantity: INTEGER,
       actual_weight: DECIMAL(10,3),
       unit_price: DECIMAL(10,2),  -- ⭐ مجمد من البورصة
       total_price: DECIMAL(10,2),
       -- ...
   }
   ```

4. **إتمام الجلسة (status = 'completed')**
   ```
   النظام يحسب تلقائياً:
   
   total_value = Σ(actual_weight × unit_price)
   base_points = FLOOR(total_value × 100)
   
   → base_points تُسجل في الجلسة (معلق حتى التسوية)
   → تُسجل في points_transactions (type: 'wallet', status: 'pending')
   ```

5. **اعتماد الجلسة (Settlement)**
   ```
   عند اعتماد الجلسة من الداش بورد:
   
   → wallet_balance += (base_points / 100)
   → is_settled = true
   → points_transactions يتم تحديثه
   ```

#### مثال عملي:

```
العميل: سارة أحمد
جمعت: 50 كيلو بلاستيك
سعر البورصة المجمد: 5 ج.م/كيلو

الحساب:
total_value = 50 × 5 = 250 ج.م
base_points = 250 × 100 = 25,000 نقطة

النتيجة:
→ wallet_balance += 250 ج.م (بعد الاعتماد)
→ العميل يمكنه السحب بعد الاعتماد
```

---

### السيناريو 2: السحب النقدي (Cash Withdrawal)

#### الخطوات:

1. **العميل يطلب سحب نقدي من التطبيق**
   ```
   - يدخل المبلغ المطلوب (مثلاً: 100 ج.م)
   - يضغط "طلب سحب"
   ```

2. **إنشاء طلب استبدال**
   ```sql
   points_redemptions {
       id: UUID,
       customer_id: UUID,
       amount_egp: 100.00,
       redemption_type: 'cash',
       status: 'pending',
       -- ...
   }
   ```

3. **التحقق من الشروط**
   ```
   النظام يتحقق:
   ✅ لا توجد جلسات غير مقفلة (is_settled = false)
   ✅ wallet_balance >= المبلغ المطلوب
   ✅ لا يوجد قيود يومية/شهرية
   ```

4. **معالجة الطلب من الداش بورد**
   ```
   عند الموافقة:
   
   → wallet_balance -= 100 ج.م
   → points_redemptions.status = 'completed'
   → points_transactions (type: 'wallet', amount: -10,000)
   → التحويل البنكي/المحفظة
   ```

#### مثال عملي:

```
العميل: محمد علي
wallet_balance: 200 ج.م
طلب سحب: 150 ج.م

التحقق:
✅ wallet_balance (200) >= المطلوب (150)
✅ لا جلسات معلقة

النتيجة:
→ wallet_balance = 200 - 150 = 50 ج.م
→ طلب السحب: pending → completed
→ التحويل البنكي يتم
```

---

### السيناريو 3: الاستبدال في المتجر (Store Redemption)

#### الخطوات:

1. **العميل يختار منتج من المتجر**
   ```
   المنتج: قميص كارمش
   السعر: 5,000 نقطة
   ```

2. **التحقق من الرصيد**
   ```
   الرصيد الإجمالي = (wallet_balance × 100) + store_points
   
   إذا الرصيد >= سعر المنتج:
   → يمكن الشراء
   ```

3. **الشراء (أولوية: store_points أولاً)**
   ```sql
   -- استخدام نقاط المتجر أولاً
   IF store_points >= product_price THEN
       store_points -= product_price
   ELSE
       -- استخدام wallet_balance للباقي
       remaining = product_price - store_points
       wallet_amount = remaining / 100
       wallet_balance -= wallet_amount
       store_points = 0
   END IF
   ```

4. **تسجيل المعاملة**
   ```sql
   points_redemptions {
       redemption_type: 'product',
       points_redeemed: 5,000,
       status: 'completed',
   }
   
   points_transactions {
       type: 'store' أو 'wallet',
       amount: -5,000,
   }
   ```

#### مثال عملي:

```
العميل: فاطمة خالد
wallet_balance: 50 ج.م (5,000 نقطة)
store_points: 3,000 نقطة
المنتج: 8,000 نقطة

الحساب:
1. استخدام store_points: 3,000 نقطة
2. الباقي: 8,000 - 3,000 = 5,000 نقطة
3. استخدام wallet: 5,000 / 100 = 50 ج.م

النتيجة:
→ store_points = 0
→ wallet_balance = 0
→ المنتج يتم تسليمه
```

---

### السيناريو 4: الجلسات المعلقة (Pending Sessions)

#### المشكلة:

```
العميل جمع مخلفات اليوم:
├─ الجلسة: completed ✅
├─ wallet_balance: 0 ج.م (لم يتم الاعتماد بعد)
└─ is_settled: false ⚠️

العميل يريد سحب 50 ج.م:
❌ لا يمكن - الجلسة غير مقفلة
```

#### الحل:

1. **من الداش بورد:**
   ```
   → اعتماد الجلسة (Settle Session)
   → wallet_balance += المبلغ
   → is_settled = true
   → الآن يمكن السحب
   ```

2. **من التطبيق (للعميل):**
   ```
   عرض رسالة:
   "لديك جلسة قيد المراجعة. سيتم تفعيل رصيدك قريباً."
   ```

## ⚠️ حالات خاصة ومشاكل محتملة

### الحالة 1: جلسة غير مقفلة + طلب سحب

**المشكلة:**
```
العميل لديه:
- wallet_balance: 100 ج.م (متاح)
- جلسة معلقة: 50 ج.م (غير مقفلة)

طلب سحب: 120 ج.م

النتيجة:
❌ لا يمكن السحب - يوجد جلسة غير مقفلة
```

**الحل:**
```
1. اعتماد الجلسة أولاً
2. ثم السماح بالسحب
```

---

### الحالة 2: رفض طلب السحب

**المشكلة:**
```
طلب سحب: 100 ج.م
تم رفضه من الداش بورد

النتيجة:
→ wallet_balance تم خصمه بالفعل
```

**الحل:**
```
عند الرفض:
→ refund_redemption()
→ wallet_balance += 100 ج.م (إرجاع)
→ points_redemptions.status = 'rejected'
```

---

### الحالة 3: تعديل وزن الجلسة بعد الاعتماد

**المشكلة:**
```
الجلسة: is_settled = true
wallet_balance: تم إضافة 100 ج.م
تم اكتشاف خطأ في الوزن

النتيجة:
⚠️ لا يمكن تعديل الجلسة المقفلة
```

**الحل:**
```
1. إنشاء جلسة تصحيحية
2. أو تعديل يدوي من الداش بورد
3. تسجيل في points_transactions (type: 'admin_adjustment')
```

---

### الحالة 4: استبدال منتج ثم إلغاء الطلب

**المشكلة:**
```
العميل اشترى منتج بـ 5,000 نقطة
تم خصم من store_points
العميل يريد إلغاء الطلب

النتيجة:
→ store_points تم خصمه بالفعل
```

**الحل:**
```
عند الإلغاء:
→ store_points += 5,000 (إرجاع)
→ points_redemptions.status = 'cancelled'
→ points_transactions (type: 'refund')
```

الحساب:
total_value = 30 × 3 = 90 ج.م

قبل الاعتماد:
wallet_balance: 0 ج.م
store_points: 0 نقطة
is_settled: false

بعد الاعتماد:
wallet_balance: 90 ج.م ✅
store_points: 0 نقطة

**العميل:** سارة نشطة  
**الجلسة 1:** 50 كيلو بلاستيك × 5 ج.م = 250 ج.م (معتمدة)  
**الجلسة 2:** 40 كيلو ورق × 3 ج.م = 120 ج.م (معلقة)

```
الحالة الحالية:
wallet_balance: 250 ج.م (من الجلسة 1)
store_points: 0 نقطة
جلسة معلقة: 120 ج.م (غير معتمدة)

العميل يريد سحب 200 ج.م:
✅ wallet_balance (250) >= المطلوب (200)
✅ لا جلسات معلقة تمنع السحب
→ يمكن السحب ✅

بعد السحب:
wallet_balance: 50 ج.م
store_points: 0 نقطة
```

---

### المثال 3: عميل مع نقاط متجر

**wallet_balance:** 100 ج.م

```
الرصيد الإجمالي:
= (100 × 100) + 10,000
= 10,000 + 10,000
= 20,000 نقطة

العميل يريد شراء منتج بـ 15,000 نقطة:

الحساب:
1. استخدام store_points: 10,000 نقطة
2. الباقي: 15,000 - 10,000 = 5,000 نقطة
3. استخدام wallet: 5,000 / 100 = 50 ج.م

النتيجة:
store_points: 0 نقطة

```

### المثال 5: جلسة معلقة + طلب سحب

## 📋 السياسات والإجراءات

### السياسة 1: اعتماد الجلسات

**المدة القصوى:** 24 ساعة من إتمام الجلسة  
**المسؤول:** فريق الإدارة في الداش بورد  
**الإجراء:**

1. مراجعة الجلسة المكتملة
2. التحقق من البيانات (الوزن، السعر، القيمة)
3. اعتماد الجلسة (Settle Session)
4. إشعار العميل (اختياري)

---

### السياسة 2: معالجة طلبات السحب

**المدة القصوى:** 48 ساعة من الطلب  
**المسؤول:** فريق المالية  
**الإجراء:**

1. التحقق من الجلسات المعلقة
2. التحقق من الرصيد
3. التحقق من القيود اليومية/الشهرية
4. الموافقة/الرفض
5. التحويل البنكي (في حالة الموافقة)

---

### السياسة 3: إرجاع النقاط

**الحالات:**
- رفض طلب سحب
- إلغاء طلب منتج
- خطأ في الحساب

**الإجراء:**

1. التحقق من السبب
2. استخدام `refund_redemption()` function
3. إرجاع النقاط للمحفظة/المتجر
4. تسجيل في `points_transactions`

---

### السياسة 4: تعديلات إدارية

**الحالات:**
- تصحيح خطأ في الحساب
- مكافأة خاصة
- خصم إداري

**الإجراء:**

1. إنشاء سجل في `points_transactions`
   - `type`: 'wallet' أو 'store'
   - `source`: 'admin_adjustment'
   - `description`: سبب التعديل
2. تحديث `wallet_balance` أو `store_points`
3. إشعار العميل (اختياري)

## ❓ FAQ إداري

### س1: ماذا يحدث إذا رفض العميل استلام المخلفات بعد الجلسة؟

**الجواب:**
```
- الجلسة تبقى pending
- لا يتم اعتمادها
- wallet_balance لا يتغير
- يمكن إلغاء الجلسة من الداش بورد
```

---

### س2: هل يمكن للعميل سحب wallet_balance قبل اعتماد الجلسة؟

**الجواب:**
```
❌ لا - النظام يمنع السحب إذا:
- يوجد جلسة completed + is_settled = false
- حتى لو wallet_balance كافٍ
```

---

### س3: ماذا يحدث إذا تغير سعر البورصة بعد الجلسة؟

**الجواب:**
```
✅ لا مشكلة - السعر مجمد في unit_price
- unit_price يُثبت عند إنشاء العنصر
- لا يتأثر بتغييرات البورصة لاحقاً
```

---

### س4: هل يمكن للعميل تحويل store_points إلى wallet_balance؟

**الجواب:**
```
❌ لا - store_points للاستبدال في المتجر فقط
- لا يمكن تحويلها إلى wallet_balance
- لا يمكن سحبها نقدياً
```

---

### س5: ماذا يحدث إذا كان wallet_balance سالباً؟

**الجواب:**
```
⚠️ لا يجب أن يحدث هذا
- النظام يمنع السحب إذا wallet_balance < المطلوب
- إذا حدث خطأ: تعديل إداري فوري
```

---

### س6: كيف نتعامل مع شكوى عميل عن نقاط مفقودة؟

**الإجراء:**

1. **التحقق من `points_transactions`**
   ```sql
   SELECT * FROM points_transactions
   WHERE profile_id = 'customer_id'
   ORDER BY created_at DESC;
   ```

2. **التحقق من الجلسات**
   ```sql
   SELECT * FROM waste_collection_sessions
   WHERE customer_id = 'customer_id'
   ORDER BY created_at DESC;
   ```

3. **التحقق من طلبات الاستبدال**
   ```sql
   SELECT * FROM points_redemptions
   WHERE customer_id = 'customer_id'
   ORDER BY created_at DESC;
   ```

4. **إذا كان هناك خطأ:**
   - تعديل إداري
   - تسجيل في `points_transactions`
   - إشعار العميل

```sql
-- الجلسات المكتملة غير المعتمدة
SELECT 
    COUNT(*) as pending_sessions,
    SUM(base_points / 100) as pending_amount_egp
FROM waste_collection_sessions
WHERE status = 'completed'
  AND is_settled = false;
```

---

### التقرير 3: طلبات السحب المعلقة

```sql
-- طلبات السحب النقدي المعلقة
SELECT 
    COUNT(*) as pending_withdrawals,
    SUM(amount_egp) as pending_amount_egp
FROM points_redemptions
WHERE redemption_type = 'cash'
  AND status = 'pending';
```

---

### التقرير 4: حركة النقاط (Audit Log)

```sql
-- جميع المعاملات المالية
SELECT 
    type,
    SUM(amount) as total_points,
    COUNT(*) as transaction_count
FROM points_transactions
WHERE created_at >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY type;
```

## 🎯 الخلاصة التنفيذية

### المبادئ الأساسية:

1. **💰 wallet_balance = قيمة مالية حقيقية**
   - قابلة للسحب النقدي
   - كل نقطة = 0.01 جنيه
   - تحتاج اعتماد الجلسة قبل السحب

2. **🎁 store_points = نقاط تحفيزية**
   - غير قابلة للسحب النقدي
   - للاستبدال في المتجر فقط
   - لا يمكن تحويلها إلى wallet_balance

3. **🔒 الأمان:**
   - لا سحب قبل اعتماد الجلسة
   - تجميد السعر عند الجلسة
   - Audit Log لكل معاملة

4. **📊 الشفافية:**
   - كل معاملة مسجلة في `points_transactions`
   - يمكن تتبع كل حركة مالية
   - تقارير شاملة متاحة
