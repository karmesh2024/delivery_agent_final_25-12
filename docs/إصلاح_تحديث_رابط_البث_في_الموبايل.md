# إصلاح تحديث رابط البث في تطبيق الموبايل

**التاريخ:** 2025-01-22  
**الإصدار:** V1.5.1  
**المشكلة:** الموبايل لا يحدث رابط البث عند تغييره في الداش بورد

---

## 🔴 المشكلة

عند تغيير رابط البث في الداش بورد، الموبايل لا يزال يستمع للرابط القديم.

**السبب:**
1. الموبايل يقرأ الرابط **مرة واحدة** عند بدء الاستماع ويحتفظ به في cache
2. الموبايل **لا يتابع** Real-time updates لتحديثات `club_activities`
3. حتى لو تم تحديث الرابط في قاعدة البيانات، الموبايل لا يعرف بذلك

---

## ✅ الحل المطلوب

### 1. قراءة الرابط من قاعدة البيانات (بدون cache)

**المشكلة الحالية:**
```dart
// ❌ الكود الحالي (يقرأ الرابط مرة واحدة فقط)
Future<void> startListening() async {
  final stream = await getCurrentStream(); // قراءة واحدة
  final streamUrl = stream.listen_url ?? stream.stream_url;
  audioPlayer.setUrl(streamUrl); // حفظ في cache
  await audioPlayer.play();
}
```

**الحل:**
```dart
// ✅ قراءة الرابط من قاعدة البيانات في كل مرة
Future<String?> getCurrentStreamUrl() async {
  final response = await supabase
    .from('club_activities')
    .select('listen_url, stream_url')
    .eq('activity_type', 'radio_stream')
    .eq('is_active', true)
    .eq('is_live', true)
    .maybeSingle();
  
  if (response == null) return null;
  
  // استخدام listen_url أولاً، ثم stream_url كـ fallback
  return response['listen_url'] ?? response['stream_url'];
}

// استخدام الرابط المحدث
Future<void> updateStreamUrl() async {
  final newUrl = await getCurrentStreamUrl();
  if (newUrl != null && newUrl != audioPlayer.url) {
    final wasPlaying = audioPlayer.playing;
    await audioPlayer.stop();
    await audioPlayer.setUrl(newUrl);
    if (wasPlaying) {
      await audioPlayer.play();
    }
  }
}
```

### 2. إضافة Real-time Subscription (مهم جداً)

**هذا هو الحل الأساسي للمشكلة:**

```dart
import 'package:supabase_flutter/supabase_flutter.dart';

class RadioStreamService {
  final SupabaseClient supabase;
  RealtimeChannel? _streamChannel;
  String? _currentStreamId;
  
  RadioStreamService(this.supabase);
  
  // ✅ إعداد Real-time subscription لتحديثات رابط البث
  void setupStreamUrlSubscription(String streamId, Function(String newUrl) onUrlUpdated) {
    // إلغاء الاشتراك السابق إن وجد
    if (_streamChannel != null) {
      supabase.removeChannel(_streamChannel!);
    }
    
    _currentStreamId = streamId;
    
    _streamChannel = supabase
      .channel('radio_stream_${streamId}')
      .onPostgresChanges(
        event: PostgresChangeEvent.update,
        schema: 'public',
        table: 'club_activities',
        filter: PostgresChangeFilter(
          type: PostgresChangeFilterType.eq,
          column: 'id',
          value: streamId,
        ),
        callback: (payload) {
          final updatedStream = payload.newRecord;
          final newStreamUrl = updatedStream['listen_url'] ?? updatedStream['stream_url'];
          
          print('[Radio Stream] Stream URL updated: $newStreamUrl');
          
          // استدعاء callback لتحديث audio player
          onUrlUpdated(newStreamUrl);
        },
      )
      .subscribe();
    
    print('[Radio Stream] Subscription setup for stream: $streamId');
  }
  
  // إلغاء الاشتراك
  void dispose() {
    if (_streamChannel != null) {
      supabase.removeChannel(_streamChannel!);
      _streamChannel = null;
    }
  }
}
```

### 3. استخدام Real-time في Audio Player

```dart
class RadioPlayerController {
  final AudioPlayer audioPlayer;
  final RadioStreamService streamService;
  String? currentStreamId;
  bool isPlaying = false;
  
  RadioPlayerController(this.audioPlayer, this.streamService);
  
  // ✅ بدء الاستماع مع Real-time subscription
  Future<void> startListening(String streamId) async {
    currentStreamId = streamId;
    
    // 1. قراءة الرابط الحالي من قاعدة البيانات
    final streamUrl = await getCurrentStreamUrl(streamId);
    if (streamUrl == null) {
      throw Exception('لا يوجد بث نشط');
    }
    
    // 2. بدء الاستماع
    await audioPlayer.setUrl(streamUrl);
    await audioPlayer.play();
    isPlaying = true;
    
    // 3. إعداد Real-time subscription لتحديثات الرابط
    streamService.setupStreamUrlSubscription(streamId, (newUrl) {
      _handleStreamUrlUpdate(newUrl);
    });
  }
  
  // ✅ معالجة تحديث رابط البث
  Future<void> _handleStreamUrlUpdate(String newUrl) async {
    if (newUrl == audioPlayer.url) {
      return; // نفس الرابط، لا حاجة للتحديث
    }
    
    print('[Radio Player] Updating stream URL: $newUrl');
    
    final wasPlaying = isPlaying;
    
    // إيقاف البث القديم
    await audioPlayer.stop();
    
    // تحديث الرابط الجديد
    await audioPlayer.setUrl(newUrl);
    
    // إعادة التشغيل إذا كان البث يعمل
    if (wasPlaying) {
      await audioPlayer.play();
      print('[Radio Player] Stream URL updated and resumed');
    } else {
      print('[Radio Player] Stream URL updated (paused)');
    }
  }
  
  // إيقاف الاستماع
  Future<void> stopListening() async {
    await audioPlayer.stop();
    isPlaying = false;
    streamService.dispose();
    currentStreamId = null;
  }
  
  // قراءة الرابط الحالي من قاعدة البيانات
  Future<String?> getCurrentStreamUrl(String streamId) async {
    final response = await streamService.supabase
      .from('club_activities')
      .select('listen_url, stream_url')
      .eq('id', streamId)
      .eq('is_active', true)
      .eq('is_live', true)
      .maybeSingle();
    
    if (response == null) return null;
    
    return response['listen_url'] ?? response['stream_url'];
  }
}
```

### 4. مثال كامل للاستخدام

```dart
class RadioPage extends StatefulWidget {
  @override
  _RadioPageState createState() => _RadioPageState();
}

class _RadioPageState extends State<RadioPage> {
  final AudioPlayer _audioPlayer = AudioPlayer();
  late final RadioStreamService _streamService;
  late final RadioPlayerController _playerController;
  String? _currentStreamId;
  
  @override
  void initState() {
    super.initState();
    _streamService = RadioStreamService(Supabase.instance.client);
    _playerController = RadioPlayerController(_audioPlayer, _streamService);
    _loadCurrentStream();
  }
  
  Future<void> _loadCurrentStream() async {
    // جلب البث الحالي
    final stream = await _getCurrentStream();
    if (stream != null) {
      setState(() {
        _currentStreamId = stream['id'];
      });
    }
  }
  
  Future<Map<String, dynamic>?> _getCurrentStream() async {
    final response = await Supabase.instance.client
      .from('club_activities')
      .select()
      .eq('activity_type', 'radio_stream')
      .eq('is_active', true)
      .eq('is_live', true)
      .maybeSingle();
    
    return response;
  }
  
  Future<void> _startListening() async {
    if (_currentStreamId == null) return;
    
    try {
      await _playerController.startListening(_currentStreamId!);
      setState(() {});
    } catch (e) {
      print('Error starting listening: $e');
    }
  }
  
  Future<void> _stopListening() async {
    await _playerController.stopListening();
    setState(() {});
  }
  
  @override
  void dispose() {
    _playerController.stopListening();
    _audioPlayer.dispose();
    super.dispose();
  }
  
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Column(
        children: [
          // UI للتحكم في البث
          ElevatedButton(
            onPressed: _playerController.isPlaying ? _stopListening : _startListening,
            child: Text(_playerController.isPlaying ? 'إيقاف' : 'تشغيل'),
          ),
        ],
      ),
    );
  }
}
```

## 🔍 التحقق من التنفيذ

### 1. اختبار في الموبايل

1. **ابدأ الاستماع للراديو**
2. **في الداش بورد، اضغط على "تحديث الرابط"**
3. **أدخل رابط جديد**
4. **تحقق من Console في الموبايل - يجب أن ترى:**
   ```
   [Radio Stream] Stream URL updated: http://new-url.com/stream.mp3
   [Radio Player] Updating stream URL: http://new-url.com/stream.mp3
   [Radio Player] Stream URL updated and resumed
   ```

### 2. التحقق من قاعدة البيانات

```sql
-- في Supabase SQL Editor
SELECT id, stream_url, listen_url, updated_at
FROM club_activities
WHERE activity_type = 'radio_stream'
  AND is_active = true
  AND is_live = true;
```

### 3. اختبار Real-time Subscription

- افتح Console في الموبايل
- راقب الرسائل: `[Radio Stream] Stream URL updated`
- تحقق من أن audio player يتحدث تلقائياً

## ⚠️ ملاحظات مهمة

### 1. Real-time Subscription مطلوب

**بدون Real-time subscription، الموبايل لن يعرف بتحديث الرابط تلقائياً.**

### 2. قراءة الرابط من قاعدة البيانات

**لا تستخدم cache للرابط.** يجب قراءة الرابط من قاعدة البيانات في كل مرة.

### 3. معالجة تغيير الرابط

**عند تحديث الرابط:**
- أوقف البث القديم
- حدث الرابط في audio player
- أعد التشغيل إذا كان البث يعمل

### 4. إلغاء الاشتراك

**تأكد من إلغاء Real-time subscription عند:**
- إيقاف الاستماع
- إغلاق الصفحة
- تغيير البث

## 🎯 الخطوات التالية

1. **إضافة Real-time Subscription:**
   - تابع تحديثات `club_activities` عبر Supabase Realtime
   - عند تحديث الرابط، قم بتحديث audio player تلقائياً

2. **قراءة الرابط من قاعدة البيانات:**
   - لا تستخدم cache للرابط
   - اقرأ `listen_url` من `club_activities` عند بدء الاستماع

3. **معالجة تغيير الرابط:**
   - عند تحديث الرابط، أوقف البث القديم
   - ابدأ البث الجديد بالرابط المحدث
   - أبلغ المستخدم بتغيير الرابط (اختياري)
