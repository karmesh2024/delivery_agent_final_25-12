## 🎯 نظرة عامة على النظام الجديد

### الهدف الرئيسي

بناء نظام نقاط مرن ومستدام لنادي Scope Zone يسمح بـ:

1. **فصل واضح** بين نقاط المخلفات ونقاط النادي
2. **إدارة مرنة** لمعدلات النقاط لكل خدمة
3. **اعتماد شهري** للنقاط المكتسبة (PENDING → AVAILABLE)
4. **ربط مركزي** بجوائز وهدايا موحدة
5. **تحكم كامل** في التكاليف والاستدامة المالية

### المبادئ الأساسية

```
✅ نقاط النادي = أداة تحفيز + وصول لمزايا (ليست فلوس)
✅ كل خدمة لها معدل نقاط قابل للتعديل
✅ النقاط تُعتمد شهرياً (PENDING → AVAILABLE)
✅ الجوائز لها تكلفة نقاط ثابتة (مستقلة عن المصدر)
✅ الشريك "النظام" يحدد معدلات النقاط للخدمات
✅ الرعاة يموّلون الجوائز فقط (لا يحددون معدلات النقاط)
```

## 🧠 الفهم الصحيح للنظام

### 1. الفصل بين أنظمة النقاط

```
┌─────────────────────────────────────────────────────────┐
│ نظام نقاط المخلفات (منفصل تماماً)                      │
│ ─────────────────────────────────────────────────────── │
│ • مصدر: بيع المخلفات                                    │
│ • القيمة: قيمة نقدية فعلية                              │
│ • الاستخدام: تحويل إلى رصيد مالي                        │
│ • يمكن تحويلها إلى نقاط النادي (30% مثلاً)             │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ نظام نقاط النادي (متعدد المصادر)                        │
│ ─────────────────────────────────────────────────────── │
│ • مصدر 1: راديو كارمش (1 نقطة/دقيقة)                   │
│ • مصدر 2: حملة بيئية (10 نقاط/مشاركة)                  │
│ • مصدر 3: مشاهدة إعلان (2 نقطة/إعلان)                  │
│ • مصدر 4: حضور فعالية (50 نقطة/فعالية)                 │
│ • الاستخدام: استبدال جوائز وهدايا                      │
│ • الحالة: PENDING → AVAILABLE (شهرياً)                  │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ نظام الجوائز (الهدف المشترك)                            │
│ ─────────────────────────────────────────────────────── │
│ • كرت شحن 50 جنيه = 500 نقطة                            │
│ • منتج من شريك = 300 نقطة                               │
│ • دخول سحب شهري = 150 نقطة                              │
│ • النقاط تأتي من أي مصدر (راديو، حملة، إلخ)            │
└─────────────────────────────────────────────────────────┘
```

### 2. أنواع الشركاء

#### أ. الشريك "النظام" (System Partner)
```
المسؤوليات:
├── تقديم خدمات النادي (الراديو، الحملات، الفعاليات)
├── تحديد معدلات النقاط لكل خدمة
├── إدارة الاعتمادات الشهرية
└── تمويل الجوائز الأساسية

مثال:
- خدمة الراديو: 1 نقطة/دقيقة
- خدمة الحملة: 10 نقاط/مشاركة
- خدمة الإعلان: 2 نقطة/إعلان
```

#### ب. الرعاة (Sponsors)
```
المسؤوليات:
├── تمويل الجوائز (منتجات، خدمات، خصومات)
├── تقديم منتجاتهم كجوائز
└── لا يحددون معدلات النقاط

مثال:
- شركة X تموّل: "كرت شحن 50 جنيه = 500 نقطة"
- شركة Y تموّل: "منتج تنظيف = 300 نقطة"
- المستخدم يجمع النقاط من أي مصدر ويستبدل
```

### 3. دورة حياة النقاط

```
┌─────────────────────────────────────────────────────────┐
│ 1. كسب النقاط (EARNED)                                  │
│    └── تذهب إلى: PENDING                                │
│    └── مثال: استمع 30 دقيقة = 30 نقطة PENDING          │
└─────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│ 2. الاعتماد الشهري (ACTIVATED)                         │
│    └── PENDING → AVAILABLE                              │
│    └── مثال: 30 نقطة PENDING → 30 نقطة AVAILABLE       │
└─────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│ 3. الاستبدال (USED)                                     │
│    └── AVAILABLE → USED                                 │
│    └── مثال: استبدال جائزة 80 نقطة                      │
│        └── 30 نقطة AVAILABLE → 0                        │
│        └── باقي 50 نقطة تبقى PENDING                    │
└─────────────────────────────────────────────────────────┘
```

## 🏗️ الهيكل المقترح

### 1. الهيكل العام

```
club_service_points_config (إعدادات معدلات النقاط)
    ├── service_name: 'radio_stream'
    ├── base_points_rate: 1.0
    └── points_rate_type: 'per_minute'
            ↓
club_activities (البث/النشاط)
    ├── service_config_id → club_service_points_config
    ├── custom_points_rate (اختياري - معدل مخصص)
    └── points_reward (legacy - للتوافق)
            ↓
radio_listeners / campaign_participants (المشاركون)
    ├── user_id → new_profiles
    ├── activity_id → club_activities
    └── points_earned (محسوبة من الإعدادات)
            ↓
club_points_wallet (محفظة النقاط)
    ├── pending_points (مكتسبة، لم تُعتمد)
    ├── available_points (معتمدة، جاهزة للاستخدام)
    ├── used_points (مستخدمة)
    └── lifetime_points (إجمالي المكتسبة)
            ↓
club_points_transactions (معاملات النقاط)
    ├── transaction_type: 'EARNED' | 'ACTIVATED' | 'USED'
    ├── source: 'radio_stream' | 'campaign' | 'ad_view'
    └── reference_id → club_activities.id
            ↓
club_rewards (الجوائز)
    ├── points_required (تكلفة ثابتة)
    └── partner_id → club_partners (الراعي المموّل)
            ↓
club_reward_redemptions (الاستبدالات)
    ├── user_id → new_profiles
    ├── reward_id → club_rewards
    ├── points_spent (من available_points)
    └── delivery_method: 'COLLECTION_POINT' | 'DIGITAL'
```

### 2. العلاقات بين الجداول

```
new_profiles (الأعضاء)
    ├── 1:1 → club_memberships
    ├── 1:1 → club_points_wallet
    ├── 1:N → club_points_transactions
    ├── 1:N → radio_listeners
    └── 1:N → club_reward_redemptions

club_service_points_config (إعدادات الخدمات)
    ├── 1:N → club_activities (service_config_id)
    └── N:1 → club_partners (partner_id) - NULL = النظام

club_activities (الأنشطة)
    ├── N:1 → club_service_points_config
    ├── N:1 → club_partners (partner_id - الراعي)
    └── 1:N → radio_listeners

club_points_wallet (المحفظة)
    └── 1:N → club_points_transactions

club_rewards (الجوائز)
    ├── N:1 → club_partners (partner_id - الراعي المموّل)
    └── 1:N → club_reward_redemptions

collection_points (منافذ الاستلام)
    └── 1:N → club_reward_redemptions (collection_point_id)
```

### 1. جدول إعدادات معدلات النقاط (`club_service_points_config`)

```sql
CREATE TABLE IF NOT EXISTS club_service_points_config (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- معلومات الخدمة
  service_name VARCHAR(100) NOT NULL UNIQUE,
  service_title VARCHAR(255) NOT NULL,
  service_type VARCHAR(50) NOT NULL CHECK (service_type IN (
    'radio',           -- بث راديو
    'campaign',        -- حملة بيئية
    'ad_view',         -- مشاهدة إعلان
    'event',           -- حضور فعالية
    'challenge',       -- تحدٍ أو مسابقة
    'referral',        -- إحالة صديق
    'manual',          -- إدخال يدوي
    'other'            -- خدمات أخرى
  )),
  
  -- إعدادات معدل النقاط
  points_rate_type VARCHAR(20) NOT NULL CHECK (points_rate_type IN (
    'per_minute',      -- نقطة/دقيقة (للراديو)
    'per_action',      -- نقطة/عمل (للحملات)
    'per_item',        -- نقطة/عنصر
    'fixed',           -- نقاط ثابتة
    'percentage'       -- نسبة من قيمة
  )),
  
  base_points_rate DECIMAL(10, 2) NOT NULL DEFAULT 1.0,
  min_points INTEGER DEFAULT 0,
  max_points INTEGER, -- NULL = غير محدود
  max_points_per_day INTEGER, -- NULL = غير محدود
  
  -- إعدادات الشريك
  partner_id UUID REFERENCES club_partners(id), -- NULL = الشريك "النظام"
  is_system_service BOOLEAN DEFAULT TRUE,
  
  -- إعدادات إضافية
  multiplier_factor DECIMAL(5, 2) DEFAULT 1.0, -- معامل الضرب (للمستويات)
  bonus_enabled BOOLEAN DEFAULT FALSE,
  bonus_conditions JSONB, -- {"after_minutes": 30, "bonus_points": 10}
  
  -- الإعدادات العامة
  is_active BOOLEAN DEFAULT TRUE,
  description TEXT,
  created_by UUID REFERENCES new_profiles(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  
  UNIQUE(service_name)
);

CREATE INDEX idx_service_points_config_type ON club_service_points_config(service_type);
CREATE INDEX idx_service_points_config_partner ON club_service_points_config(partner_id);
CREATE INDEX idx_service_points_config_active ON club_service_points_config(is_active);

COMMENT ON TABLE club_service_points_config IS 'إعدادات معدلات النقاط لكل خدمة في النادي';
COMMENT ON COLUMN club_service_points_config.service_name IS 'اسم الخدمة الفني (radio_stream, campaign_xyz)';
COMMENT ON COLUMN club_service_points_config.is_system_service IS 'true = خدمة النظام (الشريك "النظام")، false = خدمة شريك معين';
COMMENT ON COLUMN club_service_points_config.base_points_rate IS 'معدل النقاط الأساسي (نقطة/دقيقة، نقطة/عمل، إلخ)';
COMMENT ON COLUMN club_service_points_config.bonus_conditions IS 'شروط المكافآت الإضافية: {"after_minutes": 30, "bonus_points": 10}';
```

```sql
COMMENT ON COLUMN club_points_wallet.available_points IS 'نقاط معتمدة جاهزة للاستخدام (AVAILABLE)';
COMMENT ON COLUMN club_points_wallet.used_points IS 'نقاط تم استخدامها (USED)';
```

#### ✅ قرار تصميم مهم: إلغاء `points_balance` كعمود أساسي (لتقليل الأخطاء)

**المشكلة في `points_balance`**: وجوده كعمود “حسابي” بجانب `available_points` و`used_points` يفتح باب الـ double accounting (خصم من available + زيادة used + ثم تحديث balance) ويزيد احتمالات الـ bugs.

**التصميم النهائي المقترح**:
- **`available_points` هو الرصيد الجاهز للاستخدام**.
- `pending_points` و`used_points` لأغراض التتبع والمحاسبة.
- `lifetime_points` إجمالي ما تم اكتسابه (والتحويلات إن رغبت).

**Backward compatibility (اختياري)**:
- بدل عمود `points_balance`، نعمل **View**:

```sql
CREATE OR REPLACE VIEW club_points_wallet_v AS
SELECT
  *,
  available_points AS points_balance
FROM club_points_wallet;
```

> لو عندك كود قديم يعتمد على `points_balance`، خليه يقرأ من `club_points_wallet_v` بدل الجدول مباشرة.

## ⚙️ Functions والخدمات

### 1. Function: `get_service_points_rate`

```sql
CREATE OR REPLACE FUNCTION get_service_points_rate(
  p_service_name VARCHAR,
  p_activity_id UUID DEFAULT NULL
) RETURNS DECIMAL(10, 2)
LANGUAGE plpgsql
STABLE
AS $$
DECLARE
  v_rate DECIMAL(10, 2);
  v_custom_rate DECIMAL(10, 2);
BEGIN
  -- 1. التحقق من وجود معدل مخصص في النشاط
  IF p_activity_id IS NOT NULL THEN
    SELECT custom_points_rate INTO v_custom_rate
    FROM club_activities
    WHERE id = p_activity_id AND custom_points_rate IS NOT NULL;
    
    IF v_custom_rate IS NOT NULL THEN
      RETURN v_custom_rate;
    END IF;
  END IF;
  
  -- 2. الحصول من إعدادات الخدمة
  SELECT base_points_rate INTO v_rate
  FROM club_service_points_config
  WHERE service_name = p_service_name 
    AND is_active = TRUE;
  
  RETURN COALESCE(v_rate, 1.0); -- Default: 1 نقطة
END;
$$;

COMMENT ON FUNCTION get_service_points_rate IS 'الحصول على معدل النقاط لخدمة معينة (يفضل المعدل المخصص في النشاط)';
```

### 2. Function: `calculate_service_points`

```sql
CREATE OR REPLACE FUNCTION calculate_service_points(
  p_service_name VARCHAR,
  p_activity_id UUID DEFAULT NULL,
  p_amount DECIMAL DEFAULT 1.0 -- المدة بالدقائق أو عدد الأعمال
) RETURNS INTEGER
LANGUAGE plpgsql
STABLE
AS $$
DECLARE
  v_config club_service_points_config%ROWTYPE;
  v_rate DECIMAL(10, 2);
  v_points INTEGER;
  v_custom_rate DECIMAL(10, 2);
BEGIN
  -- 1. الحصول على إعدادات الخدمة
  SELECT * INTO v_config
  FROM club_service_points_config
  WHERE service_name = p_service_name AND is_active = TRUE;
  
  IF NOT FOUND THEN
    RAISE EXCEPTION 'الخدمة غير موجودة أو غير نشطة: %', p_service_name;
  END IF;
  
  -- 2. التحقق من معدل مخصص في النشاط
  IF p_activity_id IS NOT NULL THEN
    SELECT custom_points_rate INTO v_custom_rate
    FROM club_activities
    WHERE id = p_activity_id;
    
    IF v_custom_rate IS NOT NULL THEN
      v_rate := v_custom_rate;
    ELSE
      v_rate := v_config.base_points_rate;
    END IF;
  ELSE
    v_rate := v_config.base_points_rate;
  END IF;
  
  -- 3. حساب النقاط حسب نوع المعدل
  CASE v_config.points_rate_type
    WHEN 'per_minute' THEN
      v_points := FLOOR(p_amount * v_rate);
    WHEN 'per_action' THEN
      v_points := FLOOR(p_amount * v_rate);
    WHEN 'fixed' THEN
      v_points := FLOOR(v_rate);
    ELSE
      v_points := FLOOR(p_amount * v_rate);
  END CASE;
  
  -- 4. تطبيق الحدود
  IF v_config.min_points IS NOT NULL AND v_points < v_config.min_points THEN
    v_points := v_config.min_points;
  END IF;
  
  IF v_config.max_points IS NOT NULL AND v_points > v_config.max_points THEN
    v_points := v_config.max_points;
  END IF;
  
  RETURN v_points;
END;
$$;

COMMENT ON FUNCTION calculate_service_points IS 'حساب النقاط المستحقة لخدمة معينة مع تطبيق الحدود والشروط';
```

### 3. Function: `update_club_points_wallet` (محدثة)

```sql
CREATE OR REPLACE FUNCTION update_club_points_wallet(
  p_user_id UUID,
  p_points INTEGER,
  p_transaction_type VARCHAR,
  p_reason VARCHAR DEFAULT NULL,
  p_source VARCHAR DEFAULT NULL,
  p_description TEXT DEFAULT NULL,
  p_created_by UUID DEFAULT NULL,
  p_reference_id UUID DEFAULT NULL -- radio_id, campaign_id, etc.
) RETURNS UUID
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_wallet_id UUID;
  v_transaction_id UUID;
  v_available_before INTEGER;
  v_available_after INTEGER;
  v_lifetime_before INTEGER;
  v_lifetime_after INTEGER;
  v_pending_before INTEGER;
  v_used_before INTEGER;
  v_pending_after INTEGER;
  v_used_after INTEGER;
BEGIN
  -- 1. إنشاء/الحصول على المحفظة
  INSERT INTO club_points_wallet (user_id, pending_points, available_points, used_points, lifetime_points)
  VALUES (p_user_id, 0, 0, 0, 0)
  ON CONFLICT (user_id) DO NOTHING;

  SELECT id, pending_points, available_points, used_points, lifetime_points
  INTO v_wallet_id, v_pending_before, v_available_before, v_used_before, v_lifetime_before
  FROM club_points_wallet
  WHERE user_id = p_user_id;

  -- 2. تحديث المحفظة حسب نوع المعاملة
  IF p_transaction_type = 'EARNED' THEN
    -- النقاط تُضاف إلى PENDING
    UPDATE club_points_wallet
    SET 
      pending_points = pending_points + p_points,
      lifetime_points = lifetime_points + p_points,
      updated_at = NOW()
    WHERE id = v_wallet_id;

    SELECT pending_points, available_points, used_points, lifetime_points
    INTO v_pending_after, v_available_after, v_used_after, v_lifetime_after
    FROM club_points_wallet
    WHERE id = v_wallet_id;

  ELSIF p_transaction_type = 'ACTIVATED' THEN
    -- تحويل PENDING → AVAILABLE
    UPDATE club_points_wallet
    SET 
      pending_points = GREATEST(0, pending_points - p_points),
      available_points = available_points + p_points,
      updated_at = NOW()
    WHERE id = v_wallet_id;

    SELECT pending_points, available_points, used_points, lifetime_points
    INTO v_pending_after, v_available_after, v_used_after, v_lifetime_after
    FROM club_points_wallet
    WHERE id = v_wallet_id;

  ELSIF p_transaction_type = 'USED' THEN
    -- خصم من AVAILABLE وتحديث USED
    UPDATE club_points_wallet
    SET 
      available_points = GREATEST(0, available_points - ABS(p_points)),
      used_points = used_points + ABS(p_points),
      updated_at = NOW()
    WHERE id = v_wallet_id;

    SELECT pending_points, available_points, used_points, lifetime_points
    INTO v_pending_after, v_available_after, v_used_after, v_lifetime_after
    FROM club_points_wallet
    WHERE id = v_wallet_id;

  ELSIF p_transaction_type = 'CONVERTED' THEN
    -- تحويل من نقاط المخلفات → AVAILABLE مباشرة (مميزة)
    UPDATE club_points_wallet
    SET 
      available_points = available_points + p_points,
      lifetime_points = lifetime_points + p_points,
      updated_at = NOW()
    WHERE id = v_wallet_id;

    SELECT pending_points, available_points, used_points, lifetime_points
    INTO v_pending_after, v_available_after, v_used_after, v_lifetime_after
    FROM club_points_wallet
    WHERE id = v_wallet_id;

  ELSE
    -- ADJUSTED, BONUS, EXPIRED - نفس المنطق القديم
    UPDATE club_points_wallet
    SET 
      -- في حالات التعديل/المكافأة/انتهاء الصلاحية، القرار يعتمد على السياسة:
      -- الافتراضي هنا: تعديل على available_points فقط
      available_points = GREATEST(0, available_points + p_points),
      lifetime_points = lifetime_points + GREATEST(0, p_points),
      updated_at = NOW()
    WHERE id = v_wallet_id;

    SELECT pending_points, available_points, used_points, lifetime_points
    INTO v_pending_after, v_available_after, v_used_after, v_lifetime_after
    FROM club_points_wallet
    WHERE id = v_wallet_id;
  END IF;

  v_available_after := COALESCE(v_available_after, 0);
  v_lifetime_after := COALESCE(v_lifetime_after, v_lifetime_before);

  -- الحقول المطلوبة للـ transactions
  -- سنخزّن “قبل/بعد” على مستوى available_points (الرصيد القابل للاستخدام)
  v_available_before := COALESCE(v_available_before, 0);

  -- 3. إنشاء سجل المعاملة
  INSERT INTO club_points_transactions (
    user_id,
    transaction_type,
    points,
    points_before,
    points_after,
    reason,
    source,
    description,
    created_by,
    reference_id
  ) VALUES (
    p_user_id,
    p_transaction_type,
    p_points,
    v_available_before,
    v_available_after,
    p_reason,
    p_source,
    p_description,
    p_created_by,
    p_reference_id
  )
  RETURNING id INTO v_transaction_id;

  RETURN v_transaction_id;
END;
$$;

COMMENT ON FUNCTION update_club_points_wallet IS 'تحديث محفظة نقاط النادي مع دعم PENDING/AVAILABLE/USED';
```

### 4. Function: `activate_monthly_points`

```sql
CREATE OR REPLACE FUNCTION activate_monthly_points(
  p_settlement_month VARCHAR(7), -- '2026-01'
  p_processed_by UUID,
  p_notes TEXT DEFAULT NULL
) RETURNS UUID
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_settlement_id UUID;
  v_total_pending INTEGER;
  v_total_activated INTEGER;
  v_users_count INTEGER;
  v_user_record RECORD;
BEGIN
  -- 1. إنشاء سجل الاعتماد الشهري
  INSERT INTO monthly_points_settlement (
    settlement_month,
    processed_by,
    notes,
    status
  ) VALUES (
    p_settlement_month,
    p_processed_by,
    p_notes,
    'processing'
  )
  RETURNING id INTO v_settlement_id;

  -- 2. حساب إجمالي النقاط المعلقة والمستخدمين المتأثرين
  SELECT 
    COALESCE(SUM(pending_points), 0),
    COUNT(DISTINCT user_id)
  INTO v_total_pending, v_users_count
  FROM club_points_wallet
  WHERE pending_points > 0;

  -- 3. تحويل جميع النقاط المعلقة إلى متاحة
  -- ✅ مهم: ممنوع double update.
  -- هنا نعتمد على update_club_points_wallet وحدها لتعديل المحفظة + إنشاء معاملة.
  v_total_activated := 0;
  
  FOR v_user_record IN 
    SELECT user_id, pending_points
    FROM club_points_wallet
    WHERE pending_points > 0
  LOOP
    -- إنشاء معاملة ACTIVATED
    PERFORM update_club_points_wallet(
      v_user_record.user_id,
      v_user_record.pending_points,
      'ACTIVATED',
      'اعتماد شهري',
      'monthly_settlement',
      'اعتماد نقاط شهر ' || p_settlement_month,
      p_processed_by,
      NULL
    );

    v_total_activated := v_total_activated + v_user_record.pending_points;
  END LOOP;

  -- 4. تحديث سجل الاعتماد
  UPDATE monthly_points_settlement
  SET 
    total_pending_points = v_total_pending,
    total_activated_points = v_total_activated,
    total_users_affected = v_users_count,
    status = 'completed',
    processed_at = NOW()
  WHERE id = v_settlement_id;

  RETURN v_settlement_id;
END;
$$;

COMMENT ON FUNCTION activate_monthly_points IS 'اعتماد جميع النقاط المعلقة (PENDING) وتحويلها إلى متاحة (AVAILABLE)';
```

## ❓ الأسئلة الشائعة

### س1: لماذا PENDING و AVAILABLE؟
**ج:** للتحكم في التكاليف والاستدامة المالية. يمكن للأدمن تأجيل الاعتماد أو تقليل النسبة حسب الميزانية.

### س2: متى تُعتمد النقاط؟
**ج:** نهاية كل شهر (مثلاً: 31 يناير). يمكن للأدمن تغيير التاريخ حسب الحاجة.

### س3: ماذا لو رفض المستخدم GPS؟
**ج:** يتم استخدام العنوان الافتراضي أو اختيار الحي. الموقع لا يؤثر على النقاط، فقط للتتبع الجغرافي.

### س4: هل يمكن تعديل معدل النقاط بعد بدء البث؟
**ج:** نعم، لكن التعديل يؤثر على النقاط الجديدة فقط. النقاط المكتسبة سابقاً محفوظة.

### س5: كيف يتم تمويل الجوائز؟
**ج:** من 3 مصادر:
1. الرعاة (يموّلون منتجاتهم)
2. فائض منظومة المخلفات
3. ميزانية النادي

### س6: هل يمكن استبدال جائزة بنقاط معلقة؟
**ج:** لا، فقط النقاط المعتمدة (AVAILABLE) يمكن استخدامها. النقاط المعلقة (PENDING) تنتظر الاعتماد.

### س7: ماذا يحدث إذا لم يتم الاعتماد الشهري؟
**ج:** النقاط تبقى في PENDING حتى يتم الاعتماد. يمكن للأدمن اعتمادها يدوياً في أي وقت.

### س8: هل يمكن إلغاء الاعتماد؟
**ج:** لا، الاعتماد نهائي. لكن يمكن للأدمن تعديل النقاط يدوياً (ADJUSTED) في حالات خاصة.
