# توثيق تطبيق العميل (Customer Karmesh)

## نظرة عامة

تطبيق العميل (Customer Karmesh) هو تطبيق محمول مطور باستخدام إطار عمل Flutter وبرمجة Dart، مصمم للعملاء الراغبين في بيع المخلفات القابلة للتدوير والاستفادة من نظام النقاط والمكافآت. يوفر التطبيق واجهة مستخدم سلسة ومتكاملة مع خدمات Supabase في الخلفية لإدارة البيانات والمصادقة.

## الهيكل المعماري

يتبع تطبيق العميل نمط البنية المعمارية Clean Architecture مع نمط BLoC (Business Logic Component) لإدارة حالة التطبيق، مما يضمن فصل المسؤوليات وقابلية الاختبار والصيانة.

### المكونات الرئيسية

1. **Core**: المكونات الأساسية والمشتركة
   - `common`: كونفيجرات وودجات مشتركة
   - `constants`: ثوابت التطبيق (ألوان، نصوص، ...)
   - `data`: نماذج البيانات والبيئة
   - `error`: معالجة الاستثناءات والأخطاء
   - `network`: خدمات الشبكة والاتصال
   - `presentation`: مكونات العرض المشتركة
   - `route`: إدارة التنقل والمسارات
   - `services`: الخدمات الأساسية
   - `utils`: أدوات مساعدة

2. **Futures**: الميزات الوظيفية للتطبيق
   - `auth_by_phone`: المصادقة عبر رقم الهاتف
   - `basketWastePage`: إدارة سلة النفايات
   - `CartPage`: إدارة سلة التسوق
   - `categoriesProducts`: فئات المنتجات والمخلفات
   - `home`: الصفحة الرئيسية
   - `intro/onBoarding`: صفحات الترحيب والتعريف
   - `invoiceFinal`: إدارة الفواتير
   - `map`: خدمات الخرائط والمواقع
   - `orders`: إدارة الطلبات
   - `users/profile`: إدارة ملف المستخدم

3. **مكونات أخرى**:
   - `app.dart`: تكوين التطبيق الرئيسي
   - `init_dependencies.dart`: تسجيل تبعيات التطبيق (حقن التبعية)
   - `main.dart`: نقطة البداية للتطبيق

## تدفق البيانات

1. **نقطة البداية**: يبدأ التطبيق في ملف `main.dart` حيث يتم تهيئة التبعيات وتحديد المسار الأولي بناءً على حالة المستخدم.

2. **إدارة الحالة**: يستخدم التطبيق مكتبة `flutter_bloc` لإدارة حالة التطبيق، مع مكونات BLoC لكل ميزة رئيسية.

3. **التفاعل مع قاعدة البيانات**: يتم التواصل مع Supabase من خلال مصادر البيانات (DataSources) التي تتفاعل مع API وقاعدة البيانات.

4. **المستودعات**: طبقة الوسيط بين مصادر البيانات وطبقة العرض، وتوفر واجهة متسقة للوصول إلى البيانات.

## تنفيذ الميزات الرئيسية

### 1. المصادقة وإدارة الحساب

يستخدم التطبيق `AuthBlocCustomer` للتعامل مع عمليات المصادقة وإدارة جلسات المستخدمين:

```dart
BlocProvider<AuthBlocCustomer>(
  create: (_) => serviceLocator<AuthBlocCustomer>(),
  ...
)
```

العملية تبدأ من الشاشات في مجلد `auth_by_phone` حيث يتم التحقق من الهاتف وإدارة رموز التحقق.

### 2. استعراض فئات المخلفات

يستخدم التطبيق `CategoryBloc` للتعامل مع بيانات الفئات والفئات الفرعية من خلال `CategoryRepository`:

```dart
BlocProvider(
  create: (context) {
    final wasteDataSource = WasteDataSource(serviceLocator<SupabaseClient>());
    return CategoryBloc(CategoryRepository(wasteDataSource));
  },
)
```

### 3. إدارة الطلبات

يتم إدارة الطلبات من خلال `OrdersBloc` الذي يتعامل مع `customer_orders` و`delivery_orders`:

```dart
BlocProvider(create: (_) => serviceLocator<OrdersBloc>()),
```

### 4. إدارة سلة المخلفات

يتم إدارة سلة المخلفات من خلال `BasketWasteBloc` و`CartCubit`:

```dart
BlocProvider(create: (_) => BasketWasteBloc()),
BlocProvider(create: (_) => CartCubit()),
```

## تحديات التطوير والحلول

1. **إدارة الحالة**: استخدام نمط BLoC لفصل منطق الأعمال عن واجهة المستخدم.
2. **المصادقة**: تنفيذ المصادقة عبر الهاتف مع رموز التحقق.
3. **العمل دون اتصال**: تنفيذ آلية تخزين مؤقت للعمل في وضع عدم الاتصال.
4. **تتبع الموقع**: دمج خدمات الموقع الجغرافي والخرائط.
5. **أمان البيانات**: تنفيذ سياسات الوصول المناسبة في Supabase.

## الجداول والدوال غير المستخدمة

على الرغم من وجود العديد من الجداول في قاعدة البيانات، فإن بعضها غير مستخدم حاليًا في تطبيق العميل ويمكن الاستفادة منها في المستقبل:

1. **`analytics_dashboard`**: يمكن استخدامه لعرض إحصائيات للعملاء المميزين.
2. **`system_notifications`**: يمكن استخدامه لإرسال إشعارات نظام مخصصة.
3. **`customer_ratings`**: يمكن تفعيله لتمكين العملاء من تقييم خدمة التوصيل.
4. **`payment_qr_codes`**: يمكن استخدامه لتنفيذ نظام دفع عبر رموز QR.
5. **`wallet_transactions`**: يمكن تفعيله لتقديم نظام محفظة إلكترونية كامل.

## الخلاصة

تطبيق العميل (Customer Karmesh) هو تطبيق شامل يوفر واجهة سهلة الاستخدام للعملاء لإدارة مخلفاتهم والاستفادة من نظام الحوافز. يتميز التطبيق بهيكل معماري نظيف ومنظم يسهل صيانته وتطويره في المستقبل.

التفاعل مع قاعدة البيانات يتم من خلال Supabase مع مجموعة من الجداول والدوال والتريجرز المصممة خصيصًا لتلبية احتياجات تطبيق العميل، مع إمكانية التوسع في المستقبل لتقديم المزيد من الميزات.
