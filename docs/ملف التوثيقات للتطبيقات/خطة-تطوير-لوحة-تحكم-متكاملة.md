# خطة تطوير شاملة للوحة تحكم متكاملة لإدارة تطبيقي العميل ومندوب التوصيل

## مقدمة

تهدف هذه الوثيقة إلى تقديم خطة تطوير شاملة ومفصلة لتحويل لوحة التحكم الحالية (Delivery-Agent-Dashboard) إلى نظام إدارة ومراقبة متكامل لكل من تطبيق العميل وتطبيق مندوب التوصيل (Delivery Karmesh). تستند هذه الخطة إلى تحليل متعمق لقواعد البيانات المستخدمة في كلا التطبيقين، ونقاط الربط بينهما، وإمكانيات التكامل المستقبلية.

تراعي هذه الخطة الوضع الحالي للمشروع الذي يمر بمرحلة إعادة هيكلة لاعتماد نهج التصميم الموجه بالنطاقات (Domain-Driven Design) واستخدام Redux لإدارة الحالة، وتقترح خارطة طريق متكاملة تجمع بين استكمال عملية إعادة الهيكلة وتطوير وظائف متقدمة للإدارة والمراقبة.

## الوضع الحالي للمشروع

### مشروع لوحة التحكم

- **البنية التحتية**: يستخدم المشروع Next.js مع TypeScript، ويخضع حالياً لإعادة هيكلة لاعتماد نهج DDD وRedux
- **حالة إعادة الهيكلة**: بعض النطاقات (مثل الطلبات والمندوبين والخرائط) مكتملة التحويل، بينما لا تزال نطاقات أخرى قيد العمل
- **التحديات الحالية**: عدم اتساق استخدام Redux بين الصفحات المختلفة وتنوع طرق الوصول للبيانات

### قواعد البيانات في التطبيقين

- **النظام**: يستخدم كلا التطبيقين Supabase المبنية على PostgreSQL
- **الجداول المشتركة**: تتشارك التطبيقات في مجموعة من الجداول الأساسية (الطلبات، العملاء، المندوبين، جمع المخلفات)
- **نقاط الربط**: تتمثل نقاط الربط الرئيسية في دورة حياة الطلب من إنشائه بواسطة العميل حتى إكماله بواسطة المندوب

## رؤية لوحة التحكم المتكاملة

نسعى لتطوير لوحة تحكم متكاملة تتيح للمشرفين إدارة ومراقبة كامل النظام البيئي بدءاً من تسجيل العملاء، ومروراً بإدارة الطلبات وتخصيصها للمندوبين، وانتهاءً بتتبع عمليات جمع المخلفات والفوترة. يجب أن توفر اللوحة رؤية شاملة لعمليات النظام في الوقت الفعلي، مع إمكانيات تحليل البيانات التاريخية لاتخاذ قرارات أفضل.

##### أ. إنشاء طبقة API موحدة لجميع النطاقات

```typescript
// نموذج لطبقة API موحدة لنطاق الرسائل
// src/domains/messages/api/messagesApi.ts

import { supabase } from '@/lib/supabase';
import { Message, Conversation } from '@/domains/messages/types';

export const messagesApi = {
  getConversations: async (): Promise<Conversation[]> => {
    const { data, error } = await supabase
      .from('conversations')
      .select('*')
      .order('created_at', { ascending: false });
    
    if (error) throw error;
    return data || [];
  },
  
  getMessagesByConversation: async (conversationId: string): Promise<Message[]> => {
    const { data, error } = await supabase
      .from('messages')
      .select('*')
      .eq('conversation_id', conversationId)
      .order('created_at', { ascending: true });
    
    if (error) throw error;
    return data || [];
  },
  
  sendMessage: async (message: Message): Promise<Message> => {
    const { data, error } = await supabase
      .from('messages')
      .insert(message)
      .select()
      .single();
    
    if (error) throw error;
    return data;
  },
};
```

##### ب. تطوير شرائح Redux للنطاقات المتبقية

```typescript
// نموذج لشريحة Redux لنطاق الرسائل
// src/domains/messages/store/messagesSlice.ts

import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import { messagesApi } from '../api/messagesApi';
import { Message, Conversation } from '../types';

export const fetchConversations = createAsyncThunk(
  'messages/fetchConversations',
  async () => {
    return await messagesApi.getConversations();
  }
);

export const fetchMessages = createAsyncThunk(
  'messages/fetchMessages',
  async (conversationId: string) => {
    return await messagesApi.getMessagesByConversation(conversationId);
  }
);

export const sendMessage = createAsyncThunk(
  'messages/sendMessage',
  async (message: Message) => {
    return await messagesApi.sendMessage(message);
  }
);

interface MessagesState {
  conversations: Conversation[];
  messages: Message[];
  selectedConversation: string | null;
  loading: boolean;
  error: string | null;
}

const initialState: MessagesState = {
  conversations: [],
  messages: [],
  selectedConversation: null,
  loading: false,
  error: null,
};

const messagesSlice = createSlice({
  name: 'messages',
  initialState,
  reducers: {
    setSelectedConversation: (state, action) => {
      state.selectedConversation = action.payload;
    },
  },
  extraReducers: (builder) => {
    builder
      .addCase(fetchConversations.pending, (state) => {
        state.loading = true;
        state.error = null;
      })
      .addCase(fetchConversations.fulfilled, (state, action) => {
        state.loading = false;
        state.conversations = action.payload;
      })
      .addCase(fetchConversations.rejected, (state, action) => {
        state.loading = false;
        state.error = action.error.message || 'فشل في جلب المحادثات';
      })
      .addCase(fetchMessages.pending, (state) => {
        state.loading = true;
        state.error = null;
      })
      .addCase(fetchMessages.fulfilled, (state, action) => {
        state.loading = false;
        state.messages = action.payload;
      })
      .addCase(fetchMessages.rejected, (state, action) => {
        state.loading = false;
        state.error = action.error.message || 'فشل في جلب الرسائل';
      })
      .addCase(sendMessage.fulfilled, (state, action) => {
        state.messages.push(action.payload);
      });
  },
});

export const { setSelectedConversation } = messagesSlice.actions;
export default messagesSlice.reducer;
```

##### د. تحويل الصفحات المتبقية لاستخدام Redux

```typescript
// نموذج لتحويل صفحة الرسائل
// src/domains/messages/pages/MessagesPage.tsx

"use client";

import { useEffect, useState } from "react";
import { useAppDispatch, useAppSelector } from "@/store/hooks";
import { fetchConversations, fetchMessages, setSelectedConversation } from "../store/messagesSlice";
import { ConversationList } from "../components/ConversationList";
import { MessageList } from "../components/MessageList";
import { MessageInput } from "../components/MessageInput";

export default function MessagesPage() {
  const dispatch = useAppDispatch();
  const { 
    conversations, 
    messages, 
    selectedConversation,
    loading
  } = useAppSelector(state => state.messages);

  useEffect(() => {
    dispatch(fetchConversations());
  }, [dispatch]);

  useEffect(() => {
    if (selectedConversation) {
      dispatch(fetchMessages(selectedConversation));
    }
  }, [dispatch, selectedConversation]);

  const handleConversationSelect = (conversationId: string) => {
    dispatch(setSelectedConversation(conversationId));
  };

  return (
    <div className="grid grid-cols-3 gap-4 h-full">
      <div className="col-span-1 border-l">
        <ConversationList 
          conversations={conversations}
          selectedId={selectedConversation}
          onSelect={handleConversationSelect}
          loading={loading}
        />
      </div>
      <div className="col-span-2 flex flex-col h-full">
        {selectedConversation ? (
          <>
            <div className="flex-1 overflow-y-auto">
              <MessageList messages={messages} loading={loading} />
            </div>
            <div className="py-4">
              <MessageInput conversationId={selectedConversation} />
            </div>
          </>
        ) : (
          <div className="flex items-center justify-center h-full">
            <p className="text-gray-500">اختر محادثة للبدء</p>
          </div>
        )}
      </div>
    </div>
  );
}
```

### المرحلة الثانية: تطوير نطاقات جديدة للإدارة المتكاملة (2-3 أشهر)

#### 1. نطاق إدارة العملاء الشامل (Customer Management)

##### أ. مكونات واجهة المستخدم

- **قائمة العملاء**: عرض جميع العملاء مع إمكانية البحث والتصفية
- **صفحة تفاصيل العميل**: عرض معلومات تفصيلية عن العميل وتاريخ الطلبات
- **خريطة عناوين العميل**: عرض عناوين العميل على الخريطة
- **إحصائيات العميل**: عرض إحصائيات العميل (عدد الطلبات، المبالغ، النقاط)

##### ب. وظائف إدارة العملاء

- إدارة حسابات العملاء (تفعيل، تعليق، إيقاف)
- إدارة برنامج الولاء والنقاط
- تتبع نشاط العميل وإصدار تقارير

#### 2. نطاق إدارة المندوبين المتكامل (Agent Management)

##### أ. مكونات واجهة المستخدم

- **قائمة المندوبين**: عرض جميع المندوبين مع حالاتهم الحالية
- **صفحة تفاصيل المندوب**: عرض معلومات تفصيلية عن المندوب
- **لوحة معلومات المندوب**: عرض إحصائيات وأداء المندوب
- **تتبع المندوب**: عرض موقع المندوب الحالي وتاريخ تحركاته

##### ب. وظائف إدارة المندوبين

- إدارة دورة حياة المندوب (تسجيل، اعتماد، تعليق)
- تقييم أداء المندوبين وتصنيفهم
- إدارة المدفوعات والحوافز

#### 3. نطاق إدارة الطلبات المتكامل (Order Management)

##### أ. مكونات واجهة المستخدم

- **قائمة الطلبات**: عرض جميع الطلبات مع إمكانية البحث والتصفية
- **صفحة تفاصيل الطلب**: عرض معلومات تفصيلية عن الطلب
- **تاريخ حالة الطلب**: عرض تاريخ تغييرات حالة الطلب
- **خريطة تتبع الطلب**: عرض مسار توصيل الطلب

##### ب. وظائف إدارة الطلبات

- إنشاء وتعديل وإلغاء الطلبات
- تخصيص الطلبات للمندوبين (يدوياً أو تلقائياً)
- تتبع حالة الطلب في الوقت الفعلي
- إدارة جدولة الطلبات والتذكيرات

#### 4. نطاق إدارة جمع المخلفات (Waste Collection)

##### أ. مكونات واجهة المستخدم

- **قائمة جلسات جمع المخلفات**: عرض جميع جلسات جمع المخلفات
- **صفحة تفاصيل جلسة الجمع**: عرض معلومات تفصيلية عن جلسة الجمع
- **إحصائيات جمع المخلفات**: عرض إحصائيات جمع المخلفات حسب الفئة والمنطقة
- **إدارة فئات المخلفات**: تعديل وإضافة فئات وأسعار المخلفات

##### ب. وظائف إدارة جمع المخلفات

- تتبع عمليات جمع المخلفات
- إدارة فواتير المخلفات
- تقارير وإحصائيات جمع المخلفات
- إدارة مخزون المخلفات المجمعة
