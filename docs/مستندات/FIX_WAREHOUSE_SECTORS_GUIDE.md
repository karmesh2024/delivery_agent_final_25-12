# دليل إصلاح مشكلة جدول warehouse_sectors

## المشكلة
يوجد تضارب في تعريف جدول `warehouse_sectors` في ملفات SQL المختلفة:
- في بعض الملفات يتم تعريفه كجدول ربط بين المخازن والقطاعات
- في ملفات أخرى يتم تعريفه كجدول القطاعات الأساسي

هذا يسبب خطأ 400 عند محاولة الوصول إلى الجدول.

## الحل
تم إنشاء ملف `fix_warehouse_sectors_schema.sql` لحل هذه المشكلة.

### الخطوات:

1. **تطبيق الإصلاح على قاعدة البيانات:**
   ```sql
   -- تشغيل ملف الإصلاح
   \i fix_warehouse_sectors_schema.sql
   ```

2. **التحقق من النتيجة:**
   ```sql
   -- التحقق من وجود جدول القطاعات
   SELECT * FROM warehouse_sectors;
   
   -- التحقق من وجود جدول الربط
   SELECT * FROM warehouse_sector_assignments;
   ```

3. **اختبار التطبيق:**
   - انتقل إلى صفحة `/warehouse-management/admin-settings/sectors`
   - يجب أن تظهر القطاعات الافتراضية (8 قطاعات)
   - يجب أن تختفي رسائل الخطأ 400

## الملفات المحدثة

### 1. `fix_warehouse_sectors_schema.sql`
- إعادة تسمية الجدول الموجود إلى `warehouse_sector_assignments_old`
- إنشاء جدول `warehouse_sectors` الجديد للقطاعات
- إنشاء جدول `warehouse_sector_assignments` للربط
- إدراج القطاعات الافتراضية

### 2. `src/domains/warehouse-management/services/warehouseService.ts`
- تحديث دالة `getSectors()` للعمل مع المخطط الجديد
- إضافة معالجة أفضل للأخطاء
- إضافة رسائل تسجيل مفصلة

## القطاعات الافتراضية
1. القطاع الصناعي (IND)
2. القطاع التجاري (COM)
3. القطاع الزراعي (AGR)
4. القطاع الطبي (MED)
5. القطاع المنزلي (RES)
6. القطاع الحكومي (GOV)
7. القطاع التعليمي (EDU)
8. القطاع السياحي (TOU)

## ملاحظات مهمة
- تم الحفاظ على التوافق مع النظام القديم عبر جدول `waste_sectors`
- تم إنشاء سياسات أمان (RLS) مناسبة
- تم إضافة فهارس لتحسين الأداء
- يمكن حذف الجدول القديم بعد التأكد من عمل النظام
