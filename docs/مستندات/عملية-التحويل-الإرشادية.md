# دليل عملية تحويل المشروع إلى الهيكل الجديد

## ما تم إنجازه حتى الآن

قمنا بنجاح بإتمام المرحلة الأولى من خطة إعادة هيكلة المشروع، والتي تضمنت:

1. **إنشاء هيكل مجلدات جديد** يتبع نهج Domain-Driven Design:
   - `src/domains/` - مجلدات منفصلة لكل مجال عمل (orders, agents)
   - `src/core/` - للمكونات الأساسية المشتركة
   - `src/shared/` - للمكونات المشتركة بين المجالات
   - `src/store/` - لإدارة حالة Redux المركزية

2. **تثبيت مكتبات Redux اللازمة**:
   - `redux`
   - `react-redux`
   - `@reduxjs/toolkit`

3. **إعداد البنية الأساسية لـ Redux**:
   - إنشاء متجر Redux الرئيسي (`store/index.ts`)
   - إنشاء Redux hooks المخصصة (`store/hooks.ts`)
   - إنشاء Provider للتطبيق (`app/providers.tsx`)

4. **إنشاء واجهات API موحدة**:
   - `domains/orders/api/ordersApi.ts` - واجهة موحدة للتعامل مع بيانات الطلبات
   - `domains/agents/api/agentsApi.ts` - واجهة موحدة للتعامل مع بيانات المندوبين

5. **نقل وتحديث مكونات رئيسية**:
   - مكونات المندوبين في `domains/agents/components/`
   - مكونات الطلبات في `domains/orders/components/`

6. **إنشاء نماذج للصفحات المحدثة**:
   - `domains/agents/pages/AgentsPage.tsx` - صفحة المندوبين المحدثة
   - `domains/orders/pages/OrdersPage.tsx` - صفحة الطلبات المحدثة

   ```typescript
   // src/store/index.ts
   import { configureStore } from '@reduxjs/toolkit';
   import ordersReducer from '@/domains/orders/store/ordersSlice';
   import agentsReducer from '@/domains/agents/store/agentsSlice';

   export const store = configureStore({
     reducer: {
       orders: ordersReducer,
       agents: agentsReducer,
     }
   });

   export type RootState = ReturnType<typeof store.getState>;
   export type AppDispatch = typeof store.dispatch;
   ```

2. **تفعيل Redux Provider** في ملف `app/providers.tsx`:
   ```typescript
   // src/app/providers.tsx
   'use client';

   import { Provider } from 'react-redux';
   import { store } from '@/store';

   export function Providers({ children }: { children: React.ReactNode }) {
     return (
       <Provider store={store}>
         {children}
       </Provider>
     );
   }
   ```

1. **`src/domains/orders/api/ordersApi.ts`**:

   import { supabase } from '@/lib/supabase';
   import { DeliveryOrder } from '@/types';

   export const ordersApi = {
     // اختبار الاتصال بقاعدة البيانات
     async testConnection() {
       try {
         // فحص إمكانية الوصول إلى جدول الطلبات
         const { count, error } = await supabase
           .from('delivery_orders')
           .select('*', { count: 'exact', head: true });

         return { 
           success: !error,
           count,
           message: error ? error.message : null
         };
       } catch (e) {
         return { 
           success: false,
           count: 0,
           message: e instanceof Error ? e.message : 'خطأ غير معروف'
         };
       }
     },

     // جلب كافة الطلبات
     async getDeliveryOrders(): Promise<DeliveryOrder[]> {
       const { data, error } = await supabase
         .from('delivery_orders')
         .select(`
           *,
           agent:delivery_boy_id (
             id, name, phone, avatar_url, status, location
           ),
           customer:customer_id (*)
         `)
         .order('created_at', { ascending: false });

       if (error) {
         console.error("Error fetching orders:", error);
         return [];
       }

       return data || [];
     },

     // جلب طلب محدد حسب المعرف
     async getOrderById(orderId: string): Promise<DeliveryOrder | null> {
       const { data, error } = await supabase
         .from('delivery_orders')
         .select(`
           *,
           agent:delivery_boy_id (
             id, name, phone, avatar_url, status, location
           ),
           customer:customer_id (*)
         `)
         .eq('id', orderId)
         .single();

       if (error) {
         console.error(`Error fetching order ${orderId}:`, error);
         return null;
       }

       return data;
     },

     // مزيد من الوظائف...
   };

### شرائح Redux

1. **`src/domains/orders/store/ordersSlice.ts`**:
   ```typescript
   import { createSlice, createAsyncThunk, PayloadAction } from '@reduxjs/toolkit';
   import { ordersApi } from '../api/ordersApi';
   import { DeliveryOrder, OrderStatus } from '@/types';

   interface OrdersState {
     items: DeliveryOrder[];
     filteredItems: DeliveryOrder[];
     selectedOrder: DeliveryOrder | null;
     status: 'idle' | 'loading' | 'succeeded' | 'failed';
     error: string | null;
     activeFilter: OrderStatus | 'all';
     searchQuery: string;
   }

   const initialState: OrdersState = {
     items: [],
     filteredItems: [],
     selectedOrder: null,
     status: 'idle',
     error: null,
     activeFilter: 'all',
     searchQuery: '',
   };

   // Async thunks
   export const fetchOrders = createAsyncThunk(
     'orders/fetchOrders',
     async () => {
       return await ordersApi.getDeliveryOrders();
     }
   );

   export const fetchOrderById = createAsyncThunk(
     'orders/fetchOrderById',
     async (orderId: string) => {
       return await ordersApi.getOrderById(orderId);
     }
   );

   const ordersSlice = createSlice({
     name: 'orders',
     initialState,
     reducers: {
       setActiveFilter: (state, action: PayloadAction<OrderStatus | 'all'>) => {
         state.activeFilter = action.payload;
         applyFilters(state);
       },
       setSearchQuery: (state, action: PayloadAction<string>) => {
         state.searchQuery = action.payload;
         applyFilters(state);
       },
       selectOrder: (state, action: PayloadAction<string>) => {
         state.selectedOrder = state.items.find(order => order.id === action.payload) || null;
       },
       clearSelectedOrder: (state) => {
         state.selectedOrder = null;
       },
     },
     extraReducers: (builder) => {
       builder
         .addCase(fetchOrders.pending, (state) => {
           state.status = 'loading';
         })
         .addCase(fetchOrders.fulfilled, (state, action) => {
           state.status = 'succeeded';
           state.items = action.payload;
           applyFilters(state);
         })
         .addCase(fetchOrders.rejected, (state, action) => {
           state.status = 'failed';
           state.error = action.error.message || 'حدث خطأ أثناء جلب الطلبات';
         })
         .addCase(fetchOrderById.fulfilled, (state, action) => {
           if (action.payload) {
             state.selectedOrder = action.payload;
           }
         });
     },
   });

   // Helper function to apply filters
   function applyFilters(state: OrdersState) {
     let filtered = state.items;
     
     // تصفية حسب الحالة
     if (state.activeFilter !== 'all') {
       filtered = filtered.filter(order => order.status === state.activeFilter);
     }
     
     // تصفية حسب البحث
     if (state.searchQuery) {
       const query = state.searchQuery.toLowerCase();
       filtered = filtered.filter(order => 
         (order.order_number?.toLowerCase().includes(query)) ||
         (order.customer?.name?.toLowerCase().includes(query)) ||
         (order.customer?.phone?.toLowerCase().includes(query)) ||
         (order.delivery_address?.city?.toLowerCase().includes(query))
       );
     }
     
     state.filteredItems = filtered;
   }

   export const { setActiveFilter, setSearchQuery, selectOrder, clearSelectedOrder } = ordersSlice.actions;
   export default ordersSlice.reducer;
   ```

## مقارنة بين الهيكل القديم والجديد

| الهيكل القديم | الهيكل الجديد | الميزات |
|--------------|--------------|---------|
| مكونات مختلطة في `components/` | مكونات منظمة حسب المجال في `domains/` | فصل أوضح للمسؤوليات |
| منطق API مدمج في المكونات | واجهات API موحدة | سهولة الصيانة والاختبار |
| إدارة حالة محلية باستخدام useState | إدارة حالة مركزية باستخدام Redux | تزامن أفضل للبيانات |
| مكونات مرتبطة بشدة | مكونات فضفاضة الارتباط | سهولة إعادة الاستخدام |
| صعوبة التوسع | سهولة إضافة مجالات جديدة | دعم أفضل للنمو المستقبلي |

## نصائح للتحويل

1. **البدء بمكون واحد في كل مرة**:
   - تحويل الصفحات والمكونات تدريجياً
   - اختبار كل تغيير قبل المتابعة

2. **استخدام النماذج المقدمة**:
   - الاسترشاد بالمكونات المحولة بالفعل
   - اتباع نفس نمط التنظيم والترميز

3. **الحفاظ على توافقية الواجهات**:
   - عدم تغيير الواجهات البرمجية للمكونات المحولة
   - الحفاظ على أسماء الخصائص والوظائف

4. **التسمية المتسقة**:
   - استخدام نفس اصطلاحات التسمية في كل المجالات
   - تجنب الاختصارات والأسماء الغامضة

## الخلاصة

يوفر هذا الدليل خارطة طريق شاملة لإتمام عملية تحويل المشروع إلى الهيكل الجديد. باتباع هذه الخطوات، يمكن تحويل المشروع تدريجياً مع الحفاظ على عمله بشكل طبيعي أثناء التحويل.

البنية الجديدة ستعزز:
- **القابلية للصيانة** - فصل واضح للمسؤوليات
- **القابلية للتوسع** - سهولة إضافة مجالات وميزات جديدة
- **إعادة الاستخدام** - مكونات قابلة للتركيب
- **الأداء** - تحسين أداء التطبيق عبر إدارة الحالة المركزية
