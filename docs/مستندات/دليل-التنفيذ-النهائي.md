# دليل التنفيذ النهائي لإعادة هيكلة المشروع

## ما تم إنجازه

قمنا بإنشاء هيكل جديد للمشروع يتبع نهج Domain-Driven Design، مع تطبيق Redux كحل مركزي لإدارة الحالة. الهيكل الجديد يفصل بين مجالات العمل المختلفة ويوفر بنية أكثر قابلية للتوسع.

### 1. هيكل المجلدات الجديد

تم إنشاء الهيكل التالي:

```
src/
├── core/             # المكونات الأساسية المشتركة
│   └── api/          # واجهة متابعة Supabase
│
├── domains/          # مجالات العمل مقسمة حسب الوظيفة
│   ├── orders/       # مجال الطلبات
│   │   ├── api/      # واجهة API الطلبات
│   │   ├── components/ # مكونات الطلبات
│   │   └── store/    # شريحة Redux للطلبات
│   │
│   └── agents/       # مجال المندوبين
│       ├── api/      # واجهة API المندوبين
│       ├── components/ # مكونات المندوبين
│       └── store/    # شريحة Redux للمندوبين
│
├── shared/           # المكونات المشتركة بين المجالات
│   └── components/   # مكونات UI المشتركة
│
└── store/            # متجر Redux المركزي
    ├── index.ts      # تكوين المتجر الرئيسي
    └── hooks.ts      # hooks مخصصة لاستخدام Redux
```

### 2. واجهات API موحدة

تم إنشاء واجهات API موحدة لكل مجال:

- **ordersApi**: واجهة موحدة للتعامل مع بيانات الطلبات
- **agentsApi**: واجهة موحدة للتعامل مع بيانات المندوبين

هذه الواجهات تعمل كطبقة وسيطة بين المكونات وقاعدة البيانات، مما يسهل تغيير مصدر البيانات في المستقبل.

### 3. إدارة الحالة باستخدام Redux

تم إعداد Redux مع:

- متجر مركزي في `src/store/index.ts`
- hooks مخصصة في `src/store/hooks.ts`
- شرائح Redux لكل مجال:
  - `ordersSlice`: لإدارة حالة الطلبات
  - `agentsSlice`: لإدارة حالة المندوبين

### 4. مكونات نموذجية

تم إنشاء مكونات نموذجية تستخدم البنية الجديدة:

- `OrderDetail`: مكون محدث لعرض تفاصيل الطلب
- `OrdersPageExample`: نموذج لكيفية استخدام Redux في صفحة الطلبات
- `AgentsPageExample`: نموذج لكيفية استخدام Redux في صفحة المندوبين

## الخطوات المتبقية

لإكمال عملية التحويل، يجب اتباع الخطوات التالية:

### المرحلة 1: نقل المكونات الحالية (1-2 أسبوع)

1. نقل مكونات الطلبات المتبقية من `components/orders` إلى `domains/orders/components`
2. نقل مكونات المندوبين المتبقية من `components/agents` إلى `domains/agents/components`
3. نقل مكونات الخرائط والتتبع إلى المجلدات المناسبة
4. تحديث الاستيرادات في الصفحات لتشير إلى المكونات في مواقعها الجديدة

### المرحلة 2: تفعيل Redux (1-2 أسبوع)

1. إلغاء تعليق استيراد واستخدام المخفضات في `store/index.ts`
2. إلغاء تعليق استخدام Redux Provider في `app/providers.tsx`
3. تحويل صفحة واحدة في كل مرة لاستخدام شرائح Redux بدلاً من `useState` المحلية
4. البدء بصفحة الطلبات، ثم صفحة المندوبين

### المرحلة 3: إضافة مجالات جديدة (حسب الحاجة)

لإضافة مجالات جديدة (مثل النظام المالي أو نظام الموارد البشرية):

1. إنشاء بنية مجلدات جديدة تحت `domains/` (مثل `domains/finance/` أو `domains/hr/`)
2. إنشاء واجهة API موحدة للمجال الجديد (مثل `financeApi.ts` أو `hrApi.ts`)
3. إنشاء شريحة Redux للمجال الجديد (مثل `financeSlice.ts` أو `hrSlice.ts`)
4. إضافة المخفض الجديد إلى متجر Redux الرئيسي
5. بناء المكونات الجديدة باستخدام البنية الجديدة

## مثال عملي: تحويل صفحة الطلبات

لتحويل صفحة الطلبات الحالية لاستخدام البنية الجديدة، اتبع الخطوات التالية:

### 1. نقل المكونات

انقل المكونات المستخدمة في صفحة الطلبات إلى المجلدات الجديدة:

```tsx
// src/domains/orders/components/OrdersPage.tsx
'use client';

import { useEffect } from 'react';
import { useAppDispatch, useAppSelector } from '@/store/hooks';
import { fetchOrders } from '../store/ordersSlice';
import { DashboardLayout } from '@/shared/components/layouts/DashboardLayout';

export default function OrdersPage() {
  const dispatch = useAppDispatch();
  const { items: orders, status, error } = useAppSelector(state => state.orders);
  
  useEffect(() => {
    dispatch(fetchOrders());
  }, [dispatch]);
  
  // باقي الكود...
}
```

### 2. تحديث صفحة الطلبات الرئيسية

حدّث ملف `app/orders/page.tsx` لاستخدام المكون الجديد:

```tsx
// src/app/orders/page.tsx
import OrdersPage from '@/domains/orders/components/OrdersPage';

export default function OrdersRoute() {
  return <OrdersPage />;
}
```

### 3. إلغاء تعليق المخفضات في متجر Redux

عدّل ملف `store/index.ts` لإلغاء تعليق مخفض الطلبات:

```ts
// src/store/index.ts
import { configureStore } from '@reduxjs/toolkit';

// استيراد المخفضات
import ordersReducer from '@/domains/orders/store/ordersSlice';
// import agentsReducer from '@/domains/agents/store/agentsSlice';

export const store = configureStore({
  reducer: {
    orders: ordersReducer,
    // agents: agentsReducer,
  },
  middleware: (getDefaultMiddleware) => 
    getDefaultMiddleware({
      serializableCheck: false,
    }),
  devTools: process.env.NODE_ENV !== 'production',
});

export type RootState = ReturnType<typeof store.getState>;
export type AppDispatch = typeof store.dispatch;
```

### 4. تفعيل Redux Provider

عدّل ملف `app/providers.tsx` لتفعيل Redux Provider:

```tsx
// src/app/providers.tsx
'use client';

import { Provider } from 'react-redux';
import { store } from '@/store';

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <Provider store={store}>
      {children}
    </Provider>
  );
}
```

### 5. تحديث Providers في layout.tsx

تأكد من تفعيل Providers في `app/layout.tsx`:

```tsx
// src/app/layout.tsx
import { Providers } from './providers';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ar" dir="rtl">
      <body>
        <Providers>
          {children}
        </Providers>
      </body>
    </html>
  );
}
```

## الخلاصة

لقد بدأنا عملية إعادة هيكلة المشروع لتحسين بنيته واستخدام Redux لإدارة الحالة المركزية. الخطوات التالية هي تحويل المكونات الحالية تدريجياً للاستفادة من البنية الجديدة، ثم إضافة مجالات جديدة حسب الحاجة.

يوفر هذا النهج التدريجي القدرة على تنفيذ التحويل بأقل قدر من التعطيل للعمل اليومي، مع الحفاظ على إمكانية التوسع في المستقبل.
