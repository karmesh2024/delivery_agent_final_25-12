# دليل النشر على Vercel 🚀

هذا الدليل يشرح كيفية نشر تطبيق "بيكب" (Delivery Agent Dashboard) على منصة
Vercel.

## 1. المتطلبات المسبقة

- حساب على [Vercel](https://vercel.com).
- حساب على [GitHub](https://github.com) (أو GitLab/Bitbucket) يحتوي على كود
  المشروع.
- مشروع Supabase جاهز (تم إعداده مسبقاً).

## 2. إعداد متغيرات البيئة (Environment Variables)

قبل النشر، يجب تجهيز القيم التالية من مشروع Supabase الخاص بك:

| اسم المتغير                     | الوصف                        | هل هو سري؟           |
| ------------------------------- | ---------------------------- | -------------------- |
| `NEXT_PUBLIC_SUPABASE_URL`      | رابط مشروع Supabase          | ❌ لا (عام)          |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | المفتاح العام (Anon Key)     | ❌ لا (عام)          |
| `SUPABASE_SERVICE_ROLE_KEY`     | المفتاح السري (Service Role) | ✅ **نعم (سري جداً)** |

> **ملاحظة هامة:** `SUPABASE_SERVICE_ROLE_KEY` مطلوب لأننا نستخدمه في بعض وظائف
> الـ API (مثل `get-current-user.ts`). تأكد من عدم تسريبه أبداً في كود الواجهة
> الأمامية.

## 3. خطوات النشر (Deployment Steps)

### الطريقة الأسهل: عبر لوحة تحكم Vercel (Web UI)

1. اذهب إلى [Vercel Dashboard](https://vercel.com/dashboard).
2. اضغط على **"Add New..."** ثم اختر **"Project"**.
3. اختر مستودع GitHub الخاص بالمشروع واضغط **"Import"**.
4. في صفحة الإعدادات (Configure Project):
   - **Framework Preset:** سيتعرف Vercel تلقائياً على أنه `Next.js`.
   - **Root Directory:** اتركه كما هو (`./`) إلا إذا كان المشروع في مجلد فرعي.
   - **Environment Variables:**
     - اضغط لفتح القائمة.
     - أضف المتغيرات الثلاثة المذكورة أعلاه (الاسم والقيمة).
5. اضغط على **"Deploy"**.

### الطريقة المتقدمة: عبر سطر الأوامر (Vercel CLI)

إذا كنت تفضل استخدام التيرمينال:

1. تأكد من تثبيت Vercel CLI:
   ```bash
   npm i -g vercel
   ```
2. سجل الدخول:
   ```bash
   vercel login
   ```
3. من داخل مجلد المشروع، نفذ الأمر:
   ```bash
   vercel
   ```
4. اتبع التعليمات على الشاشة. عندما يسألك عن إعدادات المشروع، وافق على
   الافتراضيات.
5. للنشر في بيئة الإنتاج (Production)، استخدم:
   ```bash
   vercel --prod
   ```

## 4. ما بعد النشر (Post-Deployment Checks)

بعد انتهاء النشر، قم بزيارة الرابط الذي يوفره Vercel وتأكد من:

1. **تسجيل الدخول:** جرب تسجيل الدخول بحساب مندوب أو مسؤول.
2. **البيانات:** تأكد من ظهور البيانات (الطلبات، المندوبين) بشكل صحيح.
3. **الأخطاء:** افتح Console المتصفح وتأكد من عدم وجود أخطاء حمراء (Red Errors).

## 5. استكشاف الأخطاء وإصلاحها (Troubleshooting)

- **خطأ 500 في الـ API:** غالباً بسبب نقص أو خطأ في متغيرات البيئة. راجع قسم
  Environment Variables في إعدادات المشروع على Vercel.
- **البيانات لا تظهر:** تأكد من سياسات RLS في Supabase. (لقد قمنا بتأمينها مسبقاً،
  لكن تأكد من أن المستخدم لديه الصلاحيات المناسبة).
- **Build Failed:** راجع سجلات البناء (Build Logs) في Vercel لمعرفة سبب الفشل
  (غالباً خطأ في TypeScript أو ESLint).
