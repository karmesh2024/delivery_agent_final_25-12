# خطة تطوير هيكلة المشروع

## الهيكلة المقترحة للمشروع

نقترح إعادة هيكلة المشروع بناءً على مبادئ Domain-Driven Design (DDD) لدعم التوسع المستقبلي في المجالات التالية:
- نظام توصيل الطلبات (الموجود حالياً)
- نظام الموارد البشرية والرواتب (مستقبلي)
- النظام المالي (مستقبلي)
- النظام اللوجستي (مستقبلي)

## الهيكل العام المقترح للمشروع

```
src/
├── core/                   # المكونات الأساسية المشتركة
```

## خطة التنفيذ التدريجية

### المرحلة الأولى: إعادة تنظيم الكود الحالي (1-2 أسابيع)

1. **إنشاء البنية الأساسية للمشروع**:
   - إنشاء المجلدات الرئيسية: `core`, `domains`, `shared`
   - نقل المكونات المشتركة إلى المجلدات المناسبة

2. **إعادة تنظيم نظام التوصيل الحالي**:
   - نقل كود التوصيل الحالي إلى مجلد `domains/delivery`
   - تنظيم الملفات حسب الهيكل الجديد دون تغييرات جذرية في المنطق

3. **تحديد وتنظيم المكونات المشتركة**:
   - نقل مكونات UI المشتركة إلى `shared/components`
   - تحديد الأنواع المشتركة ونقلها إلى `shared/types`

### المرحلة الثانية: تنفيذ إدارة الحالة المركزية (2-3 أسابيع)

1. **إضافة Redux Toolkit للمشروع**:
   ```bash
   npm install redux react-redux @reduxjs/toolkit redux-thunk
   ```

2. **إنشاء هيكل إدارة الحالة**:
   - إنشاء متجر Redux الرئيسي
   - تنفيذ شرائح (slices) للبيانات الرئيسية: الطلبات، المندوبين، المستخدمين

3. **ترحيل إدارة البيانات الحالية إلى Redux**:
   - تحديث المكونات لاستخدام Redux بدلاً من حالة React المحلية
   - ضمان تدفق البيانات المتناسق عبر التطبيق

### المرحلة الثالثة: تحسين طبقة API (1-2 أسابيع)

1. **إنشاء مكتبة API موحدة**:
   - تنفيذ عميل API مركزي باستخدام RTK Query
   - دمج بيانات Supabase في طبقة API

2. **تنظيم مكالمات API حسب النطاق**:
   - تجميع خدمات API للتوصيل في `domains/delivery/api`
   - إعداد البنية التحتية لنطاقات أخرى مستقبلية

### المرحلة الرابعة: التجهيز للتوسع المستقبلي (1-2 أسابيع)

1. **إعداد هيكل المصادقة والأدوار**:
   - تنفيذ نظام أدوار وصلاحيات مرن
   - إعداد حماية المسارات بناءً على الأدوار

2. **إعداد نظام التدويل**:
   - تنفيذ البنية التحتية للتعددية اللغوية
   - إعداد ملفات الترجمة الأساسية

3. **إعداد الاختبارات الأساسية**:
   - إنشاء اختبارات وحدة للوظائف الأساسية
   - إنشاء اختبارات تكامل للمكونات الرئيسية

## كيفية البدء الآن

### الخطوات الفورية (خلال الأسبوع الأول)

1. **تقييم الكود الحالي**:
   - تحليل المكونات الموجودة وتصنيفها (مكونات خاصة بالتوصيل / مكونات مشتركة)
   - تحديد الاعتماديات بين المكونات

2. **إنشاء هيكل المجلدات الأساسي**:
   ```bash
   mkdir -p src/{core,domains/delivery,shared}
   ```

3. **البدء بنقل الملفات المشتركة**:
   - نقل مكونات UI الأساسية (مثل الأزرار، الجداول، النماذج) إلى `shared/components`
   - نقل الأنواع المشتركة إلى `shared/types`

4. **تنفيذ مكون واحد باستخدام Redux**:
   - اختيار مكون واحد (مثل قائمة الطلبات) لتحويله إلى Redux
   - استخدامه كمثال للمكونات الأخرى

### مثال عملي للبدء: تحويل إدارة الطلبات إلى النظام الجديد

1. **إنشاء ملف Redux Slice للطلبات**:
   ```typescript
   // src/domains/delivery/store/ordersSlice.ts
   import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
   import { supabase } from '@/lib/supabase';

   export const fetchOrders = createAsyncThunk(
     'orders/fetchOrders',
     async () => {
       const { data, error } = await supabase
         .from('delivery_orders')
         .select('*');
       
       if (error) throw error;
       return data;
     }
   );

   const ordersSlice = createSlice({
     name: 'orders',
     initialState: {
       items: [],
       status: 'idle',
       error: null
     },
     reducers: {},
     extraReducers: (builder) => {
       builder
         .addCase(fetchOrders.pending, (state) => {
           state.status = 'loading';
         })
         .addCase(fetchOrders.fulfilled, (state, action) => {
           state.status = 'succeeded';
           state.items = action.payload;
         })
         .addCase(fetchOrders.rejected, (state, action) => {
           state.status = 'failed';
           state.error = action.error.message;
         });
     }
   });

   export default ordersSlice.reducer;
   ```

2. **إنشاء متجر Redux الرئيسي**:
   ```typescript
   // src/core/store/index.ts
   import { configureStore } from '@reduxjs/toolkit';
   import ordersReducer from '@/domains/delivery/store/ordersSlice';

   export const store = configureStore({
     reducer: {
       orders: ordersReducer,
       // يمكن إضافة المزيد من المخفضات هنا
     }
   });

   export type RootState = ReturnType<typeof store.getState>;
   export type AppDispatch = typeof store.dispatch;
   ```

3. **تحديث مكون صفحة الطلبات**:
   ```tsx
   // src/domains/delivery/pages/OrdersPage.tsx
   import { useEffect } from 'react';
   import { useDispatch, useSelector } from 'react-redux';
   import { fetchOrders } from '@/domains/delivery/store/ordersSlice';
   import { RootState, AppDispatch } from '@/core/store';

   export default function OrdersPage() {
     const dispatch = useDispatch<AppDispatch>();
     const { items, status, error } = useSelector((state: RootState) => state.orders);
     
     useEffect(() => {
       if (status === 'idle') {
         dispatch(fetchOrders());
       }
     }, [status, dispatch]);
     
     if (status === 'loading') {
       return <div>جاري التحميل...</div>;
     }
     
     if (status === 'failed') {
       return <div>خطأ: {error}</div>;
     }
     
     return (
       <div>
         <h1>الطلبات</h1>
         <ul>
           {items.map(order => (
             <li key={order.id}>{order.id}</li>
           ))}
         </ul>
       </div>
     );
   }
   ```

## الخلاصة

هذه الخطة تسمح بإعادة هيكلة المشروع بشكل تدريجي دون الحاجة لإيقاف التطوير أو إعادة كتابة الكود بالكامل. التركيز في البداية على إعادة التنظيم ثم إضافة Redux سيضع الأساس اللازم للتوسع المستقبلي، مع الحفاظ على استمرارية عمل المشروع الحالي.

من خلال اتباع هذه الخطة، سيكون المشروع جاهزًا لإضافة الأنظمة الجديدة (الموارد البشرية، المالية، اللوجستية) دون الحاجة لتغييرات جذرية في البنية الأساسية.
