# إعداد جداول كتالوج المخلفات

## 📋 الجداول المطلوبة

### 1. **جدول الوحدات (units)**
- يحتوي على الوحدات المختلفة (كيلوجرام، لتر، قطعة، إلخ)
- يحل مشكلة خطأ 400 في جلب الوحدات

### 2. **جدول مصادر المخلفات (waste_sources)**
- يحتوي على مصادر المخلفات المختلفة
- يحل سؤال "من أين يأتي المصدر"

### 3. **جدول الفئات الرئيسية (waste_main_categories)**
- فئات المخلفات الرئيسية (بلاستيك، معادن، ورق، إلخ)

### 4. **جدول الفئات الفرعية (waste_sub_categories)**
- فئات فرعية لكل فئة رئيسية

## 🚀 طرق الإنشاء

### **الطريقة 1: استخدام السكريبت (الأسهل)**
```bash
npm run create:waste-tables
```

### **الطريقة 2: استخدام Supabase Dashboard**
1. اذهب إلى Supabase Dashboard
2. اختر مشروعك
3. اذهب إلى "SQL Editor"
4. انسخ محتوى ملف `create_waste_catalog_tables.sql`
5. اضغط "Run"

### **الطريقة 3: استخدام Supabase CLI**
```bash
supabase db reset
# أو
supabase db push
```

## 📊 البيانات المُضافة

### **الوحدات (8 وحدات):**
- كيلوجرام، جرام، طن (وزن)
- لتر، متر مكعب (حجم)
- قطعة (عدد)
- متر، سنتيمتر (أبعاد)

### **مصادر المخلفات (8 مصادر):**
- منتج تالف
- منتج منتهي الصلاحية
- عبوات فارغة
- مرتجعات
- بقايا إنتاج/تغليف
- مواد تغليف
- مخلفات مكتبية
- أخرى

### **الفئات الرئيسية (8 فئات):**
- بلاستيك
- معادن
- كرتون/ورق
- زجاج
- أقمشة/نسيج
- مواد عضوية
- إلكترونيات
- أخرى

### **الفئات الفرعية (20+ فئة فرعية):**
- زجاجات بلاستيكية، أكياس بلاستيكية
- علب معدنية، خردة معدنية
- كرتون، ورق مكتبي
- زجاجات زجاجية، حاويات زجاجية
- ملابس، أغطية وفرش
- مخلفات طعام، مخلفات حدائق
- إلكترونيات صغيرة وكبيرة
- مخلفات مختلطة

## ✅ بعد الإنشاء

بعد إنشاء الجداول:
1. ✅ لن تظهر أخطاء "خطأ في جلب الوحدات"
2. ✅ ستظهر خيارات المصدر في القائمة المنسدلة
3. ✅ ستعمل الفئات الرئيسية والفرعية بشكل صحيح
4. ✅ سيكون النظام مستقراً ومتكاملاً

## 🔧 استكشاف الأخطاء

### **إذا فشل السكريبت:**
1. تأكد من وجود متغيرات البيئة
2. تأكد من صلاحيات قاعدة البيانات
3. جرب الطريقة اليدوية

### **إذا لم تظهر البيانات:**
1. تحقق من RLS policies
2. تأكد من اتصال قاعدة البيانات
3. تحقق من console logs

## 📞 الدعم

إذا واجهت أي مشاكل:
1. تحقق من console logs
