# تعليمات تطبيق النظام الهرمي للمخازن

## الخطوات المطلوبة:

### 1. تطبيق التحديثات على قاعدة البيانات

قم بتشغيل أحد الملفات التالية في قاعدة البيانات:

**الملف المبسط (موصى به للاختبار السريع):**
```sql
\i quick_warehouse_hierarchy_setup.sql
```

**الملف الكامل (للتطبيق النهائي):**
```sql
\i apply_warehouse_hierarchy.sql
```

### 2. التحقق من عمل النظام

1. تأكد من تشغيل الخادم:
   ```bash
   npm run dev
   ```

2. انتقل إلى الرابط:
   ```
   http://localhost:3000/warehouse-management/warehouses/tree
   ```

### 3. إذا واجهت مشاكل

**مشكلة: "لا توجد مخازن في النظام"**
- تأكد من تطبيق ملف SQL على قاعدة البيانات
- تحقق من وجود بيانات تجريبية

**مشكلة: خطأ في جلب البيانات**
- تحقق من اتصال قاعدة البيانات
- تأكد من وجود الدوال المطلوبة (get_warehouse_tree)

**مشكلة: صفحة لا تظهر**
- تحقق من وجود الملف: `src/app/warehouse-management/warehouses/tree/page.tsx`
- تأكد من عدم وجود أخطاء في الكونسول

### 4. اختبار النظام

1. **إضافة مخزن جديد:**
   - انتقل إلى: `http://localhost:3000/warehouse-management/warehouses/new`
   - اختر مستوى هرمي (رئيسي/مدينة/فرعي)
   - اختر المخزن الأب إذا لم يكن رئيسي

2. **عرض الشجرة:**
   - انتقل إلى: `http://localhost:3000/warehouse-management/warehouses/tree`
   - يجب أن ترى المخازن مع المستويات الهرمية

3. **قائمة المخازن:**
   - انتقل إلى: `http://localhost:3000/warehouse-management/warehouses`
   - يجب أن ترى عمود المستوى الهرمي

### 5. البيانات التجريبية

بعد تطبيق ملف SQL، ستجد البيانات التالية:

- **مخزن شمال الدلتا الرئيسي** (مستوى رئيسي)
  - **مخزن الإسكندرية الرئيسي** (مستوى مدينة)
    - **مخزن محرم بك شرق** (مستوى منطقة)

### 6. استكشاف الأخطاء

إذا لم تعمل الصفحة، تحقق من:

1. **كونسول المتصفح** - ابحث عن أخطاء JavaScript
2. **كونسول الخادم** - ابحث عن أخطاء في الخادم
3. **قاعدة البيانات** - تأكد من وجود الدوال والجداول المطلوبة

### 7. الدعم

إذا استمرت المشاكل، تأكد من:
- تطبيق جميع التحديثات على قاعدة البيانات
- عدم وجود أخطاء في الملفات
- عمل اتصال قاعدة البيانات بشكل صحيح
