# خطة تنفيذية لإعادة هيكلة المشروع الحالي

بعد مراجعة المشروع الحالي، أقدم خطة تنفيذية عملية لإعادة هيكلته وتطبيق Redux كحل لإدارة الحالة، مع المحافظة على وظائف المشروع وضمان قابليته للتوسع المستقبلي.

## تحليل الوضع الحالي

### هيكل المشروع
المشروع الحالي يتبع بنية Next.js القياسية مع app router:
- `src/app/`: يحتوي على صفحات التطبيق
- `src/components/`: يحتوي على مكونات UI المشتركة
- `src/lib/`: يحتوي على خدمات الاتصال بقاعدة البيانات والوظائف المساعدة
- `src/types/`: يحتوي على تعريفات الأنواع

### إدارة الحالة الحالية
تعتمد إدارة الحالة حالياً على:
- استخدام `useState` و `useEffect` داخل المكونات
- جلب البيانات مباشرة من Supabase في كل مكون يحتاجها
- تمرير البيانات كخصائص (props) بين المكونات المتداخلة

**مثال من صفحة الطلبات الحالية:**
```javascript
// src/app/orders/page.tsx
export default function OrdersPage() {
  const [deliveryOrders, setDeliveryOrders] = useState<DeliveryOrder[]>([]);
  const [filteredOrders, setFilteredOrders] = useState<DeliveryOrder[]>([]);
  const [loading, setLoading] = useState(true);
  // المزيد من حالات المكون...

  // جلب البيانات من Supabase
  useEffect(() => {
    const loadOrders = async () => {
      setLoading(true);
      try {
        const data = await getDeliveryOrders();
        setDeliveryOrders(data);
        setFilteredOrders(data);
      } catch (error) {
        console.error("خطأ في تحميل بيانات الطلبات:", error);
        toast.error("Failed to load order data");
      } finally {
        setLoading(false);
      }
    };

    loadOrders();
  }, []);
  
  // المزيد من كود المكون...
}
```

## خطة إعادة الهيكلة

أقترح خطة تنفيذية مقسمة إلى 4 مراحل، يمكن تنفيذها تدريجياً مع الحفاظ على عمل المشروع خلال فترة التطوير:

### المرحلة 1: إعادة تنظيم المشروع (1 أسبوع)

5. **إنشاء شرائح Redux (Redux Slices)**:

```typescript
// src/store/slices/ordersSlice.ts
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import { ordersApi } from '@/domains/delivery/api/orders';
import { DeliveryOrder } from '@/types';

// حالة Redux الأولية
interface OrdersState {
  items: DeliveryOrder[];
  filteredItems: DeliveryOrder[];
  selectedOrder: DeliveryOrder | null;
  activeFilter: string;
  searchTerm: string;
  loading: boolean;
  error: string | null;
  // المزيد من حالات الطلبات...
}

const initialState: OrdersState = {
  items: [],
  filteredItems: [],
  selectedOrder: null,
  activeFilter: 'all',
  searchTerm: '',
  loading: false,
  error: null
};

// Async Thunks - لجلب البيانات بشكل غير متزامن
export const fetchOrders = createAsyncThunk(
  'orders/fetchAll',
  async (_, { rejectWithValue }) => {
    try {
      return await ordersApi.getAll();
    } catch (error) {
      return rejectWithValue(error instanceof Error ? error.message : 'فشل في جلب الطلبات');
    }
  }
);

export const fetchOrderById = createAsyncThunk(
  'orders/fetchById',
  async (id: string, { rejectWithValue }) => {
    try {
      return await ordersApi.getById(id);
    } catch (error) {
      return rejectWithValue(error instanceof Error ? error.message : 'فشل في جلب الطلب');
    }
  }
);

// شريحة Redux للطلبات
const ordersSlice = createSlice({
  name: 'orders',
  initialState,
  reducers: {
    setActiveFilter: (state, action) => {
      state.activeFilter = action.payload;
      // تحديث القائمة المصفاة
      filterOrders(state);
    },
    setSearchTerm: (state, action) => {
      state.searchTerm = action.payload;
      // تحديث القائمة المصفاة
      filterOrders(state);
    },
    selectOrder: (state, action) => {
      state.selectedOrder = action.payload;
    },
    // المزيد من المخفضين...
  },
  extraReducers: (builder) => {
    builder
      // التعامل مع حالات جلب الطلبات
      .addCase(fetchOrders.pending, (state) => {
        state.loading = true;
        state.error = null;
      })
      .addCase(fetchOrders.fulfilled, (state, action) => {
        state.items = action.payload;
        state.loading = false;
        // تحديث القائمة المصفاة
        filterOrders(state);
      })
      .addCase(fetchOrders.rejected, (state, action) => {
        state.loading = false;
        state.error = action.payload as string;
      })
      // التعامل مع حالات جلب طلب محدد
      .addCase(fetchOrderById.pending, (state) => {
        state.loading = true;
        state.error = null;
      })
      .addCase(fetchOrderById.fulfilled, (state, action) => {
        state.selectedOrder = action.payload;
        state.loading = false;
      })
      .addCase(fetchOrderById.rejected, (state, action) => {
        state.loading = false;
        state.error = action.payload as string;
      });
  }
});

// وظيفة مساعدة لتصفية الطلبات بناءً على الفلاتر الحالية
function filterOrders(state: OrdersState) {
  let filtered = [...state.items];
  
  // فلترة حسب الحالة
  if (state.activeFilter !== 'all') {
    filtered = filtered.filter(order => order.status === state.activeFilter);
  }
  
  // فلترة حسب كلمة البحث
  if (state.searchTerm) {
    const term = state.searchTerm.toLowerCase();
    filtered = filtered.filter(
      order => 
        order.order_number.toLowerCase().includes(term) ||
        (order.customer_name && order.customer_name.toLowerCase().includes(term)) ||
        order.pickup_address.toLowerCase().includes(term) ||
        order.delivery_address.toLowerCase().includes(term)
    );
  }
  
  state.filteredItems = filtered;
}

export const { setActiveFilter, setSearchTerm, selectOrder } = ordersSlice.actions;
export default ordersSlice.reducer;
```

## ميزات هذه الخطة

1. **تنفيذ تدريجي**: يمكن تنفيذ الخطة بشكل تدريجي دون تعطيل التطوير.
2. **استعداد للتوسع**: الهيكل الجديد يدعم إضافة أنظمة جديدة في المستقبل.
3. **فصل المخاوف**: فصل واضح بين منطق العمل وواجهة المستخدم والبيانات.
4. **مرونة وصيانة**: تحسين قابلية الصيانة والتطوير المستقبلي.
5. **تجنب الأخطاء**: تقليل الأخطاء من خلال تركيز إدارة الحالة في مكان واحد.

هذه الخطة تتيح تحويل المشروع الحالي تدريجياً إلى هيكل أكثر قوة وقابلية للتوسع، مع الاستفادة من مزايا Redux لإدارة الحالة، مما يضع أساساً متيناً للتطورات المستقبلية.
