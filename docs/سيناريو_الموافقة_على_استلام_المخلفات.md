# سيناريو الموافقة على استلام المخلفات

## 📋 نظرة عامة

هذا المستند يوضح **السيناريو الكامل** لما يحدث بعد الموافقة على تأكيد التحقق من استلام المخلفات.

---

## 🔄 التدفق الكامل

### المرحلة 1: إنشاء طلب الاستلام
```
1. إنشاء طلب استلام جديد
   ↓
2. الحالة: pending_verification
   ↓
3. البيانات المحفوظة في: waste_receiving_approval_requests
```

### المرحلة 2: التحقق (Verification)
```
1. مدير المخلفات يضغط "التحقق"
   ↓
2. فتح dialog للتحقق من:
   - الكمية المحققة لكل مخلف
   - درجة الجودة (A, B, C, D)
   - ملاحظات على كل مخلف
   - ملاحظات عامة للتحقق
   ↓
3. تحديث الطلب:
   - status → 'verified'
   - verified_by → user_id
   - verified_at → timestamp
   - verification_notes → ملاحظات التحقق
   - waste_items → تحديث بالكميات والجودة المحققة
   ↓
4. الحالة: verified
```

### المرحلة 3: الموافقة (Approval) ⚠️ **يحتاج تنفيذ**
```
1. مدير المخلفات يضغط "موافقة"
   ↓
2. تحديث الطلب:
   - status → 'approved'
   - approved_by → user_id
   - approved_at → timestamp
   - approval_notes → ملاحظات الموافقة
   ↓
3. **إضافة حركات المخزون** (يحتاج تنفيذ):
   لكل مخلف في waste_items:
   - إضافة سجل في inventory_movements:
     * warehouse_id: من الطلب
     * product_id: من waste_data_admin (يحتاج تحويل waste_no → UUID)
     * movement_type: 'in'
     * quantity: الكمية المحققة
     * unit: الوحدة
     * source_type: 'waste_receiving'
     * source_id: request.id
     * notes: ملاحظات التحقق
   ↓
4. **تحديث المخزون تلقائياً** (بواسطة Trigger):
   - Trigger: trg_update_inventory
   - الدالة: update_inventory_on_movement()
   - التحديث في: warehouse_inventory
     * إذا كان السجل موجود: quantity += الكمية الجديدة
     * إذا كان السجل غير موجود: إنشاء سجل جديد
   ↓
5. **تحديث إجمالي المخزون في المخزن**:
   - تحديث warehouses.current_stock
   ↓
6. الحالة: approved
```

## 🔧 التنفيذ المطلوب

### 1. تحديث `approveReceiving` في `receivingApprovalService.ts`

```typescript
async approveReceiving(
  requestId: string,
  approverId: string,
  notes?: string
): Promise<boolean> {
  try {
    // ... (الكود الحالي للتحقق من الطلب)
    
    // تحديث حالة الطلب
    const { error: updateError } = await supabase!
      .from('waste_receiving_approval_requests')
      .update({
        status: 'approved',
        approved_by: approverId,
        approval_notes: notes,
        approved_at: new Date().toISOString(),
      })
      .eq('id', requestId);

    if (updateError) {
      // ... handle error
    }

    // ✅ إضافة حركات المخزون لكل مخلف
    const wasteItems = request.waste_items as Array<{
      waste_material_id: string; // waste_no من catalog_waste_materials
      quantity: number;
      unit: string;
      quality_grade?: string;
      notes?: string;
    }>;

    for (const item of wasteItems) {
      // 1. البحث عن product_id من waste_data_admin باستخدام waste_no
      const { data: wasteMaterial } = await supabase!
        .from('catalog_waste_materials')
        .select('waste_no')
        .eq('waste_no', item.waste_material_id)
        .single();

      if (!wasteMaterial) {
        console.warn(`لم يتم العثور على مخلف: ${item.waste_material_id}`);
        continue;
      }

      // 2. البحث عن waste_data_admin.id من waste_no
      // ملاحظة: قد تحتاج إلى ربط مباشر أو جدول وسيط
      // الحل المؤقت: استخدام waste_no كـ source_id مباشرة
      
      // 3. إضافة حركة المخزون
      const { error: movementError } = await supabase!
        .from('inventory_movements')
        .insert({
          warehouse_id: request.warehouse_id,
          product_id: null, // ⚠️ يحتاج حل: تحويل waste_no إلى UUID
          movement_type: 'in',
          quantity: item.quantity,
          unit: item.unit,
          source_type: 'waste_receiving',
          source_id: requestId,
          notes: `استلام مخلفات: ${item.waste_material_id} - ${item.notes || ''}`,
          created_by: approverId,
        });

      if (movementError) {
        console.error('خطأ في إضافة حركة المخزون:', movementError);
        // لا نوقف العملية، نكمل مع باقي المخلفات
      }
    }

    toast.success('تمت الموافقة على استلام المخلفات بنجاح');
    return true;
  } catch (error) {
    // ... handle error
  }
}
```

### 2. حل مشكلة `product_id`

**المشكلة**: 
- `waste_items` يحتوي على `waste_material_id` (waste_no من `catalog_waste_materials`)
- `inventory_movements.product_id` يحتاج `UUID` من `waste_data_admin`

**الحلول المقترحة**:

#### الحل 1: استخدام `waste_no` مباشرة في `source_id`
```typescript
// في inventory_movements:
source_type: 'waste_receiving',
source_id: `${requestId}:${item.waste_material_id}`, // يجمع request ID و waste_no
product_id: null, // يمكن أن يكون null مؤقتاً
```

#### الحل 2: البحث عن `waste_data_admin.id` من `waste_no`
```typescript
// البحث في waste_data_admin عن السجل المرتبط بـ waste_no
// قد يحتاج إلى جدول وسيط أو علاقة مباشرة
const { data: wasteData } = await supabase!
  .from('waste_data_admin')
  .select('id')
  .eq('waste_no', item.waste_material_id) // إذا كان هناك حقل waste_no
  .single();

if (wasteData) {
  product_id: wasteData.id;
}
```

#### الحل 3: إضافة حقل `waste_material_id` في `inventory_movements`
```sql
ALTER TABLE inventory_movements 
ADD COLUMN waste_material_id TEXT REFERENCES catalog_waste_materials(waste_no);
```

## 🔄 Trigger التلقائي

### `trg_update_inventory`
```sql
CREATE TRIGGER trg_update_inventory
AFTER INSERT ON inventory_movements
FOR EACH ROW
EXECUTE FUNCTION update_inventory_on_movement();
```

**ما يفعله**:
1. إذا `movement_type = 'in'`:
   - إضافة/تحديث `warehouse_inventory`
   - `quantity += NEW.quantity`
2. تحديث `warehouses.current_stock`
