## السؤال: هل يتم قراءة الوقت المتوقع من قواعد البيانات؟

**الإجابة:** نعم، **يتم قراءة الوقت المتوقع مباشرة من قاعدة البيانات** (V1.4).

## 📊 مصدر البيانات

### الجدول: `radio_listeners`

البيانات الأساسية تأتي من جدول `radio_listeners` عبر RPC Function `get_active_radio_listeners`:

```sql
SELECT 
  id,
  user_id,
  activity_id,
  started_listening_at,  -- ✅ وقت بدء الاستماع (من قاعدة البيانات)
  last_active_at,        -- ✅ آخر نشاط (من قاعدة البيانات)
  duration_minutes,      -- ⚠️ مدة تقريبية (قد تكون قديمة)
  points_earned,
  is_active,
  location
FROM radio_listeners
WHERE activity_id = :activityId
  AND is_active = true
  AND last_active_at > NOW() - INTERVAL '5 minutes';
```

## 🔄 كيف يتم حساب الوقت المتوقع؟

### الخطوة 1: جلب البيانات من قاعدة البيانات

```typescript
// في clubRadioService.getActiveListenersWithDetails()
const { data: listeners } = await supabase.rpc('get_active_radio_listeners', {
  p_activity_id: activityId
});
```

**البيانات القادمة من قاعدة البيانات:**
- `started_listening_at`: وقت بدء الاستماع (مثال: `"2025-01-22T10:00:00Z"`)
- `last_active_at`: آخر نشاط (مثال: `"2025-01-22T10:05:00Z"`)
- `duration_minutes`: مدة تقريبية (قد تكون قديمة، مثال: `0`)

### الخطوة 2: قراءة الوقت المتوقع مباشرة من قاعدة البيانات (V1.4)

```typescript
// في page.tsx - عرض قائمة المستمعين (V1.4)
{activeListeners.map((listener) => {
  // V1.4: قراءة مباشرة من قاعدة البيانات (يتم تحديثها من الموبايل كل 60 ثانية)
  const durationMinutes = listener.duration_minutes || 0;  // ✅ من قاعدة البيانات مباشرة
  const estimatedPoints = listener.points_earned || 0;     // ✅ من قاعدة البيانات مباشرة
  
  // التحقق من حالة الاتصال (آخر تحديث أقل من 5 دقائق = نشط)
  const isOnline = listener.last_active_at 
    ? (Date.now() - new Date(listener.last_active_at).getTime()) < 5 * 60 * 1000
    : false;
  
  // عرض الوقت المتوقع
  return (
    <div>
      <p>{listener.user_name}</p>
      <span>{durationMinutes > 0 ? `${durationMinutes} دقيقة` : 'بدأ للتو'}</span>  {/* ✅ من قاعدة البيانات */}
      {!isOnline && <span>🔴 انقطع الاتصال</span>}
      <span>⚡ {estimatedPoints} نقطة (متوقع)</span>  {/* ✅ من قاعدة البيانات */}
    </div>
  );
})}
```

**❌ الكود القديم (لا يُستخدم بعد V1.4):**
```typescript
// ❌ لا تحسب المدة من started_listening_at
const calculatedMinutes = Math.floor(
  (Date.now() - new Date(listener.started_listening_at).getTime()) / 60000
);
```

## 📋 الخلاصة

### ✅ ما يأتي من قاعدة البيانات (V1.4):

1. **`duration_minutes`** (من `radio_listeners`) ✅ **الأساسي**
   - عدد الدقائق الفعلية التي قضاها المستخدم في الاستماع
   - **يتم تحديثه من الموبايل كل 60 ثانية**
   - يُستخدم مباشرة للعرض (بدون حساب)

2. **`points_earned`** (من `radio_listeners`) ✅ **الأساسي**
   - عدد النقاط المتوقعة (Calculated Points)
   - **يتم تحديثه من الموبايل كل 60 ثانية**
   - يُستخدم مباشرة للعرض (بدون حساب)

3. **`last_active_at`** (من `radio_listeners`)
   - آخر لحظة أرسل فيها الموبايل تحديثاً
   - يُستخدم للتحقق من حالة الاتصال (نشط/انقطع)

4. **`started_listening_at`** (من `radio_listeners`)
   - وقت بدء الاستماع
   - للتوثيق فقط (لا يُستخدم في الحساب)

### ❌ ما لا يتم حسابه في Frontend بعد V1.4:

- **لا يتم حساب `duration_minutes` في Frontend**
- **لا يتم حساب `points_earned` في Frontend**
- **كل القيم تأتي جاهزة من قاعدة البيانات**

## 📝 ملاحظات مهمة (V1.4)

1. **`duration_minutes` في قاعدة البيانات:**
   - ✅ **يتم تحديثه من الموبايل كل 60 ثانية**
   - ✅ **يُستخدم مباشرة للعرض (بدون حساب)**
   - ⚠️ قد يكون `0` في أول دقيقة (يعرض "بدأ للتو")

2. **`points_earned` في قاعدة البيانات:**
   - ✅ **يتم تحديثه من الموبايل كل 60 ثانية**
   - ✅ **يُستخدم مباشرة للعرض (بدون حساب)**
   - ✅ **يعكس نفس الأرقام التي يراها المستخدم في التطبيق**

3. **دقة البيانات:**
   - الموبايل يحسب البيانات بناءً على حالة مشغل الصوت (Audio Player State)
   - إذا قام المستخدم بإيقاف الصوت مؤقتاً، يتوقف العداد في الموبايل وبالتالي في الداش بورد
   - هذا هو السلوك المطلوب

4. **حالة الاتصال:**
   - إذا كان `last_active_at` أقدم من 5 دقائق → "🔴 انقطع الاتصال"
   - إذا كان `last_active_at` حديث (أقل من 5 دقائق) → "🟢 نشط"

5. **لا داعي للقلق بشأن:**
   - اختلاف المناطق الزمنية (Timezones) - المدة تأتي جاهزة كرقم صحيح
   - الحساب في Frontend - كل القيم تأتي جاهزة من قاعدة البيانات
