# نظام الصلاحيات والموافقات لإدارة المخلفات

## 📋 نظرة عامة

تم إنشاء نظام شامل للصلاحيات والموافقات لإدارة المخلفات يضمن:
- **التحكم في الكتالوج**: فقط إدارة المخلفات يمكنها إضافة/تعديل/حذف المخلفات
- **التسعير المراقب**: مدير المخلفات يضع التسعير الأساسي، والتغييرات الكبيرة تحتاج موافقة
- **التحقق من الاستلام**: الدليفري بوي يسلم، وإدارة المخلفات تتحقق وتوافق

---

## 🔐 الصلاحيات

### صلاحيات الكتالوج
- `waste:catalog:create` - إضافة مخلفات جديدة
- `waste:catalog:update` - تعديل مخلفات موجودة
- `waste:catalog:delete` - حذف مخلفات
- `waste:catalog:view` - عرض الكتالوج 

### صلاحيات التسعير
- `waste:pricing:set_base` - وضع السعر الأساسي
- `waste:pricing:update_small` - تحديثات صغيرة (< 10%)
- `waste:pricing:update_large` - تحديثات كبيرة (>= 10%) - تحتاج موافقة
- `waste:pricing:approve` - الموافقة على التغييرات الكبيرة

### صلاحيات الاستلام
- `waste:receiving:view` - عرض طلبات الاستلام
- `waste:receiving:verify` - التحقق من جودة المخلفات
- `waste:receiving:approve` - الموافقة على الاستلام

### صلاحيات أخرى
- `waste:exchange:view` - عرض البورصة
- `waste:exchange:manage` - إدارة البورصة
- `waste:partners:view` - عرض الشركاء
- `waste:partners:manage` - إدارة الشركاء
- `waste:warehouse:monitor` - مراقبة المخازن

## ✅ نظام الموافقات

### 1. موافقة تغيير الأسعار

#### آلية العمل:
1. **التغييرات الصغيرة (< 10%)**: تطبق مباشرة بدون موافقة
2. **التغييرات الكبيرة (>= 10%)**: تحتاج موافقة من مدير المخلفات

#### التدفق:
```
المستخدم يطلب تغيير السعر
    ↓
النظام يحسب نسبة التغيير
    ↓
إذا >= 10% → إنشاء طلب موافقة
    ↓
مدير المخلفات يراجع الطلب
    ↓
الموافقة → تطبيق التغيير تلقائياً
الرفض → إلغاء الطلب
```

#### الجدول: `waste_price_approval_requests`
- `waste_material_id` - معرف المخلفات
- `old_price` - السعر القديم
- `new_price` - السعر الجديد
- `price_change_percentage` - نسبة التغيير
- `reason` - سبب التغيير
- `status` - الحالة (pending, approved, rejected)

### 2. موافقة استلام المخلفات

#### آلية العمل:
1. **الدليفري بوي**: يسلم المخلفات ويطلب الموافقة
2. **إدارة المخلفات**: تتحقق من الجودة والكمية
3. **الموافقة**: بعد التحقق، يتم الموافقة وتحديث المخزون

#### التدفق:
```
الدليفري بوي يسلم المخلفات
    ↓
إنشاء طلب استلام (pending_verification)
    ↓
إدارة المخلفات تتحقق (verified)
    ↓
الموافقة النهائية (approved)
    ↓
تحديث المخزون تلقائياً
```

#### الجدول: `waste_receiving_approval_requests`
- `delivery_agent_id` - معرف الدليفري بوي
- `waste_items` - قائمة المخلفات (JSONB)
- `status` - الحالة (pending_verification, verified, approved, rejected)

## 🛠️ الاستخدام

### التحقق من الصلاحيات

```typescript
import { canManageCatalog, canManagePricing } from '@/domains/waste-management/services/wasteManagementPermissions';

// التحقق من صلاحية إضافة مخلفات
const permission = await canManageCatalog(userId, 'create');
if (!permission.allowed) {
  toast.error(permission.reason);
  return;
}
```

### إنشاء طلب موافقة على السعر

```typescript
import { priceApprovalService } from '@/domains/waste-management/services/priceApprovalService';

const approval = await priceApprovalService.createApprovalRequest({
  waste_material_id: 'WASTE-001',
  stock_exchange_id: 123,
  approval_type: 'price_change',
  old_price: 100,
  new_price: 120,
  price_change_percentage: 20,
  reason: 'زيادة الطلب في السوق',
  requested_by: userId,
  status: 'pending',
});
```

### الموافقة على طلب

```typescript
// الموافقة
await priceApprovalService.approveRequest(requestId, approverId, 'موافق');

// الرفض
await priceApprovalService.rejectRequest(requestId, approverId, 'السعر مرتفع جداً');
```

### استلام المخلفات

```typescript
import { receivingApprovalService } from '@/domains/waste-management/services/receivingApprovalService';

// إنشاء طلب استلام
const request = await receivingApprovalService.createReceivingRequest({
  delivery_agent_id: agentId,
  waste_items: [
    {
      waste_material_id: 'WASTE-001',
      quantity: 100,
      unit: 'kg',
      quality_grade: 'A',
    }
  ],
});

// التحقق
await receivingApprovalService.verifyReceiving(requestId, verifierId, {
  waste_items: [...],
  verification_notes: 'الجودة ممتازة',
});

// الموافقة
await receivingApprovalService.approveReceiving(requestId, approverId, 'تم الموافقة');
```
