# سيناريو توحيد المصدر والصلاحيات — الحالة الحالية

هذا المستند يوضح **هل النظام اكتمل وتوحد**، ويشرح **السيناريو الكامل** كما هو مُنفَّذ حالياً: مصدر البيانات، المزامنة، الصلاحيات، وما تبقى اختيارياً.

---

## 1. هل النظام اكتمل وتوحد؟

| الجانب | الحالة | الوصف |
|--------|--------|--------|
| **مصدر البيانات** | ✅ مُوحَّد | الفئات والمنتجات (مخلفات) لها **مصدر واحد** في قاعدة البيانات: `waste_main_categories`، `waste_sub_categories`، `waste_data_admin`. جميع الواجهات تقرأ وتكتب من نفس الجداول. |
| **التحكم بالواجهة (من يعدّل؟)** | ✅ مُوحَّد بالكامل | **إدارة التنظيم والتسلسل** = المكان **الوحيد** للتعديل (الفئات، المنتجات، النقاط، إعدادات السلة). **إدارة الفئات والمنتجات** و**إدارة المخلفات > الفئات** = **عرض فقط للمراجعة** (بدون أزرار إضافة/تعديل/حذف؛ مع رابط «التعديل من إدارة التنظيم والتسلسل»). |
| **مزامنة الكتالوج** | ✅ مُنفَّذة | عند إنشاء/تعديل/حذف منتج في أي مسار → مزامنة تلقائية مع `catalog_waste_materials` (إنشاء، تحديث، أو تعطيل). |
| **مزامنة التسعير** | ✅ مُنفَّذة | تعديل سعر الفئة الفرعية في «أسعار الفئات الفرعية» يحدّث أسعار المنتجات في `stock_exchange` ويُسجَّل في `exchange_price_history`. |
| **سلوك الحذف** | ✅ مُنفَّذ | حذف فئة/فئة فرعية لها منتجات → حوار يختار المستخدم: «حذف الفئة فقط (المنتجات تبقى بدون ربط)» أو «حذف الفئة وجميع المنتجات». إنشاء فئة فرعية جديدة يمكن معها إدخال **سعر تبادل أولي** (شراء/بيع). |
| **إعدادات النقاط والباسكت** | ✅ مُدرَجَة | **النقاط والمكافآت** و**إعدادات السلة (الباسكت)** مُدرَجَتان في إدارة التنظيم والتسلسل كتبويبين: «النقاط والمكافآت» (نفس واجهة إعدادات النقاط للمخلفات)، «إعدادات السلة» (اختيار الفئة الرئيسية + عرض/إضافة إعدادات السلة). |

**خلاصة:**  
- **نعم:** المصدر **مُوحَّد**، و**إدارة التنظيم والتسلسل** هي المكان **الوحيد** للتعديل (الفئات، المنتجات، النقاط، إعدادات السلة).  
- **إدارة الفئات والمنتجات** و**إدارة المخلفات > الفئات** = **عرض فقط للمراجعة** مع رابط إلى إدارة التنظيم والتسلسل.

## 2. السيناريو الكامل من إنشاء المنتج إلى إدارة المخلفات

### 2.1 مصدر البيانات الواحد

```
┌─────────────────────────────────────────────────────────────────────────┐
│  مصدر واحد في قاعدة البيانات                                             │
│  waste_main_categories → waste_sub_categories → waste_data_admin         │
└─────────────────────────────────────────────────────────────────────────┘
         │                    │                         │
         │ يقرأ/يكتب          │ يقرأ فقط               │ يقرأ فقط
         ▼                    ▼                         ▼
┌─────────────────────────────┐  ┌─────────────────────────┐  ┌──────────────────────────────┐
│ إدارة التنظيم والتسلسل      │  │ إدارة الفئات والمنتجات  │  │ إدارة المخلفات > الفئات      │
│ /general-management/        │  │ /product-categories     │  │ /waste-management/categories │
│   organization-structure    │  │ (عرض فقط للمراجعة)      │  │ (عرض فقط للمراجعة)           │
│ • الفئات، المنتجات         │  │ رابط → التنظيم والتسلسل │  │ رابط → التنظيم والتسلسل      │
│ • النقاط والمكافآت          │  └─────────────────────────┘  └──────────────────────────────┘
│ • إعدادات السلة (الباسكت)   │
└─────────────────────────────┘
```

- **نفس الجداول** لكل المسارات.  
- **التعديل** من **إدارة التنظيم والتسلسل** فقط؛ الواجهتان الأخريان للمراجعة مع رابط للتعديل.

### 2.2 عند إنشاء منتج (من أي مسار يملك صلاحية التعديل)

```
المستخدم يضغط «إضافة منتج» (من product-categories أو waste-management/categories أو التنظيم والتسلسل)
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  productService.createProduct(product)                          │
│  • INSERT INTO waste_data_admin (name, subcategory_id, ...)      │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  wasteCatalogSyncService.syncNewProduct(product)                │
│  • إنشاء سجل في catalog_waste_materials                         │
│  • تحديث waste_data_admin.catalog_waste_id                      │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
المنتج يظهر فوراً في:
  • /product-categories (قائمة المنتجات للفئة الفرعية)
  • /waste-management/categories (نفس القائمة)
  • أي واجهة تقرأ من waste_data_admin أو من الكتالوج
```

### 2.3 عند تعديل منتج

- **تعديل من الواجهة:** تحديث `waste_data_admin`.  
- **Trigger في قاعدة البيانات:** عند تحديث `waste_data_admin` يُحدَّث تلقائياً السجل المقابل في `catalog_waste_materials` (الفئة، السعر المتوقع، الوزن، الملاحظات، إلخ).  
- **تعديل سعر الفئة الفرعية:** من «أسعار الفئات الفرعية» يُحدَّث `subcategory_exchange_price` ثم تُحدَّث أسعار المنتجات في `stock_exchange` ويُسجَّل في `exchange_price_history`.

### 2.4 عند حذف فئة أو فئة فرعية لها منتجات

- تظهر **نافذة اختيار:**  
  - «حذف الفئة فقط — المنتجات تبقى بدون ربط»، أو  
  - «حذف الفئة وجميع المنتجات».  
- في الحالة الثانية: تعطيل سجلات الكتالوج المرتبطة، حذف المنتجات من `waste_data_admin`، ثم حذف الفئة الفرعية أو الرئيسية.

### 2.5 إنشاء فئة فرعية جديدة

- من واجهة الفئات الفرعية يمكن إدخال **سعر تبادل أولي** (شراء/بيع) عند الإنشاء، فيُسجَّل في `subcategory_exchange_price` مباشرة.

## 3. من يعدّل ومن يعرض فقط؟

- **إدارة التنظيم والتسلسل** (`/general-management/organization-structure`) = المكان **الوحيد** للتعديل:
  - الفئات الأساسية والفرعية والمنتجات (المخلفات)
  - النقاط والمكافآت (تبويب «النقاط والمكافآت»)
  - إعدادات السلة — الباسكت (تبويب «إعدادات السلة»)
- **إدارة الفئات والمنتجات** (`/product-categories`) و**إدارة المخلفات > الفئات** (`/waste-management/categories`) = **عرض فقط للمراجعة**:
  - لا أزرار إضافة/تعديل/حذف
  - رسالة «للمراجعة فقط. التعديل من إدارة التنظيم والتسلسل» مع رابط إلى الصفحة
  - صفحات الفئات الفرعية والمنتجات من نفس المسارين تعمل بنفس المنطق (عرض فقط عند فتحها من هذين المسارين)

## 4. ما تم تنفيذه تقنياً (مرجع سريع)

| العنصر | الملف / الموقع |
|--------|-----------------|
| عرض فقط (إدارة الفئات والمنتجات / إدارة المخلفات > الفئات) | `src/app/product-categories/page.tsx`، `src/app/waste-management/categories/page.tsx` — بدون أزرار تعديل؛ رسالة + رابط إلى إدارة التنظيم والتسلسل. |
| عرض فقط (الفئات الفرعية والمنتجات من نفس المسارين) | `SubCategoriesListPage`، `ProductsListPage` — دعم `viewOnly`؛ عند `viewOnly={true}` (من product-categories و waste-management) = عرض فقط + رابط. |
| إنشاء الصلاحيات في قاعدة البيانات (للاستخدام المستقبلي إن لزم) | `supabase/migrations/20260129_add_category_edit_permissions.sql` |
| مزامنة المنتج مع الكتالوج | `wasteCatalogSyncService` — `syncNewProduct`، `syncUpdatedProduct`، `syncDeletedProduct`؛ واستدعاؤها من `productService` عند إنشاء/تحديث/حذف المنتج |
| Trigger تحديث الكتالوج عند تعديل منتج | `supabase/migrations/20260202_trigger_sync_waste_data_admin_to_catalog.sql` |
| مزامنة أسعار الفئة الفرعية مع البورصة | `subcategoryExchangePriceService` — تحديث `stock_exchange` و`exchange_price_history` |
| حذف فئة/فئة فرعية مع المنتجات | `categoryService.deleteSubCategoryWithProducts`، `deleteCategoryWithProducts`؛ وحوار التأكيد في `SubCategoriesListPage` و`CategoriesListPage` |
| إنشاء فئة فرعية مع سعر تبادل أولي | `categoryService.createSubCategoryWithInitialExchangePrice`؛ وحقول اختيارية في واجهة إضافة فئة فرعية |
| النقاط والمكافآت في التنظيم والتسلسل | تبويب «النقاط والمكافآت» في `OrganizationStructurePage` — يعرض `PointsSettingsPage` (إعدادات النقاط للمخلفات). |
| إعدادات السلة في التنظيم والتسلسل | تبويب «إعدادات السلة» في `OrganizationStructurePage` — اختيار الفئة الرئيسية، عرض `CategoryBasketConfigDetails`، وإضافة إعداد عبر `CategoryBasketConfigForm` في حوار. |

## 5. الخلاصة

- **النظام حالياً:** مصدر البيانات **مُوحَّد**، و**إدارة التنظيم والتسلسل** هي المكان **الوحيد** للتعديل (الفئات، المنتجات، النقاط، إعدادات السلة).  
- **إدارة الفئات والمنتجات** و**إدارة المخلفات > الفئات** = **عرض فقط للمراجعة** مع رابط إلى إدارة التنظيم والتسلسل.  
- **السيناريو:** إنشاء/تعديل/حذف الفئات والمنتجات والنقاط وإعدادات السلة من **إدارة التنظيم والتسلسل** فقط؛ المراجعة من المسارين الآخرين.
