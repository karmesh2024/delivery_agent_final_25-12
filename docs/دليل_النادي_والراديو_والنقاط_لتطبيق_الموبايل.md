## 🎯 1. النادي (Club Zone) - المفهوم الإداري

### 1.1 ما هو نادي Scope Zone؟

**نادي Scope Zone** هو نظام مكافآت وتحفيز داخل تطبيق كارمش يوفر:

- 🎁 **جوائز وهدايا** يمكن استبدالها بالنقاط
- 📻 **راديو كارمش** للحصول على نقاط أثناء الاستماع
- 🤝 **شراكات** مع رعاة ومؤسسات محلية
- ⭐ **نظام نقاط** مبسط وآمن (V1.3)

### 1.2 الفرق بين رصيد المخلفات ورصيد النادي

#### 🔵 رصيد المخلفات (Economic Balance)
- **الوظيفة**: حق اقتصادي كامل للمستخدم
- **الاستخدام**: يتحول كاش فوراً أو يُستخدم للشراء في المتجر الإلكتروني
- **الاعتماد**: لا يخضع لاعتماد شهري
- **العبارة الثابتة**: *"رصيد المخلفات هو حقك الاقتصادي الكامل."*

#### 🟢 رصيد النادي (Club Rewards Balance)
- **الوظيفة**: نظام مكافآت وتحفيز
- **الاستخدام**: جوائز، خصومات، سحوبات، خدمات
- **الاعتماد**: يُعتمد شهرياً (رصيد هذا الشهر → رصيد متاح)
- **العبارة الثابتة**: *"نقاط النادي نظام مكافآت وتحفيز، وليست مقابل مالي."*

> ⚠️ **مهم جداً**: تجنّب كلمة "تحويل" بين الرصيدين في UI. استخدم: "مكافأة"، "تقدير"، "رصيد تحفيزي".

## 🎵 2. الراديو (راديو كارمش) - آلية العمل

### 2.1 ما هو راديو كارمش؟

**راديو كارمش** هو راديو صوتي مباشر يُبث داخل تطبيق كارمش يوفر:

- 📻 **بث مباشر 24/7** من AzuraCast Server
- 🎵 **موسيقى وإعلانات** صوتية
- ⭐ **نقاط للمستمعين** الحاضرين أثناء البث
- 📍 **تتبع المستمعين** على الخريطة (اختياري)

### 2.2 بنية قاعدة البيانات للراديو

#### جدول `club_activities` (نشاط الراديو)

```sql
CREATE TABLE club_activities (
  id UUID PRIMARY KEY,
  activity_type VARCHAR(50),              -- 'radio_stream'
  title VARCHAR(255),                     -- 'راديو كارمش'
  scheduled_at TIMESTAMPTZ,               -- وقت البث
  stream_url TEXT,                        -- رابط البث من AzuraCast
  listen_url TEXT,                        -- رابط الاستماع للـ Player
  station_id INTEGER,                     -- معرف المحطة في AzuraCast
  current_listeners INTEGER DEFAULT 0,    -- عدد المستمعين الحاليين
  max_listeners INTEGER DEFAULT 1000,     -- الحد الأقصى للمستمعين
  is_live BOOLEAN DEFAULT FALSE,          -- هل البث مباشر الآن؟
  stream_type VARCHAR(20) DEFAULT 'mp3',  -- 'mp3', 'aac', 'ogg'
  description TEXT,
  partner_id UUID REFERENCES club_partners(id),  -- الراعي (إن وجد)
  points_reward INTEGER DEFAULT 0,        -- نقاط مكافأة (مثلاً: 1 نقطة/دقيقة)
  planned_duration_minutes INTEGER,       -- المدة المخططة بالدقائق
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**العلاقات:**
- `partner_id` → `club_partners.id` (الراعي)
- `id` → `radio_listeners.activity_id` (المستمعون)
- `id` → `radio_user_sessions.activity_id` (جلسات الاستماع)

#### جدول `radio_listeners` (المستمعون الحاليون)

```sql
CREATE TABLE radio_listeners (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  activity_id UUID REFERENCES club_activities(id),
  started_listening_at TIMESTAMPTZ DEFAULT NOW(),  -- وقت بدء الاستماع
  last_active_at TIMESTAMPTZ DEFAULT NOW(),        -- آخر نشاط
  duration_minutes INTEGER DEFAULT 0,               -- مدة الاستماع
  points_earned INTEGER DEFAULT 0,                  -- النقاط المكتسبة
  is_active BOOLEAN DEFAULT TRUE,                   -- هل ما زال يستمع؟
  location JSONB,                                    -- الموقع الجغرافي (GPS/District)
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, activity_id)
);
```

**العلاقات:**
- `user_id` → `new_profiles.id` (المستخدم)
- `activity_id` → `club_activities.id` (نشاط الراديو)

**الاستخدام:**
- تتبع المستمعين الحاليين فقط (Realtime)
- يتم تحديث `last_active_at` كل دقيقة أثناء الاستماع
- يتم تحديث `is_active = false` عند إيقاف الاستماع

**ملاحظة مهمة:**
- حقل `location JSONB` مذكور في المستند لكن قد لا يكون موجوداً في قاعدة البيانات حالياً
- إذا كان الموقع مطلوباً، يجب إضافته عبر migration:
  ```sql
  ALTER TABLE radio_listeners
  ADD COLUMN IF NOT EXISTS location JSONB;
  -- مثال: {"latitude": 31.2000, "longitude": 29.9000, "city": "الإسكندرية"}
  ```

#### جدول `radio_user_sessions` (جلسات الاستماع V1.3)

```sql
CREATE TABLE radio_user_sessions (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  activity_id UUID REFERENCES club_activities(id),
  start_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),  -- وقت بدء الجلسة
  stop_time TIMESTAMPTZ,                          -- وقت إيقاف الجلسة
  duration_minutes INTEGER,                        -- المدة بالدقائق
  points_earned INTEGER DEFAULT 0,                 -- النقاط المكتسبة
  status VARCHAR(20) DEFAULT 'active',             -- 'active' أو 'closed'
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**العلاقات:**
- `user_id` → `new_profiles.id` (المستخدم)
- `activity_id` → `club_activities.id` (نشاط الراديو)

**الاستخدام في V1.3:**
- **Session-Based Points**: النقاط تُمنح مرة واحدة عند إيقاف الجلسة
- **لا منح نقاط كل دقيقة**: لتقليل التلاعب والـ Bugs
- **Auto-close**: الجلسات المتروكة تُغلق تلقائياً بعد 4 ساعات

### 2.3 السيناريو الإداري للراديو

#### من جانب الأدمن (Dashboard):

1. **بدء البث المباشر:**
   - يفتح `/club-zone/radio`
   - يضغط "بدء البث المباشر"
   - يدخل رابط البث من AzuraCast (مثلاً: `http://radio.karmesh.eg:8000/radio.mp3`)
   - يحدد معدل النقاط (مثلاً: 1 نقطة/دقيقة)
   - يتم إنشاء سجل في `club_activities` مع `is_live = true`

2. **إيقاف البث:**
   - يضغط "إيقاف البث"
   - يتم تحديث `is_live = false` في `club_activities`
   - يتم إغلاق جميع الجلسات النشطة (`radio_user_sessions.status = 'closed'`)

3. **مراقبة المستمعين:**
   - يرى قائمة المستمعين الحاليين من `radio_listeners`
   - يرى عدد المستمعين على الخريطة (إن كان لديهم موقع GPS)

#### من جانب المستخدم (Mobile App):

1. **بدء الاستماع:**
   - يفتح شاشة "راديو كارمش" في التطبيق
   - يرى البث الحالي إن كان `is_live = true`
   - يضغط "تشغيل"
   - يتم إنشاء سجل في `radio_listeners` مع `is_active = true`
   - يتم إنشاء جلسة في `radio_user_sessions` مع `status = 'active'`
   - يبدأ العداد (Preview) لإظهار النقاط المتوقعة

2. **أثناء الاستماع:**
   - يتم تحديث `last_active_at` في `radio_listeners` كل دقيقة
   - يتم عرض عداد Preview للنقاط (غير فعلي، فقط للعرض)
   - **لا يتم منح نقاط** خلال الاستماع (V1.3)

3. **إيقاف الاستماع:**
   - يضغط "إيقاف"
   - يتم تحديث `is_active = false` في `radio_listeners`
   - يتم تحديث `stop_time` في `radio_user_sessions`
   - يتم حساب `duration_minutes = (stop_time - start_time) / 60`
   - يتم حساب النقاط: `points_earned = duration_minutes × points_rate`
   - يتم منح النقاط **مرة واحدة** في `club_points_wallet.pending_points`
   - يتم تحديث `status = 'closed'` في `radio_user_sessions`

## ⭐ 3. نظام النقاط (Points System V1.3)

### 3.1 المبادئ الأساسية في V1.3

- **3 حالات فقط للنقاط:**
  - `PENDING` → نقاط مكتسبة لم تُعتمد بعد (تظهر للمستخدم كـ "رصيد هذا الشهر")
  - `AVAILABLE` → نقاط جاهزة للاستخدام (بعد الاعتماد الشهري)
  - `USED` → نقاط تم استخدامها في جوائز

- **الاعتماد الشهري:**
  - يتم يدوياً من قبل الأدمن في نهاية كل شهر
  - يحول `pending_points` → `available_points`

- **Session-Based Radio Points:**
  - النقاط تُمنح مرة واحدة عند إيقاف الجلسة
  - لا يوجد منح نقاط كل دقيقة

### 3.2 بنية قاعدة البيانات للنقاط

#### جدول `club_points_wallet` (محفظة النقاط)

```sql
CREATE TABLE club_points_wallet (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL UNIQUE REFERENCES new_profiles(id),
  points_balance INTEGER DEFAULT 0,        -- الرصيد الإجمالي (للتوافق القديم)
  lifetime_points INTEGER DEFAULT 0,       -- إجمالي النقاط المكتسبة مدى الحياة
  pending_points INTEGER DEFAULT 0,        -- نقاط قيد الاعتماد (رصيد هذا الشهر)
  available_points INTEGER DEFAULT 0,      -- نقاط جاهزة للاستخدام (رصيد متاح)
  used_points INTEGER DEFAULT 0,           -- نقاط مستخدمة
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**العلاقات:**
- `user_id` → `new_profiles.id` (المستخدم)

**الحقول المهمة:**
- `pending_points`: النقاط المكتسبة خلال الشهر الحالي (لم تُعتمد بعد)
- `available_points`: النقاط المعتمدة والجاهزة للاستخدام
- `used_points`: النقاط التي تم استبدالها في الجوائز
- `lifetime_points`: إجمالي النقاط المكتسبة منذ البداية (للإحصائيات)

#### جدول `club_points_transactions` (معاملات النقاط)

```sql
CREATE TABLE club_points_transactions (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  transaction_type VARCHAR(20) NOT NULL CHECK (
    transaction_type IN ('EARNED', 'ACTIVATED', 'USED')
  ),
  points INTEGER NOT NULL,                  -- القيمة (موجب للنقاط المكتسبة، سالب للخصم)
  balance_before INTEGER,                   -- الرصيد قبل المعاملة
  balance_after INTEGER,                    -- الرصيد بعد المعاملة
  reason VARCHAR(255),                      -- السبب (مثلاً: 'اعتماد شهري')
  source VARCHAR(50),                       -- المصدر: 'radio_stream', 'monthly_settlement', 'waste_collection'
  description TEXT,                         -- وصف تفصيلي
  created_by UUID REFERENCES new_profiles(id),  -- من قام بالعملية (للأدمن)
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**العلاقات:**
- `user_id` → `new_profiles.id` (المستخدم)
- `created_by` → `new_profiles.id` (الأدمن أو النظام)

**أنواع المعاملات (V1.3):**
- `EARNED`: كسب نقاط (تذهب إلى `pending_points`)
  - مثال: الاستماع للراديو، مكافأة من الأدمن
- `ACTIVATED`: تسوية شهرية (نقل من `pending_points` إلى `available_points`)
  - مثال: اعتماد نقاط شهر يناير
- `USED`: استخدام نقاط (خصم من `available_points`)
  - مثال: استبدال جائزة

**مصادر النقاط (source):**
- `radio_stream`: من الاستماع للراديو
- `monthly_settlement`: من الاعتماد الشهري
- `waste_collection`: من تحويل نقاط المخلفات
- `admin_bonus`: مكافأة إدارية
- `reward_redeem`: استبدال جائزة

#### جدول `monthly_points_settlement` (التسوية الشهرية)

## 🔄 4. السيناريوهات البرمجية

### 4.1 سيناريو الاستماع للراديو (Flutter → Backend)

#### الخطوة 1: جلب البث الحالي

```dart
// Flutter Code Example
Future<RadioStream?> getCurrentStream() async {
  final response = await supabase
    .from('club_activities')
    .select()
    .eq('activity_type', 'radio_stream')
    .eq('is_active', true)
    .eq('is_live', true)
    .maybeSingle();
  
  if (response == null) return null;
  
  return RadioStream.fromJson(response);
}
```

**SQL Equivalent:**
```sql
SELECT * FROM club_activities
WHERE activity_type = 'radio_stream'
  AND is_active = true
  AND is_live = true
LIMIT 1;
```

#### الخطوة 2: بدء الاستماع

```dart
// Flutter Code Example
Future<String?> startListening(String activityId, String userId) async {
  // 1) إنشاء سجل في radio_listeners
  final listenerResponse = await supabase
    .from('radio_listeners')
    .upsert({
      'user_id': userId,
      'activity_id': activityId,
      'started_listening_at': DateTime.now().toIso8601String(),
      'last_active_at': DateTime.now().toIso8601String(),
      'is_active': true,
      'duration_minutes': 0,
      'points_earned': 0,
    })
    .select()
    .single();
  
  // 2) إنشاء جلسة في radio_user_sessions
  final sessionResponse = await supabase
    .from('radio_user_sessions')
    .insert({
      'user_id': userId,
      'activity_id': activityId,
      'start_time': DateTime.now().toIso8601String(),
      'status': 'active',
    })
    .select()
    .single();
  
  return sessionResponse['id'] as String;
}
```

**SQL Equivalent:**
```sql
-- 1) إنشاء/تحديث radio_listeners
INSERT INTO radio_listeners (user_id, activity_id, started_listening_at, last_active_at, is_active)
VALUES (:userId, :activityId, NOW(), NOW(), true)
ON CONFLICT (user_id, activity_id) 
DO UPDATE SET 
  is_active = true,
  started_listening_at = NOW(),
  last_active_at = NOW();

-- 2) إنشاء radio_user_sessions
INSERT INTO radio_user_sessions (user_id, activity_id, start_time, status)
VALUES (:userId, :activityId, NOW(), 'active')
RETURNING id;
```

**RPC Function: `redeem_club_reward`**

```sql
CREATE OR REPLACE FUNCTION redeem_club_reward(
  p_user_id UUID,
  p_reward_id UUID
)
RETURNS JSONB
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_reward RECORD;
  v_points_required INTEGER;
  v_available_points INTEGER;
  v_redemption_code VARCHAR(100);
  v_redemption_id UUID;
BEGIN
  -- 1) جلب معلومات الجائزة
  SELECT * INTO v_reward
  FROM club_rewards
  WHERE id = p_reward_id
    AND is_active = true
    AND (quantity_available IS NULL OR quantity_redeemed < quantity_available);
  
  IF NOT FOUND THEN
    RAISE EXCEPTION 'الجائزة غير متاحة';
  END IF;
  
  v_points_required := v_reward.points_required;
  
  -- 2) التحقق من الرصيد المتاح
  SELECT available_points INTO v_available_points
  FROM club_points_wallet
  WHERE user_id = p_user_id;
  
  IF v_available_points < v_points_required THEN
    RAISE EXCEPTION 'رصيد النقاط غير كافي';
  END IF;
  
  -- 3) إنشاء كود الاستبدال
  v_redemption_code := 'REDEEM-' || UPPER(SUBSTRING(REPLACE(gen_random_uuid()::TEXT, '-', ''), 1, 12));
  
  -- 4) إنشاء سجل الاستبدال
  INSERT INTO club_reward_redemptions (
    user_id, reward_id, points_spent, redemption_code,
    redemption_type, redemption_data, status, expires_at
  ) VALUES (
    p_user_id,
    p_reward_id,
    v_points_required,
    v_redemption_code,
    v_reward.reward_type,
    CASE 
      WHEN v_reward.reward_type = 'wallet_credit' THEN
        jsonb_build_object('amount', v_reward.points_required, 'currency', 'EGP')
      ELSE NULL
    END,
    'pending',
    CASE 
      WHEN v_reward.reward_type = 'wallet_credit' THEN NULL
      ELSE NOW() + INTERVAL '30 days'
    END
  ) RETURNING id INTO v_redemption_id;
  
  -- 5) خصم النقاط من available_points
  PERFORM update_club_points_wallet(
    p_user_id,
    -v_points_required,
    'USED',
    'استبدال جائزة',
    'reward_redeem',
    'استبدال: ' || v_reward.title,
    NULL
  );
  
  -- 6) تحديث quantity_redeemed في club_rewards
  UPDATE club_rewards
  SET quantity_redeemed = COALESCE(quantity_redeemed, 0) + 1,
      updated_at = NOW()
  WHERE id = p_reward_id;
  
  -- 7) تحديث حالة الاستبدال إلى completed
  UPDATE club_reward_redemptions
  SET status = 'completed',
      redeemed_at = NOW()
  WHERE id = v_redemption_id;
  
  RETURN jsonb_build_object(
    'redemption_id', v_redemption_id,
    'redemption_code', v_redemption_code,
    'points_spent', v_points_required
  );
END;
$$;
```

### 4.5 سيناريو تحويل نقاط المخلفات إلى نقاط النادي

#### الخطوة 1: إنشاء طلب تحويل (من Flutter)

```dart
// Flutter Code Example
Future<String?> createRecyclingConversionRequest(
  String userId,
  int recyclingPoints,
) async {
  // 1) جلب نسبة التحويل من club_settings
  final settings = await supabase
    .from('club_settings')
    .select()
    .eq('key', 'waste_to_club_conversion')
    .single();
  
  final rate = (settings['value'] as Map)['rate'] as double; // 0.3
  final clubPointsExpected = (recyclingPoints * rate).floor();
  
  // 2) التحقق من السقف الشهري
  final capSettings = await supabase
    .from('club_settings')
    .select()
    .eq('key', 'monthly_waste_conversion_cap')
    .single();
  
  final monthlyCap = (capSettings['value'] as Map)['points'] as int; // 2000
  
  // حساب ما تم تحويله هذا الشهر
  final currentMonth = DateTime.now().toIso8601String().substring(0, 7); // '2025-01'
  final monthlyConverted = await supabase
    .from('club_points_transactions')
    .select('points')
    .eq('user_id', userId)
    .eq('source', 'waste_collection')
    .gte('created_at', '$currentMonth-01')
    .lt('created_at', '${currentMonth}-32');
  
  final totalConverted = (monthlyConverted as List)
    .fold<int>(0, (sum, t) => sum + (t['points'] as int));
  
  if (totalConverted + clubPointsExpected > monthlyCap) {
    throw Exception('تم الوصول للسقف الشهري ($monthlyCap نقطة)');
  }
  
  // 3) إنشاء طلب التحويل
  final request = await supabase
    .from('recycling_conversion_requests')
    .insert({
      'user_id': userId,
      'recycling_points': recyclingPoints,
      'club_points_expected': clubPointsExpected,
      'status': 'PENDING',
    })
    .select()
    .single();
  
  return request['id'] as String;
}
```

#### الخطوة 2: معالجة الطلب (من Dashboard - Admin Only)

**RPC Function: `process_recycling_conversion_request`**

```sql
CREATE OR REPLACE FUNCTION process_recycling_conversion_request(
  p_request_id UUID,
  p_action VARCHAR, -- 'APPROVE' or 'REJECT'
  p_processed_by UUID,
  p_rejection_reason TEXT DEFAULT NULL
)
RETURNS JSONB
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_request RECORD;
  v_settings RECORD;
  v_rate DECIMAL(10,2);
  v_monthly_cap INTEGER;
  v_monthly_converted INTEGER;
  v_current_month VARCHAR(7);
BEGIN
  -- 1) جلب الطلب
  SELECT * INTO v_request
  FROM recycling_conversion_requests
  WHERE id = p_request_id
    AND status = 'PENDING';
  
  IF NOT FOUND THEN
    RAISE EXCEPTION 'الطلب غير موجود أو تم معالجته مسبقاً';
  END IF;
  
  -- 2) إذا كان الرفض
  IF p_action = 'REJECT' THEN
    UPDATE recycling_conversion_requests
    SET 
      status = 'REJECTED',
      processed_by = p_processed_by,
      processed_at = NOW(),
      rejection_reason = p_rejection_reason
    WHERE id = p_request_id;
    
    RETURN jsonb_build_object('status', 'rejected', 'message', 'تم رفض الطلب');
  END IF;
  
  -- 3) إذا كان القبول - التحقق من السقف الشهري
  v_current_month := TO_CHAR(NOW(), 'YYYY-MM');
  
  SELECT value->>'rate' INTO v_rate
  FROM club_settings
  WHERE key = 'waste_to_club_conversion';
  
  SELECT (value->>'points')::INTEGER INTO v_monthly_cap
  FROM club_settings
  WHERE key = 'monthly_waste_conversion_cap';
  
  -- حساب ما تم تحويله هذا الشهر
  SELECT COALESCE(SUM(points), 0) INTO v_monthly_converted
  FROM club_points_transactions
  WHERE user_id = v_request.user_id
    AND source = 'waste_collection'
    AND TO_CHAR(created_at, 'YYYY-MM') = v_current_month;
  
  IF v_monthly_converted + v_request.club_points_expected > v_monthly_cap THEN
    RAISE EXCEPTION 'تم الوصول للسقف الشهري (%)', v_monthly_cap;
  END IF;
  
  -- 4) منح النقاط (تذهب إلى pending_points)
  PERFORM update_club_points_wallet(
    v_request.user_id,
    v_request.club_points_expected,
    'EARNED',
    'تحويل نقاط المخلفات',
    'waste_collection',
    'تحويل ' || v_request.recycling_points || ' نقطة مخلفات إلى ' || 
    v_request.club_points_expected || ' نقطة نادي',
    p_processed_by
  );
  
  -- 5) تحديث حالة الطلب
  UPDATE recycling_conversion_requests
  SET 
    status = 'APPROVED',
    processed_by = p_processed_by,
    processed_at = NOW()
  WHERE id = p_request_id;
  
  -- ملاحظة: خصم النقاط من نظام المخلفات يتم في منظومة منفصلة
  
  RETURN jsonb_build_object(
    'status', 'approved',
    'club_points_awarded', v_request.club_points_expected,
    'message', 'تم منح النقاط بنجاح'
  );
END;
$$;
```

```dart
  @override
  void initState() {
    super.initState();
    loadCurrentStream();
  }

  Future<void> loadCurrentStream() async {
    final stream = await getCurrentStream();
    setState(() {
      currentStream = stream;
    });
  }

  Future<void> startListening() async {
    if (currentStream == null) return;
    
    final userId = await getCurrentUserId();
    sessionId = await startListening(
      currentStream!.id,
      userId,
    );
    
    setState(() {
      isPlaying = true;
    });
    
    // تحديث عداد Preview كل دقيقة
    updateTimer = Timer.periodic(Duration(minutes: 1), (timer) {
      setState(() {
        listeningMinutes++;
        estimatedPoints = listeningMinutes * (currentStream?.pointsPerMinute ?? 1);
      });
    });
  }

  Future<void> stopListening() async {
    if (sessionId == null) return;
    
    updateTimer?.cancel();
    
    final userId = await getCurrentUserId();
    final pointsEarned = await stopListeningAndEarnPoints(
      sessionId!,
      userId,
      currentStream!.id,
    );
    
    setState(() {
      isPlaying = false;
      sessionId = null;
      listeningMinutes = 0;
      estimatedPoints = 0;
    });
    
    // عرض رسالة النجاح
    showDialog(
      context: context,
      builder: (context) => AlertDialog(
        title: Text('تم الحصول على النقاط!'),
        content: Text('لقد حصلت على $pointsEarned نقطة من الاستماع للراديو'),
        actions: [
          TextButton(
            onPressed: () => Navigator.pop(context),
            child: Text('حسناً'),
          ),
        ],
      ),
    );
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('راديو كارمش')),
      body: currentStream == null
          ? Center(child: Text('لا يوجد بث مباشر حالياً'))
          : Column(
              children: [

                ElevatedButton(
                  onPressed: isPlaying ? stopListening : startListening,
```

- ✅ أخبر المستخدم: "رصيد هذا الشهر سيُعتمد نهاية الشهر"

### 8.4 قواعد العمل (Business Rules)

**قواعد مهمة يجب اتباعها:**

1. **نسبة تحويل نقاط المخلفات:**
   - النسبة الثابتة: **30%** (0.3)
   - موجودة في `club_settings` كـ `waste_to_club_conversion`
   - يجب قراءتها من قاعدة البيانات وليس Hardcode

2. **السقف الشهري لتحويل نقاط المخلفات:**
   - الحد الأقصى: **2000 نقطة نادي** لكل مستخدم في الشهر
   - موجود في `club_settings` كـ `monthly_waste_conversion_cap`
   - يتم التحقق منه تلقائياً في دالة `process_recycling_conversion_request`

3. **استبدال الجوائز:**
   - يتم الخصم من `available_points` فقط (ليس `pending_points`)
   - يتم إنشاء `redemption_code` فريد لكل استبدال
   - يتم تحديث `quantity_redeemed` في `club_rewards` تلقائياً

4. **نقاط الراديو:**
   - تُمنح مرة واحدة عند إيقاف الجلسة فقط (Session-Based)
   - تذهب إلى `pending_points` (رصيد هذا الشهر)
   - تُعتمد شهرياً مع باقي النقاط المعلقة

### 12.2 جداول إضافية للراديو

#### جدول سجلات البث (`radio_sessions`)

```sql
CREATE TABLE radio_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  activity_id UUID NOT NULL REFERENCES club_activities(id) ON DELETE CASCADE,
  started_at TIMESTAMPTZ DEFAULT NOW(),
  ended_at TIMESTAMPTZ,
  total_listeners INTEGER DEFAULT 0,
  peak_listeners INTEGER DEFAULT 0,
  total_duration_minutes INTEGER DEFAULT 0,
  partner_id UUID REFERENCES club_partners(id),
  status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'completed', 'cancelled')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**الفرق بين `radio_sessions` و `radio_user_sessions`:**
- `radio_sessions`: سجلات جلسات البث المباشر (للإحصائيات الإدارية)
  - يُنشأ سجل واحد لكل بث مباشر
  - يُستخدم لتتبع إحصائيات البث (عدد المستمعين، المدة، إلخ)
- `radio_user_sessions`: جلسات الاستماع لكل مستخدم (لنظام النقاط)
  - يُنشأ سجل لكل مستخدم عند بدء الاستماع
  - يُستخدم لحساب النقاط المستحقة لكل مستخدم

**الاستخدام:** للـ Dashboard فقط (إحصائيات البث)
