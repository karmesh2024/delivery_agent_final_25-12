## 🎯 نظرة عامة

نظام راديو كارمش هو جزء متكامل من نادي Scope Zone، يهدف إلى:
- **تعزيز التفاعل** بين الأعضاء والنادي من خلال البث المباشر
- **مكافأة الأعضاء** بالنقاط مقابل الاستماع للبث
- **تتبع النشاط** والتفاعل مع المحتوى
- **تحليل البيانات** الجغرافية للمستمعين

---

## 🔗 علاقة الراديو بالنادي

### 1. التكامل مع نظام النادي

الراديو مرتبط مباشرة بنظام النادي من خلال:

#### أ. جدول `club_activities`
- كل بث راديو هو **نشاط** في النادي (`activity_type = 'radio_stream'`)
- يمكن ربط البث بـ **راعي** (`partner_id`) من جدول `club_partners`
- البث له **عنوان** و**وصف** و**مدة محددة**

#### ب. جدول `radio_listeners`
- يربط بين **الأعضاء** (`user_id` من `new_profiles`) و**البث** (`activity_id`)
- يتتبع **مدة الاستماع** و**النقاط المكتسبة**
- يسجل **الموقع الجغرافي** للمستمع (GPS أو الحي)

#### ج. نظام النقاط
- النقاط تُضاف تلقائياً إلى **محفظة نقاط النادي** (`club_points_wallet`)
- كل معاملة تُسجل في `club_points_transactions` مع:
  - `transaction_type = 'EARNED'`
  - `reason = 'radio_listening'`
  - `source = 'radio_stream'`

### 2. العلاقات بين الجداول

```
club_activities (البث)
    ├── partner_id → club_partners (الراعي)
    └── id → radio_listeners.activity_id

radio_listeners (المستمعون)
    ├── user_id → new_profiles.id (العضو)
    └── activity_id → club_activities.id (البث)

club_points_wallet (محفظة النقاط)
    └── user_id → new_profiles.id (العضو)

club_points_transactions (معاملات النقاط)
    ├── wallet_id → club_points_wallet.id
    └── reason = 'radio_listening'
```

## 💰 سيناريو النقاط بالتفصيل

### 1. إعداد البث (من قبل الأدمن)

عند إنشاء بث جديد، الأدمن يحدد:

```typescript
{
  title: "راديو كارمش - البث المباشر",
  stream_url: "http://radio.example.com:8000/stream.mp3",
  listen_url: "http://radio.example.com:8000/stream.mp3",
  points_per_minute: 1,  // عدد النقاط لكل دقيقة استماع
  planned_duration_minutes: 30,  // مدة البث (اختياري)
  max_listeners: 1000,
  partner_id: "uuid-الراعي"  // اختياري
}
```

**ملاحظات مهمة:**
- `points_per_minute`: القيمة الافتراضية هي **1 نقطة لكل دقيقة**
- يمكن للأدمن تغيير هذه القيمة (0-10 نقاط)
- `planned_duration_minutes`: إذا تم تحديدها، سيتم إيقاف البث تلقائياً بعد هذه المدة

### 2. بدء الاستماع (من قبل العضو)

عندما يضغط العضو على زر **"تشغيل"**:

#### أ. طلب إذن الموقع
1. النظام يطلب **إذن GPS** من المستخدم
2. إذا وافق:
   - يتم حفظ الموقع الدقيق (`location_source = 'gps'`)
   - يتم حفظ `latitude`, `longitude`, `accuracy`
3. إذا رفض:
   - يتم عرض قائمة **أحياء الإسكندرية**
   - المستخدم يختار حي (مثل: محرم بك)
   - النظام يضع موقع عشوائي داخل الحي (`location_source = 'district'`)
4. إذا رفض كل شيء:
   - يتم استخدام **العنوان الافتراضي** من `customer_addresses` (`location_source = 'address'`)

#### ب. تسجيل بدء الاستماع
```sql
INSERT INTO radio_listeners (
  user_id,
  activity_id,
  started_listening_at,
  last_active_at,
  is_active,
  current_latitude,
  current_longitude,
  location_source,
  location_updated_at
) VALUES (...)
```

### 3. تحديث النشاط (Heartbeat)

كل دقيقة أثناء الاستماع:

#### أ. تحديث `last_active_at`
```typescript
// يتم استدعاء updateListenerActivity() كل دقيقة
UPDATE radio_listeners
SET last_active_at = NOW(), is_active = TRUE
WHERE user_id = ? AND activity_id = ?
```

#### ب. حساب النقاط المستحقة
```typescript
const durationMinutes = Math.floor(
  (NOW() - started_listening_at) / 60000
);

const pointsPerMinute = currentStream.points_per_minute || 1;
const totalPointsEarned = durationMinutes * pointsPerMinute;
const pointsToAward = totalPointsEarned - (listener.points_earned || 0);
```

#### ج. منح النقاط (إذا كانت هناك نقاط جديدة)
```sql
-- استدعاء Function قاعدة البيانات
SELECT update_club_points_wallet(
  p_user_id := listener.user_id,
  p_points := pointsToAward,
  p_transaction_type := 'EARNED',
  p_reason := 'radio_listening',
  p_source := 'radio_stream',
  p_description := 'نقاط الاستماع للراديو - ' || durationMinutes || ' دقيقة'
);
```

#### د. تحديث سجل المستمع
```sql
UPDATE radio_listeners
SET 
  points_earned = totalPointsEarned,
  duration_minutes = durationMinutes
WHERE id = listener.id;
```

### 4. إيقاف الاستماع

عندما يضغط العضو على زر **"إيقاف"** أو يغلق الصفحة:

#### أ. حساب النقاط النهائية
```typescript
const durationMinutes = Math.floor(
  (NOW() - started_listening_at) / 60000
);

const pointsPerMinute = currentStream.points_per_minute || 1;
const finalPoints = durationMinutes * pointsPerMinute;
const pointsToAward = finalPoints - (listener.points_earned || 0);
```

#### ب. منح النقاط المتبقية
```sql
-- إذا كان هناك نقاط لم تُمنح بعد
IF pointsToAward > 0 THEN
  SELECT update_club_points_wallet(...);
END IF;
```

#### ج. تحديث حالة المستمع
```sql
UPDATE radio_listeners
SET 
  is_active = FALSE,
  duration_minutes = durationMinutes,
  points_earned = finalPoints
WHERE user_id = ? AND activity_id = ?
```

#### د. عرض ملخص النقاط
- يتم عرض Dialog للمستخدم:
  - **النقاط المكتسبة**: `finalPoints`
  - **مدة الاستماع**: `durationMinutes` دقيقة
  - **رسالة تأكيد**: "تم إضافة النقاط تلقائياً إلى محفظة نقاط النادي"

### 5. إيقاف البث تلقائياً

إذا تم تحديد `planned_duration_minutes`:

1. عند بدء البث، يتم جدولة **Timer** في لوحة التحكم
2. بعد انتهاء المدة:
   - يتم إيقاف البث تلقائياً
   - يتم إيقاف جميع المستمعين النشطين
   - يتم منح النقاط النهائية لكل مستمع

## 🏗️ الهيكل التقني

### 1. الجداول الرئيسية

#### أ. `club_activities`
```sql
CREATE TABLE club_activities (
  id UUID PRIMARY KEY,
  activity_type VARCHAR DEFAULT 'radio_stream',
  title TEXT NOT NULL,
  description TEXT,
  scheduled_at TIMESTAMPTZ,
  partner_id UUID REFERENCES club_partners(id),
  stream_url TEXT,
  listen_url TEXT,
  stream_type VARCHAR DEFAULT 'mp3',
  max_listeners INTEGER DEFAULT 1000,
  current_listeners INTEGER DEFAULT 0,
  points_reward INTEGER DEFAULT 1,  -- النقاط لكل دقيقة
  planned_duration_minutes INTEGER,  -- مدة البث (اختياري)
  is_active BOOLEAN DEFAULT TRUE,
  is_live BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### ب. `radio_listeners`
```sql
CREATE TABLE radio_listeners (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  activity_id UUID REFERENCES club_activities(id),
  started_listening_at TIMESTAMPTZ DEFAULT NOW(),
  last_active_at TIMESTAMPTZ DEFAULT NOW(),
  duration_minutes INTEGER DEFAULT 0,
  points_earned INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  -- حقول الموقع
  current_latitude DOUBLE PRECISION,
  current_longitude DOUBLE PRECISION,
  location_updated_at TIMESTAMPTZ,
  location_source VARCHAR(20) DEFAULT 'gps',  -- 'gps' | 'district' | 'address'
  location_accuracy DOUBLE PRECISION,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, activity_id)
);
```

#### ج. `club_points_wallet`
```sql
CREATE TABLE club_points_wallet (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  total_points INTEGER DEFAULT 0,
  available_points INTEGER DEFAULT 0,
  pending_points INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### د. `club_points_transactions`
```sql
CREATE TABLE club_points_transactions (
  id UUID PRIMARY KEY,
  wallet_id UUID REFERENCES club_points_wallet(id),
  points INTEGER NOT NULL,
  transaction_type VARCHAR NOT NULL,  -- 'EARNED' | 'USED' | 'TRANSFERRED'
  reason VARCHAR,  -- 'radio_listening' | 'reward_redemption' | ...
  source VARCHAR,  -- 'radio_stream' | 'reward' | ...
  description TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

```sql
  p_points INTEGER,
  p_transaction_type VARCHAR,
  p_reason VARCHAR DEFAULT NULL,
  p_source VARCHAR DEFAULT NULL,
  p_description TEXT DEFAULT NULL

LANGUAGE plpgsql

-- 1. إنشاء/تحديث محفظة النقاط
-- 2. إضافة معاملة جديدة

```

#### ب. `get_active_radio_listeners`
```sql

  p_activity_id UUID

-- Function تقوم بـ:
```

## 🎬 سيناريوهات الاستخدام

### السيناريو 1: بث عادي (30 دقيقة، نقطة واحدة لكل دقيقة)

1. **الأدمن ينشئ البث**:
   - `points_per_minute = 1`
   - `planned_duration_minutes = 30`

2. **المستخدم يستمع لمدة 15 دقيقة**:
   - النقاط المكتسبة: `15 × 1 = 15 نقطة`
   - يتم منحها تلقائياً كل دقيقة

3. **المستخدم يوقف الاستماع**:
   - يتم منح النقاط النهائية (15 نقطة)
   - يتم عرض ملخص: "مبروك! حصلت على 15 نقطة"

### السيناريو 2: بث خاص (5 نقاط لكل دقيقة)

1. **الأدمن ينشئ بث خاص**:
   - `points_per_minute = 5`
   - `planned_duration_minutes = 60`

2. **المستخدم يستمع لمدة 20 دقيقة**:
   - النقاط المكتسبة: `20 × 5 = 100 نقطة`
   - يتم منحها تلقائياً كل دقيقة

3. **البث ينتهي تلقائياً بعد 60 دقيقة**:
   - جميع المستمعين يحصلون على نقاطهم النهائية
   - المستمع الذي استمع 60 دقيقة يحصل على `60 × 5 = 300 نقطة`

### السيناريو 3: مستمع بدون GPS

1. **المستخدم يرفض GPS**:
   - يتم عرض قائمة أحياء الإسكندرية
   - المستخدم يختار "محرم بك"

2. **النظام يضع موقع عشوائي**:
   - `location_source = 'district'`
   - موقع عشوائي داخل حدود محرم بك

3. **النقاط تُمنح بشكل طبيعي**:
   - الموقع لا يؤثر على النقاط
   - فقط للتتبع الجغرافي

## ❓ الأسئلة الشائعة

### س1: متى تُمنح النقاط؟
**ج:** النقاط تُمنح **تلقائياً كل دقيقة** أثناء الاستماع، وعند إيقاف الاستماع يتم منح النقاط المتبقية.

### س2: ماذا لو أغلق المستخدم المتصفح؟
**ج:** عند إيقاف الصوت أو إغلاق الصفحة، يتم استدعاء `stopListening()` تلقائياً ومنح النقاط النهائية.

### س3: هل يمكن للمستخدم الحصول على نقاط بدون الاستماع الفعلي؟
**ج:** لا، النظام يتتبع `last_active_at` كل دقيقة. إذا لم يتم تحديث النشاط خلال 5 دقائق، يعتبر المستمع غير نشط.

### س4: كيف يتم حساب النقاط؟
**ج:** 
```
النقاط = مدة الاستماع (بالدقائق) × النقاط لكل دقيقة
```

### س5: ماذا لو استمع المستخدم لنفس البث مرتين؟
**ج:** كل مرة يتم إنشاء سجل جديد في `radio_listeners` (بسبب `UNIQUE(user_id, activity_id)` يتم تحديث السجل الموجود).

### س6: هل يمكن للأدمن تغيير النقاط بعد بدء البث؟
**ج:** لا، `points_per_minute` يتم تحديده عند إنشاء البث ولا يمكن تغييره بعد البدء.

### س7: ماذا يحدث إذا انقطع الإنترنت؟
**ج:** 
- إذا انقطع الإنترنت لأكثر من 5 دقائق، يعتبر المستمع غير نشط
- النقاط المكتسبة حتى لحظة الانقطاع محفوظة
- عند إعادة الاتصال، يتم استئناف الاستماع من حيث توقف

### س8: كيف يتم تتبع الموقع؟
**ج:** 
- **GPS**: موقع دقيق من الجهاز (`location_source = 'gps'`)
- **الحي**: موقع عشوائي داخل الحي المختار (`location_source = 'district'`)
- **العنوان**: العنوان الافتراضي المسجل (`location_source = 'address'`)
