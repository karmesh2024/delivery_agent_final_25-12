## 1. نظرة عامة على النظام

### 1.1 ما هو نظام النقاط في الراديو؟

**نظام النقاط في راديو كارمش** هو نظام مكافآت يحفز المستخدمين على الاستماع للراديو من خلال منحهم نقاط مقابل كل دقيقة استماع.

### 1.2 المبادئ الأساسية (V1.3)

- ✅ **Session-Based Points**: النقاط تُمنح **مرة واحدة** عند إيقاف الجلسة فقط
- ✅ **Preview Counter**: عداد النقاط المتوقعة يُعرض أثناء الاستماع (للتحفيز فقط)
- ✅ **لا منح نقاط أثناء الاستماع**: لتقليل التلاعب والـ Bugs
- ✅ **Auto-close**: الجلسات المتروكة تُغلق تلقائياً بعد 4 ساعات

### 1.3 دورة حياة النقاط

```
بدء الاستماع → عداد Preview (متوقع) → إيقاف الاستماع → منح النقاط → pending_points → اعتماد شهري → available_points
```

## 2. فكرة النقاط وعلاقتها بالراديو

### 2.1 كيف تربط النقاط بالراديو؟

**العلاقة:**
- 📻 **البث المباشر** (`club_activities.is_live = true`) → يسمح للمستخدمين بالاستماع
- 👂 **الاستماع** (`radio_user_sessions`) → يُنشأ سجل لكل مستمع
- ⏱️ **المدة** (`duration_minutes`) → تُحسب من `start_time` إلى `stop_time`
- ⭐ **النقاط** (`points_earned`) → تُحسب بناءً على المدة × معدل النقاط

### 2.2 معدل النقاط

**معدل النقاط** (`points_per_minute`) يُحدد من قبل الأدمن عند بدء البث:
- القيمة الافتراضية: **1 نقطة لكل دقيقة**
- يمكن تغييرها: 0.5، 1، 2، إلخ
- تُحفظ في `club_activities.points_reward` أو `club_service_points_config`

### 2.3 مثال عملي

**سيناريو:**
- المستخدم يستمع للراديو لمدة **30 دقيقة**
- معدل النقاط: **1 نقطة/دقيقة**
- النقاط المكتسبة: **30 نقطة**

**ما يحدث:**
1. عند الإيقاف: تُحسب النقاط = 30 × 1 = **30 نقطة**
2. تُضاف إلى `club_points_wallet.pending_points`
3. تُسجل في `club_points_transactions` كـ `EARNED`
4. تظهر للمستخدم كـ "رصيد هذا الشهر"

## 3. كيف يعمل النظام على مستوى قاعدة البيانات

### 3.1 الجداول الرئيسية

#### جدول `club_activities` (نشاط الراديو)

```sql
CREATE TABLE club_activities (
  id UUID PRIMARY KEY,
  activity_type VARCHAR(50) DEFAULT 'radio_stream',
  title VARCHAR(255) DEFAULT 'راديو كارمش',
  stream_url TEXT,                    -- رابط البث من AzuraCast
  listen_url TEXT,                    -- رابط الاستماع
  is_live BOOLEAN DEFAULT FALSE,      -- هل البث مباشر الآن؟
  points_per_minute INTEGER DEFAULT 1, -- معدل النقاط (1 نقطة/دقيقة)
  current_listeners INTEGER DEFAULT 0,
  max_listeners INTEGER DEFAULT 1000,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**الحقول المهمة:**
- `is_live`: يحدد ما إذا كان البث مباشراً (يسمح بالاستماع)
- `points_per_minute`: معدل النقاط لكل دقيقة استماع
- `stream_url`: رابط البث الصوتي

#### جدول `radio_listeners` (المستمعون الحاليون - Realtime)

**الاستخدام:**
- تتبع المستمعين النشطين فقط (Realtime)
- يتم تحديث `last_active_at` كل دقيقة (Heartbeat)
- يتم تحديث `is_active = false` عند إيقاف الاستماع
- **ملاحظة:** `points_earned` هنا تقريبي فقط، النقاط الفعلية تُحسب من `radio_user_sessions`

#### جدول `radio_user_sessions` (جلسات الاستماع - V1.3)

```sql
CREATE TABLE radio_user_sessions (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  activity_id UUID REFERENCES club_activities(id),
  start_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),  -- وقت بدء الجلسة
  stop_time TIMESTAMPTZ,                          -- وقت إيقاف الجلسة
  duration_minutes INTEGER,                        -- المدة بالدقائق (تُحسب عند الإيقاف)
  points_earned INTEGER DEFAULT 0,                 -- النقاط المكتسبة (تُحسب عند الإيقاف)
  estimated_points DECIMAL(10, 2) DEFAULT 0,       -- النقاط المتوقعة (مرجع تاريخي - يُحدث كل دقيقة)
  status VARCHAR(20) DEFAULT 'active',             -- 'active' أو 'closed'
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**الحقول المهمة:**
- `start_time`: وقت بدء الجلسة (يُستخدم لحساب النقاط الفعلية)
- `estimated_points`: النقاط المتوقعة (مرجع تاريخي - يُحدث كل دقيقة أثناء الاستماع)
- `points_earned`: النقاط الفعلية (تُحسب مرة واحدة عند الإيقاف من `start_time`)

**الاستخدام في V1.3:**
- **Session-Based Points**: النقاط تُمنح مرة واحدة عند إيقاف الجلسة
- **لا منح نقاط كل دقيقة**: لتقليل التلاعب
- **Auto-close**: الجلسات المتروكة تُغلق تلقائياً بعد 4 ساعات

#### جدول `club_points_wallet` (محفظة النقاط)

```sql
CREATE TABLE club_points_wallet (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL UNIQUE REFERENCES new_profiles(id),
  pending_points INTEGER DEFAULT 0,        -- نقاط قيد الاعتماد (رصيد هذا الشهر)
  available_points INTEGER DEFAULT 0,      -- نقاط جاهزة للاستخدام (رصيد متاح)
  used_points INTEGER DEFAULT 0,           -- نقاط مستخدمة
  lifetime_points INTEGER DEFAULT 0,       -- إجمالي النقاط المكتسبة مدى الحياة
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**الحقول المهمة:**
- `pending_points`: النقاط المكتسبة من الراديو تذهب هنا أولاً
- `available_points`: النقاط المعتمدة والجاهزة للاستخدام
- `used_points`: النقاط المستخدمة في الجوائز

#### جدول `club_points_transactions` (معاملات النقاط)

```sql
CREATE TABLE club_points_transactions (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES new_profiles(id),
  transaction_type VARCHAR(20) NOT NULL CHECK (
    transaction_type IN ('EARNED', 'ACTIVATED', 'USED')
  ),
  points INTEGER NOT NULL,                  -- القيمة (موجب للنقاط المكتسبة)
  reason VARCHAR(255),                      -- السبب
  source VARCHAR(50),                       -- المصدر: 'radio_stream'
  description TEXT,                         -- وصف تفصيلي
  created_by UUID REFERENCES new_profiles(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**أنواع المعاملات:**
- `EARNED`: كسب نقاط (من الراديو) → تذهب إلى `pending_points`
- `ACTIVATED`: اعتماد شهري → نقل من `pending_points` إلى `available_points`
- `USED`: استخدام نقاط (استبدال جائزة) → خصم من `available_points`

### 3.2 الدوال (RPC Functions)

#### `stop_user_listening_session_and_award`

**الوصف:** إيقاف جلسة الاستماع وحساب النقاط ومنحها مرة واحدة

**المعاملات:**
- `p_session_id`: معرف الجلسة (UUID)
- `p_user_id`: معرف المستخدم (UUID)
- `p_activity_id`: معرف نشاط الراديو (UUID)

**ما تفعله:**
1. تجلب `start_time` من `radio_user_sessions`
2. تحسب `duration_minutes = (NOW() - start_time) / 60`
3. تحسب `points_earned = calculate_service_points('radio_stream', duration_minutes)`
4. تحدث الجلسة: `stop_time`, `duration_minutes`, `points_earned`, `status = 'closed'`
5. تمنح النقاط: `update_club_points_wallet(..., 'EARNED', 'radio_stream', ...)`
6. ترجع: `{minutes_listened, points_earned}`

**الكود الكامل:**
```sql
CREATE OR REPLACE FUNCTION stop_user_listening_session_and_award(
  p_session_id UUID,
  p_user_id UUID,
  p_activity_id UUID
)
RETURNS JSONB
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_start_time TIMESTAMPTZ;
  v_stop_time TIMESTAMPTZ;
  v_duration_minutes INTEGER;
  v_points_earned INTEGER;
BEGIN
  -- 1) جلب وقت البدء
  SELECT start_time INTO v_start_time
  FROM radio_user_sessions
  WHERE id = p_session_id AND user_id = p_user_id;
  
  IF v_start_time IS NULL THEN
    RAISE EXCEPTION 'Session not found';
  END IF;
  
  -- 2) حساب المدة
  v_stop_time := NOW();
  v_duration_minutes := FLOOR(EXTRACT(EPOCH FROM (v_stop_time - v_start_time)) / 60);
  
  -- 3) حساب النقاط
  v_points_earned := calculate_service_points('radio_stream', v_duration_minutes);
  
  -- 4) تحديث الجلسة
  UPDATE radio_user_sessions
  SET 
    stop_time = v_stop_time,
    duration_minutes = v_duration_minutes,
    points_earned = v_points_earned,
    status = 'closed',
    updated_at = NOW()
  WHERE id = p_session_id;
  
  -- 5) منح النقاط (تذهب إلى pending_points)
  PERFORM update_club_points_wallet(
    p_user_id,
    v_points_earned,
    'EARNED',
    'استماع راديو كارمش',
    'radio_stream',
    'نقاط من الاستماع للراديو: ' || v_duration_minutes || ' دقيقة',
    NULL
  );
  
  RETURN jsonb_build_object(
    'minutes_listened', v_duration_minutes,
    'points_earned', v_points_earned
  );
END;
$$;
```

#### `calculate_service_points`

**الوصف:** حساب النقاط بناءً على اسم الخدمة والمدة

**المعاملات:**
- `p_service_name`: اسم الخدمة (مثلاً: 'radio_stream')
- `p_minutes`: المدة بالدقائق (INTEGER)

**ما تفعله:**
1. تجلب معدل النقاط من `club_service_points_config` أو `club_activities`
2. تحسب: `points = minutes × points_rate`
3. ترجع النقاط المحسوبة

```dart
    .upsert({

      'activity_id': activityId,
      'start_time': DateTime.now().toIso8601String(),
      'status': 'active',
    })
    .select()
    .single();
  
  return sessionResponse['id'] as String;
}
```

### 4.4 سيناريو Auto-close (إغلاق تلقائي)

**ما يحدث:**
- إذا ترك المستخدم التطبيق مفتوحاً ولم يوقف الاستماع
- بعد **4 ساعات**، يتم إغلاق الجلسة تلقائياً
- يتم حساب النقاط ومنحها بناءً على المدة الفعلية

**من Backend (Cron Job):**

```sql
-- دالة لإغلاق الجلسات المتروكة
CREATE OR REPLACE FUNCTION close_abandoned_radio_user_sessions()
RETURNS INTEGER
LANGUAGE plpgsql
AS $$
DECLARE
  v_count INTEGER;
BEGIN
  -- إغلاق الجلسات النشطة الأقدم من 4 ساعات
  WITH closed_sessions AS (
    UPDATE radio_user_sessions
    SET 
      stop_time = NOW(),
      duration_minutes = FLOOR(EXTRACT(EPOCH FROM (NOW() - start_time)) / 60),
      status = 'closed',
      updated_at = NOW()
    WHERE status = 'active'
      AND start_time < NOW() - INTERVAL '4 hours'
    RETURNING id, user_id, activity_id, duration_minutes
  )
  SELECT COUNT(*) INTO v_count FROM closed_sessions;
  
  -- منح النقاط للجلسات المغلقة
  -- (يتم استدعاء stop_user_listening_session_and_award لكل جلسة)
  
  RETURN v_count;
END;
$$;
```

## 5. الأنيميشن لعداد النقاط المتوقعة

### 5.1 كيف يعمل الأنيميشن في التطبيق الحالي (Next.js)

#### المبدأ الأساسي

**الأنيميشن يعمل على مرحلتين:**

1. **حساب النقاط المتوقعة** (كل ثانية)
2. **أنيميشن سلس للعداد** (كل 100ms)

#### 📊 شرح تفصيلي لآلية العمل

**الهدف:** عرض عداد نقاط متوقع يتحرك بسلاسة أثناء الاستماع للراديو، مع تأثيرات بصرية عند الاقتراب من نقطة كاملة.

**البنية:**
- **`estimatedPoints`**: القيمة المستهدفة (تُحسب كل ثانية)
- **`animatedPoints`**: القيمة المعروضة (تُحدث كل 100ms بشكل سلس)
- **`isNearPoint`**: حالة بصرية عند الاقتراب من نقطة كاملة

#### 🔍 مصدر البيانات: من أين يأتي الوقت المتوقع؟

**السؤال المهم:** هل يتم احتساب الوقت المتوقع من قواعد البيانات أم من Frontend؟

**الإجابة:**

1. **`start_time` في قاعدة البيانات:**
   - ✅ **يُسجل** في جدول `radio_user_sessions.start_time` عند بدء الاستماع
   - يتم إنشاؤه من Backend عند استدعاء `startUserListeningSession()`
   - مثال: `2025-01-22T10:00:00.000Z`

2. **`sessionStartAt` في Frontend:**
   - ✅ **يُسجل محلياً** في Frontend (State) عند بدء الاستماع
   - يأتي من `new Date().toISOString()` في Frontend مباشرة

```typescript
useEffect(() => {

    setEstimatedPoints(0);
    setAnimatedPoints(0);
    setCurrentListeningMinutes(0);
    setIsNearPoint(false);
    return;

  const updateEstimatedPoints = () => {
    const now = new Date();  // الوقت الحالي من Frontend
    const start = new Date(sessionStartAt);  // وقت البدء من Frontend State (ليس من قاعدة البيانات)
    const diffMs = now.getTime() - start.getTime();
    const diffSeconds = diffMs / 1000;
    const diffMinutes = diffSeconds / 60;  // مثال: 2.5 دقيقة
    const currentMinute = Math.floor(diffMinutes);
    setCurrentListeningMinutes(currentMinute);
    
    const pointsRate = currentStream?.points_per_minute || 1;
    const estimated = diffMinutes * pointsRate;  // مثال: 2.5 × 1 = 2.5 نقطة
    setEstimatedPoints(estimated);  // ✅ يُسجل في قاعدة البيانات كل دقيقة (مرجع تاريخي)

    const nextPoint = Math.ceil(estimated);  // مثال: Math.ceil(2.5) = 3
    const distanceToNextPoint = nextPoint - estimated;  // مثال: 3 - 2.5 = 0.5
    const secondsToNextPoint = distanceToNextPoint * 60 / pointsRate;  // مثال: 0.5 × 60 / 1 = 30 ثانية
    setIsNearPoint(secondsToNextPoint <= 10 && secondsToNextPoint > 0);  // إذا كان أقل من 10 ثواني

    // تحديث estimated_points في قاعدة البيانات كل دقيقة (مرجع تاريخي)
    if (currentMinute > lastSavedMinute && userSessionId) {
      lastSavedMinute = currentMinute;
      // تحديث قاعدة البيانات بشكل غير متزامن (لا ننتظر النتيجة)
      clubRadioService.updateEstimatedPoints(userSessionId, estimated).catch((err) => {
        console.warn('[Radio] Failed to update estimated_points:', err);
      });
    }
  };

  updateEstimatedPoints();
  const interval = setInterval(updateEstimatedPoints, 1000);  // كل ثانية

  return () => clearInterval(interval);
}, [sessionStartAt, isPlaying, currentStream?.points_per_minute]);
```

```typescript
useEffect(() => {
  if (!isPlaying || !sessionStartAt) {
    setAnimatedPoints(0);
    return;
  }

  const animate = () => {
    setAnimatedPoints((current) => {
      const targetPoints = estimatedPoints;  // القيمة المستهدفة (من المرحلة 1)
      const diff = targetPoints - current;   // الفرق بين الهدف والقيمة الحالية

      if (Math.abs(diff) < 0.01) {
        return targetPoints;
      }

      // Easing Function: نتحرك 20% من المسافة في كل خطوة
      // هذا يخلق تأثير "تباطؤ تدريجي" (Ease Out)
      const step = diff * 0.2;
      return current + step;
    });
  };

  const interval = setInterval(animate, 100);  // كل 100ms
  return () => clearInterval(interval);
}, [estimatedPoints, isPlaying, sessionStartAt]);
```

**شرح المرحلة 2:**

1. **المبدأ:**
   - `estimatedPoints` (الهدف) يُحدث كل ثانية من المرحلة 1

2. **Easing Function (دالة التباطؤ):**
   - `diff = targetPoints - current`: الفرق بين الهدف والقيمة الحالية
   - `step = diff * 0.2`: نتحرك 20% من المسافة في كل خطوة

3. **مثال عملي:**

   ```
   القيمة الحالية: 4.0 نقطة
   ```

4. **لماذا 20%؟**
   - نسبة متوازنة: ليست سريعة جداً (مثل 50%) ولا بطيئة جداً (مثل 10%)
   - تخلق تأثير سلس وطبيعي

```dart
class RadioService {
  final SupabaseClient supabase;
  
  RadioService(this.supabase);
  
  /// جلب البث الحالي
  Future<RadioStream?> getCurrentStream() async {
    final response = await supabase
        .from('club_activities')
        .select()
        .eq('activity_type', 'radio_stream')
        .eq('is_active', true)
        .eq('is_live', true)
        .maybeSingle();
    
    if (response == null) return null;
    return RadioStream.fromJson(response);
  }
  
  /// بدء الاستماع
  Future<RadioSession> startListening(String userId, String activityId) async {
    // 1) إنشاء/تحديث radio_listeners
    await supabase.from('radio_listeners').upsert({
      'user_id': userId,
      'activity_id': activityId,
      'started_listening_at': DateTime.now().toIso8601String(),
      'last_active_at': DateTime.now().toIso8601String(),
      'is_active': true,
      'duration_minutes': 0,
      'points_earned': 0,
    });
    
    // 2) إنشاء جلسة
    final sessionResponse = await supabase
        .from('radio_user_sessions')
        .insert({
          'user_id': userId,
          'activity_id': activityId,
          'start_time': DateTime.now().toIso8601String(),
          'status': 'active',
        })
        .select()
        .single();
    
    return RadioSession.fromJson(sessionResponse);
  }
  
  /// تحديث آخر نشاط (Heartbeat)
  Future<void> updateLastActive(String userId, String activityId) async {
    await supabase
        .from('radio_listeners')
        .update({
          'last_active_at': DateTime.now().toIso8601String(),
        })
        .eq('user_id', userId)
        .eq('activity_id', activityId)
        .eq('is_active', true);
  }
  
  /// إيقاف الاستماع ومنح النقاط
  Future<StopListeningResult> stopListening(
    String sessionId,
    String userId,
    String activityId,
  ) async {
    // 1) إيقاف الاستماع في radio_listeners
    await supabase
        .from('radio_listeners')
        .update({'is_active': false})
        .eq('user_id', userId)
        .eq('activity_id', activityId);
    
    // 2) استدعاء RPC Function
    final result = await supabase.rpc(
      'stop_user_listening_session_and_award',
      params: {
        'p_session_id': sessionId,
        'p_user_id': userId,
        'p_activity_id': activityId,
      },
    );
    
    return StopListeningResult(
      minutesListened: result['minutes_listened'] as int,
      pointsEarned: result['points_earned'] as int,
    );
  }
}
```

```dart
  await audioPlayer.play(UrlSource(stream.listenUrl));
  
  // 5) بدء Heartbeat
  Timer.periodic(Duration(minutes: 1), (timer) {
    radioService.updateLastActive(user.id, stream.id);
  });
  
  // 6) بدء عداد النقاط المتوقعة
  setState(() {
    currentSession = session;
    sessionStartTime = session.startTime;
    isPlaying = true;
  });
}
```

### 7.3 مثال كامل: عداد النقاط المتوقعة مع الأنيميشن

```dart
class AnimatedPointsCounter extends StatefulWidget {
  final DateTime startTime;
  final int pointsPerMinute;
  final bool isActive;
  
  const AnimatedPointsCounter({
    Key? key,
    required this.startTime,
    required this.pointsPerMinute,
    required this.isActive,
  }) : super(key: key);
  
  @override
  _AnimatedPointsCounterState createState() => _AnimatedPointsCounterState();
}

class _AnimatedPointsCounterState extends State<AnimatedPointsCounter> {
  Timer? _calculationTimer;
  Timer? _animationTimer;
  double _animatedPoints = 0.0;
  double _targetPoints = 0.0;
  bool _isNearPoint = false;
  bool _isMilestone = false;
  
  @override
  void initState() {
    super.initState();
    if (widget.isActive) {
      _startAnimation();
    }
  }
  
  @override
  void didUpdateWidget(AnimatedPointsCounter oldWidget) {
    super.didUpdateWidget(oldWidget);
    if (widget.isActive && !oldWidget.isActive) {
      _startAnimation();
    } else if (!widget.isActive && oldWidget.isActive) {
      _stopAnimation();
    }
  }
  
  void _startAnimation() {
    int lastWholePoint = 0;
    
    // حساب النقاط المتوقعة (كل ثانية)
    _calculationTimer = Timer.periodic(Duration(seconds: 1), (timer) {
      if (!mounted || !widget.isActive) {
        timer.cancel();
        return;
      }
      
      final now = DateTime.now();
      final duration = now.difference(widget.startTime);
      final minutes = duration.inMinutes + (duration.inSeconds % 60) / 60.0;
      final newTarget = minutes * widget.pointsPerMinute;
      
      // التحقق من الوصول لنقطة كاملة
      final newWholePoint = newTarget.floor();
      if (newWholePoint > lastWholePoint) {
        _triggerMilestoneAnimation();
        lastWholePoint = newWholePoint;
      }
      
      setState(() {
        _targetPoints = newTarget;
        _isNearPoint = _checkNearPoint(newTarget);
      });
    });
    
    // أنيميشن سلس (كل 100ms)
    _animationTimer = Timer.periodic(Duration(milliseconds: 100), (timer) {
      if (!mounted || !widget.isActive) {
        timer.cancel();
        return;
      }
      
      setState(() {
        final diff = _targetPoints - _animatedPoints;
        
        if (diff.abs() < 0.01) {
          _animatedPoints = _targetPoints;
        } else {
          // Easing: 20% من المسافة في كل خطوة
          final step = diff * 0.2;
          _animatedPoints += step;
        }
      });
    });
  }
  
  void _stopAnimation() {
    _calculationTimer?.cancel();
    _animationTimer?.cancel();
  }
  
  bool _checkNearPoint(double points) {
    final nextPoint = points.ceil();
    final distanceToNextPoint = nextPoint - points;
    final secondsToNextPoint = (distanceToNextPoint * 60) / widget.pointsPerMinute;
    return secondsToNextPoint <= 10 && secondsToNextPoint > 0;
  }
  
  void _triggerMilestoneAnimation() {
    setState(() {
      _isMilestone = true;
    });
    
    Future.delayed(Duration(milliseconds: 500), () {
      if (mounted) {
        setState(() {
          _isMilestone = false;
        });
      }
    });
  }
  
  @override
  void dispose() {
    _stopAnimation();
    super.dispose();
  }
  
  @override
  Widget build(BuildContext context) {
    return AnimatedContainer(
      duration: Duration(milliseconds: 300),
      padding: EdgeInsets.all(16),
      decoration: BoxDecoration(
        color: _isNearPoint
            ? Colors.green.withOpacity(0.1)
            : _isMilestone
                ? Colors.blue.withOpacity(0.1)
                : Colors.grey.withOpacity(0.1),
        borderRadius: BorderRadius.circular(12),
        border: _isNearPoint
            ? Border.all(color: Colors.green, width: 2)
            : _isMilestone
                ? Border.all(color: Colors.blue, width: 2)
                : null,
      ),
      child: Column(
        children: [
          Text(
            widget.isActive ? 'النقاط المتوقعة' : 'النقاط المكتسبة',
            style: TextStyle(
              fontSize: 14,
              color: Colors.grey[700],
            ),
          ),
          SizedBox(height: 8),
          AnimatedScale(
            scale: _isMilestone ? 1.2 : 1.0,
            duration: Duration(milliseconds: 300),
            child: Text(
              _animatedPoints.toStringAsFixed(1),
              style: TextStyle(
                fontSize: 48,
                fontWeight: FontWeight.bold,
                color: _isNearPoint
                    ? Colors.green
                    : _isMilestone
                        ? Colors.blue
                        : Colors.black,
              ),
            ),
          ),
          if (widget.isActive) ...[
            SizedBox(height: 4),
            Text(
              '⚠️ تُمنح عند الإيقاف',
              style: TextStyle(
                fontSize: 12,
                color: Colors.orange,
              ),
            ),
          ],
          if (_isNearPoint) ...[
            SizedBox(height: 4),
            Text(
              '🎯 قريب من نقطة!',
              style: TextStyle(
                fontSize: 12,
                color: Colors.green,
                fontWeight: FontWeight.bold,
              ),
            ),
          ],
        ],
      ),
    );
  }
}
```

## 8. ملخص شامل

### 8.1 دورة حياة النقاط الكاملة

```
1. بدء البث (من الأدمن)
   └─> club_activities.is_live = true
   
2. بدء الاستماع (من المستخدم)
   └─> radio_listeners.is_active = true
   └─> radio_user_sessions.status = 'active'
   └─> عداد Preview يبدأ (للعرض فقط)
   
3. أثناء الاستماع
   └─> Heartbeat كل دقيقة (تحديث last_active_at)
   └─> عداد Preview يُحدث كل ثانية
   └─> أنيميشن سلس كل 100ms
   └─> ⚠️ لا منح نقاط (Preview فقط)
   
4. إيقاف الاستماع
   └─> radio_listeners.is_active = false
   └─> استدعاء stop_user_listening_session_and_award
   └─> حساب duration_minutes
   └─> حساب points_earned
   └─> منح النقاط إلى pending_points
   └─> تسجيل في club_points_transactions
   
5. الاعتماد الشهري (من الأدمن)
   └─> pending_points → available_points
   └─> تسجيل في club_points_transactions كـ ACTIVATED
```

### 8.2 النقاط المهمة

1. ✅ **Session-Based**: النقاط تُمنح مرة واحدة عند الإيقاف فقط
2. ✅ **Preview Counter**: عداد النقاط المتوقعة للتحفيز فقط
3. ✅ **Heartbeat**: تحديث `last_active_at` كل دقيقة
4. ✅ **Auto-close**: الجلسات المتروكة تُغلق بعد 4 ساعات
5. ✅ **Pending Points**: النقاط تذهب إلى `pending_points` أولاً
6. ✅ **Monthly Activation**: الاعتماد الشهري يدوياً من الأدمن

### 8.3 قواعد العمل

- **معدل النقاط**: يُحدد من قبل الأدمن (افتراضي: 1 نقطة/دقيقة)
- **الحد الأدنى**: لا يوجد حد أدنى (حتى ثانية واحدة تُحسب)
- **الحد الأقصى**: لا يوجد حد أقصى (لكن الجلسات تُغلق بعد 4 ساعات)
- **التلاعب**: غير ممكن (النقاط تُمنح مرة واحدة فقط عند الإيقاف)
