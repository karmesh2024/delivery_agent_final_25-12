## 🆕 تأثير آخر التحديثات على تطبيق العميل

### 1) الربحية (Profitability)
- ✅ **لا تأثير مباشر على واجهات العميل**: الربحية تخص الإدارة واللوحات فقط.
- ✅ قد ينعكس **بشكل غير مباشر** في جودة التسعير/العروض، لكن **لا يوجد API لازم للعميل**.

### 2) حدود السحب (Withdrawal Limits) + التحقق قبل الطلب
- ✅ أصبح السحب النقدي محكوم بـ:
  - **وجود جلسات مكتملة غير مُعتمدة** (Unsettled Sessions) → يمنع السحب.
  - **حدود يومية/أسبوعية/شهرية** (Amount + Count).
- ✅ في تطبيق العميل: لا تُنشئ سجل سحب مباشرة في `points_redemptions`، بل استخدم **RPC** `withdraw_from_wallet`.
- ✅ اختياري (لـ UX أفضل): اعرض رسالة واضحة قبل الإرسال عبر **RPC** `check_withdrawal_limits`.

### 3) قواعد نقاط المتجر (store_points rules)
- ✅ `store_points` أصبحت تُحسب عبر **قواعد ديناميكية** تُدار من الداشبورد (ترحيب/فئات/موسمي/نسبة/قيمة ثابتة).
- ✅ في تطبيق العميل: **لا تحسب البونص محلياً**. اعرض فقط الناتج النهائي القادم من النظام (جلسات/رصيد/سجل معاملات).

### 4) أهم تحسينيْن لتجربة العميل (مطلوبين)
1) **شفافية استباقية بدون مصطلحات تقنية**:
   - لا نعرض كلمات مثل Pending/Settlement.
   - لكن نعرض “جلسة قيد المراجعة” + “جاهز خلال 24 ساعة” قبل أن يصطدم العميل عند السحب.
2) **نحوّل القيود إلى ميزة**:
   - حدود السحب تُعرض كجزء من “برنامج مستويات” (Bronze/Silver/Gold) حتى لو كان تطبيقه تدريجي.

**المسار:** `/balance` أو `/wallet`

**التصميم:**
```
┌─────────────────────────────────┐
│                                 │
│    💰 رصيدك الحالي              │
│                                 │
│        15,000 نقطة              │
│                                 │
│      (~150 ج.م)                 │

├─────────────────────────────────┤
│                                 │
```

**الكود:**
```typescript
// Screen: BalanceScreen.tsx
import { useState, useEffect } from 'react';
import { supabase } from '@/lib/supabase';

export default function BalanceScreen() {
  const [balance, setBalance] = useState(0);
  const [walletBalance, setWalletBalance] = useState(0); // بالجنيه (للعرض التقديري/السحب)
  const [pendingSessions, setPendingSessions] = useState(0); // جلسات قيد المراجعة (بدون كلمة Pending)
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    loadBalance();
  }, []);
  
  const loadBalance = async () => {
    try {
      const { data: { user } } = await supabase.auth.getUser();
      
      if (!user) return;
      
      // ✅ الأفضل: جلب ملخص الرصيد من Function (بدلاً من قراءة الجدول مباشرة)
      // هذا يضمن توافق الحسابات حتى لو تغيرت قواعد البونص/القيود لاحقاً
      const { data, error } = await supabase.rpc('get_customer_balance_summary', {
        p_customer_id: user.id
      });
      
      if (error) throw error;
      
      // حساب الرصيد الإجمالي (نقاط) للعرض كرقم واحد
      // ملاحظة: wallet_balance = قيمة نقدية بالجنيه، store_points = نقاط بونص
      const totalPoints = (data.wallet_balance * 100) + (data.store_points || 0);
      
      setBalance(totalPoints);
      setWalletBalance(data.wallet_balance || 0);
      setPendingSessions(data.pending_sessions || 0);
    } catch (error) {
      console.error('Error loading balance:', error);
    } finally {
      setLoading(false);
    }
  };
  
  if (loading) {
    return <LoadingSpinner />;
  }
  
  // ✅ القيمة النقدية المتاحة للسحب (الأدق): wallet_balance من الـ RPC
  // (تجنب تحويل store_points إلى جنيه لأنها ليست مالاً قابلاً للسحب)
  const estimatedValue = walletBalance;
  
  return (
    <View style={styles.container}>
      <Text style={styles.title}>رصيدك الحالي</Text>
      
      <Text style={styles.balance}>{balance.toLocaleString()} نقطة</Text>
      
      <Text style={styles.estimated}>
        (~{estimatedValue.toFixed(2)} ج.م)
      </Text>

      {/* ✅ شفافية استباقية (بدون مصطلحات تقنية) */}
      {pendingSessions > 0 && (
        <View style={styles.infoBox}>
          <Text style={styles.infoTitle}>🕐 جلستك قيد المراجعة</Text>
          <Text style={styles.infoText}>
            لديك {pendingSessions} جلسة قيد المراجعة{'\n'}
            سيكون رصيدك جاهز للسحب خلال 24 ساعة
          </Text>
        </View>
      )}
      
      <View style={styles.actions}>
        <Button 
          title="استبدال نقدي"
          onPress={() => navigation.navigate('Withdraw')}
        />
        
        <Button 
          title="شراء من المتجر"
          onPress={() => navigation.navigate('Store')}
        />
        
        <Button 
          title="الهدايا"
          onPress={() => navigation.navigate('Gifts')}
        />
        
        <Button 
          title="التبرع"
          onPress={() => navigation.navigate('Donate')}
        />
      </View>
    </View>
  );
}
```

**ملاحظة UX مهمة (حلّ مشكلة “ليه ظهر الرصيد لو مش متاح؟”):**
- لا تمنع عرض “رصيدك الحالي” (نقاط).
- لكن عند وجود جلسات قيد المراجعة: أضف الـ **InfoBanner** أعلاه في نفس صفحة الرصيد + في صفحة السحب.

### 2. صفحة طلب السحب (Withdraw Request)

**المسار:** `/withdraw`

**التصميم:**
```
┌─────────────────────────────────┐
│ 💸 طلب سحب نقدي                 │
├─────────────────────────────────┤
│                                 │
│ رصيدك: 15,000 نقطة              │
│                                 │
│ المبلغ المطلوب:                │
│ [____] جنيه                     │
│                                 │
│ طريقة الاستلام:                │
│ ○ تحويل بنكي                   │
│ ○ محفظة إلكترونية              │
│                                 │
│ ⚠️ سيتم المعالجة خلال 24 ساعة │
│                                 │
│ [إرسال الطلب]                  │
└─────────────────────────────────┘
```

**ملاحظة مهمة عن طريقة الاستلام (تحويل/محفظة):**
- طريقة الاستلام **ليست جزءاً من نظام النقاط نفسه**.
- يمكنك حفظها في:
  - `new_profiles` (مثلاً حقول `preferred_withdraw_method`/`payout_info`) أو
  - جدول مستقل مثل `customer_payout_accounts`،
  - أو إرسالها كـ `notes` عبر Endpoint مخصص (إن وُجد).

**الكود:**
```typescript
// Screen: WithdrawScreen.tsx
export default function WithdrawScreen() {
  const [amount, setAmount] = useState('');
  const [paymentMethod, setPaymentMethod] = useState('bank');
  const [loading, setLoading] = useState(false);
  const [pendingSessions, setPendingSessions] = useState(0);
  const [walletBalance, setWalletBalance] = useState(0); // للعرض والتحقق الودي
  
  const handleWithdraw = async () => {
    if (!amount || parseFloat(amount) <= 0) {
      showError('يرجى إدخال مبلغ صحيح');
      return;
    }
    
    setLoading(true);
    
    try {
      const { data: { user } } = await supabase.auth.getUser();
      
      if (!user) {
        throw new Error('يجب تسجيل الدخول');
      }

      // ✅ (اختياري) جلب ملخص سريع لعرض رسالة استباقية (بدون انتظار فشل العملية)
      const { data: summary, error: summaryError } = await supabase.rpc('get_customer_balance_summary', {
        p_customer_id: user.id
      });
      if (summaryError) throw summaryError;
      setPendingSessions(summary.pending_sessions || 0);
      setWalletBalance(summary.wallet_balance || 0);

      // ✅ إذا يوجد جلسات قيد المراجعة: أخبره بلغة طبيعية (بدون تقنية)
      if ((summary.pending_sessions || 0) > 0) {
        showError('لديك جلسة قيد المراجعة. سيكون السحب متاح خلال 24 ساعة.');
        return;
      }
      
      // ✅ (اختياري) تحقّق مسبق لعرض رسالة واضحة قبل الإرسال
      const { data: limitCheck, error: limitError } = await supabase.rpc('check_withdrawal_limits', {
        p_customer_id: user.id,
        p_amount_egp: parseFloat(amount),
      });
      if (limitError) throw limitError;
      if (limitCheck?.allowed === false) {
        showError(limitCheck.reason || 'لا يمكن تنفيذ السحب حالياً');
        return;
      }

      // ✅ مهم: لا تقم بعمل INSERT مباشر في points_redemptions من الموبايل
      // استخدم Function لأنها تطبق:
      // - منع السحب مع وجود جلسات غير مُعتمدة
      // - حدود السحب (يومي/أسبوعي/شهري)
      // - خصم الرصيد وحجزه + تسجيل Audit Log
      const { data: redemptionId, error } = await supabase.rpc('withdraw_from_wallet', {
        p_customer_id: user.id,
        p_amount_egp: parseFloat(amount),
      });
      
      if (error) throw error;
      
      // رسالة نجاح
      showSuccess('تم إرسال طلب السحب. سيتم المعالجة خلال 24 ساعة.');
      
      // العودة للصفحة السابقة
      navigation.goBack();
      
    } catch (error: any) {
      console.error('Withdraw error:', error);
      
      // رسائل خطأ واضحة
      if (error.message.includes('جلسة غير مقفلة')) {
        showError('جلستك قيد المراجعة. سيكون رصيدك جاهز للسحب خلال 24 ساعة.');
      } else if (error.message.includes('تجاوز الحد')) {
        showError('وصلت للحد المسموح للسحب. جرّب مبلغ أقل أو لاحقاً.');
      } else if (error.message.includes('غير كافٍ')) {
        showError('رصيدك غير كافٍ');
      } else {
        showError('حدث خطأ. يرجى المحاولة لاحقاً.');
      }
    } finally {
      setLoading(false);
    }
  };
  
  return (
    <View style={styles.container}>
      <Text style={styles.title}>طلب سحب نقدي</Text>
      
      <Text style={styles.balanceLabel}>رصيدك: {balance} نقطة</Text>

      {/* ✅ شفافية استباقية */}
      {pendingSessions > 0 && (
        <View style={styles.infoBox}>
          <Text style={styles.infoTitle}>🕐 جلستك قيد المراجعة</Text>
          <Text style={styles.infoText}>سيكون السحب متاح خلال 24 ساعة</Text>
        </View>
      )}
      
      <TextInput
        style={styles.input}
        placeholder="المبلغ بالجنيه"
        value={amount}
        onChangeText={setAmount}
        keyboardType="numeric"
      />
      
      <Text style={styles.methodLabel}>طريقة الاستلام:</Text>
      
      <RadioButton
        value="bank"
        selected={paymentMethod === 'bank'}
        onSelect={() => setPaymentMethod('bank')}
        label="تحويل بنكي"
      />
      
      <RadioButton
        value="wallet"
        selected={paymentMethod === 'wallet'}
        onSelect={() => setPaymentMethod('wallet')}
        label="محفظة إلكترونية"
      />
      
      <Text style={styles.note}>
        ⚠️ سيتم المعالجة خلال 24 ساعة
      </Text>
      
      <Button
        title="إرسال الطلب"
        onPress={handleWithdraw}
        loading={loading}
      />
    </View>
  );
}
```

**⚠️ مهم جداً:**
- لا تعمل `INSERT` مباشر في `points_redemptions` من الموبايل
- استخدم `supabase.rpc('withdraw_from_wallet')` لإنشاء الطلب وتطبيق القيود تلقائياً
- رسالة واضحة: "سيتم المعالجة خلال 24 ساعة"
- لا عرض تفاصيل تقنية (is_settled, pending, etc.)

```typescript
// Screen: SessionsHistoryScreen.tsx
export default function SessionsHistoryScreen() {
  const [sessions, setSessions] = useState([]);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    loadSessions();
  }, []);
  
  const loadSessions = async () => {
    try {
      const { data: { user } } = await supabase.auth.getUser();
      
      if (!user) return;
      
      // جلب الجلسات المكتملة فقط
      const { data, error } = await supabase
        .from('waste_collection_sessions')
        .select(`
          id,
          status,
          total_points,
          base_points,
          bonus_points,
          created_at,
          completed_at
        `)
        .eq('customer_id', user.id)
        .eq('status', 'completed')
        .order('completed_at', { ascending: false })
        .limit(20);
      
      if (error) throw error;
      
      // تحويل البيانات لعرض بسيط
      const formattedSessions = data.map(session => ({
        id: session.id,
        date: formatDate(session.completed_at),
        points: session.total_points || (session.base_points + (session.bonus_points || 0)),
        status: 'completed' // ✅ دائماً "تم التجميع"
      }));
      
      setSessions(formattedSessions);
    } catch (error) {
      console.error('Error loading sessions:', error);
    } finally {
      setLoading(false);
    }
  };
  
  return (
    <View style={styles.container}>
      <Text style={styles.title}>تاريخ التجميع</Text>
      
      {loading ? (
        <LoadingSpinner />
      ) : (
        <FlatList
          data={sessions}
          renderItem={({ item }) => (
            <SessionCard
              date={item.date}
              points={item.points}
              onPress={() => navigation.navigate('SessionDetails', { id: item.id })}
            />
```

## 🔔 إشعارات استباقية (مستحسنة جداً)

### الهدف
- العميل **لا يكتشف** “المراجعة 24 ساعة” عند السحب فقط.
- بل يصله توضيح مبكر بعد الجلسة، وتنبيه عند جاهزية السحب.

### إشعار 1: بعد اكتمال الجلسة (خلال دقائق)
> “🎉 تم جمع مخلفاتك بنجاح! حصلت على X نقطة. 🕐 سيكون رصيدك جاهز للسحب خلال 24 ساعة”

### إشعار 2: بعد اعتماد الجلسة (Settlement)
> “💰 رصيدك جاهز الآن! يمكنك السحب أو الاستبدال”

### طرق التنفيذ (اختر الأسهل)
1) **Realtime من Supabase**:
   - اشترك على تغييرات صفوف `waste_collection_sessions` الخاصة بالمستخدم.
   - عندما تصبح `status='completed'` → إشعار 1
   - عندما تصبح `is_settled=true` → إشعار 2
2) **Notifications Table + Push Provider (FCM/APNs)**:
   - Backend يكتب في جدول `notifications` ثم Edge Function ترسل Push.

> ملاحظة: لا نضيف تعقيد في واجهة العميل، فقط “رسائل ذكية في الوقت المناسب”.

أضف زر بسيط: **“كيف يعمل النظام؟”** داخل صفحة الرصيد.

### أمثلة أسئلة/إجابات جاهزة
1) **متى يكون رصيدي جاهز للسحب؟**

3) **هل كل النقاط يمكن تحويلها لنقد؟**  
   الجزء النقدي مرتبط بمحفظتك. قد تحصل أيضاً على نقاط بونص للمتجر تُستخدم في الاستبدالات داخل المتجر.

```typescript
// utils/formatBalance.ts

/**
 * تنسيق الرصيد للعرض
 * @param walletBalance - رصيد المحفظة بالجنيه
 * @param storePoints - نقاط المتجر
 * @returns الرصيد الإجمالي بالنقاط
 */
export function formatBalance(
  walletBalance: number,
  storePoints: number
): number {
  // حساب الرصيد الإجمالي (نقاط)
  return (walletBalance * 100) + storePoints;
}

/**
 * تنسيق الرصيد للعرض مع القيمة التقديرية
 */
export function formatBalanceWithValue(
  walletBalance: number,
  storePoints: number
): { points: number; estimatedValue: number } {
  const points = formatBalance(walletBalance, storePoints);
  const estimatedValue = points / 100;
  
  return { points, estimatedValue };
}

/**
 * تنسيق التاريخ للعرض
 */
export function formatDate(dateString: string): string {
  const date = new Date(dateString);
  const options: Intl.DateTimeFormatOptions = {
    year: 'numeric',
    month: 'long',
    day: 'numeric'
  };
  
  return date.toLocaleDateString('ar-EG', options);
}
```
