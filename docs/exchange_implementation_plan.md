# خطة تنفيذ نظام البورصة وإدارة الأسعار

## نظرة عامة

الهدف هو تمكين تحديث أسعار المخلفات يدويًا من قبل المسؤولين، وعرض هذه الأسعار
المحدثة يوميًا للعملاء عبر التطبيق.

## المرحلة الأولى: الأساسيات والتحكم اليدوي

تهدف هذه المرحلة إلى إنشاء البنية التحتية الأساسية وتمكين التعديل اليدوي للأسعار.

- [x] **تحليل المتطلبات وقاعدة البيانات**
  - مراجعة `docs\نظام_البورصة_المتكامل_النهائي.md`.
  - التحقق من جدول `stock_exchange` في Supabase.

- [x] **تحديث خدمة البورصة (`exchangeService.ts`)**
  - تحديث واجهة `StockExchange` لتشمل `buy_price`, `sell_price`, `region_id`.
  - إزالة منطق التحديث التلقائي مؤقتًا للتركيز على التحديث اليدوي.
  - دعم إضافة وتحديث المنتجات مع الحفاظ على `region_id` (افتراضي: القاهرة).

- [x] **تحديث واجهة إدارة البورصة (`ExchangeDashboardPage.tsx`)**
  - عرض `buy_price` بدلاً من `current_price`.
  - تمكين تعديل `buy_price` وحفظه في قاعدة البيانات.
  - عرض حالة الطلب (Category/Supply/Demand).

- [x] **إنشاء API لتطبيق العملاء**
  - المسار: `/api/exchange/prices`
  - الوظيفة: جلب قائمة الأسعار الحالية (`buy_price`) للمنتجات النشطة.
  - التنسيق: JSON بسيط يحتوي على الاسم، السعر، الوحدة، ونسبة التغير.

- [x] **التحقق من البيانات**
  - إضافة بيانات تجريبية لجدول `stock_exchange` للتأكد من عمل النظام.
